Теория — Redux/RTK не «silver bullet»

Цитата — Redux FAQ

«Not all apps need Redux. It's important to understand the kind of app you're building, the kinds of problems that you need to solve, and what tools can best solve the problems you're facing. Redux is a valuable tool for certain situations but it's not the only or best tool for every situation.»
— Redux FAQ — When should I use Redux
«Не каждому приложению нужен Redux. Важно понимать, какого типа приложение вы строите, какие задачи решаете, и какие инструменты лучше подходят. Redux — ценный инструмент в определённых ситуациях, но не единственный и не лучший для каждой.»

Критерии для Redux/RTK

  • State нужен многим компонентам, разнесённым по дереву.
  • Есть cross-cutting concerns: analytics, notifications, WebSocket, undo/redo.
  • Команда больше 3 человек — важна единая convention.
  • Ценны time-travel DevTools для отладки сложных багов.

Когда Redux избыточен

  • Только local UI-state: модалки, чекбоксы, формы → useState/useReducer.
  • Только server state: всё приложение — это «загрузить-показать-обновить» → React Query или RTK Query (без slice'ов).
  • Мелкий SPA без серьёзной логики — Zustand решит за 5 минут то, на что Redux тратит 50.

RTK Query vs React Query — почти одно и то же

Оба решают server state: cache, dedup, invalidation, polling, optimistic updates. RTK Query — выбор, если у вас уже Redux (интеграция бесплатная, один store, один DevTools). React Query — выбор, если Redux не нужен вовсе. По возможностям они примерно равны; по API React Query чуть гибче (лучше документация для edge cases), RTK Query — строже и type-safe.

Подводные камни

Redux для форм — почти всегда ошибка. Форма — это local state до submit. Redux-формы (old style) заставляют диспатчить action на каждый keystroke — медленно и шумно в DevTools. Используйте react-hook-form / Formik / обычный useState.
RTK Query + React Query одновременно — redundancy. Не добавляйте React Query в Redux-проект «потому что красивее API». Выберите одно. Два кэша, два инвалидатора, два DevTools — двойная сложность.
Zustand ≠ скрытый Redux. Zustand не эквивалентен Redux по возможностям: нет middleware-экосистемы уровня Redux, DevTools только через плагин, time-travel ограниченный. Если вам нужны listener/saga/RTK Query — возвращайтесь в Redux.
