Как Immer превращает мутацию в новый объект

Immer создаёт Proxy-обёртку над state (draft). Когда вы «мутируете» draft, Immer на самом деле записывает в внутренний map изменений. После reducer'а он строит новый объект только для изменённых путей. Непроизменные ветки — та же ссылка (structural sharing).

Структура state

{ counter: { value: 0 }, user: { name: 'Alice', age: 30 }, tags: ['a', 'b'], items: { ids: [], byId: {} }, }

Действия

Ссылочное равенство до/после

Зелёный = та же ссылка (===), красный = новая ссылка. После state.counter.value += 1 — только ветка counter обновится.

Теория — Immer

      

        
Из доки RTK

        

          «createReducer uses Immer under the hood to let you write "mutating" immutable update logic.
          Inside a case reducer, you can modify the state directly, and Immer will produce a safely copied new state.»
          
— Writing Reducers with Immer
        

        

          «createReducer использует Immer под капотом, чтобы позволить писать «мутирующую»
          иммутабельную логику обновления. Внутри case-reducer'а вы можете модифицировать state напрямую,
          а Immer произведёт безопасно скопированный новый state.»
        


        
Упрощённая реализация produce

        

import { produce } from 'immer'

function reducer(state, action) {
  return produce(state, (draft) => {
    // draft — это Proxy(state)
    if (action.type === 'increment') {
      draft.count += 1   // set-trap на Proxy
    }
    // ничего не возвращаем → Immer сам построит новый state
  })
}

// createReducer делает примерно это:
const wrappedReducer = (state, action) => {
  return produce(state, (draft) => {
    for (const caseReducer of matchingReducers) {
      caseReducer(draft, action)
    }
  })
}
        


        
Как работает Proxy draft

        
    
          
  1. produce создаёт Proxy над вашим state.
    2. 
          
  2. При чтении (get-trap) Proxy возвращает значение или создаёт вложенный Proxy.
    3. 
          
  3. При записи (set-trap) Proxy ставит modified: true на ноду и запоминает изменение.
    4. 
          
  4. После reducer'а Immer рекурсивно обходит draft:
            
      
              
    • если нода не modified — возвращает оригинал (тот же объект);
      • 
              
    • если modified — создаёт копию и применяет изменения.
      • 
            
    
          
    5. 
        


        
Structural sharing

        

          Ключевая оптимизация: непроизменные ветки state остаются с той же ссылкой.
          Это критично для React-Redux: селекторы, возвращающие ссылку на немутировавшую ветку,
          не триггерят ре-рендер.
        


        
Возврат из reducer'а

        

// Вариант 1: мутация draft, без return (самый частый)
(state, action) => {
  state.value += 1
}

// Вариант 2: возврат нового значения
(state, action) => {
  return { ...state, value: state.value + 1 }
}

// Вариант 3: возврат совершенно нового state
(state, action) => {
  return createInitialState()
}
        

        

          Но нельзя смешивать мутацию draft и return — подробнее в уроке 27.
        


        
Исключение: draft + primitive

        

          Если state — это примитив (number, string), «мутация» невозможна. Вы обязаны
          return newValue:
        

        

const reducer = createReducer(0, (builder) => {
  builder.addCase(increment, (state) => state + 1)  // обязательно return
})
        


        
Производительность

        

          Immer имеет оверхед ~2-5x на создание/обход Proxy по сравнению с ручным копированием. Для
          большинства reducer'ов это незаметно (микросекунды). Для больших массивов (1000+) можно
          использовать обычный иммутабельный return без мутации.
        


        
setAutoFreeze

        

          Immer по умолчанию замораживает (Object.freeze) результирующий state. Это ловит
          случайные мутации вне reducer'а. RTK включает autoFreeze на уровне produce — вы не можете
          state.value = 999 после dispatch.
        


        
original() и current()

        

          Внутри reducer'а у вас draft — не тот же объект, что state. Если нужен оригинал
          или снэпшот draft'а — используйте утилиты Immer (урок 27):
        

        

import { current, original } from '@reduxjs/toolkit'

(state, action) => {
  console.log(original(state))   // исходный объект до мутаций
  state.x = 1
  console.log(current(state))    // snapshot draft'а в данный момент
}
