Шпаргалка по секции A (быстрая сверка)

Пять правил, которые нужно запомнить

  1. RTK — это обёртка, не замена Redux. Те же action → dispatch → reducer → store, те же три принципа (SSOT, read-only state, pure reducers). Просто меньше boilerplate.
  2. Три боли, которые решает RTK дословно из доки: "Configuring a Redux store is too complicated", "I have to add a lot of packages", "Redux requires too much boilerplate code".
  3. Три точки входа: @reduxjs/toolkit (core), @reduxjs/toolkit/query (createApi без React), @reduxjs/toolkit/query/react (createApi с auto-generated хуками). Разделены, чтобы RTK работал в Node/Svelte/Vue без peer react.
  4. Внутри RTK: redux, immer, reselect, redux-thunk (все как dependencies). react и react-redux — peer dependencies, нужно ставить отдельно.
  5. Bundle: core ≈ 17 kB min+gzip. Полный RTK + RTK Query + хуки ≈ 28 kB. Классический стек (redux+thunk+reselect+immer+react-redux) ≈ 22.5 kB — разница всего 5-6 kB в обмен на x3 меньше кода.

Шпаргалка импортов

// Core — всё кроме createApi import { configureStore, createSlice, createAsyncThunk, createEntityAdapter, createListenerMiddleware, createSelector, nanoid } from '@reduxjs/toolkit' // RTK Query без React import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query' // RTK Query С автогенерацией хуков import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react' // → api.useGetPostsQuery, api.useAddPostMutation
