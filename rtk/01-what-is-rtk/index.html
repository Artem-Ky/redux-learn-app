Тот же counter — классика vs RTK
Classic Redux
import { createStore, combineReducers, applyMiddleware, compose } from 'redux'
import thunk from 'redux-thunk'

const INCREMENT = 'counter/incremented'
const DECREMENT = 'counter/decremented'

const increment = () => ({ type: INCREMENT })
const decrement = () => ({ type: DECREMENT })

const initial = { value: 0 }

function counterReducer(state = initial, action) {
  switch (action.type) {
    case INCREMENT: return { ...state, value: state.value + 1 }
    case DECREMENT: return { ...state, value: state.value - 1 }
    default: return state
  }
}

const rootReducer = combineReducers({ counter: counterReducer })
const composeEnhancers = window.__REDUX_DEVTOOLS_EXTENSION_COMPOSE__ || compose
const store = createStore(rootReducer, composeEnhancers(applyMiddleware(thunk)))

Redux Toolkit
import { configureStore, createSlice } from '@reduxjs/toolkit'

const counterSlice = createSlice({
  name: 'counter',
  initialState: { value: 0 },
  reducers: {
    increment: (state) => { state.value++ },
    decrement: (state) => { state.value-- },
  },
})

export const { increment, decrement } = counterSlice.actions

const store = configureStore({
  reducer: { counter: counterSlice.reducer },
})

Теория — Что такое Redux Toolkit?

Определение

Redux Toolkit (RTK) — это официальный, мнения-имеющий, всё-включающий набор инструментов для эффективной разработки на Redux. Это рекомендованный способ писать Redux-логику сегодня. Раньше в команду Redux входил пакет redux — сегодня в команду входит @reduxjs/toolkit.

«The Redux Toolkit package is intended to be the standard way to write Redux logic. It was originally created to help address three common concerns about Redux: "Configuring a Redux store is too complicated", "I have to add a lot of packages to get Redux to do anything useful", "Redux requires too much boilerplate code".»
— Getting Started — Purpose
«Пакет Redux Toolkit задуман как стандартный способ писать Redux-логику. Он был создан, чтобы решить три типичные жалобы на Redux: «Настройка Redux store слишком сложная», «Чтобы получить от Redux хоть какую-то пользу, приходится ставить много пакетов», «Redux требует слишком много шаблонного кода».»

Три проблемы, которые решает RTK

  1. Сложная настройка store. В классике нужно вручную собрать combineReducers, applyMiddleware, composeEnhancers, подключить DevTools — 20-30 строк копипасты. RTK вместо этого предоставляет один вызов configureStore, который делает всё за вас и даёт хорошие defaults.
  2. Много отдельных пакетов. Раньше нужны были redux, redux-thunk, reselect, immer, redux-devtools-extension. Сейчас всё это входит в один @reduxjs/toolkit и реэкспортируется (например createSelector — это re-export из reselect).
  3. Boilerplate. Константы action types, action creators-обёртки, switch-reducer, spread-каскады для вложенных обновлений. RTK генерирует action types и creators автоматически из createSlice и позволяет писать reducer'ы в "мутирующем" стиле через Immer.

Что НЕ изменилось

RTK — это обёртка поверх классического Redux. Он не заменяет идеи Redux: action → dispatch → reducer → store → subscribers. Тот же однонаправленный поток данных, те же три принципа (Single Source of Truth, State is Read-Only, Changes via Pure Functions). Просто меньше ручного кода вокруг тех же концепций.

RTK Query как бонус

Внутри @reduxjs/toolkit также есть RTK Query — отдельный модуль для data fetching и кэширования. Доступен через точку входа @reduxjs/toolkit/query/react. Опциональный: можно использовать только core RTK без него. Подробно мы разберём его в секциях K и L.

«Redux Toolkit also includes a powerful data fetching and caching capability that we've dubbed "RTK Query". It's included in the package as a separate set of entry points. It's optional, but can eliminate the need to hand-write data fetching logic yourself.»
— Getting Started
«Redux Toolkit также включает мощный механизм загрузки и кеширования данных, который мы назвали «RTK Query». Он входит в пакет как отдельный набор точек входа. Это опционально, но позволяет полностью отказаться от ручного написания логики загрузки данных.»

Кому полезно

  • Новичкам — RTK не имеет смысла учить отдельно; начинать рекомендуется сразу с него.
  • Существующим Redux-проектам — миграция инкрементальная (см. урок 77).
  • Любому, кто пишет Redux в 2024+ — это де-факто стандарт.
