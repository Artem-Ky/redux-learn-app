Теория — Что именно убирает RTK

Что убрано в RTK-варианте

  1. Action types — нет констант const INCREMENT = 'counter/INCREMENT'. createSlice генерирует имена сам: ${'counter/increment'}.
  2. Action creators — нет вручную написанных const increment = () => ({ type: ... }). createSlice возвращает их в slice.actions.
  3. switch в reducer — нет switch (action.type) { case INCREMENT: return ... }. Каждый ключ в reducers: { ... } — это case.
  4. Spread/copy для immutability — нет return { ...state, value: state.value + 1 }. Immer внутри createSlice позволяет писать state.value++.
  5. combineReducers вручную — configureStore делает это сам если передан объект.
  6. applyMiddleware(thunk) — thunk включён по умолчанию.
  7. Redux DevTools Extension setup — configureStore подключает devtools автоматически в development.

Что осталось одинаковым

  • Концепция store / action / reducer — RTK не меняет паттерн.
  • store.dispatch(action) — точно так же, action — обычный объект {type, payload}.
  • store.subscribe(listener) — подписка на изменения.
  • В React: <Provider store={store}>, useSelector, useDispatch.

Цитата из доки

«createSlice() automatically generates action creators and action types that correspond to the reducers and state. The Redux state for the slice will be added to the Redux store using the slice name as the key.»
— createSlice docs
«createSlice() автоматически генерирует action creators и action types, соответствующие reducer'ам и state. Redux state этой slice будет добавлен в store с использованием name slice в качестве ключа.»

Что произойдёт в DevTools

В classic-версии вы увидите actions с типом counter/INCREMENT (в строгом верхнем кейсе, как принято в classic-стиле). В RTK-версии — counter/increment (camelCase ключ slice). Сама структура state одинаковая: { counter: { value: 0 } }.

Подводные камни при переходе

Не пишите классические action types в createSlice. Если вы хотите слушать «свой» action из другой slice — используйте extraReducers. Если хотите СВОЙ имя action — используйте createAction отдельно. createSlice генерирует имена сам и переопределить их нельзя.

Вывод

RTK не заменяет Redux. Это удобная обёртка над теми же концепциями. Все знания про reducer/action/store остаются актуальными — RTK просто убирает 70% повторяющегося кода. В следующих секциях разберём как именно это делается под капотом.
