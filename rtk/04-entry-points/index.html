3 точки входа — разные импорты

Один пакет @reduxjs/toolkit — три точки входа. Каждая экспортирует свой набор. Подходящую выбирают по тому, нужен ли RTK Query и используете ли вы React.

@reduxjs/toolkit
Core RTK без RTK Query
  • configureStore
  • createSlice
  • createReducer
  • createAction
  • createAsyncThunk
  • createEntityAdapter
  • combineSlices
  • createSelector
  • createListenerMiddleware
  • nanoid, isAnyOf, Tuple, ...
  • autoBatchEnhancer, prepareAutoBatched
@reduxjs/toolkit/query
RTK Query без React-хуков
  • createApi
  • fetchBaseQuery
  • setupListeners
  • retry
  • buildCreateApi (advanced)
  • coreModule (advanced)

Для vanilla JS / других UI-фреймворков (Svelte, Vue, ванильный JS).

@reduxjs/toolkit/query/react
RTK Query с auto-generated React hooks
  • createApi (с reactHooksModule)
  • fetchBaseQuery
  • setupListeners
  • retry
  • ApiProvider
  • useDispatch, useSelector (re-export react-redux)

Главное отличие: createApi из этого entry автоматически генерирует use*Query / use*Mutation хуки для каждого endpoint.

@reduxjs/toolkit/query/react
+ react bindings + auto hooks
@reduxjs/toolkit/query
+ createApi + fetchBaseQuery + setupListeners
@reduxjs/toolkit
core: configureStore + slice + thunk + entity + listener + ...

Теория — Архитектура entry points

Дословно из доки

«RTK Query is included within the installation of the core Redux Toolkit package. It is available via either of the two entry points: @reduxjs/toolkit/query or @reduxjs/toolkit/query/react (React-specific entry point that automatically generates hooks corresponding to the defined endpoints).»
— RTK Query Overview
«RTK Query поставляется вместе с core-пакетом Redux Toolkit. Он доступен через одну из двух точек входа: @reduxjs/toolkit/query или @reduxjs/toolkit/query/react (React-специфичная точка входа, автоматически генерирующая хуки для каждого определённого endpoint).»

Почему не один entry?

Если бы хуки React всегда экспортировались из @reduxjs/toolkit, то react стал бы обязательной peer-зависимостью. А @reduxjs/toolkit можно использовать в Node, Svelte, Vue, ванильном JS — и react там не нужен. Поэтому React-специфичная функциональность вынесена в отдельный entry /query/react, который импортирует react только если вы его используете.

Что значит "/query/react генерирует хуки"

Когда вы делаете createApi(...) из @reduxjs/toolkit/query/react — RTK подключает специальный модуль reactHooksModule, который для каждого endpoint вашего API генерирует React-хук с именем по правилу:

  • build.query({ ... }) с ключом getPosts → useGetPostsQuery
  • build.mutation({ ... }) с ключом addPost → useAddPostMutation
  • build.infiniteQuery(...) с ключом getMessages → useGetMessagesInfiniteQuery

Если же вы импортируете createApi из @reduxjs/toolkit/query (без /react) — этих хуков просто не будет. Будут только api.endpoints.getPosts.initiate(arg) (для ручного dispatch), api.endpoints.getPosts.select(arg) (selector) и middleware.

Источник: package.json

В packages/toolkit/package.json объявлено три "exports":

"exports": { ".": { /* core */ "import": "./dist/redux-toolkit.modern.mjs" }, "./query": { "import": "./dist/query/rtk-query.modern.mjs" }, "./query/react": { "import": "./dist/query/react/rtk-query-react.modern.mjs" } }

Подводный камень

Не смешивайте! Если у вас один проект использует одновременно createApi из /query и из /query/react — это два разных набора функций (с разными reactHooksModule-привязками). Лучше всегда использовать один entry для всего проекта. Также configureStore при duplicateMiddlewareCheck: true (default) выдаст warning, если случайно подключили middleware от двух разных API.

Что использовать?

  • React-проект с RTK Query — почти всегда @reduxjs/toolkit + @reduxjs/toolkit/query/react.
  • React-проект без RTK Query — только @reduxjs/toolkit.
  • Не-React проект (Node, Svelte, Vue) с RTK Query — @reduxjs/toolkit + @reduxjs/toolkit/query.
