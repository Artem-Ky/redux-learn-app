4 default middleware. Включайте/выключайте каждый

configureStore автоматически подключает 4 middleware. В development — все 4. В production — только thunk (invariants отключены ради производительности). Внизу — кастомизация через callback (getDefaultMiddleware) => getDefaultMiddleware({...}).

1
thunk
Позволяет dispatch функций. Для async-логики и доступа к getState.
DEV + PROD
2
immutableStateInvariantMiddleware
Глубокое сравнение state до/после reducer. Throw если state мутирован.
DEV ONLY
3
serializableStateInvariantMiddleware
Проверяет action.payload и state на не-сериализуемые значения (Date, Map, Promise).
DEV ONLY
4
actionCreatorInvariantMiddleware
Throw если вы dispatch'нули actionCreator (функцию) вместо action (объекта).
DEV ONLY
Теория — getDefaultMiddleware изнутри

Из доки

«getDefaultMiddleware returns an array containing the default list of middleware included with the store created by configureStore. By default, this includes: thunk, immutableStateInvariant, serializableStateInvariant, and actionCreatorInvariant (in development mode only).»
— getDefaultMiddleware API
«getDefaultMiddleware возвращает массив со списком middleware, которые по умолчанию включены в store, созданный configureStore. По умолчанию это: thunk, immutableStateInvariant, serializableStateInvariant и actionCreatorInvariant (последние три — только в development).»

Кастомизация через callback

Опция middleware в configureStore — это функция, которой передают getDefaultMiddleware. Вы возвращаете Tuple из middleware.

// Добавить свой middleware к default configureStore({ reducer, middleware: (getDefaultMiddleware) => getDefaultMiddleware().concat(myLogger), }) // Поставить свой middleware ДО default configureStore({ reducer, middleware: (getDefaultMiddleware) => getDefaultMiddleware().prepend(listenerMiddleware.middleware), }) // Отключить отдельные default middleware через опции configureStore({ reducer, middleware: (getDefaultMiddleware) => getDefaultMiddleware({ thunk: false, // убрать thunk полностью immutableCheck: false, // убрать immutable check serializableCheck: { ignoredActions: ['auth/setUser'] } }), })

Варианты опций каждого middleware

  • thunk: false или thunk: { extraArgument: api }
  • immutableCheck: false или { warnAfter: 32, ignoredPaths: ['some.path'] }
  • serializableCheck: false или { ignoredActions, ignoredActionPaths, ignoredPaths, warnAfter, isSerializable }
  • actionCreatorCheck: false

Production vs development

В production-сборке (process.env.NODE_ENV === 'production') RTK автоматически исключает immutable/serializable/action invariant — ради скорости. В бандл попадает только thunk. Это не нужно настраивать вручную.

Подводный камень: warnAfter

immutable и serializable middleware рекурсивно обходят весь state. Если у вас большой state, это может занимать несколько мс на каждый dispatch. По умолчанию warnAfter: 32 ms. При превышении — warning в консоль. Решения: ignoredPaths, увеличить warnAfter, или отключить middleware для конкретного slice.

Подводный камень: Tuple, не Array

В TypeScript метод concat на обычном массиве сужает тип до Middleware[] и теряет информацию о типах thunk/extra. RTK 2.x ввёл Tuple — расширение массива с правильными типами concat/prepend. Если вы делаете middleware: () => [thunk, myMw] — TS выдаст ошибку. Используйте Tuple или getDefaultMiddleware().concat(...).
