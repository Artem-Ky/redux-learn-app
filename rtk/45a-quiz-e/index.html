Пять правил, которые нужно запомнить

  1. createSelector кэширует по ссылке аргументов input-селекторов. Если вернёшь новый массив/объект — кеш всегда miss.
  2. Начиная с reselect v5, дефолтный memoizer и у createSelector, и у createDraftSafeSelector — weakMapMemoize (дерево WeakMap, N-уровней по аргументам).
  3. createDraftSafeSelector оборачивает селектор так: isDraft(value) ? current(value) : value. Это нужно, если селектор вызывается внутри reducer'а (куда приходит draft). Обычный createSelector закеширует draft-Proxy и будет отдавать stale значение.
  4. Параметризованный селектор selectById(state, id) на weakMapMemoize имеет эффективно бесконечный кеш — каждый id получает свою ветку. На дефолтном lruMemoize({size:1}) — постоянный miss при переборе id.
  5. useAppSelector, useAppDispatch через withTypes<S,D>() — это чисто TS-обёртка, identity в рантайме. Никакой валидации.

Где использовать что

// 1. Обычный селектор, один аргумент state const selectAll = createSelector([s => s.items], items => items.filter(...)) // 2. Параметризованный — передаём id const selectById = createSelector( [(s: RootState) => s.items, (_: RootState, id: string) => id], (items, id) => items.find(t => t.id === id) ) // ← на default lruMemoize(size:1) развалится при разных id в одном рендере. // Лечится: передать opt { memoize: weakMapMemoize } или сделать factory. // 3. Внутри reducer'а (редкий кейс!) reducers: { recompute: (state) => { state.avg = selectAvg(state) // ← selectAvg ДОЛЖЕН быть createDraftSafeSelector } }
