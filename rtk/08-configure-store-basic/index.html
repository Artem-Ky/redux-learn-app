Теория — что такое configureStore

Сигнатура

function configureStore<S, A, M, E>(options: ConfigureStoreOptions<S, A, M, E>): EnhancedStore

Из документации

«A friendly abstraction over the standard Redux createStore function that adds good defaults to the store setup for a better development experience.»
— configureStore API
«Дружелюбная абстракция над стандартной Redux-функцией createStore, которая добавляет хорошие умолчания при настройке store ради лучшего опыта разработки.»

Что configureStore делает за вас (5 пунктов)

1. combineReducers — если вы передали объект-карту slice'ов, он сам соединит их в rootReducer
2. applyMiddleware — добавит default middleware (thunk + 3 invariant)
3. composeWithDevTools — подключит Redux DevTools Extension в development
4. autoBatchEnhancer — добавит batching subscribers
5. preloadedState — поддержит hydration из localStorage / SSR
6. duplicateMiddlewareCheck — предупредит про повторное подключение того же middleware

Все опции configureStore

interface ConfigureStoreOptions { reducer: Reducer | ReducersMapObject // функция или объект-карта middleware?: (getDefaultMiddleware) => Tuple // callback с дефолтами devTools?: boolean | DevToolsOptions // включить/настроить devtools preloadedState?: DeepPartial<State> // начальное состояние enhancers?: (getDefaultEnhancers) => Tuple // доп. enhancers }

Источник

packages/toolkit/src/configureStore.ts — здесь живёт реализация. Подробно разберём в уроке 09.

Подводный камень

Не вызывайте configureStore несколько раз для одного приложения. Один store на приложение — это ключевой принцип Redux. Если нужны несколько изолированных state — используйте combineSlices или несколько Provider с одним store, разделяя ключи.
