Что тут происходит

Долгий «fetch» (2 секунды). Кнопка Start запускает, кнопка Cancel вызывает promise.abort(). Thunk получает AbortError, в DevTools появляется rejected с meta.aborted === true. Второй кейс — внешний AbortController, пробрасывается через dispatch(thunk(arg, { signal: externalController.signal })).

Три способа отмены

// Способ 1. Вернуть promise от dispatch и вызвать .abort() const promise = store.dispatch(fetchUser(1)) promise.abort('user left page') // Способ 2. thunkAPI.abort() изнутри (самоотмена) const fetchUser = createAsyncThunk('user/fetch', async (id, { abort, signal }) => { if (id < 0) abort('invalid id') return await fetch(`/api/${id}`, { signal }) }) // Способ 3. Внешний AbortController — пробрасываем signal в config-object const controller = new AbortController() store.dispatch(fetchUser(1, { signal: controller.signal })) // ← второй аргумент! controller.abort()

Что получается в reducer

b.addCase(fetchUser.rejected, (s, a) => { if (a.meta.aborted) { // aborted === true + error.name === 'AbortError' s.canceled = true } else if (a.meta.condition) { // см. урок 59 } else { s.error = a.error.message ?? 'unknown' } })

Подводные камни

1. Signal нужно передать в fetch САМОМУ. RTK не может заглянуть внутрь вашего fetch/axios. Без { signal } сеть продолжит работать, даже если rejected-action уже задиспатчен. Это утечка: ответ придёт, вы его проигнорируете, но трафик ушёл.
2. abort() идемпотентен. Повторный вызов — no-op. AbortController спокойно принимает несколько .abort(). Удобно для cleanup в React-компонентах.
3. После fulfilled abort — ничего не даст. Race выиграл payload, rejected(aborted) НЕ будет задиспатчен. Не пытайтесь отменить завершённый thunk — это silent no-op (правильное поведение, но первое, что ломает голову новичку).
4. External signal — config-object, не второй аргумент thunk'а. dispatch(thunk(arg, { signal })) — это сигнатура AsyncThunkDispatchConfig. Её часто путают с thunkApiConfig.
5. DOMException vs Error. В браузере canonical: new DOMException('Aborted', 'AbortError'). В Node 20+ — тоже. Но new Error() с err.name = 'AbortError' — тоже считается (RTK проверяет name === 'AbortError').
