Unmount — обратный путь

1. Component unmount → useEffect cleanup → promise.unsubscribe() 2. Dispatch unsubscribeQueryResult → refCount-- 3. Если refCount === 0 → setTimeout(removeQueryResult, keepUnusedDataFor * 1000) 4. Если таймер догорел → dispatch(removeQueryResult) 5. Reducer: delete state.queries[cacheKey], delete state.provided.* 6. cacheEntryRemoved.resolve() — твой onCacheEntryAdded продолжает → ws.close()

buildInitiate.ts — ключевое решение fetch vs cache

// упрощённо из buildInitiate.ts const initiate = (queryArgs, { subscribe = true, forceRefetch, subscriptionOptions }) => (dispatch, getState) => { const cacheKey = serializeQueryArgs({ endpointName, queryArgs }) const stateBefore = getState()[reducerPath].queries[cacheKey] let shouldFetch = stateBefore === undefined || stateBefore.status === 'uninitialized' || forceRefetch === true || (typeof forceRefetch === 'number' && stateBefore.fulfilledTimeStamp && (Date.now() - stateBefore.fulfilledTimeStamp) / 1000 >= forceRefetch) || stateBefore.status === 'rejected' if (shouldFetch) { const thunk = queryThunk({ queryArgs, endpointName, subscribe, ... }) return dispatch(thunk) // запускаем fetch } if (subscribe) { dispatch(subscribeQueryResult({ queryCacheKey: cacheKey, requestId, options })) } return selectFromResult || selectByCache(cacheKey) // cached }

buildSlice.ts — reducer queries

// упрощённо queries: createReducer({}, (builder) => { builder .addCase(queryThunk.pending, (draft, action) => { const { arg: { queryCacheKey, originalArgs, endpointName } } = action.meta draft[queryCacheKey] = { status: 'pending', endpointName, originalArgs, requestId: action.meta.requestId, startedTimeStamp: action.meta.startedTimeStamp, } }) .addCase(queryThunk.fulfilled, (draft, action) => { const entry = draft[action.meta.arg.queryCacheKey] entry.status = 'fulfilled' entry.data = action.payload entry.fulfilledTimeStamp = Date.now() }) .addCase(queryThunk.rejected, (draft, action) => { const entry = draft[action.meta.arg.queryCacheKey] entry.status = 'rejected' entry.error = action.payload ?? action.error }) .addCase(removeQueryResult, (draft, action) => { delete draft[action.payload.queryCacheKey] }) })

Подводные камни понимания

1. Subscribe vs fetch — разные концепции.
Subscribe — «я подписчик, не удаляй entry». Fetch — «запроси данные». Subscribe без fetch = cache HIT. Fetch без subscribe = prefetch.

2. cacheKey — это endpoint + args, не endpoint + requestId.
RequestId уникален на каждый fetch, cacheKey — на каждый уникальный набор args. Нельзя сверять по requestId для «какой компонент заказал» — по cacheKey.

3. onQueryStarted запускается ПАРАЛЛЕЛЬНО с fetch, не после.
Поэтому в нём можно делать optimistic update. Ждать завершения — через await queryFulfilled.

4. cacheDataLoaded.resolve один раз — на первый fulfilled.
Повторные fulfilled (refetch) — НЕ триггерят её снова. Это именно «первые данные в cache».

5. Двух identical-arg useQuery — два subscription, один fetch.
Это дедупликация на уровне cacheKey. Но если один делает refetch(), второй автоматически получит новые данные.

6. Все actions RTKQ имеют префикс {reducerPath}/.
Default: api/. В middleware сверяешь по type.startsWith('api/') или через matchers (api.endpoints.X.matchFulfilled).
