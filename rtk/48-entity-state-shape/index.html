Анимация: ids и entities всегда меняются в паре

Слева — ids[] (порядок). Справа — entities (Record<id, T>). При любом CRUD-действии оба поля обновляются атомарно (это и есть инвариант нормализации). Нажимай кнопки и смотри, как pill'ы и карточки вспыхивают одновременно.

state.users.ids: string[] 0
state.users.entities: Record<string, User> 0
Теория — 48. Форма EntityState<T, Id>

Тип из packages/toolkit/src/entities/models.ts

// https://github.com/reduxjs/redux-toolkit/blob/master/packages/toolkit/src/entities/models.ts export type EntityId = number | string export type Comparer<T> = (a: T, b: T) => number export type IdSelector<T, Id> = (model: T) => Id export type Update<T, Id> = { id: Id; changes: Partial<T> } export interface EntityState<T, Id extends EntityId> { ids: Id[] entities: Record<Id, T> }

Инициализация — entity_state.ts

export function getInitialEntityState<T, Id extends EntityId>(): EntityState<T, Id> { return { ids: [], entities: {} as Record<Id, T>, } }

GitHub: entities/entity_state.ts

Инвариант

  • ids.length === Object.keys(entities).length — всегда равны после CRUD.
  • Порядок в ids определяет, в каком порядке selectAll возвращает массив.
  • Для любого id из ids гарантированно есть entities[id].

Цитата — Redux docs (Normalizing State Shape)

«Our state now has an ‘entities’ slice that contains the actual data, with the individual items stored by ID, and an ‘ids’ array that represents the ‘current’ set of items.»
— Designing a Normalized State
«Наш state теперь содержит срез entities с самими данными по id и массив ids, который представляет текущий порядок/набор элементов.»

Подводные камни

1. entities — это Record, а не Map.
RTK использует plain object. Это значит: ключи будут строками (JavaScript coerce), даже если вы передали number. В типах Id может быть number, но внутри: entities['42'] === entities[42]. Из-за этого нельзя использовать в качестве id объекты или символы — только string | number (type EntityId = number | string).

2. Branded id (nominal types). Если вы хотите type-safe id вроде type BookId = string & { __brand: 'BookId' } — используйте второй generic:
type BookId = string & { readonly __brand: 'BookId' } const booksAdapter = createEntityAdapter<Book, BookId>({ selectId: (b) => b.isbn as BookId, // selectId ОБЯЗАТЕЛЕН при 2-м generic'е })
Первая перегрузка createEntityAdapter<T, Id> делает selectId required, вторая (только <T>) — нет, но требует T extends { id: EntityId }.

3. Не мутируйте entities вручную. Если забыть и добавить ключ в entities, не добавив в ids — selectAll не увидит новую запись (он итерирует ids). Всегда используйте adapter-методы.
