Immer не копирует то, чего не трогал

Когда вы dispatch(action), Immer запускается ТОЛЬКО на той slice, чей reducer отработал. Остальные slices в RootState получают ту же ссылку. Это значит: useSelector(s => s.untouchedSlice) вернёт старое === → React не ререндерит. Это бесплатный кэш — вы ничего не настраивали, он работает всегда.

Нажмите «users/added» — только у state.users ссылка поменяется, у остальных останется. Нажмите «users/noop» — Immer увидит, что ничего не мутировали в draft'е, и вернёт оригинал. Ссылка не изменится даже у state.users.

Теория — 70b. Почему это кэш

Ментальная модель

Ваш RootState — это дерево. После dispatch'а:

  • Если reducer slice не отработал — ссылка на эту slice такая же, что и раньше.
  • Если reducer отработал, но ничего не изменил в draft — Immer вернёт оригинал. Ссылка та же.
  • Если reducer изменил одно поле в глубине — Immer скопирует только путь до этого поля. Всё остальное — по старым ссылкам.

Что из этого следует для кэша

// Компонент А читает только users: function UsersList() { const users = useSelector(s => s.users) // будет === пока не было users/* action ... } // Компонент B читает только todos: function TodosList() { const todos = useSelector(s => s.todos) // === пока не было todos/* action ... } // dispatch(users/added) → useSelector UsersList пересчитает, TodosList — нет. // Это бесплатный кэш, встроенный в модель Immer.

Что ломает structural sharing

1. Новый массив каждый раз (s => s.users.map(u => u.name)) — это уже Layer 2, memoized selector нужен. Каждый dispatch любой slice'а даст новый массив, компонент ререндерит.
2. Inline объект в selector'е (s => ({ name: s.user.name, isAdmin: s.user.role === 'admin' })) — та же проблема: новый объект каждый раз. Либо createSelector, либо shallowEqual в useSelector.
3. Прямая мутация без Immer (state.users.push(x); return state) — ссылка та же, но state изменился. React не заметит изменений. Это баг.
4. Замена неизменённого объекта (state.users = [...state.users] без причины) — ссылка новая, structural sharing сломан. Лишние ререндеры.

Структура Immer'а

// Псевдокод для понимания: function produce(base, recipe) { const draft = createProxy(base) recipe(draft) if (!wasModified(draft)) return base // ← та же ссылка! return finalize(draft) // ← новая ссылка, но shared sub-trees }

Рассуждение: selector — почему === достаточно в 80% случаев

useSelector(s => s.users) — простая форма. Она работает когда:
• Вы возвращаете ссылку на уже существующую часть state (не новый объект).
• State мутируется через Immer (который хранит structural sharing invariant).

Как только вы трансформируете данные (filter/map/merge), === ломается — нужен Layer 2 (memoized selectors, следующий урок).

В RTKQ

Layer 1 остаётся полностью. RTKQ хранит свой cache в обычном slice (state.api.queries), и structural sharing работает там точно так же. Поэтому useGetFooQuery().data — стабильная ссылка, если ответ не изменился. selectFromResult из RTKQ как раз использует этот принцип.
