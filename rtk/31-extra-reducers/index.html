Теория — extraReducers

Из доки

«extraReducers allows the slice to handle action types defined elsewhere in the application, including other createSlice-generated actions, createAsyncThunk actions, or actions from external libraries.»
— extraReducers API
«extraReducers позволяет slice'у обрабатывать типы actions, определённые где-то ещё в приложении: включая actions из других createSlice, createAsyncThunk, или из внешних библиотек.»

Сигнатура

const slice = createSlice({ name: 'feature', initialState, reducers: { ... }, // СВОИ actions extraReducers: (builder) => { // ЧУЖИЕ actions builder .addCase(externalAction, handler) .addCase('literal/string/type', handler) .addMatcher(matcher, handler) .addDefaultCase(handler) } })

Builder тот же, что в createReducer (уроки 24-25).

Зачем extraReducers нужен

  1. Cross-slice events: userLoggedOut сбрасывает auth, cart, profile.
  2. createAsyncThunk integration: thunk имеет 3 actions (pending/fulfilled/rejected), их слушаете в slice через extraReducers.
  3. Глобальные ошибки: addMatcher(isRejected, handler) — единое место для error handling.
  4. Reset при init: action '@@INIT' или 'app/reset'.

extraReducers vs reducers — сравнение

// reducers — генерирует actions reducers: { add: (s, a) => { ... } } // → создаётся slice.actions.add (creator) и type 'feature/add' // extraReducers — НЕ генерирует actions extraReducers: (builder) => { builder.addCase(externalAction, (s, a) => { ... }) } // → ничего не создаётся, slice только реагирует на externalAction

Старая object-form deprecated

В RTK 1.x был старый синтаксис extraReducers: { [actionType]: handler }. В RTK 2.x он удалён. Используйте только builder-callback.

Типичный паттерн с createAsyncThunk

const fetchUser = createAsyncThunk('user/fetch', async (id) => { const r = await fetch(`/api/users/${id}`) return r.json() }) const userSlice = createSlice({ name: 'user', initialState: { data: null, loading: false, error: null }, reducers: {}, extraReducers: (builder) => { builder .addCase(fetchUser.pending, (s) => { s.loading = true; s.error = null }) .addCase(fetchUser.fulfilled, (s, a) => { s.loading = false; s.data = a.payload }) .addCase(fetchUser.rejected, (s, a) => { s.loading = false; s.error = a.error.message }) } })

Что нельзя

В extraReducers нельзя ссылаться на slice.actions.xxx — slice ещё не построен! Внешние actions должны быть определены ДО slice (или в другом файле).

extraReducers И reducers — порядок

createSlice сначала добавляет в builder addCase из reducers, потом из extraReducers. Если у вас в обоих местах addCase для одного action.type — сработает последний (extraReducers), но это невероятный сценарий: actions в reducers и extraReducers всегда разные.
