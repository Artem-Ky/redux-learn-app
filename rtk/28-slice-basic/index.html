createSlice — три обязательных поля

createSlice объединяет actions + reducer + initialState в один объект. Это главный API RTK для определения state-логики. 90% всего написанного на RTK использует createSlice.

import { createSlice } from '@reduxjs/toolkit' const counterSlice = createSlice({ name: 'counter', // 1. Префикс для всех action.type initialState: { value: 0, history: [] }, // 2. Начальный state reducers: { // 3. Map reducer-функций increment: (state) => { state.value++ }, decrement: (state) => { state.value--; state.history.push('-') }, addBy: (state, action) => { state.value += action.payload }, reset: () => ({ value: 0, history: [] }) // можно и return } }) // На выходе: объект с этими полями ↓ counterSlice.name // 'counter' counterSlice.actions // { increment, decrement, addBy, reset } counterSlice.reducer // (state, action) => newState counterSlice.caseReducers // { increment: fn, decrement: fn, ... } — оригинальные функции counterSlice.getInitialState() // { value: 0, history: [] }

Теория — createSlice

Из доки

«createSlice accepts an object of reducer functions, a slice name, and an initial state value, and automatically generates a slice reducer with corresponding action creators and action types.»
— createSlice API
«createSlice принимает объект reducer-функций, имя slice'а и начальное состояние и автоматически генерирует slice-reducer с соответствующими action creator'ами и action type'ами.»

Сигнатура (упрощённо)

interface CreateSliceOptions<S, R, N> { name: N // уникальный префикс initialState: S | (() => S) // state или lazy-функция reducers: R // { [name]: caseReducer | { reducer, prepare } } extraReducers?: (builder) => void // слушать чужие actions (урок 31) selectors?: { [name]: (state) => T } // встроенные селекторы (урок 33) reducerPath?: string // для combineSlices (урок 37) }

Что генерируется автоматически

  1. action.type = ${name}/${reducerKey} (например 'counter/increment').
  2. action creator для каждого ключа в reducers (доступен в slice.actions).
  3. reducer — функция (state, action) => newState.
  4. caseReducers — оригинальные функции, не обёрнутые в Immer (для тестов).
  5. match на каждом action creator (как у createAction).

Внутри: createSlice ≈ createReducer + createAction[]

// createSlice внутри делает примерно следующее (упрощённо): function createSlice({ name, initialState, reducers }) { const actions = {} const caseReducers = {} for (const [key, caseReducer] of Object.entries(reducers)) { const type = `${name}/${key}` actions[key] = createAction(type) caseReducers[key] = caseReducer } const reducer = createReducer(initialState, (b) => { for (const [key, caseReducer] of Object.entries(caseReducers)) { b.addCase(actions[key], caseReducer) } }) return { name, actions, reducer, caseReducers, getInitialState: () => initialState } }

createSlice.ts (полный исходник) — там около 600 строк, потому что есть extraReducers, selectors, lazy injection и куча TS-магии.

Подключение в store

const store = configureStore({ reducer: { counter: counterSlice.reducer, // ключ → state.counter user: userSlice.reducer, // ключ → state.user } }) // Использование const { increment, addBy } = counterSlice.actions store.dispatch(increment()) store.dispatch(addBy(5))

Подводный камень: name должен быть уникальным

name используется как префикс action.type. Если у двух slice'ов одинаковый name — actions будут конфликтовать. Соглашение: name = ключ в store ('counter' ↔ state.counter).

initialState может быть функцией

const slice = createSlice({ name: 'session', initialState: () => ({ // lazy sessionId: nanoid(), startedAt: Date.now(), }), reducers: { ... } })

Функция вызывается один раз при первом @@INIT. Полезно если initialState зависит от side-effects (timestamp, localStorage).
