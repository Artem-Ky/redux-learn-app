«Я забыл скобки» — частая ошибка

Это самый молодой default middleware (добавлен в RTK 1.7). Ловит одну конкретную ошибку: dispatch самой функции actionCreator вместо результата её вызова. Без него reducer молча игнорирует action (потому что type === undefined), и баг сложно заметить.

1. Забыли скобки

// неправильно
dispatch(increment)    // ← без ()
// reducer молча скипает

2. Корректный вызов

// правильно
dispatch(increment())  // ← со ()
// → { type: 'counter/increment' }

3. bindActionCreators забыт

// при connect используют map
const mapDispatch = { increment }
// → бинды НЕ применены, dispatch получает функцию

4. .toString() для использования

// actionCreator имеет toString() → 'counter/increment'
[increment.type]: handler  // в objectReducer
[increment]: handler        // тоже работает
— ожидание dispatch —
Теория — actionCreatorInvariant

Из доки

«actionCreatorInvariantMiddleware checks to see if dispatch was called with an action creator (a function returned from createAction) instead of the action itself. This is a common mistake, since action creators look like simple functions.»
— actionCreatorCheck docs
«actionCreatorInvariantMiddleware проверяет, не был ли dispatch вызван с самим actionCreator (функцией, возвращённой из createAction) вместо action. Это типичная ошибка, потому что actionCreator выглядит как обычная функция.»

Реализация (исходник)

// actionCreatorInvariantMiddleware.ts (упрощённо) export const createActionCreatorInvariantMiddleware = (options) => { return () => (next) => (action) => { if (isActionCreator(action)) { const msg = `Detected an action creator with \`type\` property being dispatched, instead of an action object! ...` console.warn(msg) } return next(action) } } // isActionCreator — проверяет наличие .type на функции function isActionCreator(action: unknown): action is ActionCreator { return ( typeof action === 'function' && 'type' in action && typeof (action as { type?: unknown }).type === 'string' ) }

Поведение зависит от версии: в современных RTK middleware пишет console.warn, но проходит action дальше. В redux ванильном такой dispatch вызовет ошибку «Actions must be plain objects. Use custom middleware for async actions.»

Опции

interface ActionCreatorInvariantMiddlewareOptions { isActionCreator?: (action: unknown) => boolean // можно подменить функцию-проверку, если у вас свои actionCreators }

Почему это важно

Без middleware баг с забытыми скобками не вызывает ошибку: action.type === undefined, reducer его не обрабатывает, state не меняется. Вы видите «приложение не реагирует» и часами ищете причину. Middleware говорит сразу: «вы передали функцию, не action».

Подводный камень: thunk и actionCreatorInvariant

У thunk-actions тоже есть .type (например, fetchTodos.fulfilled.type). Но thunk сам как функция — без .type. Поэтому middleware не путает их с обычными actionCreators. Если же вы делаете dispatch(fetchTodos.fulfilled) вместо dispatch(fetchTodos.fulfilled({...})) — middleware поймает.

Совмещение с thunk

Порядок default middleware: actionCreatorInvariant → immutable → serializable → thunk. Сначала проверки, потом thunk обрабатывает функции. Если actionCreator проходит — он считается функцией, и thunk попытается её выполнить. Это и происходит без middleware: «thunk выполнил вашу функцию, ничего не изменилось».
