Динамическое расширение api — endpoints из chunks

Стартуем с пустого createApi. Жми «загрузить feature» — динамический import() модуля, который вызывает api.injectEndpoints(...). В api.endpoints появляются новые записи, хуки становятся доступны.

Теория — 99. injectEndpoints + code splitting

Цитата — Code Splitting

«injectEndpoints accepts a collection of endpoints, adds them to the API slice, and returns the same API with updated types, so you can use the newly-added endpoints. Any endpoint added using injectEndpoints will stay in the cache unless explicitly removed via initial: overrideExisting.»
— RTKQ · Code Splitting

Паттерн — тонкий root-api + feature modules

// src/store/api.ts — общий root (без endpoints) export const api = createApi({ reducerPath: 'api', baseQuery: fetchBaseQuery({ baseUrl: '/api' }), tagTypes: ['Post', 'User', 'Comment'], // 👀 ВСЕ возможные теги тут endpoints: () => ({}), // пусто })
// src/features/posts/postsApi.ts — feature module import { api } from '../../store/api' export const postsApi = api.injectEndpoints({ endpoints: (build) => ({ getPosts: build.query<Post[], void>({ query: () => 'posts', providesTags: [{ type: 'Post', id: 'LIST' }], }), addPost: build.mutation<Post, NewPost>({ query: (body) => ({ url: 'posts', method: 'POST', body }), invalidatesTags: [{ type: 'Post', id: 'LIST' }], }), }), overrideExisting: false, // дефолт — warn если endpoint уже был }) export const { useGetPostsQuery, useAddPostMutation } = postsApi
// src/features/posts/PostsPage.tsx — ленивый модуль const PostsPage = lazy(() => import('./features/posts/PostsPage')) // При первом рендере PostsPage: // 1. Vite/webpack загружает chunk features/posts/* // 2. Executing модуль — вызывается api.injectEndpoints(...) // 3. В store появляются новые reducer-пути (если endpoint был с injectInto кастомной slice'и) // 4. Хуки работают.

Эффект на бандл

Без split: root bundle тащит все endpoints + transport + utils. С split: root имеет только createApi({ endpoints: () => ({}) }) — это ~3kb. Остальное — по мере надобности.

Enhanced api — расширить сгенерированное из OpenAPI

// если используешь @rtk-query/codegen-openapi — он генерит "пустые" endpoints. // Ты дополняешь тегами и transformResponse через enhanceEndpoints: import { api as generatedApi } from './generated/api' export const api = generatedApi.enhanceEndpoints({ addTagTypes: ['Post', 'User'], endpoints: { getPosts: { providesTags: [{ type: 'Post', id: 'LIST' }], transformResponse: (r: { items: Post[] }) => r.items, }, addPost: { invalidatesTags: [{ type: 'Post', id: 'LIST' }], }, }, })

Источник — buildCreateApi.ts

// упрощённо из createApi: const api = { reducerPath, reducer, middleware, endpoints: {}, injectEndpoints({ endpoints, overrideExisting = false }) { const newDefs = endpoints(builder) for (const [name, def] of Object.entries(newDefs)) { if (api.endpoints[name] && !overrideExisting) { if (__DEV__) console.warn(`Endpoint ${name} already exists`) continue } api.endpoints[name] = buildEndpoint(def) } return api // тот же объект — можно деструктурировать хуки }, }

Подводные камни

1. tagTypes нужно объявить В ROOT createApi.
Если в injectEndpoints используешь тег 'Post', а в root tagTypes его нет — RTKQ выдаст warning. Объявляй ВСЕ теги заранее в root или используй addTagTypes при enhanceEndpoints.

2. Endpoint навсегда остаётся в api после inject.
Нет механизма «unload feature» — endpoint продолжает занимать место в api.endpoints. Это не утечка данных (cache умирает по keepUnusedDataFor), но TS типы и middleware остаются.

3. overrideExisting: false → silent skip в продакшене.
В prod warning не логируется, endpoint просто не добавляется. Если у тебя два модуля вызывают inject одного имени — второй проиграет. Для hot-reload поставь overrideExisting: true или 'throw'.

4. useGetXQuery типизируется ТОЛЬКО из injectEndpoints return.
import { useGetPostsQuery } from './postsApi' — работает. api.useGetPostsQuery (корневой) — вернёт undefined, потому что тип root api не знает про inject'нутые endpoints.

5. initialState reducer не перестраивается.
injectEndpoints НЕ меняет shape state (в отличие от combineSlices.inject). Он добавляет только endpoints definitions и hooks. State.api структура та же.

6. Если root createApi импортируется из каждого feature — дубли.
Держи root в одном файле (store/api.ts), из features импортируй ЕГО, не создавай новые createApi. Иначе будет несколько reducerPath.
