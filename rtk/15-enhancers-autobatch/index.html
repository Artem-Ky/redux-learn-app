100 dispatch'ей подряд: с autoBatch и без

autoBatchEnhancer объединяет уведомления subscribers, отмеченных как «low priority» (через SHOULD_AUTOBATCH в meta). Каждый subscribers вызывается один раз на пачку, не на каждый отдельный dispatch. Это ускоряет React-рендеры в 5-10 раз.

Без autoBatch

0 subscriber вызовов

Каждый dispatch уведомляет subscribers. 100 actions → 100 уведомлений.

С autoBatch

0 subscriber вызовов

Subscribers с SHOULD_AUTOBATCH=true получают 1 уведомление на пачку.

В DevTools все 100 actions записаны (autoBatch не пропускает их), но subscriber вызывается только один раз на пачку — это и даёт прирост.

Теория — Enhancers + autoBatch

Что такое enhancer

Enhancer — это «обёртка над createStore». В отличие от middleware, который перехватывает actions, enhancer перехватывает все методы store: dispatch, subscribe, getState, replaceReducer. applyMiddleware — самый известный enhancer. Любой enhancer имеет сигнатуру:

type StoreEnhancer = (createStore: CreateStore) => CreateStore // например: applyMiddleware const myEnhancer: StoreEnhancer = (createStore) => (reducer, preloadedState) => { const store = createStore(reducer, preloadedState) return { ...store, dispatch: (action) => { /* кастомный dispatch */ return store.dispatch(action) } } }

Как пометить action как batchable

import { prepareAutoBatched } from '@reduxjs/toolkit' // Способ 1: через createSlice + prepare const slice = createSlice({ name: 'data', initialState: { items: [] }, reducers: { addItem: { reducer: (s, a) => { s.items.push(a.payload) }, prepare: prepareAutoBatched<Item>() } } }) // Способ 2: вручную в meta dispatch({ type: 'data/loaded', payload, meta: { SHOULD_AUTOBATCH: true } })

4 режима autoBatch

// По умолчанию: 'raf' — batched в requestAnimationFrame (~16ms) configureStore({ reducer, enhancers: (gde) => gde({ autoBatch: { type: 'raf' } // или 'tick' / 'timer' / 'callback' }) }) // 'tick' — batched в Promise.resolve().then() (микро-таска, ~0ms) // 'timer' — batched через setTimeout(0) // 'callback' — пользовательская функция batching

RTK Query и autoBatch

RTK Query использует SHOULD_AUTOBATCH для своих внутренних actions api/queryFulfilled, api/subscriptionsUpdated. Это и есть причина, почему подписаться на 50 query одновременно не вызывает 50 React-рендеров — все они батчатся в один.

Подводный камень: getDefaultEnhancers необязателен

Если вы передаёте enhancers, нужно явно вызывать getDefaultEnhancers(), иначе autoBatch и middlewareEnhancer пропадут. Правильно: enhancers: (gde) => gde().concat(myEnhancer). Неправильно: enhancers: () => [myEnhancer] — у вас не будет middleware и batching.
