Quick reference — правила Immer в RTK

Что МОЖНО в reducer'ах createSlice/createReducer

  • Мутировать draft: state.value++, state.list.push(x).
  • Мутировать вложенные объекты: state.user.name = 'X'.
  • Использовать delete state.x.
  • Возвращать новый объект (если хочется): return {{ ...state, value: 1 }}.
  • Использовать Map и Set (RTK включает enableMapSet).

Что НЕЛЬЗЯ

  • Смешивать мутацию И return: state.x = 1; return state или state.x = 1; return {{ ...state }}.
  • Сохранять draft вне reducer'а (revoked proxy).
  • Использовать Date.now(), Math.random(), nanoid() (reducer должен быть чистым).
  • console.log(state) — выводит Proxy. Используйте current(state).
  • Хранить классы с приватными полями (Immer не сможет клонировать).
  • Создавать циклические ссылки в state.

Что ЗАВИСИТ от случая

  • state = newValue — НЕ работает. state.field = newValue — работает. return newValue — работает.
  • Сохранение ссылки на под-объект: const obj = state.x; ... ; obj.value = 1 — работает (obj — тоже draft Proxy).
  • state.list = state.list.filter(...) — работает (присваивание новому массиву OK).
