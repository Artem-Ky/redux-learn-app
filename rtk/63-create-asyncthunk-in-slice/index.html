Что тут происходит

Весь fetchAll живёт внутри createSlice. Никаких createAsyncThunk + extraReducers отдельно. Creator-форма через buildCreateSlice({ creators: { asyncThunk: asyncThunkCreator } }). Обратите внимание: slice.actions.fetchAll — это и есть готовый thunk.

Теория — 63. Creator-форма для asyncThunk

Зачем нужна creator-форма

В RTK 2.0 появилась возможность писать reducers как колбэк:

reducers: (create) => ({ increment: create.reducer((s) => { s.value++ }), add: create.preparedReducer(...), fetchAll: create.asyncThunk(payloadCreator, { pending, fulfilled, rejected, settled }), })

Это — единственный способ объявить thunk внутри slice'а. Все handlers собраны в одном месте — state-машина «loading / data / error» не размазана по файлам.

Цитата из документации

«By default, Redux Toolkit's createSlice does not create an asyncThunk creator because it would inflate the bundle size for users who don't need it. You need to use buildCreateSlice and pass asyncThunkCreator to enable it:»
— createSlice — createAsyncThunk
«По умолчанию у createSlice нет asyncThunk-creator'а — это раздуло бы bundle тем, кому он не нужен. Включается он через buildCreateSlice с передачей asyncThunkCreator.»

Подключение

import { buildCreateSlice, asyncThunkCreator, nanoid, } from '@reduxjs/toolkit' // На ВСЁ приложение — один такой createSlice export const createAppSlice = buildCreateSlice({ creators: { asyncThunk: asyncThunkCreator }, }) // Далее везде вместо createSlice const usersSlice = createAppSlice({ name: 'users', initialState: { loading: 'idle', list: [], error: null }, reducers: (create) => ({ fetchAll: create.asyncThunk( async (_, { signal }) => (await fetch('/api/users', { signal })).json(), { pending: (s) => { s.loading = 'pending' }, fulfilled: (s, a) => { s.list = a.payload; s.loading = 'idle' }, rejected: (s, a) => { s.loading = 'idle'; s.error = a.error.message ?? 'unknown' }, settled: (s) => { /* опционально: cleanup */ }, }, ), }), }) export const { fetchAll } = usersSlice.actions // dispatch(fetchAll()) — работает как обычный thunk

Под капотом — что создаётся

Creator раскладывается на два объекта и кладёт их в extraReducers автоматически:

  • Создаётся «честный» createAsyncThunk с typePrefix = `${name}/${key}`;
  • handlers (pending/fulfilled/rejected/settled) регистрируются в extraReducers;
  • итоговый thunk экспортируется через slice.actions.fetchAll (не slice.thunks.fetchAll — он не существует);
  • slice.caseReducers.fetchAll НЕ содержит reducer — там не на что указывать, есть три разных handler'а.

Полная форма с options

fetchAll: create.asyncThunk( async (arg, thunkAPI) => { /* ... */ }, { pending, fulfilled, rejected, settled, options: { // обычные AsyncThunkOptions condition: (arg, { getState }) => getState().users.loading !== 'pending', idGenerator: (arg) => `u-${arg}`, dispatchConditionRejection: false, }, }, )

Подводные камни

1. Нужен buildCreateSlice (не default). Если импортируете обычный createSlice из @reduxjs/toolkit, то reducers: (create) => ({ … }) будет компилироваться, но create.asyncThunk бросит ошибку в runtime: «asyncThunk creator must be added via buildCreateSlice».
2. slice.actions.fetchAll — это thunk, но в тип ActionCreator он НЕ вписывается. Если у вас есть тип type SliceActions = typeof slice.actions, в нём fetchAll будет с AsyncThunk<…>-сигнатурой (не ActionCreator). При деструктуризации const { fetchAll } = slice.actions — TS всё понимает, но утилиты вроде bindActionCreators могут ломаться.
3. Нет отдельного fetchAll.pending импорта. В обычном флоу вы экспортируете export const fetchAll = createAsyncThunk(...) и дёргаете fetchAll.pending.type где угодно. С creator-формой — slice.actions.fetchAll.pending.type (они всё ещё есть на thunk'е), и доступны только после экспорта из slice'а. В миграции это часто забывают.
4. Bundle size — creator-форма НЕ tree-shake'ится. Если в приложении используется buildCreateSlice + asyncThunkCreator хотя бы раз, весь createAsyncThunk попадает в bundle. Обычная форма — тоже (т.к. thunk объявлен отдельно). Разница незначительная, но существует.
5. Миграция «от thunk'а» сложна. Был export const fetchUsers = createAsyncThunk(...), все dispatch'ы работали напрямую. Переехали в creator-форму → fetchUsers пропал как standalone export, пришлось менять все импорты. Учитывайте, если рефакторите legacy.
