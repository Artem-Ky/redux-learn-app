combineReducers vs combineSlices — два способа

combineSlices — новая утилита RTK 2.x. Принимает сами slice-объекты (а не .reducer у них) и автоматически использует slice.reducerPath как ключ в state. Главное — даёт .inject() для lazy-loading (урок 35).

configureStore с reducer-объектом (старый способ)

const store = configureStore({ reducer: { // нужно явно указать ключ И .reducer counter: counterSlice.reducer, user: userSlice.reducer, posts: postsSlice.reducer, } }) // state.counter, state.user, state.posts

combineSlices (RTK 2.x)

import { combineSlices } from '@reduxjs/toolkit' const rootReducer = combineSlices(counterSlice, userSlice, postsSlice) // Ключ в state = slice.reducerPath (по умолчанию = slice.name) const store = configureStore({ reducer: rootReducer }) // state.counter, state.user, state.posts // Бонус: можно подмешивать обычные reducer'ы: const rootReducer = combineSlices( counterSlice, userSlice, { theme: themeReducer, formData: formDataReducer } )

Теория — combineSlices

Из доки

«combineSlices is an enhanced version of combineReducers that allows you to combine slices and reducers, and also supports lazy-loading slices later via injectSlices.»
— combineSlices API
«combineSlices — улучшенная версия combineReducers, которая позволяет объединять slice'ы и reducer'ы, а также поддерживает позднюю lazy-загрузку slice'ов через injectSlices.»

Сигнатура

function combineSlices( ...slicesOrReducerMaps: (Slice | { [key]: Reducer })[] ): Reducer<State> & { inject: ..., selector: ..., withLazyLoadedSlices: ... }

Принимает любую комбинацию slice-объектов и/или плоских объектов { key: reducer }. Возвращает обычный root reducer с дополнительными методами для lazy-injection (урок 35).

reducerPath — что это

Каждый slice имеет поле reducerPath. По умолчанию reducerPath = name. Можно переопределить:

const usersSlice = createSlice({ name: 'users', reducerPath: 'data/users', // (плоский ключ, не вложенный) initialState: ..., reducers: { ... } }) const root = combineSlices(usersSlice) // state['data/users'] — да, ключ со слешем (но это плохая практика)

В большинстве случаев reducerPath совпадает с name. Разница важна для RTK Query (reducerPath = 'api') и для slice'ов, которые нужно положить в нестандартное место.

Смешивание slice + reducer

import { persistReducer } from 'redux-persist' const persistedTheme = persistReducer(persistConfig, themeReducer) const rootReducer = combineSlices( counterSlice, userSlice, { theme: persistedTheme, legacy: oldReducer } // объект с произвольными reducer'ами ) // state.counter, state.user, state.theme, state.legacy

Подводный камень: name vs ключ store

С combineSlices вы ОБЯЗАНЫ полагаться на slice.reducerPath. Если хотите свои ключи, используйте плоский map: combineSlices({ myKey: slice.reducer }).
