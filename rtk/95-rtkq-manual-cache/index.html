Теория — 95. Manual cache updates

Цитата — Manual Cache Updates

«updateQueryData, upsertQueryData, prefetch, invalidateTags and resetApiState are the thunk action creators for manually interacting with the cache without tying that behavior to a mutation.»
— RTKQ · api.util reference

1. updateQueryData — патч Immer-рецептом

dispatch( api.util.updateQueryData('getProducts', undefined, (draft) => { const item = draft.find((p) => p.id === 3) if (item) item.price *= 0.9 // -10% скидка }), ) // возвращает { undo, patches, inversePatches } // сеть НЕ трогается. Если entry не существует — noop. Compatible с optimistic updates.

2. upsertQueryData — вставить/перезаписать

// ситуация: юзер кликнул на товар, у нас уже его данные из списка → кладём в cache getProduct(id) const product = { id: 5, title: 'Phone', price: 999 } dispatch( api.util.upsertQueryData('getProduct', 5, product), ) // entry создана/перезаписана: status='fulfilled', data=product, fulfilledTimeStamp=Date.now() // useGetProductQuery(5) тут же вернёт HIT без сети.

3. prefetch — превентивный fetch

// вариант A — через хук (React): const prefetchProduct = usePrefetch('getProduct') <Link onMouseEnter={() => prefetchProduct(5)} /> // вариант B — через dispatch: dispatch(api.util.prefetch('getProduct', 5, { force: false, ifOlderThan: 30 })) // опции: // force: true — всегда fetch // ifOlderThan: 30 — fetch только если кеш старше 30s // subscribe: false (дефолт для prefetch) — НЕ держим подписчика // Значит: через keepUnusedDataFor данные умрут, если никто не подпишется.

4. invalidateTags — ручная инвалидация

dispatch(api.util.invalidateTags(['Product'])) dispatch(api.util.invalidateTags([{ type: 'Product', id: 3 }])) // эквивалент invalidatesTags в mutation — но триггерится кодом: // - websocket сообщил что цены обновились → invalidateTags(['Product']) // - пользователь нажал "Обновить" — invalidateTags([{type:'Product', id: 'LIST'}])

5. resetApiState — nuke всё

// logout сценарий: function onLogout() { store.dispatch(api.util.resetApiState()) // очищает queries, mutations, subscriptions, provided — ВСЁ, что под api.reducerPath // ⚠ НЕ чистит НЕ-api slices — если в другом slice лежит производное — чисти отдельно }

Источник — buildSlice.ts (updateQueryData)

// упрощённо — как RTKQ применяет рецепт updateQueryData: (endpointName, args, updateRecipe) => (dispatch, getState) => { const endpoint = api.endpoints[endpointName] const cacheKey = endpoint.internalActions.querySubState.match(state, endpointName, args) let inversePatches: Patch[] = [] const [patched, patches] = produceWithPatches( getState()[reducerPath].queries[cacheKey].data, updateRecipe, ) // диспатчим action, reducer заменяет data на patched dispatch({ type: 'api/mutateData', payload: { endpointName, args, patches } }) return { patches, inversePatches, undo: () => dispatch({ ..., patches: inversePatches }) } }

Подводные камни

1. updateQueryData noop, если entry не существует.
Самая частая ошибка. Если компонент с useGetProductsQuery не примонтирован — updateQueryData('getProducts', ...) ничего не сделает. Проверяй через api.endpoints.X.select(arg)(state).status === 'fulfilled' или используй upsertQueryData.

2. upsertQueryData не вызывает onCacheEntryAdded.
Это «искусственная» entry, она создаётся напрямую в state. Если твой onCacheEntryAdded должен открыть WS — это не сработает при upsert. Workaround: dispatch endpoints.X.initiate(arg) для реальной подписки.

3. prefetch без subscribe — данные умрут через keepUnusedDataFor.
Для «hover предзагрузки» это OK (60s достаточно для клика). Но если prefetch'ишь на старте app — без подписчика cache умрёт через минуту.

4. resetApiState может сломать UI если компоненты всё ещё смотрят.
Сразу после reset все useQuery вернут data === undefined, status === 'uninitialized'. Если компоненты это не обрабатывают — краш. Resetай на logout, когда unmount всего app.

5. invalidateTags не работает если middleware не подключен.
api.util действия — thunks, проходят через redux-thunk. Но инвалидация — через api.middleware. Если забыл .concat(api.middleware) — всё ломается.

6. updateQueryData и TS — endpointName должен быть литеральным.
const name = 'getProducts'; updateQueryData(name, ...) ← TS потеряет type inference для args. Используй string literal напрямую или as const.
