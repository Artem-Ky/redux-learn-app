slice.injectInto(rootReducer) — динамическое добавление

В большом приложении не все slice'ы нужны сразу. Lazy injection позволяет загружать reducer вместе с feature-кодом (динамический import). Размер главного бандла меньше, state растёт по мере открытия features.

// 1. Базовый rootReducer — только core slices const rootReducer = combineSlices(coreSlice).withLazyLoadedSlices<LazyState>() const store = configureStore({ reducer: rootReducer }) // 2. Позже инжектим feature slice const chartSlice = createSlice({ name: 'chart', initialState: { data: [] }, reducers: { ... } }).injectInto(rootReducer) // ↑ injectInto возвращает копию slice'а с гарантией, что reducer уже подключён // 3. Используем actions/selectors как обычно store.dispatch(chartSlice.actions.addPoint(42)) chartSlice.selectors.selectData(store.getState())

Теория — lazy injection

Из доки

«combineSlices returns an enhanced reducer with an inject method that allows you to inject a new slice/reducer at runtime. This is most useful for large applications where some slices are loaded only when needed (code splitting).»
— combineSlices — Injecting reducers
«combineSlices возвращает улучшенный reducer с методом inject, который позволяет внедрять новый slice/reducer в runtime. Полезно для больших приложений, где некоторые slice'ы загружаются по мере необходимости (code splitting).»

Два API: rootReducer.inject и slice.injectInto

// Способ 1: rootReducer.inject — для произвольных reducer'ов rootReducer.inject({ reducerPath: 'feature', reducer: featureReducer }) // Способ 2: slice.injectInto(rootReducer) — для slice'ов (рекомендуется) const injectedSlice = featureSlice.injectInto(rootReducer) // Возвращает "клон" slice'а с гарантией, что reducer добавлен // Если slice уже был injected — возвращает существующий

Под капотом

combineSlices хранит Map<reducerPath, reducer>. Inject добавляет в эту map новую запись. На следующем dispatch'е root reducer будет вызывать ВСЕ reducer'ы, включая новый. Это безопасная операция — старые ветки state остаются без изменений.

// Псевдо-код: function combineSlices(...slices) { const reducers = new Map() // добавляем initial slices function root(state, action) { const next = {} for (const [path, reducer] of reducers) { next[path] = reducer(state?.[path], action) } return next } root.inject = (slice) => { reducers.set(slice.reducerPath, slice.reducer) } return root }

Когда state injected slice'а появляется

После inject на следующем dispatch root reducer вызовет inject'нутый reducer с state[path] = undefined. Reducer вернёт initialState. Так в state добавляется новая ветка.

Чтобы спровоцировать обновление сразу, можете dispatch любой no-op action или просто dispatch({ type: '@@INIT_INJECTED' }).

withLazyLoadedSlices — типизация

В TS root state должен «знать» о всех возможных injected slices, иначе селекторы не будут типизированы. Решение — withLazyLoadedSlices<T>():

type LazyState = { chart?: ChartState, // опционально, потому что lazy cart?: CartState, search?: SearchState, } const rootReducer = combineSlices(coreSlice) .withLazyLoadedSlices<LazyState>() type RootState = ReturnType<typeof rootReducer> // RootState = { core: CoreState, chart?: ChartState, cart?: ..., search?: ... }

// chartFeature/index.ts export const chartSlice = createSlice({ ... }).injectInto(rootReducer) export ChartView from './ChartView' // App.tsx const ChartPage = lazy(() => import('./chartFeature')) // при первом рендере: // → загрузится chunk → injectInto добавит reducer → state.chart появится

Подводный камень: doubled inject

Если вызвать injectInto для одного и того же slice дважды (например, если chunk грузится дважды) — RTK не заменит reducer. Возвращается тот же объект. Reducer сохраняется (state не обнуляется).

Подводный камень: undefined в селекторах

До inject'а state.chart === undefined. Если используете chartSlice.selectors до inject — получите runtime error. Решение: всегда импортируйте slice через chartSlice.injectInto(...) в момент монтирования компонента.
