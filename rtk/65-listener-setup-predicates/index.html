Теория — 65. Как подключить и какие predicate'ы бывают

Подключение к store

«The listener middleware should be added to the Redux store using the configureStore's middleware parameter. It's recommended to prepend the listener middleware so that it runs before the default middleware like redux-thunk.»
— RTK docs — basic usage
«Listener middleware следует добавлять через параметр middleware у configureStore. Рекомендуется именно .prepend() — чтобы listener шёл раньше thunk и успевал увидеть action до того, как thunk его преобразует.»
import { configureStore, createListenerMiddleware } from '@reduxjs/toolkit' const listenerMiddleware = createListenerMiddleware() const store = configureStore({ reducer: rootReducer, middleware: (getDefault) => getDefault() .prepend(listenerMiddleware.middleware), // именно prepend, не concat })

4 формы startListening

// 1. actionCreator — самый популярный, типы action'а вывозятся из RTK action creator listener.startListening({ actionCreator: increment, effect: (action, api) => { ... } }) // 2. type — строка, когда action создан не через createAction listener.startListening({ type: 'counter/decrement', effect: (action, api) => { ... } }) // 3. matcher — проверка принадлежности к группе (isAnyOf, isAllOf, thunk.fulfilled) listener.startListening({ matcher: isAnyOf(increment, decrement), effect: ... }) // 4. predicate — полный доступ к action + currentState + originalState listener.startListening({ predicate: (action, currentState, originalState) => (currentState as RootState).counter.value >= 10 && (originalState as RootState).counter.value < 10, effect: ... })

Исходник — компиляция всех форм в единый predicate

// packages/toolkit/src/listenerMiddleware/index.ts, getListenerEntryPropsFrom const getListenerEntryPropsFrom = (options: FallbackAddListenerOptions) => { let { type, actionCreator, matcher, predicate, effect } = options if (type) { // форма 2 → predicate = createAction(type).match predicate = createAction(type).match } else if (actionCreator) { // форма 1 → сам action creator уже содержит .match type = actionCreator!.type predicate = actionCreator.match } else if (matcher) { // форма 3 → matcher сам по себе type-guard predicate = matcher } else if (predicate) { // форма 4 → ничего не делаем } else { throw new Error( 'Creating or removing a listener requires one of the known fields for matching an action' ) } assertFunction(effect, 'options.listener') return { predicate, type, effect } }

Сигнатура predicate и что он видит

Два подводных камня подключения и predicate:
  • Именно .prepend(), а не .concat(). Если сделать .concat(listener.middleware), listener увидит thunk'и уже преобразованными в функции, и результат будет непредсказуемым. prepend ставит его первым в цепочке [listener, thunk, immutable, serializable].
  • currentState и originalState могут совпадать по ссылке. Если reducer вернул тот же объект (например, incrementBy(0) при return state), currentState === originalState. Сравнивайте по значению нужного поля, а не по ссылкам root-объекта.

Как типизировать форму-4 (predicate) сразу

// без predicate TS не знает тип action listener.startListening({ predicate: (action, currentState: RootState, previousState: RootState) => currentState.counter.value >= 10 && previousState.counter.value < 10, effect: (action, api) => { // action: UnknownAction — нет type guard'а // api.getState(): RootState — типизировано благодаря withTypes (урок 70) } })
