Цитата — Frequently Used Query Hook Return Values

«currentData vs data: data is the latest returned data for any arg. currentData is the latest returned data for the current arg.»

«Когда аргумент меняется (ввод в инпуте) — isFetching=true, data содержит СТАРЫЙ результат, currentData — undefined. Для UI паджинации удобно показывать data (не "мигает"), для инвалидации — currentData.»

isLoading vs isFetching — критическая разница

Ситуация isLoading isFetching
первый mount, data=undefined true true
refetch() / polling false true
arg изменился (pikachu → charizard) false true
cache HIT — данные уже есть false false

refetch() — когда вызывать руками

  • Кнопка "Обновить" — пользователь хочет свежак.
  • После side-effect вне RTKQ — например, websocket прислал событие "post-updated".
  • НЕ для invalidation — если у тебя тэги, пусть мутация сама трггерит refetch через invalidatesTags (урок 87).
  • Возвращает thunk — можно await'ить: const r = await refetch().unwrap().

skip & skipToken

// Условный hook: не запускать, пока нет id const { data } = useGetPostByIdQuery(id ?? skipToken) // Альтернатива (хуже для TS): const { data } = useGetPostByIdQuery(id!, { skip: id == null }) // skipToken лучше — arg: TypeArg | SkipToken, TypeScript не будет требовать null-check.

Подводные камни

1. isLoading — только первая загрузка. Для спиннера используй isFetching.
Если ты пишешь {isLoading ? spinner : data} — при смене arg данные «не мигают» спиннером, пользователь видит СТАРЫЕ данные без индикации обновления. Паттерн: {isFetching ? spinner : null} + {data && <List data={data} />} = старые данные + оверлей-спиннер сверху.

2. data vs currentData.
При смене arg data продолжает показывать СТАРЫЙ успех (полезно для пагинации «keep previous data»). currentData очищается в undefined до fulfilled нового arg. Выбирай в зависимости от UX — мигание vs отсутствие мерцания.

3. refetch не меняет cache key.
Это «принудительный reload того же cacheKey». Если у тебя 5 подписчиков на один key — все увидят новые данные. Не путать с forceRefetch-опцией endpoint.

4. hook возвращает новый объект каждый рендер, но внутренние ссылки стабильны.
Нельзя делать useEffect([{hook}]) — будет бесконечный цикл. Надо useEffect([data, isFetching]). Если нужна derived-память — selectFromResult (урок 90).

5. import из /query/react.
Если импортировать из /query — hooks не будут сгенерированы. Повторяется из урока 80, но это типовая ошибка.
