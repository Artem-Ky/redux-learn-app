Типизированный store + хуки + selector — итог секции B

Здесь — готовый рабочий React-counter с полностью типизированным store. Все типы выводятся из самого store, ничего не пишется руками. useAppSelector и useAppDispatch — типизированные обёртки над useSelector/useDispatch.

Шаг 1: типы из store

import { configureStore } from '@reduxjs/toolkit' export const store = configureStore({ reducer: { counter: counterSlice.reducer } }) // → Авто-выведение типов export type RootState = ReturnType<typeof store.getState> export type AppDispatch = typeof store.dispatch export type AppStore = typeof store

Шаг 2: типизированные хуки (старый способ)

import { useDispatch, useSelector, type TypedUseSelectorHook } from 'react-redux' export const useAppDispatch: () => AppDispatch = useDispatch export const useAppSelector: TypedUseSelectorHook<RootState> = useSelector

Шаг 3: новый способ через .withTypes() (react-redux 9+)

import { useDispatch, useSelector, useStore } from 'react-redux' export const useAppDispatch = useDispatch.withTypes<AppDispatch>() export const useAppSelector = useSelector.withTypes<RootState>() export const useAppStore = useStore.withTypes<AppStore>()

Шаг 4: использование в компонентах

function Counter() { // тип value автоматически number const value = useAppSelector(state => state.counter.value) // dispatch знает про thunk const dispatch = useAppDispatch() return <button onClick={() => dispatch(incremented())}>{value}</button> }
Теория — типизация store

Из доки

«You should infer these types from the store itself, which means that they correctly update as you add more state slices or modify middleware settings. RootState and AppDispatch should be inferred from the store, not declared manually.»
— TypeScript Quick Start
«Эти типы следует выводить из самого store — тогда они автоматически обновляются при добавлении новых slice'ов или изменении middleware. RootState и AppDispatch выводятся из store, а не объявляются вручную.»

Почему .withTypes() лучше

Старый способ через generic TypedUseSelectorHook<RootState> работает, но не покрывает useDispatch (там нет аналогичного типа). Приходилось писать : () => AppDispatch = useDispatch — теряется stack trace, выводимый при ошибке.

В react-redux 9.1+ у каждого хука есть статический метод .withTypes<T>():

  • useSelector.withTypes<RootState>()
  • useDispatch.withTypes<AppDispatch>()
  • useStore.withTypes<AppStore>()
  • useEffect-style: useSyncExternalStore тоже можно типизировать.

RootState с lazy-injected slices

Если вы используете combineSlices + .inject() для code-splitting, инжектируемые slice появляются в state позже, чем вычисляется ReturnType<typeof store.getState>. Решение — WithSlice-тип: type RootState = ReturnType<typeof rootReducer> & WithSlice<typeof productsSlice>. Подробнее в уроке 35-36.

Структура файла store.ts (best practice)

// app/store.ts import { configureStore } from '@reduxjs/toolkit' import counterReducer from '../features/counter/counterSlice' export const store = configureStore({ reducer: { counter: counterReducer }, }) export type AppStore = typeof store export type RootState = ReturnType<typeof store.getState> export type AppDispatch = typeof store.dispatch // app/hooks.ts import { useDispatch, useSelector, useStore } from 'react-redux' import type { RootState, AppDispatch, AppStore } from './store' export const useAppDispatch = useDispatch.withTypes<AppDispatch>() export const useAppSelector = useSelector.withTypes<RootState>() export const useAppStore = useStore.withTypes<AppStore>()

Подводный камень: импортируйте useAppDispatch, не useDispatch

Один из самых частых багов: разработчик импортирует useDispatch напрямую из react-redux, и тогда dispatch(myThunk()) падает на TS-проверке «Argument of type 'ThunkAction' is not assignable to 'AnyAction'». Всегда импортируйте useAppDispatch из вашего hooks.ts.
