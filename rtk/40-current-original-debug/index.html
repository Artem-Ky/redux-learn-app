Лог snapshot'ов draft внутри reducer'а

Когда reducer сложный (несколько мутаций, фильтрация, обновление множества полей), полезно логировать промежуточные snapshot'ы через current(state). Без него в console.log увидите только Proxy.

import { current, original } from '@reduxjs/toolkit' reducers: { processTodos: (state) => { console.log('1. до фильтрации:', current(state)) state.items = state.items.filter(t => !t.done) console.log('2. после фильтрации:', current(state)) state.items.forEach(t => { t.priority = computePriority(t) }) console.log('3. после priority:', current(state)) console.log('оригинал:', original(state)) // исходный, без мутаций } }

Теория — current(), original(), isDraft()

current(value) — snapshot draft'а

«current takes a draft and returns a copy with the current state of that draft. Useful for logging or for passing a snapshot to other code that does not understand drafts.»
— Immer current() docs
«current принимает draft и возвращает копию с текущим состоянием этого draft'а. Полезно для логирования или передачи snapshot'а в код, который не работает с draft'ами.»

original(value) — оригинальный объект

«original returns the original object that was wrapped by the draft. Useful when you need to compare the draft to the original to detect changes, or to maintain referential equality with the original.»
— Immer original() docs
«original возвращает оригинальный объект, обёрнутый draft'ом. Полезно когда нужно сравнить draft с оригиналом для детекции изменений, или сохранить referential equality с оригиналом.»

Сравнение

Утилита Что возвращает Когда использовать
current() plain copy текущего draft'а console.log, breakpoint debugging, передача в API
original() ссылка на оригинальный объект (state до reducer'а) сравнение «было/стало», referential equality
isDraft() boolean: это Proxy? в helper'ах/селекторах, чтобы понять контекст

Стоимость

current() делает полную глубокую копию draft'а. Для большого state (тысячи элементов) это медленно. Используйте только в dev-логах или точечно для отдельных полей: current(state.user) — копия только user-ветки.

original() — для оптимизации

// Ленивая обработка: если поле не изменилось — не пересчитываем reducers: { recomputeStats: (state) => { if (state.items === original(state.items)) { // items не изменились с прошлого dispatch — не пересчитываем return } state.stats = computeStats(state.items) } }

isDraft() — пример из реального кода

function addItemSafe(stateOrDraft, item) { if (isDraft(stateOrDraft)) { stateOrDraft.items.push(item) // можно мутировать } else { return { ...stateOrDraft, items: [...stateOrDraft.items, item] } } } // В reducer'е: reducers: { addInDraft: (state, action) => { addItemSafe(state, action.payload) } } // Вне draft (в селекторе/тесте): const next = addItemSafe(plainState, item)

DevTools показывает финальный state

Custom DevToolsPanel и Redux DevTools Extension всегда показывают финальный state после reducer'а — это уже plain object (Immer заморозил его). Для intermediate snapshots нужны console.log + current().

Anti-pattern: console.log(state)

Прямой console.log(state) в reducer'е выведет Proxy(Object) {}. Если раскрыть в DevTools — содержимое не структурируется. Всегда оборачивайте в current(state) или JSON.parse(JSON.stringify(state)).
