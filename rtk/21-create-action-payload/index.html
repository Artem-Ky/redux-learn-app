Typed payload через generic createAction<P>

Generic-параметр P в createAction<P>(type) задаёт тип payload. Вызов add(5) даёт action { type, payload: 5 }. TS проверяет тип на вызове.

increment = createAction<void>('counter/increment')
без payload → вызов без аргументов
addBy = createAction<number>('counter/addBy')
number → вызов с числом
setUser = createAction<User>('user/set')
интерфейс User = {'{'}id, name{'}'}
tagged = createAction<string[], 'tag/set'>('tag/set')
второй generic — type (const string)
— ещё не было dispatch —

TS-проверки в IDE

// ← правильно addBy(5) // → { type: 'counter/addBy', payload: 5 } // ← TS error addBy('five') // Argument of type 'string' is not assignable to 'number' // ← TS error increment(5) // Expected 0 arguments, but got 1 // ← правильно: партиал объект setUser({ id: 1, name: 'Alice' }) // ← TS error: отсутствует name setUser({ id: 1 })
Теория — payload и PayloadAction

Из доки

«createAction accepts a generic type parameter for the payload. The generated action creator will be typed to accept arguments matching that type, and the resulting action will have the correct payload type.»
— createAction — TypeScript
«createAction принимает generic-параметр для типа payload. Получившийся action creator типизируется на приём аргументов этого типа, и в итоговом action поле payload получает правильный тип.»

PayloadAction

RTK экспортирует удобный тип PayloadAction<P, T = string, M = never, E = never>:

type PayloadAction<P = void, T extends string = string, M = never, E = never> = { type: T payload: P meta: M // опционально — только если задан через prepare error: E // опционально } // Используется в reducer'ах: const reducer = createSlice({ reducers: { addBy: (state, action: PayloadAction<number>) => { state.value += action.payload // тип payload = number } } })

Generic type parameter

// Без payload const reset = createAction('reset') // P = void reset() // OK reset(5) // TS warn (аргумент проигнорируется) // С payload const add = createAction<number>('add') // P = number add(5) // OK add() // TS error: missing argument // С двумя параметрами: P и T (константа) const tag = createAction<string[], 'tag/set'>('tag/set') tag.type // тип: 'tag/set' (не просто string!)

Второй generic-параметр T — это литеральный тип для type. Он важен для discriminated unions в reducer'ах: if (action.type === tag.type) сужает тип action точечно.

Default export vs named export

Не путайте: import { createAction } from '@reduxjs/toolkit' — named export. Default export у RTK нет. Импортируйте только через named.

Ограничение: один аргумент

Без prepare-callback (урок 22) createAction принимает один аргумент — это становится payload. Если нужно передать несколько параметров, используйте объект:

// ✓ правильно const updateUser = createAction<{ id: number, name: string }>('user/update') updateUser({ id: 1, name: 'Alice' }) // ✗ неправильно (будет payload = id, name потеряется) const bad = createAction<number, string>('x') // второй generic — это T, не второй аргумент

Подводный камень: undefined payload

Если вы забыли передать аргумент — payload станет undefined. В production это сложный для debugging баг. В dev TS предупредит. Для обязательного payload всегда ставьте generic (даже примитивный): createAction<number>('add').
