3 формы reducer в configureStore

configureStore принимает reducer в трёх формах. Переключайте — внизу будет создаваться разный store и DevTools покажет одинаковый результирующий state shape.

configureStore({ reducer: ... })

Теория — что configureStore делает с reducer

      

        
Из доки

        

          «reducer: A single reducer function that will be used as the root reducer, or an object of slice reducers
          that will be passed to combineReducers().»
          
— configureStore — reducer
        

        

          «reducer: либо одна reducer-функция, которая будет использована как root reducer,
          либо объект slice-reducers, который будет передан в combineReducers().»
        


        
Логика внутри (исходник)

        

// configureStore.ts (упрощённо)
let rootReducer
if (typeof reducer === 'function') {
  rootReducer = reducer                       // Форма 3: уже готовый
} else if (isPlainObject(reducer)) {
  rootReducer = combineReducers(reducer)  // Форма 1: объект → combineReducers
} else {
  throw new Error('"reducer" is a required argument...')
}
        


        
Все три формы дают одинаковый state

        

// Форма 1 — объект (RTK сам сделает combineReducers)
configureStore({
  reducer: { counter: counterReducer, todos: todosReducer }
})

// Форма 2 — вы сами вызвали combineReducers
configureStore({
  reducer: combineReducers({ counter: counterReducer, todos: todosReducer })
})

// Форма 3 — единый reducer (без combineReducers)
configureStore({
  reducer: rootReducer  // (state, action) => newState
})

// → во всех трёх state.counter и state.todos доступны
        


        
Когда использовать какую

        
    
          
  • Форма 1 (объект) — стандартная и рекомендуемая. Меньше boilerplate, RTK сам зовёт combineReducers.
    • 
          
  • Форма 2 (явный combineReducers) — если нужно сделать что-то с rootReducer (например, обернуть в reducerEnhancer для resetState).
    • 
          
  • Форма 3 (одна функция) — для крошечных приложений или нестандартных случаев (custom rootReducer, например с redux-persist).
    • 
        


        
combineSlices vs combineReducers

        

          В RTK 2.x появился combineSlices() — расширенная версия combineReducers, которая поддерживает
          lazy injection slice'ов (для code-splitting). Подробнее в уроке 34.
        


        
Подводный камень: конфликт ключей

        

          Если у вас в Объект-карте есть ключ counter, и одновременно вы делаете
          preloadedState: { counter: ... } — RTK инициализирует state из preloadedState,
          а ваш initialState slice проигнорирует. Это поведение Redux, а не баг RTK.
        


        
Подводный камень: typeof reducer === 'function'

        

          Если вы передадите массив или Map — будет error "reducer" is a required argument and must be a function or an object of functions.
          Только plain object или function принимаются.
