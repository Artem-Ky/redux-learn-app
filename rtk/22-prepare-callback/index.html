4 сценария prepare — меняют саму структуру action

prepare — второй аргумент createAction. Это функция, которая получает входные параметры и возвращает { payload, meta?, error? }. Так можно генерировать id, timestamp, прятать non-сериализуемые данные, принимать несколько аргументов.

1. nanoid — автогенерация id
const addTodo = createAction('todos/add', (text: string) => ({ payload: { id: nanoid(), text, done: false } })) addTodo('купить хлеб') // { type: 'todos/add', payload: { id: 'V1StGXR8_Z', text: '...', done: false } }
—
2. Несколько аргументов → один payload
const addUser = createAction('users/add', (name: string, age: number) => ({ payload: { name, age, id: nanoid() } })) addUser('Alice', 30)
—
3. meta — timestamp, analytics
const logged = createAction('event/logged', (message: string) => ({ payload: message, meta: { timestamp: Date.now(), source: 'user-action' } }))
—
4. error — SerializedError
const failed = createAction('fetch/failed', (err: Error) => ({ payload: err.message, error: true, meta: { name: err.name, stack: err.stack } }))
—
Теория — prepare callback

Из доки

«If provided, prepare is called as part of the action creator with whatever arguments the creator received, and should return either an object with a payload field, or an object with payload, meta, and/or error fields.»
— prepare callback docs
«Если передан, prepare вызывается как часть action creator'а с теми аргументами, что получил creator, и должен вернуть либо объект с полем payload, либо объект с полями payload, meta и/или error.»

Сигнатура

function createAction<Args, PA extends PayloadActionCreator>( type: string, prepare?: (...args: Args) => { payload: unknown, meta?: unknown, error?: unknown } ): PA // prepare — это preparation-функция, не reducer!

FSA (Flux Standard Action)

RTK следует FSA — конвенции структуры action:

interface FSA { type: string // обязательно payload?: unknown // данные action error?: boolean // true если ошибка (payload содержит Error-like объект) meta?: unknown // метаданные (id, timestamp, source) }

Строгое правило: если error: true, то payload должен соответствовать Error-shape. Это ключ для isRejectedWithValue matcher'а.

nanoid — встроенный id-generator

import { nanoid } from '@reduxjs/toolkit' // nanoid() → строка 21 символ, криптографически random // Примеры: 'V1StGXR8_Z5jdHi6B-myT' nanoid() // дефолтная длина nanoid(10) // 10 символов

nanoid — это не alias на UUID, это отдельная функция, переэкспортированная RTK из @reduxjs/toolkit. Использует crypto.getRandomValues.

Вариант в createSlice

const todoSlice = createSlice({ name: 'todos', initialState: [], reducers: { addTodo: { reducer: (state, action) => { state.push(action.payload) }, prepare: (text: string) => ({ payload: { id: nanoid(), text, done: false } }) } } }) // В компоненте: dispatch(addTodo('купить хлеб')) // передаём только text

Подводный камень: id внутри reducer'а — запрещено

Reducer должен быть чистой функцией: без Math.random, без Date.now, без nanoid. Иначе time-travel в DevTools сломается: при откате назад reducer сгенерирует новый id. Всегда делайте nanoid() в prepare, не в reducer.

Подводный камень: возврат из prepare

prepare обязан вернуть объект с полем payload. Если вернёте просто значение, получите { type, payload: undefined, meta: 'ваше-значение' } — неожиданная структура. Правильно: return { payload: value }.

Источник

createAction.ts — функция createAction, если есть prepare, делает вызов prepared = prepare(...args) и разбрасывает его поля в action.
