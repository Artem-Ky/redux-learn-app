Теория — 52. getSelectors() и createDraftSafeSelector

Что возвращает adapter.getSelectors()

// EntitySelectors<T, V, IdType> { selectIds: (state: V) => Id[] selectEntities: (state: V) => Record<Id, T> selectAll: (state: V) => T[] selectTotal: (state: V) => number selectById: (state: V, id: Id) => T | undefined }

Две перегрузки

Вызов Принимает Когда использовать
getSelectors() EntityState напрямую Unit-тесты, self-contained slice, когда пользователь сам вытаскивает подстейт
getSelectors(selector) RootState целиком Компоненты через useSelector — всегда передают RootState

Исходник — state_selectors.ts

export function createSelectorsFactory<T, Id>() { function getSelectors(selectState?, options = {}) { const { createSelector = createDraftSafeSelector as AnyCreateSelectorFunction, // ← дефолт! } = options const selectIds = (state) => state.ids const selectEntities = (state) => state.entities const selectAll = createSelector( selectIds, selectEntities, (ids, entities) => ids.map((id) => entities[id]!), ) const selectId = (_, id) => id const selectById = (entities, id) => entities[id] const selectTotal = createSelector(selectIds, (ids) => ids.length) if (!selectState) { return { selectIds, selectEntities, selectAll, selectTotal, selectById: createSelector(selectEntities, selectId, selectById), } } // globalized: оборачиваем в ещё один createSelector const selectGlobalizedEntities = createSelector(selectState, selectEntities) return { selectIds: createSelector(selectState, selectIds), selectEntities: selectGlobalizedEntities, selectAll: createSelector(selectState, selectAll), selectTotal: createSelector(selectState, selectTotal), selectById: createSelector(selectGlobalizedEntities, selectId, selectById), } } return { getSelectors } }

GitHub: state_selectors.ts

Зачем createDraftSafeSelector

Обычный reselect.createSelector кэширует по строгому === ссылки. Внутри reducer'а state — Immer draft, и после commit'а ссылка меняется; кэш всегда мимо. createDraftSafeSelector детектит draft и правильно переключает кэш. Дефолт getSelectors безопасный для draft'ов.

Ссылка: RTK · createDraftSafeSelector

Цитата — RTK docs

«If the selector function is not provided, it returns selectors that assume their input is already an EntityState. If a selector function is provided, it returns selectors that accept the top-level state and will internally call your function to find the right slice.»
«Без аргумента getSelectors() отдаёт селекторы, которые ждут на входе EntityState. Если передать функцию — селекторы принимают RootState и сами достают нужный подстейт.»

Подводные камни

1. selectById(state, id) — cache размером 1.
Дефолтный createDraftSafeSelector — это lruMemoize({ maxSize: 1 }). Если вызываешь selectById(state, 'a'), затем selectById(state, 'b'), затем снова selectById(state, 'a') — кэш каждый раз инвалидируется (последний аргумент разный), и селектор пересчитывается на каждом вызове. Для списка компонентов, каждый из которых делает selectById(state, props.id) — это cache thrashing.
Решение: передать в options createSelector с weakMapMemoize:
import { createSelectorCreator, weakMapMemoize } from 'reselect' const wmSelector = createSelectorCreator(weakMapMemoize) const sel = booksAdapter.getSelectors( (s) => s.books, { createSelector: wmSelector as any } )
weakMapMemoize — кэш по каждому аргументу отдельно, «бесконечный» (WeakMap).

2. Globalized селекторы привязаны к пути. Если ты перенёс slice в state.features.books, а старые селекторы строились от s.books — они вернут undefined.ids и упадут с TypeError. Всегда пересоздавай globalized селекторы при рефакторе store.

3. Разделяй instance-ы селекторов. Если один и тот же selectById используется в 10 компонентах — они делят cache size=1. По одному .get(id) с разным id — ничего не кэшируется. Это reselect поведение, не RTK.

4. selectAll возвращает новый массив при каждом ids.map. Он мемоизирован по ids + entities. Если хоть одно изменилось — новый массив. Значит useSelector(selectAll) в React каждый раз rerenders ровно когда нужно, но не чаще. (С draft safe — тоже.)
