Adapter отдаёт готовые addOne / addMany / setOne / setMany / setAll / upsertOne / updateOne / removeOne / removeMany / removeAll. Внутри reducer'а достаточно вызвать booksAdapter.addOne(state, action.payload) — всё остальное (Immer-мутация ids и entities) сделает RTK.

Минимальный adapter

import { createEntityAdapter, createSlice } from '@reduxjs/toolkit' interface Book { id: string; title: string; price: number } const booksAdapter = createEntityAdapter<Book>() // по умолчанию selectId = (b) => b.id const booksSlice = createSlice({ name: 'books', initialState: booksAdapter.getInitialState(), // { ids:[], entities:{} } reducers: { bookAdded: booksAdapter.addOne, // прямо в reducers! bookUpdated: booksAdapter.updateOne, bookRemoved: booksAdapter.removeOne, booksReset: booksAdapter.setAll, }, })

Методы adapter'а — обычные Immer-reducer'ы. Сигнатура: (state, entity | PayloadAction<entity>). Работают и как autonomous reducer, и как reducers.foo.

Исходник — packages/toolkit/src/entities/create_adapter.ts

export function createEntityAdapter<T>( options: EntityAdapterOptions<T, EntityId> = {}, ): EntityAdapter<T, EntityId> { const { selectId, sortComparer, }: Required<EntityAdapterOptions<T, EntityId>> = { sortComparer: false, selectId: (instance: any) => instance.id, ...options, } const stateAdapter = sortComparer ? createSortedStateAdapter(selectId, sortComparer) : createUnsortedStateAdapter(selectId) const stateFactory = createInitialStateFactory(stateAdapter) const selectorsFactory = createSelectorsFactory<T, EntityId>() return { selectId, sortComparer, ...stateFactory, // getInitialState (урок 53) ...selectorsFactory, // getSelectors (урок 52) ...stateAdapter, // addOne, updateOne, ... } }

Разбор построчно:

  • defaults. Если не передан selectId — берётся b => b.id. Если нет sortComparer — false (отсюда ветка с unsorted).
  • выбор sorted/unsorted. Сортировка — это дополнительная стоимость в каждом addOne/updateOne, поэтому RTK выносит в отдельный файл sorted_state_adapter.ts (урок 51).
  • публичный объект. Spread трёх фабрик — в итоге adapter имеет addOne, setAll, getInitialState, getSelectors, selectId, sortComparer и т.д.

Подводные камни

1. selectId обязателен, если в объекте нет поля id. TypeScript на это ругается (первая перегрузка делает selectId required), а в runtime вы получите warning «The entity passed to the selectId implementation returned undefined» (utils.ts). Пример: книга с полем isbn — нужно createEntityAdapter<Book, string>({ selectId: b => b.isbn }).

2. CRUD-методы мутируют Immer draft. Внутри createStateOperator есть проверка isDraft(state). Если state уже draft — мутируют напрямую. Если plain — оборачивают в createNextState. Это значит: booksAdapter.addOne(plainEntityState, book) в тесте вернёт новый объект (immutably). А в reducer'е — мутирует draft (не возвращайте результат!).

3. Не вызывайте adapter-метод на plain state и потом не присваивайте:
В reducer'е неправильно писать state = booksAdapter.addOne(state, book) — переменная переприсваивается в local scope, state не меняется. Правильно: booksAdapter.addOne(state, book) (возвращает state, но мы его игнорим в reducer'е — мутация уже применена).
