Команда установки

Один пакет содержит всё: core RTK + RTK Query. Никаких отдельных установок для thunk/immer/reselect — они внутри.

npm install @reduxjs/toolkit react-redux
Что попадёт в node_modules (RTK alone)
  • @reduxjs/toolkit^2.5.0
  • redux^5.0.1 (внутри RTK)
  • immer^10.0.3 (внутри RTK)
  • reselect^5.1.0 (внутри RTK)
  • redux-thunk^3.1.0 (внутри RTK)
Peer dependencies
  • react^18 || ^19
  • react-redux^9.0.0 (для useSelector)

Помечено peer — RTK не тянет react/react-redux сам. Для не-React проектов peer не нужен.

Почему RTK = 17 kB, а не сумма зависимостей

Если бы RTK просто реэкспортировал redux/immer/reselect/thunk — итог был бы ~22 kB. Но RTK собирается единым ESM-бандлом с использованием rollup, и:

  • удаляются неиспользуемые ветки кода каждой зависимости (tree-shaking внутри RTK);
  • дублирующиеся хелперы (typeof, isPlainObject) переиспользуются;
  • terser сжимает имена переменных в production-бандле;
  • современный "modern.mjs" entry без полифиллов для legacy браузеров.

Итог: RTK суммарно меньше чем составляющие отдельно.

Tree-shaking RTK

Если вы используете только configureStore и createSlice, в финальный бандл попадут только они + immer (нужен для slice-reducers) + redux core. Но не RTK Query, не entityAdapter, не thunk middleware (хотя он подключён по умолчанию — bundler удалит его, если store не использует thunks).

Самое тяжёлое в RTK — RTK Query (~9 kB). Если вы его не используете — он автоматически выпадет из бандла (Vite/webpack tree-shake по ESM-структуре пакета).

peerDependencies — что значит

RTK не тянет react и react-redux в зависимости. Они объявлены как peerDependencies: ваше приложение должно установить их само. Это нужно чтобы:
  • RTK работал без React (Node.js, Svelte, Vue);
  • не было дублей React в node_modules;
  • версия React контролировалась вами, а не RTK.

Шпаргалка установки

// React + Redux Toolkit + RTK Query (типичный кейс) npm install @reduxjs/toolkit react-redux // Не-React (Node CLI, Vue, Svelte) npm install @reduxjs/toolkit // Vite-template (полностью настроенный TS-стартер) npx degit reduxjs/redux-templates/packages/vite-template-redux my-app // Next.js npx create-next-app --example with-redux my-app

Подводный камень: версии

RTK 2.x требует react-redux 9+ и redux 5+. Если у вас старый react-redux 7/8 — обновите его. RTK 2.x также требует Node 16+ для разработки и поддерживает только современные браузеры (ES2020). Для legacy используйте RTK 1.9.x.
