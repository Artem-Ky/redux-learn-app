Шпаргалка по configureStore (быстрая сверка)

Семь правил, которые нужно запомнить

  1. configureStore — обёртка над legacy_createStore. Автоматически: combineReducers (если передан object), applyMiddleware с 4 default middleware, composeWithDevTools, autoBatchEnhancer.
  2. Default middleware = thunk + immutableCheck + serializableCheck + actionCreatorCheck. Последние три — только в development. В production — один thunk.
  3. middleware — это callback: (getDefaultMiddleware) => getDefaultMiddleware().concat(myMw). Если вернёшь обычный массив [thunk, myMw] — в RTK 2.x TS ругнётся.
  4. enhancers — аналогично: (getDefaultEnhancers) => getDefaultEnhancers().concat(myEnh). Без вызова gde() — потеряешь autoBatch и middleware.
  5. Tuple — расширение Array с типизированными concat/prepend. Обычный [...].concat(mw) сужает тип до Middleware[] и ломает thunk-dispatch overload.
  6. preloadedState — hydration при старте. Если указан, reducer при первом вызове видит его вместо undefined, initialState slice не используется для этого куска.
  7. RootState = ReturnType<typeof store.getState>; AppDispatch = typeof store.dispatch. Хуки — через useSelector.withTypes<RootState>(). В рантайме identity, только TS-обёртка.

Форма middleware-callback'а

// ✅ правильно — gdm() возвращает Tuple configureStore({ reducer, middleware: (gdm) => gdm().concat(myLogger), }) // ✅ можно prepend — listener ВСЕГДА первым configureStore({ reducer, middleware: (gdm) => gdm().prepend(listenerMw.middleware), }) // ✅ отключить одну проверку configureStore({ reducer, middleware: (gdm) => gdm({ serializableCheck: false }), }) // ❌ обычный массив — потеряет typings в RTK 2.x configureStore({ reducer, middleware: () => [thunk, myLogger], })
