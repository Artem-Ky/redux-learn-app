Теория — 49. addOne vs setOne vs upsertOne

Цитата — RTK docs

«addOne will do nothing if the provided entity already exists in the state. setOne will overwrite an existing entity if it exists. upsertOne will perform a shallow update, preserving any existing fields that are not in the payload.»
— Should I add, set, or upsert my entity?
«addOne ничего не сделает, если entity уже есть в state. setOne полностью перезапишет существующую запись. upsertOne выполнит shallow merge, сохраняя поля, которых нет в payload.»

function addOneMutably(entity: T, state: R): void { const key = selectIdValue(entity, selectId) if (key in state.entities) { return // ← no-op! } state.ids.push(key) ;(state.entities)[key] = entity } function setOneMutably(entity: T, state: R): void { const key = selectIdValue(entity, selectId) if (!(key in state.entities)) { state.ids.push(key) } ;(state.entities)[key] = entity // ← полная замена } function upsertOneMutably(entity: T, state: R): void { return upsertManyMutably([entity], state) } function upsertManyMutably(newEntities, state) { const [added, updated] = splitAddedUpdatedEntities(newEntities, selectId, state) addManyMutably(added, state) updateManyMutably(updated, state) // ← идёт через updateOne → Object.assign }

Когда какой использовать

  • addOne — только создание, не важна идемпотентность. (Пример: принять сообщение из websocket — если уже есть, не перезаписываем.)
  • setOne — «server is the truth». Пришёл свежий объект с бэка, весь — клиентский заменяется (без merge).
  • upsertOne — patch-семантика. Пришёл кусочек объекта (id + изменения) — подмешать.

Подводные камни

1. upsertOne — только shallow merge. Внутри вызывается updateOne, который делает Object.assign({}, original, changes). Для nested полей: { profile: { name: 'Bob' } } затрёт весь profile, включая age. Деталь — урок 50.

2. setOne не двигает позицию в ids. Даже если это «новая» версия, её место в массиве ids остаётся прежним (для unsorted adapter'а). Чтобы обновить порядок — используй sortComparer (урок 51) или remove + add руками.

3. addOne тихо игнорирует дубликат. Никакого warning в консоли. Если ожидаешь, что addOne «всегда сработает» — используй upsertOne. Это классическая ловушка: код кажется рабочим, но одно поле не обновилось.
