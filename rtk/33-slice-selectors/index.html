selectors внутри createSlice

В RTK 2.x появилось поле selectors в createSlice. Селекторы получают state slice'а (не root state!), а наружу экспортируются как обычные селекторы от root state. RTK сам прокидывает state[name] под капотом.

const counterSlice = createSlice({ name: 'counter', initialState: { value: 0, history: [] }, reducers: { ... }, selectors: { // state — это slice state, НЕ root selectValue: (state) => state.value, selectIsEven: (state) => state.value % 2 === 0, selectHistoryLength: (state) => state.history.length, selectHistoryItem: (state, index: number) => state.history[index], // с аргументом } }) // Использование — селекторы автоматически работают на ROOT state: counterSlice.selectors.selectValue(rootState) // rootState.counter.value counterSlice.selectors.selectIsEven(rootState) // true / false counterSlice.selectors.selectHistoryItem(rootState, 0) // первая запись

Теория — slice.selectors

Из доки

«You can pass an object of selectors to createSlice. The selectors will be defined as accepting the slice's state, but exported in slice.selectors as accepting the root state.»
— createSlice — selectors
«Можно передать в createSlice объект селекторов. Селекторы определяются как принимающие state slice'а, но в slice.selectors экспортируются как принимающие root state.»

Как RTK преобразует селекторы

// Внутри createSlice примерно так: const wrappedSelectors = {} for (const [name, sel] of Object.entries(opts.selectors)) { wrappedSelectors[name] = (rootState, ...args) => sel(rootState[opts.name], ...args) } // rootState[opts.name] — берётся ключ из reducerPath или name

selectSlice — built-in helper

У каждого slice автоматически есть slice.selectSlice — селектор, возвращающий весь state slice'а:

counterSlice.selectSlice(rootState) // rootState.counter // Если кастомизировать reducerPath, это учитывается: const slice = createSlice({ name: 'counter', reducerPath: 'features.counter', // (на самом деле это плоский ключ; вложенность через combineSlices) ... })

getSelectors() — для гибкости

Если slice не знает, где он будет лежать в root (например, lazy-injected), используйте slice.getSelectors(stateSelector). Это «фабрика», которая возвращает селекторы, получающие state через ваш stateSelector:

const { selectValue, selectIsEven } = counterSlice.getSelectors( (rootState: RootState) => rootState.someDeeplyNested.counter ) selectValue(rootState) // rootState.someDeeplyNested.counter.value

Селекторы с аргументами

selectors: { selectTodoById: (state, id: string) => state.byId[id], selectByCategory: (state, cat: string) => state.items.filter(t => t.cat === cat), } // Использование: slice.selectors.selectTodoById(rootState, 'abc')

Memoization

slice.selectors не мемоизированы. Если селектор делает дорогую трансформацию (filter / map / reduce / sort), оборачивайте его в createSelector из reselect (детально в секции E).
import { createSelector } from '@reduxjs/toolkit' selectors: { selectVisibleTodos: createSelector( (state: TodoState) => state.items, (state: TodoState) => state.filter, (items, filter) => items.filter(getFilterFn(filter)) ), }

Использование в React-Redux

function Counter() { const value = useAppSelector(counterSlice.selectors.selectValue) const isEven = useAppSelector(counterSlice.selectors.selectIsEven) return ... }

Селектор передаётся в useSelector как обычный getter — никакой магии.

Подводный камень: reducerPath не равен пути в state

slice.selectors берёт state через state[slice.reducerPath]. Если в configureStore вы записали slice под другим ключом, селекторы НЕ будут работать. Используйте slice.getSelectors(rootSelector) в этом случае.
