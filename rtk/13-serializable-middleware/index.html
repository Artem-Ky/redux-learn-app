Что считается «не-сериализуемым»

Redux ожидает, что весь state и payload экшенов сериализуются в JSON. Это нужно для: time-travel debugging в DevTools, persistence, SSR, hydration. Middleware ловит запретные значения и кричит warning в консоль (не throw).

1. Date

dispatch({ type: 'set/date', payload: new Date() })

2. Map

dispatch({ type: 'set/map', payload: new Map([['a', 1]]) })

3. Promise

dispatch({ type: 'set/promise', payload: Promise.resolve(1) })

4. Function

dispatch({ type: 'set/fn', payload: () => 'hi' })

5. class instance

class User {} dispatch({ type:'set/cls', payload: new User() })

6. Plain object — OK

dispatch({ type: 'set/plain', payload: { a: 1, b: [2,3] } })
— ожидание dispatch —

Откройте также браузерную консоль — middleware выводит точный путь до невалидного значения, например action.payload.date.

Опции конфигурации

Теория — serializable check

Из доки

«It's recommended that you only put plain serializable values, like plain objects, arrays, and primitives, into your Redux store. There are some rare cases where you may need to bend this rule... For those cases, the middleware accepts options to allow non-serializable values.»
— serializabilityMiddleware
«Рекомендуется хранить в Redux store только простые сериализуемые значения — обычные объекты, массивы и примитивы. В редких случаях это правило приходится нарушать… Для них middleware принимает опции, позволяющие пропускать не-сериализуемые значения.»

Что считается сериализуемым

  • null, undefined
  • boolean, number, string, bigint
  • plain object (созданный через {} или Object.create(null))
  • Array из сериализуемых значений

Что НЕ сериализуемо

  • Date (нужно .toISOString() при сохранении)
  • Map, Set, WeakMap, WeakSet
  • Promise, любой другой Thenable
  • Function, class instance, Symbol
  • DOM-элементы, File, Blob, FormData
  • Error (исключение — RTK Query сам сериализует ошибки в SerializedError)

Опции

interface SerializableStateInvariantMiddlewareOptions { isSerializable?: (value: unknown) => boolean // custom check getEntries?: (value: unknown) => [string, unknown][] ignoredActions?: string[] // типы actions для пропуска ignoredActionPaths?: (string | RegExp)[] // пути в action для пропуска ignoredPaths?: (string | RegExp)[] // пути в state для пропуска warnAfter?: number // warn если check > N мс ignoreState?: boolean // не проверять state, только actions ignoreActions?: boolean // не проверять actions, только state disableCache?: boolean // не кешировать результат WeakMap }

Реальный пример: redux-persist

redux-persist использует action persist/PERSIST, который содержит функцию-callback. Без настройки middleware кричит warning. Решение — добавить путь в ignoredActions:

import { PERSIST, REHYDRATE, FLUSH, PURGE, REGISTER, PAUSE } from 'redux-persist' configureStore({ reducer, middleware: (gdm) => gdm({ serializableCheck: { ignoredActions: [PERSIST, REHYDRATE, FLUSH, PURGE, REGISTER, PAUSE] } }) })

Подводный камень: Date в payload

Очень частая ошибка: dispatch(addEvent({ date: new Date() })). Решение — передавайте ISO-строку: dispatch(addEvent({ date: new Date().toISOString() })). В компоненте парсите обратно: new Date(event.date).
