«Redux Style Guide» — рекомендуемая структура

Redux team рекомендует feature folders + "ducks" pattern. Это значит: одна фича = одна папка, в которой лежат и UI, и slice, и selectors, и тесты. Никаких папок типа actions/, reducers/, selectors/.

✓ Рекомендуемая структура

src/ ├── app/ │ ├── store.ts # configureStore, RootState, AppDispatch │ └── hooks.ts # useAppDispatch, useAppSelector │ ├── features/ # КАЖДАЯ ФИЧА = СВОЯ ПАПКА │ ├── counter/ │ │ ├── counterSlice.ts # slice + actions + selectors │ │ ├── Counter.tsx # UI компонент │ │ ├── Counter.test.tsx │ │ └── counterSlice.test.ts │ │ │ ├── todos/ │ │ ├── todosSlice.ts │ │ ├── TodoList.tsx │ │ ├── TodoItem.tsx │ │ ├── AddTodo.tsx │ │ └── api.ts # thunks / API calls │ │ │ └── user/ │ ├── userSlice.ts │ ├── LoginForm.tsx │ ├── Profile.tsx │ └── authThunks.ts │ ├── api/ # RTK Query (если используется) │ ├── apiSlice.ts │ └── endpoints/ │ ├── posts.ts │ └── users.ts │ ├── shared/ # переиспользуемые компоненты │ ├── ui/ │ └── utils/ │ └── App.tsx

✗ Антипаттерны

src/ ├── actions/ # ✗ группировка по типу — устарело │ ├── counterActions.ts │ └── userActions.ts ├── reducers/ # ✗ всё разбросано │ ├── counterReducer.ts │ └── userReducer.ts ├── selectors/ # ✗ менять одну фичу = править 4 папки └── components/ # ✗ компоненты оторваны от логики

✓ counterSlice.ts (модуль = "ducks" pattern)

import { createSlice, type PayloadAction } from '@reduxjs/toolkit' interface CounterState { value: number } const initialState: CounterState = { value: 0 } const counterSlice = createSlice({ name: 'counter', initialState, reducers: { increment: (s) => { s.value++ }, addBy: (s, a: PayloadAction<number>) => { s.value += a.payload }, }, selectors: { selectValue: (s) => s.value, selectIsEven: (s) => s.value % 2 === 0, }, }) // Actions, selectors, reducer — ВСЁ из одного файла export const { increment, addBy } = counterSlice.actions export const { selectValue, selectIsEven } = counterSlice.selectors export default counterSlice.reducer

✓ Counter.tsx (UI рядом со slice)

import { useAppDispatch, useAppSelector } from '@/app/hooks' import { increment, selectValue, selectIsEven } from './counterSlice' export function Counter() { const value = useAppSelector(selectValue) const isEven = useAppSelector(selectIsEven) const dispatch = useAppDispatch() return ( <div> <p>{value} ({isEven ? 'even' : 'odd'})</p> <button onClick={() => dispatch(increment())}>+</button> </div> ) }

✓ store.ts

import { configureStore, combineSlices } from '@reduxjs/toolkit' import counterSlice from '@/features/counter/counterSlice' import todosSlice from '@/features/todos/todosSlice' import userSlice from '@/features/user/userSlice' const rootReducer = combineSlices(counterSlice, todosSlice, userSlice) .withLazyLoadedSlices<LazyState>() export const store = configureStore({ reducer: rootReducer }) export type RootState = ReturnType<typeof store.getState> export type AppDispatch = typeof store.dispatch

✓ hooks.ts (типизированные)

import { useDispatch, useSelector } from 'react-redux' import type { RootState, AppDispatch } from './store' export const useAppDispatch = useDispatch.withTypes<AppDispatch>() export const useAppSelector = useSelector.withTypes<RootState>()

Теория — Redux Style Guide

Из доки

«Structure files as feature folders, with all logic for a given feature in a single folder. This includes slices, components, selectors, and tests. Avoid actions/, reducers/ folders.»
— Redux Style Guide
«Структурируйте файлы по feature-папкам, держа всю логику конкретной фичи в одной папке. Это включает slice'ы, компоненты, селекторы и тесты. Избегайте папок actions/, reducers/.»

Преимущества feature-folder подхода

  1. Изменения локализованы: добавление поля state требует править 1 папку, не 4.
  2. Удобный code-split: React.lazy(() => import('@/features/counter')) даст и UI, и reducer за один import.
  3. Тесты рядом: counterSlice.test.ts в той же папке — легко найти.
  4. Удаление фичи: rm -rf features/oldFeature — и всё, никаких хвостов.

Что класть в slice-файл

  • Сам createSlice + типы (CounterState).
  • Локальные thunks через create.asyncThunk или отдельный createAsyncThunk.
  • Селекторы — встроенные через selectors или отдельные createSelector.
  • Дефолтный экспорт — slice.reducer.
  • Named экспорты — actions и selectors.

Что НЕ класть в slice

  • UI-компоненты — отдельные файлы.
  • API-функции fetch — отдельный api.ts.
  • Утилиты, не относящиеся к state — в shared/utils/.

"Re-ducks" pattern для крупных фич

Если фича большая, можно разбить на под-файлы:

features/orders/ ├── # index.ts — re-exports ├── ordersSlice.ts ├── ordersSelectors.ts ├── ordersThunks.ts ├── orderTypes.ts ├── OrderList.tsx ├── OrderDetails.tsx └── api.ts

Когда отступить от паттерна

  • Очень малая фича (2 reducer'а) — можно держать в одном файле без отдельной папки.
  • Кросс-фичный slice — например, theme/notifications/modals — в features/ui/ или прямо в app/.
  • RTK Query — отдельная папка api/, потому что endpoints — это разные «фичи», но один apiSlice.
