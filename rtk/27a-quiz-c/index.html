Шпаргалка по createAction/createReducer (быстрая сверка)

Шесть правил, которые нужно запомнить

  1. createAction(type) возвращает функцию с тремя полями на ней: .type, .match(action) (type-guard), .toString() = type. В template literals `${myAction}` → строка type.
  2. createAction<Payload>(type) — generic. Без prepare payload = первый аргумент creator'а.
  3. createAction(type, prepare) — prepare получает аргументы creator'а и ОБЯЗАН вернуть { payload } плюс опционально { meta, error }. Чаще всего: сгенерировать id: nanoid() внутри prepare, чтобы reducer оставался чистым.
  4. nanoid() по умолчанию = 21 символ, URL-safe, криптографически случайный (использует crypto.getRandomValues). Длину задаём: nanoid(10).
  5. Порядок в builder строгий: сначала все addCase, потом все addMatcher, и только потом ОДИН addDefaultCase. Нарушишь — runtime error.
  6. Immer-правило номер один: ИЛИ мутируй draft, ИЛИ верни новое значение — но не оба сразу. Иначе: «An immer producer returned a new value *and* modified its draft».

Точная сигнатура action creator'а

// Вариант 1 — простой: payload = аргумент creator'а const add = createAction<number>('counter/add') add(5) // → { type: 'counter/add', payload: 5 } // Вариант 2 — prepare: нормализация входа, добавление meta const addTodo = createAction('todos/add', (text: string) => ({ payload: { id: nanoid(), text, done: false }, meta: { at: Date.now() }, })) addTodo('купить хлеб') // → { type: 'todos/add', payload: { id: '...', text: '...', done: false }, meta: { at: 1234 } }

Поток в builder (addCase → addMatcher → addDefaultCase)

createReducer(initial, (b) => { b .addCase(userLoggedIn, (s, a) => { /* ... */ }) // конкретный type .addCase(userLoggedOut, (s) => { /* ... */ }) .addMatcher(isUserAction, (s, a) => { /* ... */ }) // ПОСЛЕ всех addCase .addMatcher(isRejected, (s, a) => { /* ... */ }) .addDefaultCase((s, a) => { /* fallback */ }) // ОДИН раз, в самом конце }) // Для action X выполняются ВСЕ совпавшие: матчинг через addCase сработает, // потом ВСЕ addMatcher, у кого predicate true. addDefaultCase — только // если не совпал ни один addCase/addMatcher.

Immer — правила и подводные камни

// ✅ ОК — только мутация (s) => { s.count += 1 } // ✅ ОК — только return (часто для замены всего state или для примитивов) (s) => ({ count: s.count + 1 }) // ❌ ERROR — мутация И return одновременно (s) => { s.count += 1; return { ...s, extra: 42 } } // ❌ Proxy revoked — драфт нельзя сохранять вне reducer'а let saved; (s) => { saved = s } // 🔍 current(draft) — plain snapshot в текущий момент (безопасно логировать) // 🔍 original(draft) — ссылка на исходный объект до мутаций // 🔍 isDraft(value) — true если это Immer Proxy
