Теория — slice.actions и литеральные типы

Из доки

«Each action creator generated by createSlice has the same name as the reducer function, and is paired with an action type string of the form 'sliceName/reducerName'.»
— createSlice — return value
«Каждый action creator, сгенерированный createSlice, имеет то же имя, что и reducer-функция, и связан со строкой action type вида 'sliceName/reducerName'.»

Литеральный тип action.type

В TS благодаря const-assertions и шаблонным литеральным типам, auth.actions.loginRequest.type имеет тип 'auth/loginRequest', а не просто string. Это даёт точное сужение в reducer'ах:

type Slice = typeof authSlice type Actions = ReturnType<Slice['actions'][keyof Slice['actions']]> // Actions = // | { type: 'auth/loginRequest', payload: undefined } // | { type: 'auth/loginSuccess', payload: { name: string } } // | { type: 'auth/loginFailure', payload: string } // | { type: 'auth/logout', payload: undefined } // Это ↓ — discriminated union, идеально для switch: switch (action.type) { case 'auth/loginSuccess': action.payload.name // TS знает: { name: string } }

.match — type-guard на каждом actionCreator

const { loginSuccess } = authSlice.actions if (loginSuccess.match(action)) { action.type // 'auth/loginSuccess' action.payload // { name: string } } // Внутри listenerMiddleware listener.startListening({ matcher: loginSuccess.match, effect: (action, api) => { api.getState().auth // типизирован action.payload.name // string } })

Деструктуризация — стандарт

// 99% кода RTK: export const { loginRequest, loginSuccess, loginFailure, logout } = authSlice.actions export default authSlice.reducer // Импорт в компоненте: import { loginSuccess } from './authSlice'

Что НЕ генерируется

  • Действия для extraReducers — это внешние actions, не свои.
  • Селекторы (если не указаны через опцию selectors).
  • Actions для thunks — нужно использовать createAsyncThunk или RTK 2.x reducerCreators (урок 32).

Подводный камень: переименование reducerKey ломает всё

Если вы переименуете loginRequest в requestLogin, action.type поменяется на 'auth/requestLogin'. Все existing actions в localStorage / persisted state станут не распознаваемыми. Если нужно сохранить старый action.type — используйте отдельный createAction('auth/loginRequest') и подключите через extraReducers.

actions vs caseReducers

slice.actions — это action creator'ы (функции, возвращающие action). slice.caseReducers — это оригинальные reducer-функции (без обёртки Immer). Полезно для unit-тестов:

// Тест без Immer — на оригинальных функциях: test('increment', () => { const state = { value: 0 } authSlice.caseReducers.loginSuccess(state, { type: '', payload: { name: 'A' } }) expect(state.user.name).toBe('A') // state мутирован }) // Тест через slice.reducer — Immer применит produce: test('increment via reducer', () => { const next = authSlice.reducer(initial, loginSuccess({ name: 'A' })) expect(next.user.name).toBe('A') })
