6 типичных ошибок + как их избежать

Immer делает код удобным, но прячет сложность Proxy. Вот грабли, на которые наступают большинство разработчиков. Все примеры запускаются вживую — смотрите результат.

⚠ 1. Нельзя одновременно мутировать ДРАФТ и возвращать новый объект
(state, action) => { state.value += 1 // ← мутация draft return { ...state, extra: 42 } // ← И return } // Error: An immer producer returned a new value *and* modified its draft.
—
✓ 2. Или мутация, ИЛИ return — но не оба
// Вариант A: только мутация (state, action) => { state.value += 1 } // Вариант B: только return (замена всего state) (state, action) => { return { value: state.value + 1 } } // Вариант C: return undefined — означает "оставить без изменений" (state, action) => { return undefined } // то же, что без return

✓ работает корректно

⚠ 3. Draft работает только в пределах reducer'а

Draft — это Proxy с коротким lifecycle. Сохранить его вне reducer'а нельзя — получите revoked proxy error.

let savedDraft (state, action) => { savedDraft = state // ← плохо } // Позже: savedDraft.value = 5 // TypeError: Cannot perform 'get' on a proxy that has been revoked
—
⚠ 4. Нельзя возвращать undefined явно + мутировать

Да, это запутанно. TS-правило: либо ничего не возвращать, либо возвращать значение.

(state) => { state.x = 1 return undefined // ← ок, равнозначно отсутствию return } (state) => { state.x = 1 return null // ← ОШИБКА: null !== undefined, Immer сочтёт это return'ом }
—
⚠ 5. current() vs original() — знайте различие
import { current, original } from '@reduxjs/toolkit' (state, action) => { state.count = 10 original(state.count) // исходное значение ДО всех мутаций в этом reducer'е current(state.count) // snapshot draft'а в данный момент state.count = 20 current(state.count) // 20 original(state.count) // оригинал (0 или initial) }
—
⚠ 6. console.log(state) выведет Proxy, не читаемый JSON

В консоли увидите Proxy {...}, а не полный объект. Используйте current(state) для snapshot'а.

(state, action) => { // ✗ плохо: увидите Proxy console.log('state:', state) // ✓ хорошо: увидите plain object console.log('state:', current(state)) console.log('state:', JSON.parse(JSON.stringify(state))) }
—
✓ 7. isDraft() — проверка Proxy

Если нужно в селекторе/helper понять, что пришло — draft или plain:

import { isDraft } from '@reduxjs/toolkit' isDraft(draft) // true внутри reducer'а isDraft(plainState) // false
—
Теория — правила Immer в RTK

Из доки RTK

«You can never mix "mutating" the draft with returning a new value. Either return a completely new value, or mutate the existing draft — but not both.»
— Immer Reducers — Rules
«Нельзя смешивать «мутирование» draft'а с возвратом нового значения. Либо возвращайте совершенно новое значение, либо мутируйте существующий draft — но не оба варианта одновременно.»

Правила в одном списке

  1. Не смешивайте мутацию draft и return.
  2. Не сохраняйте draft вне reducer'а (он будет revoked).
  3. Для примитивов (state = 0) обязательно return.
  4. Используйте current(state) для snapshot вместо console.log(state).
  5. Immer не замораживает рекурсивно: если вы хранили Map/Set, это остаётся изменяемым.
  6. enablePatches(), enableMapSet() — уже включены в RTK по умолчанию (с v2).

current() — как работает

import { current } from '@reduxjs/toolkit' (state, action) => { state.items.push(1) state.items.push(2) // Нужно "заморозить" состояние на этот момент, чтобы передать куда-то: const snapshot = current(state) // { items: [1, 2] } — plain object log(snapshot) // безопасно state.items.push(3) // snapshot всё ещё [1, 2], state.items уже [1, 2, 3] }

original() — как работает

import { original } from '@reduxjs/toolkit' (state, action) => { const origItems = original(state.items) // ссылка на ОРИГИНАЛ (до reducer'а) state.items.push(1) original(state.items) === origItems // всё ещё true original(state.items) === state.items // false (state.items уже draft) }

Разница: current — snapshot текущего draft'а (plain copy). original — ссылка на оригинальный объект до изменений.

Подводный камень: defaultCase внутри Immer

В addDefaultCase вы получаете draft. Если вы ничего не меняете и ничего не возвращаете — Immer вернёт оригинал (та же ссылка). Это корректно. Не пишите return state вручную — это затянет перестройку объекта.

enableMapSet — использование Map/Set в state

Immer по умолчанию не поддерживает Map/Set (как драфт). RTK активирует enableMapSet() глобально. Теперь можно:

const slice = createSlice({ name: 'cache', initialState: { entries: new Map() }, reducers: { setEntry: (s, a) => { s.entries.set(a.payload.key, a.payload.value) // works! } } })

НО! Map/Set не сериализуемы. serializableStateInvariantMiddleware выдаст warning. Для production лучше хранить как Record<string, T>.
