Теория — что configureStore делает изнутри

Дословно из доки

«configureStore wraps createStore to provide simplified configuration options and good defaults. It can automatically combine your slice reducers, adds whatever Redux middleware you supply, includes redux-thunk by default, and enables use of the Redux DevTools Extension.»
— configureStore API
«configureStore оборачивает createStore, предоставляя упрощённые опции конфигурации и хорошие умолчания. Он может автоматически собрать ваши slice-reducers, добавить любые предоставленные Redux middleware, по умолчанию включает redux-thunk и активирует Redux DevTools Extension.»

Псевдокод реализации (упрощённо)

export function configureStore(options) { const { reducer = undefined, middleware, devTools = true, preloadedState = undefined, enhancers = undefined, } = options // 1. Собрать rootReducer let rootReducer if (typeof reducer === 'function') { rootReducer = reducer } else if (isPlainObject(reducer)) { rootReducer = combineReducers(reducer) } // 2. Получить middleware (default + кастомные) let finalMiddleware = typeof middleware === 'function' ? middleware(getDefaultMiddleware) : getDefaultMiddleware() // 3. Собрать enhancers (default + кастомные) let finalEnhancers = typeof enhancers === 'function' ? enhancers(getDefaultEnhancers) : getDefaultEnhancers() // 4. composeWithDevTools если включён const composer = devTools ? composeWithDevTools({ trace: false, ...devTools }) : compose // 5. Создать store через ванильный createStore return createStore( rootReducer, preloadedState, composer(applyMiddleware(...finalMiddleware), ...finalEnhancers) ) }

Реальный исходник configureStore.ts (~150 строк)

Что добавляется бесплатно

  • thunk middleware — async actions без отдельной установки.
  • immutableStateInvariantMiddleware — ловит мутации state в reducer (только dev).
  • serializableStateInvariantMiddleware — ловит не-сериализуемые payload (только dev).
  • actionCreatorInvariantMiddleware — ловит dispatch(actionCreator) вместо dispatch(actionCreator()).
  • autoBatchEnhancer — батчит уведомления подписчиков.
  • composeWithDevToolsExtension — DevTools подключается без условий typeof window.

Подводный камень: тип store

configureStore возвращает не Store, а EnhancedStore. Разница: EnhancedStore содержит типы middleware и enhancers в дженериках. При вытаскивании типа делайте type AppDispatch = typeof store.dispatch и type RootState = ReturnType<typeof store.getState> — получите правильную сигнатуру с поддержкой thunk dispatch.

Что вы теряете без configureStore

Если вы используете createStore напрямую — у вас нет: thunk (нужно applyMiddleware(thunk)), invariant-проверок (баги пройдут в production), DevTools (нужна conditional проверка __REDUX_DEVTOOLS_EXTENSION__), batching (каждый dispatch — отдельный render). Команда Redux рекомендует: не используйте createStore напрямую, оно поэтому переименовано в legacy_createStore.
