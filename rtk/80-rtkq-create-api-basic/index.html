Сигнатура — createApi

// @reduxjs/toolkit/query const pokeApi = createApi({ reducerPath: 'pokeApi', // ключ в корне rootState baseQuery: fetchBaseQuery({ // одна "функция-транспорт" baseUrl: 'https://pokeapi.co/api/v2/', fetchFn: mockFetch as typeof fetch, // 👀 overrides global fetch — для тестов / моков }), endpoints: (build) => ({ getPokemonByName: build.query<PokemonDto, string>({ query: (name) => `pokemon/${name}`, // → pokeApi.endpoints.getPokemonByName.initiate('pikachu') }), }), }) export const { useGetPokemonByNameQuery } = pokeApi // ← в /query/react; тут мы этим не пользуемся export const pokeApi // ← сам api-объект: reducer, middleware, util, endpoints

Что возвращает createApi — расшифровка

  • api.reducer — один slice-reducer под ключ reducerPath, надо положить в configureStore({ reducer: { [api.reducerPath]: api.reducer } }).
  • api.middleware — middleware, которое выполняет baseQuery и управляет cache; обязательно через .concat(api.middleware).
  • api.endpoints.X.initiate(arg) — thunk action creator. store.dispatch(...) запустит запрос; возвращаемый promise имеет .unsubscribe().
  • api.endpoints.X.select(arg) — селектор: select(arg)(state) → { data, error, status, requestId, …}.
  • api.endpoints.X.matchPending / matchFulfilled / matchRejected — matcher'ы (как у createAsyncThunk).
  • api.util — prefetch, invalidateTags, updateQueryData, resetApiState.
  • useGetXQuery — в /query/react: hook (позже, урок 82).

Mock-сеть (наш тестовый fetchFn)

async function mockFetch(input: RequestInfo): Promise<Response> { await new Promise((r) => setTimeout(r, 400)) const url = typeof input === 'string' ? input : (input as Request).url const name = url.split('pokemon/')[1] if (!POKEDEX[name]) { return new Response(JSON.stringify({ error: 'Not found' }), { status: 404, headers: { 'content-type': 'application/json' } }) } return new Response(JSON.stringify(POKEDEX[name]), { status: 200, headers: { 'content-type': 'application/json' } }) }

fetchBaseQuery использует глобальный fetch, но мы его override через fetchFn — док. Полезно для: unit-тестов, SSR (node-fetch), локальной разработки без бэка.

Подводные камни

1. Импорт из правильного entry-point.
import { createApi } from '@reduxjs/toolkit/query' — ядро. @reduxjs/toolkit/query/react — + авто-hooks. Импортировать одновременно из обоих = два разных модуля createApi, middleware из одного не увидит reducer из другого. Ошибка на ~3 часа дебага.

2. Мок через fetchFn нужно дать типу as typeof fetch.
fetchBaseQuery ожидает фетч с полной сигнатурой DOM-fetch; наш mock типизирован проще. Каст безопасен, если мы не используем всю сигнатуру.

3. query: (arg) => url — возвращает строку или FetchArgs.
Строка = GET с baseUrl + строка. Объект { url, method, body, params, headers } — для POST/PATCH + подробностей. Ошибка: вернуть `${baseUrl}/posts` — baseUrl склеится дважды.

4. Один endpoint = один cache-тип по cacheKey от args.
Два вызова getPokemonByName('pikachu') → один cache-entry; getPokemonByName('pikachu') и getPokemonByName('charizard') — два разных cache-entry. Как строится ключ — см. урок 83.
