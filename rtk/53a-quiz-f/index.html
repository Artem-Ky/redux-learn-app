Шпаргалка по createEntityAdapter (быстрая сверка)

Шесть правил, которые нужно запомнить

  1. EntityState<T, Id> = { ids: Id[]; entities: Record<Id, T> }. Всегда оба поля, всегда синхронны после любого adapter-метода.
  2. addOne(state, entity) — no-op, если selectId(entity) уже в entities. setOne — полная замена (теряете поля, которых нет в payload). upsertOne — shallow merge: Object.assign({}, old, new).
  3. updateOne({ id, changes }) делает Object.assign(entity, changes). Nested объект в changes полностью заменит прежний nested — всё, что не перечислено в changes.profile, теряется.
  4. selectId — когда id поле называется не id (например isbn, uuid). sortComparer — пересчитывается при каждом add/update/upsert, держит ids отсортированным.
  5. adapter.getSelectors() — local: селекторы принимают EntityState напрямую. adapter.getSelectors(s => s.books) — globalized: принимают RootState.
  6. getInitialState(additionalState, entities?). Второй аргумент (seed entities) появился в RTK 2.x. Поля из additionalState ложатся НА ВЕРХ, рядом с ids/entities — плоская структура.

Где использовать что

// 1. Базовый adapter + slice const booksAdapter = createEntityAdapter<Book>() const booksSlice = createSlice({ name: 'books', initialState: booksAdapter.getInitialState({ loading: 'idle', error: null }), reducers: { bookAdded: booksAdapter.addOne }, }) // 2. Custom selectId + sortComparer const adapter = createEntityAdapter<Book, string>({ selectId: b => b.isbn, sortComparer: (a, b) => a.title.localeCompare(b.title), }) // 3. getSelectors: globalized const sel = adapter.getSelectors<RootState>(s => s.books) sel.selectAll(state) // Book[] отсортированный sel.selectById(state, id) sel.selectTotal(state) // ids.length
