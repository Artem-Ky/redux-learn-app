Теория — 103. Сравнение RTKQ / React Query / SWR

Цитата — RTKQ Comparison

«RTK Query is tightly integrated with Redux. If you already use Redux, it's a natural extension. If you don't have Redux in your project, React Query is a more lightweight alternative. SWR is the lightest, focused on simple GET caching with a Vercel/Next.js flavor.»
— RTKQ · Comparison

Архитектурные различия

── RTKQ ── Cache лежит в Redux store (state.api.queries). Reducer-based, DevTools time-travel, middleware chain. Паттерн: api.endpoints.X.initiate(arg) → thunk → baseQuery → reducer update. ── React Query ── Cache — свой QueryClient (отдельное хранилище, не Redux). Queries идентифицируются по queryKey: unknown[] (массив). Паттерн: queryClient.fetchQuery({ queryKey, queryFn }) → in-memory cache. ── SWR ── Cache — Map в useSWRConfig() контексте. Фокус на simplicity: key = URL строка, fetcher = () => fetch(url).json(). Паттерн: useSWR(key, fetcher) → stale-while-revalidate — сразу stale + обновляем.

Когда что выбрать — decision tree

У тебя уже есть Redux? ├── ДА → RTKQ. Дополнительный бандл почти 0, единый store, DevTools. └── НЕТ: Нужен ли Redux для client-state (complex UI state)? ├── ДА → установи RTK → RTKQ. └── НЕТ: Тебе нужно больше чем fetch (WS, GraphQL, custom transport)? ├── ДА → React Query (или RTKQ). └── НЕТ: Next.js / Vercel? ├── ДА → SWR (из коробки, vercel-friendly). └── НЕТ → React Query (стандарт индустрии). Специальные кейсы: - Нужна normalized cache + entity adapter → RTKQ. - Нужен super-low bundle → SWR. - Нужен DevTools с time-travel → RTKQ. - Работаете как команда на React без Redux → React Query.

Плюсы RTKQ (итого)

  • Единый store с UI state — нет двух источников истины.
  • DevTools с time-travel (React Query DT отдельная, SWR нет).
  • Tag-based инвалидация семантически понятнее queryKey-based.
  • TypeScript inference для endpoints лучший в классе.
  • Code splitting endpoints через injectEndpoints.

Минусы RTKQ

  • Бандл больше чем SWR (~17kb vs 4kb).
  • Требует Redux (хоть и один раз настроенный) — лишний learning curve без RTK.
  • Нет «stale-while-revalidate» по дефолту — данные протухают только через refetchOnMountOrArgChange: N или invalidation.
  • Кэш плоский (queries Map), normalized — вручную через createEntityAdapter.
  • Меньше community плагинов чем у React Query.
