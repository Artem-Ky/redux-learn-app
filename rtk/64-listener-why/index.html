Сценарий: после loginSuccess → fetch profile + start websocket + analytics

Типичная задача: на один action нужно запустить три независимых reactive-побочки. Ниже три реализации — thunk-цепочка, sagas (псевдокод), listenerMiddleware. Кнопка «Login» запускает живую реализацию на listener'ах — видно фазы в консоли.

// 2. redux-saga: мощная reactive-модель через generator'ы. // Плюсы: takeLatest/fork/race/cancel — всё из коробки. // Минусы: +17kB, свой язык effects, отдельная ментальная модель, // typescript-поддержка исторически слабая. function* afterLoginSaga() { while (true) { const { payload } = yield take(loginSuccess.type) yield all([ call(fetchProfile, payload.id), fork(watchWebsocket, payload.token), call(track, 'login', payload.id), ]) } }

Официальное определение

«The listener middleware lets you define "listener entries" that contain an "effect" callback with additional logic, and a way to specify when that callback should run based on dispatched actions or state changes. It's intended to be a lightweight alternative to more widely used Redux async middleware like sagas and observables. While similar to thunks in level of complexity and concept, it can be used to replicate some common saga usage patterns.»

Что важно понять из исходника

  • originalState снимается до next(action) — поэтому getOriginalState() работает только синхронно.
  • Listeners прогоняются после reducer'а — новый state уже в store.
  • Все формы подписки (actionCreator, type, matcher, predicate) компилируются в единый predicate(action, currentState, originalState).
  • Ошибка в predicate — не роняет dispatch, а уходит в onError.
Почему listenerMiddleware НЕ заменяет thunk:
  • Нет ответа на dispatch. Вы дёргаете store.dispatch(action) и не можете await'ить саму бизнес-логику. Thunk возвращает Promise — listener нет.
  • Нет аргументов, кроме payload action'а. Если нужен сложный input с несколькими полями, их придётся положить в payload, что иногда неуместно.
Почему thunk НЕ заменяет listenerMiddleware:
  • Reactive-связь «любой loginSuccess → setup» — thunk не умеет. Нужна реакция на action, которая может прилететь из любого места.
  • Нет take/condition/fork — писать вручную через subscribe+AbortController болезненно.
