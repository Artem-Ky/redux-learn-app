4 режима, одна и та же кнопка «spam 10 раз»

Одинаковый action ping() диспатчится 10 раз подряд (каждые 50мс). Во всех четырёх вкладках этот action слушает listener, но реагирует разным паттерном — смотрите, сколько раз запустился effect.

takeLatest — debounce + отмена предыдущих

listener.startListening({ actionCreator: ping, effect: async (_a, api) => { api.cancelActiveListeners() // ← отменяем всех прежних await api.delay(150) // ← debounce окно // сюда доберётся только последний api.dispatch(effectRan('takeLatest')) }, })

Ожидание: 10 dispatched, 10 started, 1 done, 9 cancelled.

takeEvery — нет cancel, все отработают

listener.startListening({ actionCreator: ping, effect: async (_a, api) => { await api.delay(150) // ← каждая копия живёт своей жизнью api.dispatch(effectRan('takeEvery')) }, })

Ожидание: 10 dispatched, 10 started, 10 done, 0 cancelled. Поведение по умолчанию.

takeLeading — первый запустился, остальные игнор

listener.startListening({ actionCreator: ping, effect: async (_a, api) => { // Отписываемся на время своей работы — новые ping\'и не будут ловиться api.unsubscribe() try { await api.delay(400) api.dispatch(effectRan('takeLeading')) } finally { api.subscribe() // ← снова готовы } }, })

Ожидание: 10 dispatched, но effect стартует только 1 раз (на первом action'е). Пока он идёт — listener снят, остальные 9 action'ов не видят entry.

throttle — unsubscribe/subscribe на окно T

listener.startListening({ actionCreator: ping, effect: async (_a, api) => { api.unsubscribe() // окно открыто api.dispatch(effectRan('throttle')) // немедленный «leading» throttle await api.delay(400) api.subscribe() // готовы снова }, })

Ожидание: 10 dispatched, effect done ≈ 1 (в пределах 400мс окна). Визуально идентично takeLeading, но разница в том, где выполняется работа: здесь leading (сразу), takeLeading — trailing (в конце окна).

Официальная док

«This middleware lets you express "when this action is dispatched, start this effect". Because the effect is just an async function that has access to cancelActiveListeners, delay, unsubscribe/subscribe, and forks, you can build the saga-like patterns on top of it.»

«Middleware выражает "когда этот action — запустить этот effect". Поскольку effect — обычная async-функция с доступом к cancelActiveListeners, delay, unsubscribe/subscribe и fork, на нём строятся saga-подобные паттерны.»

Фрагмент из документации — готовые реализации

// Debounce — ровно то же, что takeLatest // https://redux-toolkit.js.org/api/createListenerMiddleware#debouncing listener.startListening({ actionCreator: eventSourceOpened, effect: async (action, listenerApi) => { // Cancel any in-progress instances of this listener listenerApi.cancelActiveListeners() // Delay before starting actual work await listenerApi.delay(500) // Do the work doSomethingExpensive() }, })
Подводные камни паттернов:
  • throttle через unsubscribe/subscribe имеет race. Между unsubscribe() и subscribe() action может прийти в другом listener-entry (или в том же, если кто-то зовёт api.subscribe() извне). Безопаснее хранить lastRun = Date.now() в closure и в effect проверять: if (Date.now() - lastRun < T) return.
  • cancelActiveListeners отменяет все инстансы этого entry, но не fork'и. Если внутри effect'а были api.fork(...), они всё ещё крутятся до internalTaskController.abort(listenerCompleted) в finally. Для «чистого» takeLatest с fork'ами — отслеживайте tasks и вызывайте task.cancel() сами, либо используйте { autoJoin: true }.

Альтернатива throttle без race — lastRun в closure

let lastRun = 0 const THROTTLE_MS = 400 listener.startListening({ actionCreator: ping, effect: async (_a, api) => { const now = Date.now() if (now - lastRun < THROTTLE_MS) return // в окне — игнор lastRun = now // ... работа ... api.dispatch(effectRan('throttle-safe')) }, })
