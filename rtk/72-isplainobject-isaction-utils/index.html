reduxisAction(value): boolean

Проверка action'а в Redux-смысле: plain object + typeof type === 'string'.

—

RTKminiSerializeError(value): SerializedError

Сериализация ошибки в plain object для safe-сохранения в state (Redux требует serializable). Вытаскивает только поля name, message, stack, code.

—

RTKTuple — Array с сохранением типа

Подкласс Array с Symbol.species = Tuple: concat/slice/map возвращают Tuple, не обычный массив. Критично для типизации middleware и enhancers.

Исходник — isPlainObject (redux core)

// node_modules/redux/src/utils/isPlainObject.ts — весь файл, 16 строк export default function isPlainObject(obj: any): obj is object { if (typeof obj !== 'object' || obj === null) return false let proto = obj while (Object.getPrototypeOf(proto) !== null) { proto = Object.getPrototypeOf(proto) // поднимаемся к корню prototype chain } return ( Object.getPrototypeOf(obj) === proto // прямой потомок Object.prototype || Object.getPrototypeOf(obj) === null // ИЛИ без prototype (Object.create(null)) ) }

github.com/reduxjs/redux/blob/master/packages/redux/src/utils/isPlainObject.ts

Алгоритм: идём вверх по getPrototypeOf, пока не упрёмся в null. Запоминаем «последний до null» как proto. Если getPrototypeOf(obj) === proto — объект лежит на один уровень ниже root (т.е. это { ... }). Альтернатива — null сам (для Object.create(null)).

Исходник — Tuple (RTK)

// packages/toolkit/src/utils.ts, строки 29-71 (сокращено — убраны TS overloads) export class Tuple<Items extends ReadonlyArray<unknown> = []> extends Array<Items[number]> { constructor(...items: any[]) { super(...items) Object.setPrototypeOf(this, Tuple.prototype) // важно — иначе TS downlevel теряет prototype } static override get [Symbol.species]() { return Tuple as any // 🔑 map/filter/slice → Tuple, НЕ Array } override concat(...arr: any[]) { return super.concat.apply(this, arr) // Array.concat использует Symbol.species — получаем Tuple } prepend(...arr: any[]) { if (arr.length === 1 && Array.isArray(arr[0])) { return new Tuple(...arr[0].concat(this)) } return new Tuple(...arr.concat(this)) } }

Зачем Symbol.species?

Методы Array (map, filter, slice, concat) внутри смотрят на Symbol.species конструктора, чтобы решить, экземпляр какого класса создать. Переопределяем — и TypeScript сохраняет сигнатуру: Tuple<[A, B]>.concat(Tuple<[C]>) → Tuple<[A, B, C]>. Без этого concat вернул бы Array, и middleware: (g) => g().concat(custom) потерял бы precise-тип.

Подводные камни:
  • isPlainObject(Object.create(null)) === true. Хотя такой объект не имеет метода toString / hasOwnProperty, он всё равно считается plain. Почему: создан через Object.create и его прототипная цепочка имеет корректную форму (обрывается на null). Последствия: Redux примет такой state/action без warning'а, но любой obj.hasOwnProperty(...) кинет Error — используй Object.hasOwn(obj, key).
  • isPlainObject возвращает false для instance классов. Даже если класс простой — class Foo { a = 1 } — его прототип не Object.prototype, а Foo.prototype. Для state/action это корректно (Redux хочет plain), но если у тебя в payload есть new Date() — serializable middleware начнёт ругаться.
  • miniSerializeError отбрасывает не-string поля и не-whitelist ключи. Есть у тебя err.userId = 42 (number) или err.data = { ... } — эти поля потеряются. Свой serializeError: (e) => ({ ...e }) передавай через createAsyncThunk(name, fn, { serializeError }). Важно: свои поля должны быть serializable — иначе serializable-middleware ругнётся.
  • Tuple и JSON. JSON.stringify(new Tuple(1,2,3)) даст [1,2,3], но JSON.parse вернёт plain Array. Если ты хранишь Tuple в state (не надо, но вдруг) — после rehydrate получишь Array, и вся магия типизации уходит.
  • isAction не проверяет payload/meta. { type: 'X', payload: () => {} } пройдёт — будет считаться action'ом, хотя функция в payload сразу сломает DevTools time-travel. Это намеренно — Redux отделяет валидацию формы action от serializability-проверки (у них отдельный middleware).

Где внутри RTK эти утилиты используются

Утилита Где вызывается
isPlainObject configureStore → serializable-check; query → defaultSerializeQueryArgs; immer isDraftable (как гейт).
isAction middleware entry-point: прежде чем next(action), Redux core проверяет формой.
miniSerializeError createAsyncThunk default для rejected.action.error; через options.serializeError можно подменить.
Tuple getDefaultMiddleware / getDefaultEnhancers возвращают Tuple; configureStore использует .concat/.prepend для сохранения типов.
