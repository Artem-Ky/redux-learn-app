Шпаргалка по createSlice (быстрая сверка)

Семь правил, которые нужно запомнить

  1. action.type = `${slice.name}/${reducerKey}`. Переименование reducer'а ломает persisted state со старыми type'ами.
  2. name — префикс actions, reducerPath — ключ в root state. По умолчанию reducerPath = name. RTK Query — единственный частый кейс, где они различаются намеренно.
  3. reducers — СВОИ actions (генерируются creator + type). extraReducers(builder) — ЧУЖИЕ actions (от других slice, thunk, createAction) — НИЧЕГО не генерируется, только слушаем.
  4. create.asyncThunk требует buildCreateSlice({ creators: { asyncThunk: asyncThunkCreator } }). Прямой createSlice кинет ошибку. Причина — tree-shaking createAsyncThunk.
  5. combineSlices(...slices) использует slice.reducerPath как ключ. Возвращает reducer с .inject() для lazy-loading.
  6. Immer: мутируй ИЛИ возвращай, но не оба. state.x = 1; return state или state.x = 1; return { ...state } — ошибка. Чистый return { ... } работает; чистая мутация state.x = 1 без return — работает.
  7. current(draft) — plain deep copy draft'а (для console.log). original(draft) — ссылка на state ДО reducer'а. console.log(state) без current = Proxy.
