Конфигурация Redux DevTools (наша + внешняя)

configureStore передаёт опции в composeWithDevToolsExtension(options). Эти опции работают для официального Redux DevTools Extension в Chrome/FF. В нашей встроенной DevTools-панели ниже мы реализовали аналогичный actionsDenylist — попробуйте отфильтровать actions.

Теория — devTools options

Из доки

«devTools: If true, configureStore will use composeWithDevToolsExtension if available. If an object, configureStore will pass that object to composeWithDevToolsExtension.»
— devTools option
«devTools: если true, configureStore использует composeWithDevToolsExtension при наличии. Если объект — configureStore передаст его в composeWithDevToolsExtension.»

Полезные опции

configureStore({ reducer, devTools: { name: 'My App', // имя в DevTools (если несколько store) trace: true, // сохранять stack-trace каждого action traceLimit: 25, // глубина stack maxAge: 50, // сколько actions держать в истории // Фильтрация actions actionsDenylist: ['^counter/tick$'], // regex: не показывать actionsAllowlist: ['^user/'], // regex: показывать только эти predicate: (state, action) => action.type !== 'noisy', // Сериализация serialize: { options: true }, // jsan-сериализация (для Map/Set/Date) actionSanitizer: (action) => ({ ...action, password: '***' }), stateSanitizer: (state) => ({ ...state, secret: '***' }), // Импорт/экспорт shouldHotReload: true, // перезагружать историю при HMR shouldRecordChanges: true, // автостарт записи } })

actionsDenylist vs predicate

  • actionsDenylist — массив regex-строк. Простой кейс «не записывать тиковые actions».
  • actionsAllowlist — обратное: записывать только matching actions.
  • predicate(state, action) — функция. Динамическое условие, например «не записывать если state.devMode === false».

Sanitizer для секретов

actionSanitizer и stateSanitizer применяются только для DevTools: в реальный store попадает оригинальное значение. Используйте, чтобы скрыть пароли, токены, PII в логах.

devTools: false

В production обязательно ставьте devTools: false или используйте process.env.NODE_ENV !== 'production'. Иначе любой пользователь сможет посмотреть state приложения через расширение и увидеть JWT-токены, личные данные.

Подводный камень: trace

trace: true сохраняет stack-trace каждого action — очень удобно для debugging. Но включает оверхед на каждый dispatch (~3-5ms). В проде точно false, в dev включайте по необходимости.
