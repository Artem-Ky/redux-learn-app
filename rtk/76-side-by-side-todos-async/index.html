Теория — зачем нужен RTK Query

Цитата из доки RTK Query

«RTK Query is a powerful data fetching and caching tool. It is designed to simplify common cases for loading data in a web application, eliminating the need to hand-write data fetching & caching logic yourself. RTK Query is purpose-built to solve the use case of data fetching, as opposed to a general-purpose state management library.»
— RTK Query — Overview
«RTK Query — инструмент специально под задачу загрузки и кэширования данных. Он устраняет необходимость писать fetching/caching вручную. В отличие от Redux вообще, RTK Query — не general-purpose, а целевой для одной конкретной задачи: server state.»

Три уровня «что вы пишете»

В classic-версии вы руками держите loading, error, data и три action types. Забыли сбросить error перед следующим запросом — пользователь увидит старое сообщение над свежим списком.

В createAsyncThunk три action types генерируются, thunkAPI.signal даёт AbortController. Но cache и dedup всё ещё на вас.

В RTK Query сам endpoint хранит запись в state.todosApi.queries['getTodos(undefined)'] с полями status, data, error, startedTimeStamp, fulfilledTimeStamp. Второй вызов с теми же аргументами возвращает кэш.

Manual subscription vs React hook

В уроке — без React. store.dispatch(api.endpoints.getTodos.initiate()) возвращает promise-подобный объект с методом .unsubscribe(). Пока есть хотя бы один подписчик — cache entry жив; когда подписчиков ноль, по истечении keepUnusedDataFor запись удаляется. В React useGetTodosQuery() делает именно это: initiate при маунте, unsubscribe при анмаунте.

Подводные камни

Classic thunk: легко забыть сбросить error. При старте нового запроса нужно сделать error = null, иначе старая ошибка останется поверх нового fulfilled списка. Со всеми полями нужно танцевать руками — это источник регулярных багов.
createAsyncThunk — нет dedup из коробки. Пользователь жмёт кнопку 5 раз — пойдёт 5 одинаковых запросов. Нужно руками: condition: (arg, { getState }) => !getState().todos.loading. В RTK Query dedup по cache key автоматический.
RTK Query без React — subscription руками. В React хуки сами считают subscribers. Без React надо помнить const sub = dispatch(endpoints.X.initiate(arg)), потом sub.unsubscribe(). Иначе cache entry никогда не удалится.
