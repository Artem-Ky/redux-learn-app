{ reducer, prepare } — кастомизация payload в createSlice

Если reducer — простая функция, payload = аргумент action creator'а. Если нужна логика подготовки (nanoid, timestamp, несколько аргументов) — передавайте объект { reducer, prepare }.

const todosSlice = createSlice({ name: 'todos', initialState: [] as Todo[], reducers: { // Обычный reducer — payload = аргумент toggleTodo: (state, action: PayloadAction<string>) => { const t = state.find(x => x.id === action.payload) if (t) t.done = !t.done }, // Prepared reducer — кастомизированный payload addTodo: { reducer(state, action: PayloadAction<Todo>) { state.push(action.payload) }, prepare(text: string) { return { payload: { id: nanoid(), // id ← в prepare, не в reducer text, done: false, createdAt: Date.now(), } } } }, // Несколько аргументов через prepare addNote: { reducer(state, action: PayloadAction<Note, string, { user: string }>) { state.push({ ...action.payload, addedBy: action.meta.user }) }, prepare(text: string, user: string) { return { payload: { id: nanoid(), text }, meta: { user, ts: Date.now() } } } } } })

Теория — prepare в createSlice

Из доки

«If you want to customize the creation of the payload value, you may pass an object containing two keys: reducer and prepare.»
— createSlice — prepare
«Если хочется кастомизировать создание значения payload, можно передать объект с двумя ключами: reducer и prepare.»

Когда нужен prepare

  1. Генерация id — nanoid() внутри prepare, не в reducer'е (reducer должен быть чистым).
  2. Несколько аргументов — actionCreator принимает много, payload собирает в один объект.
  3. meta-поля — timestamp, source, user (не входят в payload).
  4. error — флаг для FSA-ошибок.
  5. Нормализация входа — обрезка пробелов, lowercase, валидация.

Под капотом — то же, что createAction(type, prepare)

// Это: addTodo: { reducer: (state, action) => { state.push(action.payload) }, prepare: (text) => ({ payload: { id: nanoid(), text } }) } // эквивалентно: const addTodo = createAction( 'todos/addTodo', (text) => ({ payload: { id: nanoid(), text } }) )

Несколько аргументов

// Action creator теперь принимает 2 аргумента addNote: { reducer: (state, action) => { state.push(action.payload) }, prepare: (text, user) => ({ payload: { id: nanoid(), text, addedBy: user } }) } // Использование: dispatch(addNote('привет', 'Alice'))

meta-поле

meta — поле для метаданных, которые не относятся к payload, но могут быть полезны для middleware/listener'ов: timestamp, source, requestId, analytics-теги.

prepare: (text, user) => ({ payload: { id: nanoid(), text }, meta: { addedBy: user, timestamp: Date.now(), source: 'manual' }, error: false // можно опционально }) // В reducer'е: reducer: (state, action) => { state.push({ ...action.payload, addedBy: action.meta.addedBy }) }

TypeScript: PayloadAction<P, T, M, E>

type PayloadAction< P = void, // payload T extends string = string, // type M = never, // meta E = never // error > // Использование с meta: reducer: (state, action: PayloadAction<Note, string, { user: string }>) => { action.meta.user // типизирован }

Подводный камень: prepare возвращает объект

prepare должен возвращать объект с полем payload, не само значение!
✗ prepare: (text) => { id: nanoid(), text } — неправильно
✓ prepare: (text) => ({ payload: { id: nanoid(), text } }) — правильно

Подводный камень: prepare как ручка для side-effects

prepare вызывается каждый раз, когда вы вызываете actionCreator. Не делайте там fetch, console.error, изменение DOM. Только чистая трансформация входа. Для побочных эффектов — middleware/thunks/listener.
