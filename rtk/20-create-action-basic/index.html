Интроспекция action creator

createAction(type) создаёт типизированную функцию, которая при вызове возвращает action. Но она — не просто функция: на ней есть .type, .match, .toString(). Это даёт возможность использовать её в switch, в matchers, в качестве ключей.

Создание

import { createAction } from '@reduxjs/toolkit' const tick = createAction('timer/tick')

Теория — createAction

Из доки

«A helper function for defining a Redux action type and creator. The function will call toString() with the given action type and return the type itself, so it can be used as a key in createReducer or with switch statements. Additionally, you can define a matcher via match for use with addMatcher.»
— createAction API
«Вспомогательная функция для определения типа Redux-экшена и его creator'а. При вызове toString() возвращает сам тип — это позволяет использовать её как ключ в createReducer или в switch-конструкциях. Также можно использовать match в качестве matcher'а для addMatcher.»

Сигнатура

function createAction<P = void, T extends string = string>(type: T): PayloadActionCreator<P, T> // Возвращает функцию со следующими свойствами: interface ActionCreator { (payload?: P): PayloadAction<P, T> // вызов → action type: T // строка type (const) toString(): T // возвращает type match(action: Action): action is PayloadAction<P, T> // type-guard }

Реализация (упрощённо, из исходника)

export function createAction(type) { function actionCreator(...args) { return { type, payload: args[0] } } actionCreator.type = type actionCreator.toString = () => `${type}` actionCreator.match = (action) => action?.type === type return actionCreator }

Трюк с toString()

Когда вы пишете String(tick) или используете actionCreator в качестве ключа объекта ({ [tick]: reducer }) — JS автоматически вызывает .toString(). Это возвращает 'timer/tick'. Такой трюк использовался в классическом createReducer с object-map:

const reducer = createReducer(initial, { [tick]: (state) => { state.count++ } // ключ приводится к 'timer/tick' }) // В RTK 2.x object-map deprecated — используйте builder: const reducer = createReducer(initial, (builder) => { builder.addCase(tick, (state) => { state.count++ }) })

match как type-guard

// TS сужает тип action до PayloadAction<P, T> if (tick.match(action)) { action.type // typed как 'timer/tick' action.payload // typed как P } // Это ключ к listenerMiddleware: listener.startListening({ matcher: tick.match, // ← именно так effect: (action, api) => { /* ... */ } })

Что createAction НЕ делает

createAction только создаёт action creator. Он не регистрирует его в reducer, не делает подписку, не знает про store. Это чистая функция. Связывание action ↔ state происходит в reducer (через switch, object-map или builder).

Когда использовать createAction напрямую

  • Shared actions между несколькими slice'ами (например, resetAll, userLoggedOut).
  • Actions, которые вы dispatch извне, но на которые реагирует несколько reducer'ов.
  • В 99% случаев внутри slice — пользуйтесь slice.actions.xxx, RTK сам создаёт через createAction.

Подводный камень: ключевое слово 'type'

Если вы случайно сделаете dispatch(tick) (без скобок) — вы dispatch'нете функцию. Thunk попробует её выполнить (она вернёт action), но это побочный эффект. actionCreatorInvariantMiddleware (урок 14) ловит такие случаи.
