Polling: два независимых fork'а + graceful cancel

На pollStart listener запускает два fork'а: один тикает каждую секунду (диспатчит tick), второй — каждые 2 секунды (диспатчит heartbeat). «Stop» — вызываем childTask.cancel(), у форка ловится TaskAbortError, cleanup через finally.

Теория — 67. fork, ForkedTaskAPI и parent/child abort linkage

Что такое fork

«Queues in the next microtask the execution of a task. fork is useful for running long-running side tasks that should live independently of the listener's effect callback. The forked task has its own AbortSignal, linked to the parent listener's signal: when the parent is cancelled, the child is cancelled too.»
— RTK docs — forking tasks
«fork ставит задачу в следующий microtask. Нужен для долгоиграющих побочек, которые должны жить независимо от основного effect. У fork'а свой AbortSignal, связанный с сигналом родителя: когда родитель отменён — ребёнок тоже.»

ForkedTaskAPI — что видит fork-экзекьютор

interface ForkedTaskAPI { pause<W>(waitFor: Promise<W>): Promise<W> // abort-aware await delay(timeoutMs: number): Promise<void> // abort-aware setTimeout signal: AbortSignal // сигнал именно этой child-задачи } // ForkedTask, который вернёт fork(): interface ForkedTask<T> { result: Promise<TaskResult<T>> // { status: 'ok' | 'cancelled' | 'rejected', ... } cancel(): void // отменить ребёнка }

Fork-cancellation схема

autoJoin — родитель ждёт детей

// Без autoJoin: родительский effect возвращается сразу, fork живёт «сам по себе». api.fork(async (f) => { await f.delay(1000) }) // effect закончится мгновенно, fork ещё тикает 1с. // С { autoJoin: true }: middleware.notifyListener дождётся всех autoJoin-форков // в блоке finally: `await Promise.all(autoJoinPromises)` — и только потом abort. api.fork(async (f) => { await f.delay(1000) }, { autoJoin: true })
Два критичных подводных камня fork:
  • Без autoJoin родительский effect завершается раньше. Если внутри fork'а вы диспатчите action после delay(1000), но effect заканчивается через 10мс — родительский internalTaskController.abort(listenerCompleted) в finally уронит fork через linked signal. Решение: либо { autoJoin: true }, либо await task.result.
  • Ловите отмену через try/finally, а не try/catch. catch (e) проглотит TaskAbortError, и ваш финальный dispatch(poll.result) выстрелит уже после отмены, сломав state. runTask уже заворачивает результат в { status: 'cancelled' } — просто проверяйте его; cleanup кладите в finally.

Правильный cleanup для poll-фрка

const pollTask = api.fork(async (forkApi) => { let ticks = 0 try { while (true) { await forkApi.delay(1000) // бросит TaskAbortError при cancel ticks++ api.dispatch(tick()) } } finally { // всегда выполнится — и при cancel, и при ошибке unsubscribeFromWhatever() console.log('poll stopped at tick', ticks) } }) // cancel извне: pollTask.cancel()
