Теория — CRUD с нормализованным state

      

        
Create — добавление

        

          Добавление нового элемента — это две операции: записать объект в entities
          по ключу ID и добавить этот ID в массив ids.
        


        

// ADD: entities[id] = todo, ids.push(id)
case 'todos/add': {
  const todo = action.payload
  return {
    ...state,
    entities: { ...state.entities, [todo.id]: todo },
    ids: [...state.ids, todo.id]
  }
}
        


        
Update — обновление (toggle / edit)

        

          Обновление — просто spread сущности по ID. Не нужно искать индекс в массиве,
          не нужно map всех элементов — прямой доступ entities[id].
        


        

// TOGGLE: spread entity, инвертировать completed
case 'todos/toggle': {
  const id = action.payload
  const todo = state.entities[id]
  return {
    ...state,
    entities: {
      ...state.entities,
      [id]: { ...todo, completed: !todo.completed }
    }
  }
}
        


        
Delete — удаление

        

          Удаление — delete из entities + filter из ids.
          Обе структуры должны быть обновлены синхронно.
        


        

// DELETE: delete entities[id], filter ids
case 'todos/delete': {
  const id = action.payload
  const newEntities = { ...state.entities }
  delete newEntities[id]
  return {
    ...state,
    entities: newEntities,
    ids: state.ids.filter(i => i !== id)
  }
}
        


        
Edit — редактирование текста


        

// EDIT: spread entity, новый text
case 'todos/edit': {
  const { id, text } = action.payload
  return {
    ...state,
    entities: {
      ...state.entities,
      [id]: { ...state.entities[id], text: text }
    }
  }
}
        


        
Сравнение с массивом

        

          В массиве toggle выглядит как todos.map(t => t.id === id ? {...t, completed: !t.completed} : t) —
          мы перебираем все элементы, чтобы обновить один.
          С нормализацией — обновляем напрямую по ключу: entities[id].
        


        

          «We recommend organizing your Redux store's data in a 'normalized' form.
          Among other things, this enables looking up items by their IDs without
          having to iterate over entire arrays.»
          
— Redux Docs — Normalizing State Shape
        


        

          Нормализованный CRUD проще: Add = set + push, Update = spread по ключу,
          Delete = delete + filter. Нет .find(), нет .findIndex(),
          нет .map() для обновления одного элемента.
        


        

          При delete не забудьте удалить ID из обоих мест — entities
          и ids. Осиротевший ID в массиве приведёт к ошибке
          entities[id] → undefined.
        


        
Ключевые моменты

        
    
          
  • Add: entities[id] = entity + ids.push(id)
    • 
          
  • Update: { ...entities[id], ...changes }
    • 
          
  • Delete: delete entities[id] + ids.filter()
    • 
          
  • Прямой доступ по ID вместо перебора массива
    • 
          
  • Проще и эффективнее, чем array-based CRUD
