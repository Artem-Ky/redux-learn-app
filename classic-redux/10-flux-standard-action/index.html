Теория — Flux Standard Action (FSA)

Что такое FSA?

Flux Standard Action — это соглашение о структуре action-объектов, предложенное сообществом для единообразия. FSA определяет, какие поля может содержать экшен и как они должны использоваться.

«A Flux Standard Action MUST be a plain JavaScript object, MUST have a type property. An action MAY have a payload, error, or meta property. An action MUST NOT include properties other than type, payload, error, and meta.»
— Flux Standard Action Spec

Структура FSA

{ type: 'todos/todoAdded', // ОБЯЗАТЕЛЬНОЕ — строка payload: 'Купить молоко', // необязательное — любые данные meta: { source: 'ui' }, // необязательное — метаданные error: false // необязательное — boolean }

Четыре поля FSA

  • type (обязательное) — строка, идентификатор действия
  • payload — данные, связанные с действием. Если error: true, payload ДОЛЖЕН быть объектом Error
  • error — если true, экшен представляет ошибку
  • meta — дополнительная информация, не являющаяся частью payload (источник, timestamp и т.д.)

Success vs Error action

// Success action { type: 'users/fetchCompleted', payload: { id: 1, name: 'Анна' } } // Error action { type: 'users/fetchCompleted', payload: new Error('Сервер недоступен'), error: true }

Почему сообщество предпочитает отдельные типы?

Хотя FSA предлагает использовать поле error: true для обозначения ошибок, на практике многие разработчики предпочитают отдельные action types:

// Вместо одного типа с error: true/false 'users/fetchCompleted' + error: true // Предпочитают отдельные типы 'users/fetchSucceeded' 'users/fetchFailed'

Это делает reducer проще — каждый case обрабатывает одну ситуацию, не нужно проверять action.error.

FSA — это соглашение, а не требование Redux. Однако Redux Toolkit генерирует FSA-совместимые экшены по умолчанию, поэтому знание FSA полезно.

Ключевые моменты

  • FSA — стандартизация структуры action-объектов
  • Допустимые поля: type, payload, error, meta
  • Других полей быть не должно (по спецификации)
  • На практике чаще используют отдельные action types вместо поля error
  • Redux Toolkit создаёт FSA-совместимые экшены
