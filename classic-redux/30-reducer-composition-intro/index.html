    Теория — Reducer Composition (разделение reducer'ов)
    
      
    
        
    Проблема: один reducer на всё
    
        
    
          В предыдущем уроке мы написали один reducer, обрабатывающий все действия.
          Но представьте приложение, где state содержит todos, filters,
          user, notifications... Один reducer будет огромным и
          трудно поддерживаемым.
        
    

        
    
          «It's not really the number of actions that matters, but how complex the state
          shape is. Each reducer effectively handles a slice of the state, and each reducer
          needs to know how to handle actions that apply to that particular slice.»
          
    — Redux Fundamentals, Part 3
        
    

        
    Принцип: каждый reducer управляет своим «слайсом»
    
        
    
          Reducer composition — это принцип разделения: вместо одного
          гигантского reducer мы создаём несколько маленьких, каждый из которых управляет
          своей частью (slice) state.
        
    

        
    
// БЫЛО: один большой reducer
function rootReducer(state = initialState, action) {
  switch (action.type) {
    // 10 case для todos...
    // 5 case для filters...
    // 8 case для user...
    default: return state
  }
}
        
    

        
    
// СТАЛО: разделение на слайсы
function todosReducer(state: Todo[] = [], action): Todo[] {
  // Обрабатывает только todos
}

function filtersReducer(state: FiltersState = initialFilters, action): FiltersState {
  // Обрабатывает только filters
}
        
    

        
    Ручная композиция
    
        
    
          До использования combineReducers можно объединить reducer'ы вручную.
          Корневой reducer делегирует обработку каждому слайс-reducer'у:
        
    

        
    
function rootReducer(state: AppState = initialState, action: Action): AppState {
  return {
    todos: todosReducer(state.todos, action),
    filters: filtersReducer(state.filters, action)
  }
}
        
    

        
    combineReducers — автоматическая композиция
    
        
    
          Redux предоставляет утилиту combineReducers, которая делает то же самое
          автоматически. Она принимает объект, где ключ — имя слайса, а значение — reducer:
        
    

        
    
import { combineReducers } from 'redux'

const rootReducer = combineReducers({
  todos: todosReducer,
  filters: filtersReducer
})

// Результат: state = { todos: [...], filters: { ... } }
        
    

        
    
          Нажмите кнопку «Рефакторить» в демо выше, чтобы увидеть переход от одного
          большого reducer к двум разделённым. Оба подхода дают одинаковый state —
          разница только в организации кода.
        
    

        
    Структура state после разделения
    
        
    
          Каждый слайс-reducer получает только свою часть state и
          возвращает обновлённую версию этой части:
        
    

        
    
// Полный state
{
  todos: [                        // ← todosReducer
    { id: 1, text: '...', completed: false }
  ],
  filters: {                      // ← filtersReducer
    status: 'all',
    colors: []
  }
}
        
    

        
    
          Каждый слайс-reducer не знает о существовании других слайсов.
          todosReducer получает только state.todos (массив), а не
          весь state объект. Это обеспечивает изоляцию и модульность.
        
    

        
    Подготовка к combineReducers
    
        
    
          В следующем уроке (31) мы подробно разберём combineReducers — утилиту,
          которая автоматизирует то, что мы сделали вручную. Она генерирует корневой reducer,
          вызывающий каждый слайс-reducer с его частью state.
        
    

        
    Ключевые моменты
    
        
      
          
    • Reducer composition — разделение одного reducer на несколько по слайсам
      • 
          
    • Каждый слайс-reducer управляет только своей частью state
      • 
          
    • Корневой reducer делегирует обработку слайс-reducer'ам
      • 
          
    • combineReducers автоматизирует ручную композицию
      • 
          
    • State имеет структуру { sliceName: sliceState, ... }
      • 
          
    • Оба подхода (один reducer и split) дают идентичный результат
