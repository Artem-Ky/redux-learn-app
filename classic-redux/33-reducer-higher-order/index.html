Теория — Higher-Order Reducers (фабрики)

      

        
Проблема: дублирование логики reducer'ов

        

          Представьте, что в приложении нужно несколько счётчиков с одинаковой логикой.
          Копировать код reducer'а — плохая практика. Вместо этого используется
          higher-order reducer — функция, которая создаёт reducer.
        


        

          «A higher-order reducer is a function that takes a reducer as an argument,
          and/or returns a new reducer function as a result.»
          
— Redux Docs: Reusing Reducer Logic
        


        
Reducer Factory (фабрика)

        

          createCounterReducer — фабричная функция, которая принимает имя
          счётчика и возвращает reducer, реагирующий только на actions с этим именем:
        


        

function createCounterReducer(counterName: string) {
  return function(state: number = 0, action: Action): number {
    switch (action.type) {
      case `${counterName}/incremented`:
        return state + 1
      case `${counterName}/decremented`:
        return state - 1
      default:
        return state
    }
  }
}
        


        
Динамическое создание reducer'ов

        

          Теперь мы можем создать сколько угодно независимых reducer'ов из одной фабрики:
        


        

const counterAReducer = createCounterReducer('counterA')
const counterBReducer = createCounterReducer('counterB')

const rootReducer = combineReducers({
  counterA: counterAReducer,
  counterB: counterBReducer
})

// state = { counterA: 0, counterB: 0 }
        


        
Actions привязаны к имени

        

          Каждый reducer реагирует только на «свои» actions благодаря префиксу имени.
          counterA/incremented изменяет только счётчик A,
          counterB/incremented — только B.
        


        

// Увеличить счётчик A:
store.dispatch({ type: 'counterA/incremented' })
// counterAReducer: обрабатывает ✓
// counterBReducer: игнорирует (default) ✗
        


        
Применения паттерна

        
    
          
  • Несколько экземпляров одинаковой логики (счётчики, формы, списки)
    • 
          
  • Добавление «обёрток» вокруг reducer'ов (undo/redo, логирование)
    • 
          
  • Условная обработка actions по метаданным
    • 
          
  • Создание reducer'ов из конфигурации (data-driven reducers)
    • 
        


        

          Нажимайте +/− для каждого счётчика в демо. Обратите внимание в DevTools:
          actions имеют разные префиксы (counterA/ vs counterB/),
          а каждый reducer реагирует только на «свои».
        


        

          Higher-order reducer — мощный паттерн, но не злоупотребляйте: если логика
          reducer'ов существенно различается, лучше написать их отдельно. Фабрики хороши
          для идентичной логики с разными именами.
        


        
Ключевые моменты

        
    
          
  • Higher-order reducer — функция, создающая reducer'ы
    • 
          
  • Позволяет переиспользовать логику без дублирования
    • 
          
  • Каждый созданный reducer реагирует на actions с уникальным префиксом
    • 
          
  • Комбинируется с combineReducers для создания нескольких слайсов
    • 
          
  • Применяется для однотипных компонентов: счётчики, формы, списки
