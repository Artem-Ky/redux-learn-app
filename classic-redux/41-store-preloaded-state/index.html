Теория — preloadedState (второй аргумент createStore)

      

        
Что такое preloadedState?

        

          createStore принимает необязательный второй аргумент —
          preloadedState. Это начальное значение, с которого стартует store,
          минуя значение по умолчанию из reducer.
        


        

import { legacy_createStore as createStore } from 'redux'

const store = createStore(reducer, { value: 42 })
// store.getState() → { value: 42 }
        


        

          «The preloadedState argument is the initial state passed to the Redux store
          when it is created. It is an optional argument that allows you to bootstrap
          the store with state obtained from another source.»
          
— Redux Fundamentals, Part 4
        


        
Отличие от default parameter в reducer

        

          Reducer может задавать значение по умолчанию через параметр
          state = initialState. Но если передан preloadedState,
          он перекрывает этот default — reducer получит preloadedState
          при первом вызове (@@INIT).
        


        

// Reducer с default parameter
function reducer(state = { value: 0 }, action) {
  // state = { value: 0 } используется только если
  // preloadedState НЕ передан
  ...
}

// Без preloadedState:
createStore(reducer)          // state = { value: 0 }

// С preloadedState:
createStore(reducer, { value: 42 }) // state = { value: 42 }
        


        
Основные use cases

        
    
          
  • SSR hydration — сервер рендерит HTML с данными и передаёт state через window.__PRELOADED_STATE__
    • 
          
  • localStorage persistence — сохраняем state при уходе со страницы, восстанавливаем при возврате
    • 
          
  • Тестирование — создаём store с нужным state для теста
    • 
        


        

// Пример: восстановление из localStorage
const saved = localStorage.getItem('redux-state')
const preloaded = saved ? JSON.parse(saved) : undefined

const store = createStore(reducer, preloaded)
        


        

          Нажмите «Сохранить в localStorage» в демо, затем «Загрузить из localStorage» —
          store будет пересоздан с сохранённым значением через preloadedState.
        


        

          Если вы используете combineReducers, preloadedState должен иметь
          ту же форму, что и структура reducer'ов. Каждый ключ preloadedState
          должен соответствовать ключу в combineReducers.
        


        
Ключевые моменты

        
    
          
  • preloadedState — второй аргумент createStore
    • 
          
  • Перекрывает default parameter reducer'а при инициализации
    • 
          
  • Используется для SSR, localStorage persistence, тестов
    • 
          
  • Если не передан — reducer использует свой default
    • 
          
  • Должен совпадать по форме с ожидаемым state
