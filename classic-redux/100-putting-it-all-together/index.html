Этот проект объединяет все ключевые концепции классического Redux, которые мы изучили за 100 уроков. Давайте вспомним каждую из них.

Ключевые концепции

  • Actions — простые объекты с полем type, описывающие «что произошло»
  • Action Creators — функции, возвращающие action-объекты
  • Reducers — чистые функции (state, action) => newState, обновляющие состояние иммутабельно
  • Store — единственный объект, хранящий всё состояние приложения (createStore, getState, dispatch, subscribe)
  • combineReducers — объединение нескольких slice-reducer'ов в один корневой
  • Middleware — расширение dispatch: логгеры, аналитика, обработка ошибок
  • applyMiddleware — enhancer для подключения цепочки middleware к store
  • Thunks — action-функции для async-логики: (dispatch, getState) => void
  • Selectors — функции извлечения данных из state
  • createSelector / Reselect — мемоизированные селекторы с кэшированием
  • Immer — иммутабельные обновления через «мутации» draft-объектов
  • Normalized State — хранение сущностей как { entities: {}, ids: [] }
  • DevTools — инструменты отладки: time-travel, diff, export/import

Что дальше?

Вы освоили фундамент Redux. Следующие шаги:

  • React-Redux — подключение Redux к React-приложению (Provider, useSelector, useDispatch)
  • Redux Toolkit (RTK) — современный способ писать Redux: configureStore, createSlice, createAsyncThunk
  • RTK Query — кэширование серверных данных и автоматическая синхронизация
«Redux Toolkit is our official, opinionated, batteries-included toolset for efficient Redux development. It is intended to be the standard way to write Redux logic.»
— Redux Toolkit Documentation
Знание классического Redux — бесценно. Redux Toolkit не заменяет понимание принципов, а автоматизирует шаблонный код. Всё, что вы узнали за 100 уроков, работает внутри RTK. Вы теперь понимаете, почему RTK делает то, что делает.
