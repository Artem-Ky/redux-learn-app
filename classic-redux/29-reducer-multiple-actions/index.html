    Теория — Один reducer, множество действий
    
      
    
        
    Реальные reducer'ы обрабатывают много action types
    
        
    
          В предыдущих уроках мы рассматривали каждую операцию отдельно: добавление,
          переключение, удаление, редактирование. В реальном приложении один reducer
          обрабатывает все эти действия через switch/case.
        
    

        
    
          «A reducer may handle many different action types. It should look at the
          action.type field to decide what to do for each action type, and return
          a new state based on the action.»
          
    — Redux Fundamentals, Part 3
        
    

        
    Полный reducer со всеми операциями
    
        
    
          Вот как выглядит reducer, обрабатывающий 6 различных action types для todo-списка:
        
    

        
    
function todosReducer(state: TodoState = initialState, action: TodoAction): TodoState {
  switch (action.type) {

    case 'todos/todoAdded': {
      const newTodo: Todo = {
        id: nextId++,
        text: action.payload,
        completed: false
      }
      return [...state, newTodo]
    }

    case 'todos/todoToggled':
      return state.map(todo =>
        todo.id === action.payload
          ? { ...todo, completed: !todo.completed }
          : todo
      )

    case 'todos/todoDeleted':
      return state.filter(todo =>
        todo.id !== action.payload
      )

    case 'todos/todoEdited': {
      const { id, text } = action.payload
      return state.map(todo =>
        todo.id === id ? { ...todo, text } : todo
      )
    }

    case 'todos/allCompleted':
      return state.map(todo =>
        ({ ...todo, completed: true })
      )

    case 'todos/completedCleared':
      return state.filter(todo =>
        !todo.completed
      )

    default:
      return state
  }
}
        
    

        
    Организация большого reducer
    
        
    
          Когда reducer обрабатывает 5–10+ action types, он может стать большим.
          Вот несколько советов по организации:
        
    
        
      
          
    • Группируйте case по логике: CRUD-операции вместе, bulk-операции вместе
      • 
          
    • Используйте фигурные скобки { } в case, если нужны локальные переменные
      • 
          
    • Выносите сложную логику в отдельные функции
      • 
          
    • Каждый case возвращает — не забывайте return
      • 
          
    • default всегда возвращает state без изменений
      • 
        
    

        
    Когда пора разделять reducer
    
        
    
          Если reducer становится слишком большим или обрабатывает данные из разных
          «доменов» (например, todos + filters + user), это сигнал для разделения
          на несколько reducer'ов. Об этом подробно — в следующем уроке.
        
    

        
    
// Признаки того, что reducer пора разделить:
// 1. Обрабатывает state разной «формы» (todos + filters)
// 2. Больше ~15 case
// 3. Разные части state не связаны друг с другом
// 4. Файл больше 100-150 строк
        
    

        
    
          В этом примере все 6 операций обрабатываются одним reducer — это нормально,
          потому что все они работают с одним типом данных (Todo[]).
          Проблемы начинаются, когда один reducer пытается управлять несколькими
          независимыми частями state.
        
    

        
    Bulk-операции
    
        
    
          todos/allCompleted и todos/completedCleared — примеры
          «массовых» операций. Они работают не с одним элементом, а со всеми:
        
    

        
    
// Выполнить все: map с фиксированным значением
state.map(todo => ({ ...todo, completed: true }))

// Очистить выполненные: filter по условию
state.filter(todo => !todo.completed)
        
    

        
    
          Не забывайте про default case! Если reducer получает
          неизвестный action type, он должен вернуть state без изменений. Без default
          reducer вернёт undefined, что сломает приложение.
        
    

        
    Ключевые моменты
    
        
      
          
    • Один reducer может обрабатывать множество action types через switch/case
      • 
          
    • Каждый case — самостоятельная иммутабельная операция
      • 
          
    • Паттерны: spread для добавления, map для обновления, filter для удаления
      • 
          
    • Bulk-операции (allCompleted, completedCleared) работают со всеми элементами
      • 
          
    • Если reducer слишком большой — пора разделять (следующий урок)
      • 
          
    • default всегда возвращает state
