Теория — redux-thunk: официальный middleware

      

        
Установка


        

npm install redux-thunk
        


        
Подключение


        

import { thunk } from 'redux-thunk'
import { legacy_createStore as createStore, applyMiddleware } from 'redux'

const store = createStore(reducer, applyMiddleware(thunk))
        


        

          «The Redux Thunk middleware allows you to write action creators that return
          a function instead of an action. The thunk can be used to delay the dispatch
          of an action, or to dispatch only if a certain condition is met.»
          
— redux-thunk — GitHub
        


        
Сигнатура thunk-функции

        

          Thunk-функция получает два аргумента: dispatch и getState.
          Внутри можно выполнять любую асинхронную логику.
        


        

// Thunk action creator
const fetchData = () => {
  // Возвращаем thunk-функцию
  return async (dispatch: any, getState: any) => {
    dispatch({ type: 'data/loading' })

    const response = await fetch('/api/data')
    const data = await response.json()

    dispatch({ type: 'data/loaded', payload: data })
  }
}

// dispatch thunk
store.dispatch(fetchData())
        


        
async/await в thunks

        

          Thunk-функция может быть async. Это позволяет использовать
          await для API-вызовов, таймеров и других асинхронных операций.
        


        

const loadUsers = () => async (dispatch: any) => {
  dispatch({ type: 'users/loading' })

  try {
    const res = await fetch('/api/users')
    const users = await res.json()
    dispatch({ type: 'users/loaded', payload: users })
  } catch (err) {
    dispatch({ type: 'users/error', payload: err.message })
  }
}
        


        
Сравнение с нашей реализацией

        

          Официальный redux-thunk делает практически то же самое, что наш
          5-строчный middleware из урока 69. Дополнительно он поддерживает
          extraArgument — третий аргумент, который можно передать thunk'ам.
        


        

// Наша реализация (урок 69):
const thunkMiddleware = (storeAPI) => (next) => (action) => {
  if (typeof action === 'function') {
    return action(storeAPI.dispatch, storeAPI.getState)
  }
  return next(action)
}

// Официальный redux-thunk (упрощённо):
const thunk = (storeAPI) => (next) => (action) => {
  if (typeof action === 'function') {
    return action(storeAPI.dispatch, storeAPI.getState, extraArgument)
  }
  return next(action)
}
        


        

          В Redux Toolkit redux-thunk уже включён по умолчанию — его не нужно
          устанавливать отдельно. configureStore() автоматически подключает
          thunk middleware.
        


        

          Thunk-функция не попадает в reducer — middleware перехватывает её до этого.
          Reducer по-прежнему получает только обычные action-объекты с полем type.
