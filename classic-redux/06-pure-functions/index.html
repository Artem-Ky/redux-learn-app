Интерактив — Чистая vs нечистая функция

Чистая функция pure

function add(a: number, b: number): number { return a + b }

Нечистая функция impure

let callCount = 0 function addImpure(a: number, b: number): string { callCount++ return `${a + b} (call #${callCount} at ${Date.now()})` }
Теория — Pure Functions и Side Effects

Что такое чистая функция?

Чистая функция — функция, которая:

  1. Результат зависит только от аргументов (одинаковые входы → одинаковый выход)
  2. Не вызывает побочных эффектов (не меняет внешние переменные, не делает HTTP-запросы и т.д.)
«Reducers must always follow some specific rules: They should only calculate the new state value based on the state and action arguments. They are not allowed to modify the existing state. They must not do any asynchronous logic, calculate random values, or cause other "side effects".»
— Redux Fundamentals, Part 2

Побочные эффекты (Side Effects)

Побочный эффект — любое изменение состояния или поведения, видимое за пределами возвращаемого значения:

  • Логирование в консоль
  • Сохранение файла
  • Установка таймера
  • HTTP-запросы
  • Изменение внешних переменных / мутация аргументов
  • Генерация случайных чисел (Math.random())
  • Получение текущего времени (Date.now())

Почему reducer должен быть чистым?

Предсказуемость: одни и те же входные данные (state + action) всегда дают один и тот же результат.
Тестируемость: не нужно мокать внешние зависимости.
Time-travel: DevTools могут «перематывать» и «проигрывать» экшены, получая тот же state.
Hot-reloading: можно заменить reducer на лету и пересчитать state.

Где тогда делать побочные эффекты?

Побочные эффекты живут в middleware (промежуточном ПО). Мы подробно разберём это позже — в разделе про Thunk и middleware. Reducer — это чистая математика обновления state.
