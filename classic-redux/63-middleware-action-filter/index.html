Теория — Middleware как фильтр действий

      

        
Middleware может не вызывать next()

        

          Ключевая способность middleware — остановить action, просто
          не вызвав next(action). Action не дойдёт ни до следующего middleware,
          ни до reducer'а.
        


        

          «A middleware can do anything it wants when it sees a dispatched action.
          It can do something when an action is dispatched, and then pass the action
          along to the rest of the middleware pipeline, or it can stop the action
          from being forwarded.»
          
— Redux Fundamentals, Part 4
        


        
Паттерн: условный next()

        

const actionFilter =
  (storeAPI: any) => (next: any) => (action: any): any => {
    if (action.type === 'counter/reset') {
      console.warn('BLOCKED:', action.type)
      return // не вызываем next() — action остановлен
    }
    return next(action)
  }
        


        
Почему это работает?

        
    
          
  • Middleware — «привратник» между dispatch и reducer
    • 
          
  • Если next(action) не вызван — action «исчезает»
    • 
          
  • Store не обновится, подписчики не получат уведомление
    • 
          
  • Для вызывающего кода всё выглядит нормально — dispatch вернул undefined
    • 
        


        
Blacklist vs Whitelist


        

// Blacklist — блокировать конкретные action'ы
const BLOCKED: string[] = ['counter/reset', 'admin/deleteAll']

const blacklistFilter =
  (storeAPI: any) => (next: any) => (action: any): any => {
    if (BLOCKED.includes(action.type)) return
    return next(action)
  }

// Whitelist — пропускать только разрешённые
const ALLOWED: string[] = ['counter/incremented', 'counter/decremented']

const whitelistFilter =
  (storeAPI: any) => (next: any) => (action: any): any => {
    if (!ALLOWED.includes(action.type)) return
    return next(action)
  }
        


        
Практические сценарии

        
    
          
  • Права доступа — блокировать admin-действия для обычных пользователей
    • 
          
  • Валидация — не пропускать action с невалидным payload
    • 
          
  • Дебаунс — блокировать повторные action'ы слишком быстро
    • 
          
  • Feature flags — отключать функционал без изменения компонентов
    • 
        


        

          В демо нажмите «Reset» — кнопка отправляет dispatch, но middleware перехватывает
          action и не вызывает next(). Счётчик не сбрасывается. Посмотрите
          в консоль — middleware логирует каждую попытку.
        


        

          Фильтрация action'ов — мощный инструмент, но может запутать. Если action
          dispatch'ится, но ничего не происходит — разработчик может долго искать баг.
          Всегда логируйте заблокированные action'ы!
