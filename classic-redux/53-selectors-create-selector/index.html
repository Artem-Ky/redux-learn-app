    Теория — createSelector из Reselect
    
      
    
        
    Что такое Reselect?
    
        
    
          Reselect — библиотека для создания мемоизированных селекторов.
          Она кэширует последний результат и пересчитывает его только когда
          входные данные действительно изменились.
        
    

        
    
// Установка:
npm install reselect
        
    

        
    API createSelector
    
        
    
          createSelector принимает input selectors
          и output selector:
        
    

        
    
import { createSelector } from 'reselect'

const selectTodos = (state: RootState) => state.todos

// input selector → selectTodos
// output selector → todos => todos.map(t => t.id)
const selectTodoIds = createSelector(
  selectTodos,                       // input selector
  (todos) => todos.map(t => t.id)    // output selector
)
        
    

        
    
          Redux Fundamentals, Part 7: Standard Redux Patterns:
          «Reselect provides a function called createSelector that generates
          memoized selector functions. createSelector accepts one or more
          "input selector" functions as arguments, plus an "output selector", and returns
          the new selector function.»
          
    
          redux.js.org — Memoizing Selectors with createSelector
        
    

        
    Как работает кэширование
    
        
    
          При каждом вызове createSelector сравнивает результаты
          input selectors с предыдущими. Если они совпадают по ===,
          output selector не вызывается — возвращается кэш:
        
    

        
    
// Первый вызов:
selectTodoIds(state)
  → selectTodos(state) → todos[]  // input
  → (todos) => todos.map(...)       // output вызывается ✔
  → [1, 2, 3]                     // кэшируется

// Повторный вызов (todos не изменились):
selectTodoIds(state)
  → selectTodos(state) → todos[]  // те же ===
  → output НЕ вызывается           // возврат кэша ✔
  → [1, 2, 3]                     // та же ссылка!
        
    

        
    Сравнение: без мемоизации vs с мемоизацией
    

        
    
// БЕЗ мемоизации (урок 52):
const selectTodoIds = (state) =>
  state.todos.map(t => t.id)
// Каждый вызов → новый массив → новая ссылка

// С мемоизацией (createSelector):
const selectTodoIds = createSelector(
  selectTodos,
  (todos) => todos.map(t => t.id)
)
// Если todos те же → кэшированный массив → та же ссылка
        
    

        
    
          createSelector кэширует только последний результат
          (кэш размером 1). Если input selectors чередуют разные значения, кэш будет
          «промахиваться» каждый раз. Для большинства случаев этого достаточно.
        
    

        
    
          Не вызывайте createSelector внутри компонента или функции рендера!
          Каждый вызов создаёт новый мемоизированный селектор со своим кэшем.
          Определяйте селекторы на уровне модуля (вне компонентов).
