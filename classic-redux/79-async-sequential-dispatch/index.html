Теория — Последовательные dispatch в одном thunk

      

        
Множественные await + dispatch

        

          Один thunk может содержать несколько последовательных async-операций,
          каждая из которых завершается dispatch'ем. Это позволяет моделировать
          сложные цепочки загрузки.
        


        

          «Thunks may contain any arbitrary logic, sync or async, and can call
          dispatch or getState at any time. This makes them a great place
          for putting complex synchronous logic as well as moderate async logic
          that needs access to the store.»
          
— Redux — Writing Logic with Thunks
        


        
Паттерн: chaining async operations


        

const loadAll = () => async (dispatch: any): Promise<void> => {
  // Шаг 1: загрузка пользователей
  dispatch({ type: 'users/loading' })
  const users = await fetchUsers()
  dispatch({ type: 'users/loaded', payload: users })

  // Шаг 2: загрузка постов для каждого пользователя
  for (const user of users) {
    const posts = await fetchPosts(user.id)
    dispatch({ type: 'posts/loaded', payload: { userId: user.id, posts } })
  }
}
        


        
Отслеживание прогресса

        

          Последовательные dispatch'ы позволяют обновлять UI по мере выполнения —
          показывать прогресс-бар, пошаговый статус, счётчики.
        


        

dispatch({ type: 'progress/update', payload: {
  step: 2,
  total: 5,
  message: 'Loading posts for user 2/3...'
}})
        


        
Последовательная vs параллельная загрузка

        
    
          
  • Последовательная (for...of + await) — когда следующий шаг зависит от предыдущего
    • 
          
  • Параллельная (Promise.all) — когда шаги независимы (быстрее)
    • 
        


        

// Параллельная загрузка (быстрее, но нет пошагового прогресса)
const results = await Promise.all(
  users.map(u => fetchPosts(u.id))
)
        


        

          Последовательная загрузка даёт лучший UX при отображении прогресса —
          пользователь видит, что происходит, шаг за шагом. Параллельная — быстрее,
          но прогресс скачкообразный.
        


        

          Длинные цепочки await блокируют thunk. Если пользователь может
          отменить операцию, добавьте проверку флага отмены между шагами.
        


        
Ключевые моменты

        
    
          
  • Один thunk может содержать несколько await + dispatch
    • 
          
  • Каждый dispatch обновляет state и UI в реальном времени
    • 
          
  • Удобно для цепочки зависимых запросов
    • 
          
  • Используйте for...of для последовательной, Promise.all для параллельной загрузки
