    Теория — Композиция селекторов

    Что такое композиция селекторов?

    Композиция — это паттерн, при котором один селектор вызывает другие селекторы для получения промежуточных данных, а затем комбинирует результат. Каждый селектор отвечает за одну задачу — Single Responsibility Principle.

    Redux Fundamentals, Part 7: Standard Redux Patterns: «Selectors are primarily used to encapsulate logic for looking up specific values from state, logic for actually deriving values, and improving performance by avoiding unnecessary recalculations.»
    redux.js.org — Standard Patterns: Selectors

    Паттерн: маленькие селекторы → составной

    Вместо одного большого селектора, который знает всю структуру state, мы пишем базовые селекторы для каждого среза и комбинируем их:

    // Базовые (input) селекторы — по одному на каждый срез const selectTodos = (state: RootState) => state.todos const selectFilter = (state: RootState) => state.filter // Составной (composed) селектор — использует оба базовых const selectFilteredTodos = (state: RootState): Todo[] => { const todos = selectTodos(state) const filter = selectFilter(state) switch (filter) { case 'active': return todos.filter(t => !t.completed) case 'completed': return todos.filter(t => t.completed) default: return todos } }

    Зачем разбивать на маленькие?

    • Переиспользование — selectTodos можно использовать и в других составных селекторах
    • Тестирование — каждый селектор тестируется отдельно
    • Рефакторинг — при изменении формы state меняется только базовый селектор
    • Читаемость — составной селектор описывает «что», а не «как»

    Цепочка вызовов

    При вызове selectFilteredTodos(state) происходит:

    selectFilteredTodos(state) → selectTodos(state) // получаем todos[] → selectFilter(state) // получаем 'all' | 'active' | 'completed' → фильтрация // возвращаем результат
    Держите базовые селекторы максимально простыми — одна строка, прямой доступ к полю state. Вся логика преобразования — в составных селекторах.
    Пока мы используем обычные функции. При каждом вызове selectFilteredTodos создаётся новый массив через .filter(), даже если данные не изменились. Это станет проблемой — решение рассмотрим в следующих уроках (Reselect).

    Ключевые моменты

    • Композиция = селектор вызывает другие селекторы
    • Базовые селекторы — простой доступ к полям state
    • Составные селекторы — логика фильтрации, сортировки, трансформации
    • Single Responsibility: один селектор — одна задача
    • Переиспользование и лёгкое тестирование
