Теория — Что такое Reducer?

      

        
Определение

        

          Reducer — это функция, которая принимает текущий state
          и объект action, решает, как обновить состояние (если нужно),
          и возвращает новое состояние:
        


        

// Сигнатура reducer'а
(state, action) => newState
        


        

          «A reducer is a function that receives the current state and an action object,
          decides how to update the state if necessary, and returns the new state:
          (state, action) => newState. You can think of a reducer as an event listener
          which handles events based on the received action (event) type.»
          
— Redux Fundamentals, Part 2
        


        
Аналогия с Array.reduce()

        

          Reducer в Redux назван по аналогии с Array.reduce().
          Метод reduce() «сворачивает» массив в одно значение,
          обрабатывая элементы один за другим. Redux reducer «сворачивает»
          последовательность экшенов в итоговый state:
        


        

// Array.reduce — массив → одно значение
const sum = [1, 2, 3].reduce(
  (accumulator, item) => accumulator + item,
  0  // начальное значение
)
// 0 + 1 = 1, 1 + 2 = 3, 3 + 3 = 6

// Redux reducer — экшены → итоговый state
const actions = [
  { type: 'counter/incremented' },
  { type: 'counter/incremented' },
  { type: 'counter/addAmount', payload: 5 }
]
const finalState = actions.reduce(counterReducer, { value: 0 })
// { value: 0 } → { value: 1 } → { value: 2 } → { value: 7 }
        


        
Типичная структура reducer'а

        

function counterReducer(state = { value: 0 }, action) {
  switch (action.type) {
    case 'counter/incremented':
      return { ...state, value: state.value + 1 }
    case 'counter/decremented':
      return { ...state, value: state.value - 1 }
    case 'counter/addAmount':
      return { ...state, value: state.value + action.payload }
    case 'counter/reset':
      return { value: 0 }
    default:
      return state  // неизвестные экшены — без изменений
  }
}
        


        
Правила Reducer'а

        
Reducer должен соблюдать три строгих правила:


        

          «Reducers must always follow some specific rules:
          They should only calculate the new state value based on the state and action arguments.
          They are not allowed to modify the existing state.
          They must not do any asynchronous logic, calculate random values,
          or cause other "side effects".»
          
— Redux Fundamentals, Part 3
        


        
    
          
  • Чистота: результат зависит только от state и action
    • 
          
  • Иммутабельность: нельзя изменять state напрямую, только создавать копию
    • 
          
  • Без побочных эффектов: никаких API-вызовов, Math.random(), Date.now() и т.д.
    • 
        


        

          Reducer — это чистая функция. При одинаковых state
          и action он всегда возвращает одинаковый результат.
          Никогда не мутируйте state, не вызывайте API, не генерируйте случайные значения внутри reducer'а.
        


        

          Обратите внимание на default ветку в switch —
          при неизвестном экшене reducer обязан вернуть текущий state без изменений.
          Попробуйте кнопку «???» и посмотрите, что произойдёт.
