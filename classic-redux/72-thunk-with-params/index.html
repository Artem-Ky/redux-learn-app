Теория — Thunk с параметрами

      

        
Внешняя функция принимает параметры

        

          Thunk Action Creator — это обычная функция. Она может принимать
          любые аргументы. Внутренняя thunk-функция замыкается
          (closure) на этих аргументах и использует их внутри.
        


        

function saveNewTodo(text: string) {
  // Внешняя функция получает параметр text
  return async (dispatch: any, getState: any) => {
    // Внутренняя thunk замыкается на text
    dispatch({ type: 'todos/saving' })

    const response = await fetch('/api/todos', {
      method: 'POST',
      body: JSON.stringify({ text })
    })
    const todo = await response.json()

    dispatch({ type: 'todos/todoAdded', payload: todo })
  }
}
        


        
Замыкание (Closure)

        

          Внутренняя thunk-функция имеет доступ к переменным внешней функции.
          Это стандартный механизм JavaScript — замыкание. Параметр text
          «захватывается» и доступен в момент выполнения thunk.
        


        

// Вызов: dispatch передаёт text через замыкание
store.dispatch(saveNewTodo('Купить молоко'))

// Что происходит:
// 1. saveNewTodo('Купить молоко') → возвращает thunk
// 2. thunk замкнут на text = 'Купить молоко'
// 3. middleware вызывает thunk(dispatch, getState)
// 4. внутри thunk text === 'Купить молоко'
        


        

          «We can write a function that accepts 'text' as its parameter,
          and creates the thunk function to be dispatched... then our component
          only needs to dispatch the thunk with the text value.»
          
— Redux Essentials, Part 5: Async Logic and Data Fetching
        


        
Несколько параметров

        

          Внешняя функция может принимать сколько угодно параметров — все они
          будут доступны внутри thunk через замыкание.
        


        

function updateTodo(id: number, changes: Partial<Todo>) {
  return async (dispatch: any) => {
    // id и changes доступны через замыкание
    await apiUpdateTodo(id, changes)
    dispatch({ type: 'todos/updated', payload: { id, ...changes } })
  }
}
        


        

          Параметры thunk action creator — это данные от UI (текст из формы,
          ID элемента, фильтры). Thunk получает системные аргументы
          (dispatch, getState) от middleware. Разделение обязанностей!
        


        

          Не передавайте dispatch как параметр внешней функции — его
          предоставляет middleware. Внешняя функция только для данных от пользователя.
