Теория — Arrow function синтаксис

      

        
Два способа записи

        

          Middleware можно записать через function или через стрелочные функции.
          Результат абсолютно одинаковый — это лишь вопрос синтаксиса.
        


        
function-стиль (подробный)

        

function myMiddleware(storeAPI) {
  return function wrapDispatch(next) {
    return function handleAction(action) {
      // логика
      return next(action)
    }
  }
}
        


        
arrow-стиль (компактный)

        

const myMiddleware = storeAPI => next => action => {
  // логика
  return next(action)
}
        


        

          «We could also write the middleware using ES2015 arrow functions,
          which allows us to write them in a more compact way. The arrow function
          approach is more commonly seen in Redux codebases.»
          
— Redux Fundamentals, Part 4
        


        
Неявный vs явный return

        

          Если тело функции — одно выражение, стрелочная функция делает неявный return.
          Если нужна логика (несколько строк), используйте фигурные скобки и явный return:
        


        

// Минимальный middleware — неявный return:
const passThrough = storeAPI => next => action => next(action)

// С логикой — явный return:
const logger = storeAPI => next => action => {
  console.log('action:', action)
  const result = next(action)
  console.log('new state:', storeAPI.getState())
  return result
}
        


        
Читаемость

        

          Arrow-синтаксис компактнее, но может быть сложнее для начинающих. Три уровня
          вложенных стрелок a => b => c => { } не очевидны. Рекомендуется
          сначала освоить function-стиль, а потом перейти на стрелочный.
        


        

          В экосистеме Redux arrow-синтаксис стал конвенцией. Большинство
          примеров в документации, библиотеках (redux-thunk, redux-saga) и туториалах
          используют именно его. Привыкайте читать storeAPI => next => action => { }.
        


        
Сравнение бок о бок

        

// ╔═══════════════════════════════════════╗
// ║  function стиль   →   arrow стиль     ║
// ╠═══════════════════════════════════════╣
// ║  function(storeAPI)  storeAPI =>       ║
// ║  function(next)      next =>           ║
// ║  function(action)    action =>         ║
// ╚═══════════════════════════════════════╝
        


        

          Не забывайте return next(action)! Без return результат dispatch
          будет undefined, что может сломать цепочку middleware. Стрелочная
          функция без фигурных скобок делает return автоматически, но с фигурными — нет.
