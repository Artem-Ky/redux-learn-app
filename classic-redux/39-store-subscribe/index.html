Теория — store.subscribe(listener)

      

        
Подписка на изменения state

        

          Метод store.subscribe(listener) добавляет callback-функцию, которая
          вызывается после каждого dispatch. Внутри listener можно вызвать
          getState(), чтобы получить обновлённый state.
        


        

          «Adds a change listener. It will be called any time an action is dispatched,
          and some part of the state tree may potentially have changed. You may then call
          getState() to read the current state tree inside the callback.»
          
— Redux API Reference: Store — subscribe(listener)
        


        
subscribe возвращает unsubscribe

        

          Вызов subscribe возвращает функцию для отписки. Вызов этой функции
          удаляет listener из списка — он больше не будет вызываться при dispatch.
        


        

const unsubscribe = store.subscribe(() => {
  console.log('State изменился:', store.getState())
})

store.dispatch({ type: 'counter/incremented' })
// → "State изменился: { value: 1 }"

unsubscribe()

store.dispatch({ type: 'counter/incremented' })
// → (тишина — listener отписан)
// Но state ВСЁ РАВНО обновился! getState() вернёт { value: 2 }
        


        
Unsubscribe ≠ остановка dispatch

        

          Важно: отписка от subscribe не останавливает обновление state.
          Dispatch по-прежнему вызывает reducer и обновляет state. Просто listener
          больше не уведомляется.
        


        

          Попробуйте в демо: нажмите «Отписаться», затем нажмите +1 несколько раз.
          Счётчик на экране перестанет обновляться, но если нажать «Проверить getState()» —
          вы увидите, что state продолжал меняться.
        


        
Повторная подписка

        

          После отписки можно подписаться заново, вызвав subscribe с тем же
          или новым callback. Каждый вызов subscribe создаёт новую подписку.
        


        
Low-level API

        

          subscribe — низкоуровневый API. В реальных приложениях с React
          используется react-redux, который вызывает subscribe
          внутри себя (через useSelector и Provider).
        


        

          Listener вызывается при каждом dispatch, даже если state не изменился
          (reducer вернул тот же объект). Если нужна оптимизация — сравнивайте state
          до и после внутри listener.
