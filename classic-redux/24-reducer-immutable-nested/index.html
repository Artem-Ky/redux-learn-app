Теория — Каскадное копирование вложенных структур

      

        
Проблема: глубокая вложенность

        

          Чтобы иммутабельно обновить поле на N-м уровне вложенности,
          нужно создать копию каждого уровня от корня до целевого поля.
          Это называется «каскадное копирование» (cascading copies).
        


        

// Структура: company → department → employee → salary
const state = {
  company: {
    name: 'Рога и Копыта',
    department: {
      name: 'Разработка',
      employee: {
        name: 'Иван',
        salary: 120000       // ← хотим изменить ЭТО
      }
    }
  }
}
        


        
Каскад spread-операторов

        

// Для обновления salary нужно скопировать КАЖДЫЙ уровень:
return {
  ...state,                                 // копия корня
  company: {
    ...state.company,                       // копия company
    department: {
      ...state.company.department,           // копия department
      employee: {
        ...state.company.department.employee, // копия employee
        salary: 150000                      // ← новое значение
      }
    }
  }
}
        


        

          Каждый уровень вложенности добавляет ещё один spread. При 4-5 уровнях
          код становится трудночитаемым и хрупким.
          Это одна из главных болевых точек классического Redux и основная мотивация
          для создания библиотеки Immer.
        


        
Что происходит со ссылками?

        
При правильном иммутабельном обновлении salary:

        

state === newState                                        // false ← новый корень
state.company === newState.company                       // false ← скопирован
state.company.department === newState.company.department   // false ← скопирован
state.company.department.employee === ...               // false ← скопирован
        


        

          «Unfortunately, the process of correctly applying immutable updates to
          deeply nested state is verbose and hard to read. Immutable update logic
          like this is one of the main reasons why Redux Toolkit exists,
          and why we recommend using it.»
          
— Redux: Immutable Update Patterns
        


        
Решение: Immer (забегая вперёд)

        

// С Immer (используется в Redux Toolkit):
case 'employee/salaryUpdated':
  state.company.department.employee.salary = action.payload
  // Immer сам создаст все копии за кулисами!
        


        

          Сейчас мы учим классический подход, чтобы понимать, что делает Immer
          «под капотом». В реальных проектах используйте Redux Toolkit — он включает Immer
          и избавляет от каскадного копирования.
        


        
Ключевые моменты

        
    
          
  • Для вложенных обновлений нужен spread на каждом уровне
    • 
          
  • Чем глубже вложенность — тем больше кода
    • 
          
  • Все изменённые уровни получают новые ссылки
    • 
          
  • Это боль классического Redux → мотивация для Immer
    • 
          
  • Redux Toolkit (createSlice) решает эту проблему автоматически
