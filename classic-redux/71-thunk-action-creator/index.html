Интерактив — Thunk Action Creator

Теория — Thunk Action Creator

      

        
Зачем оборачивать thunk в функцию?

        

          Thunk — это функция (dispatch, getState) => .... Но мы не передаём
          её напрямую — мы создаём action creator, который возвращает
          thunk-функцию. Это даёт переиспользуемость и единообразный интерфейс.
        


        

// Thunk Action Creator
function fetchTodos() {
  // Возвращаем thunk-функцию
  return async (dispatch: any, getState: any) => {
    dispatch({ type: 'todos/loading' })
    const response = await fetch('/api/todos')
    const todos = await response.json()
    dispatch({ type: 'todos/loaded', payload: todos })
  }
}
        


        
Как это работает

        

          fetchTodos() вызывается — возвращает thunk-функцию.
          dispatch(fetchTodos()) передаёт эту функцию в middleware.
          Thunk middleware видит, что action — это функция, и вызывает её
          с аргументами (dispatch, getState).
        


        

// Вызов thunk action creator
store.dispatch(fetchTodos())

// Что происходит внутри:
// 1. fetchTodos() → возвращает async (dispatch, getState) => { ... }
// 2. dispatch(thunkFn) → middleware перехватывает
// 3. middleware вызывает thunkFn(dispatch, getState)
        


        

          «We said that Redux reducers must never contain 'side effects'...
          However, any real app needs to do these things somewhere.
          Thunk functions... can contain side effects.»
          
— Redux Essentials, Part 5: Async Logic and Data Fetching
        


        
Переиспользуемость

        

          Thunk action creator можно вызывать в любом месте приложения.
          Логика загрузки данных инкапсулирована в одном месте — компоненты
          просто вызывают dispatch(fetchTodos()).
        


        

          Thunk Action Creator — это обычная функция, которая возвращает другую функцию.
          Внешняя функция — action creator. Внутренняя — thunk. Этот паттерн
          обеспечивает чистый и предсказуемый интерфейс для асинхронных операций.
        


        

          Не путайте: fetchTodos — это action creator,
          fetchTodos() — это результат вызова (thunk-функция),
          dispatch(fetchTodos()) — это запуск thunk через middleware.
