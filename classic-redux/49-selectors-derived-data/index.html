          Теория — Селекторы для производных данных

          Derived data — что это?

          Производные данные (derived data) — это данные, которые вычисляются на основе state, а не хранятся напрямую. Вместо того чтобы хранить в state отдельно «активные todos», мы вычисляем их через селектор.

          «Selector functions can be used to calculate derived data, allowing Redux to store the minimal possible state. Selectors are efficient. A selector is not recomputed unless one of its arguments changes.»
          — Redux Fundamentals, Part 2

          Фильтрация через селекторы

          const selectTodos = (state: AppState): Todo[] => state.todos const selectCompletedTodos = (state: AppState): Todo[] => state.todos.filter(todo => todo.completed) const selectActiveTodos = (state: AppState): Todo[] => state.todos.filter(todo => !todo.completed)

          Подсчёт через селекторы

          const selectTodoCount = (state: AppState): number => state.todos.length const selectCompletedCount = (state: AppState): number => state.todos.filter(t => t.completed).length const selectActiveCount = (state: AppState): number => state.todos.filter(t => !t.completed).length

          Почему не хранить derived data в state?

          Один из принципов Redux — хранить минимальный state. Если данные можно вычислить из уже имеющегося state, их не нужно хранить отдельно:

          • Нет дублирования — один источник правды
          • Нет рассинхронизации — вычисленные данные всегда актуальны
          • Проще reducer — не нужно обновлять производные поля
          // ❌ Плохо: дублирование в state { todos: [...], completedTodos: [...], // дубль! activeTodos: [...], // дубль! todoCount: 5 // дубль! } // ✅ Хорошо: минимальный state + селекторы { todos: [...] // остальное — через selectCompletedTodos и т.д. }
          Логика вычислений находится в селекторах, а не в компонентах. Компонент просто вызывает selectCompletedTodos(state) и получает готовый результат, не зная деталей фильтрации.

          Примеры типичных derived-селекторов

          • selectCompletedTodos — фильтрация по статусу
          • selectTodoCount — подсчёт элементов
          • selectTotalPrice — сумма цен в корзине
          • selectUnreadMessages — непрочитанные сообщения
          • selectSortedItems — отсортированный список
          Помните: каждый вызов filter или map в селекторе создаёт новый массив. Это может быть проблемой для производительности в больших приложениях. В следующих уроках мы рассмотрим мемоизацию селекторов с помощью reselect.

          Ключевые моменты

          • Derived data — данные, вычисленные из state, а не хранимые
          • Селекторы используют filter, map, reduce для вычислений
          • Храните минимальный state — вычисляйте остальное через селекторы
          • Логика фильтрации/подсчёта — в селекторах, не в компонентах
          • Каждый вызов создаёт новую ссылку (мемоизация — тема будущих уроков)
