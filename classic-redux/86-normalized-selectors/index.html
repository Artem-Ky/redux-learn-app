Теория — Селекторы для нормализованного состояния

      

        
Прямой доступ по ID — O(1)

        

          Главное преимущество нормализации — мгновенный поиск по идентификатору.
          Вместо перебора массива мы обращаемся к объекту по ключу:
        


        

// O(n) — поиск в массиве
const todo = state.todos.find(t => t.id === id)

// O(1) — прямой доступ в нормализованном состоянии
const todo = state.entities[id]
        


        
Основные селекторы

        

          Для нормализованного состояния формируется набор стандартных селекторов,
          которые покрывают все основные сценарии чтения данных:
        


        

// Получить сущность по ID — O(1)
const selectTodoById = (state: RootState, id: number) =>
  state.entities[id]

// Получить все сущности как массив
const selectAllTodos = (state: RootState) =>
  Object.values(state.entities)

// Получить массив ID
const selectTodoIds = (state: RootState) =>
  state.ids
        


        

          «Storing data items in a lookup table by item ID lets us directly find any item by its ID
          without needing to loop through an entire array.»
          
— Redux Docs: Normalizing State Shape
        


        
Конвертация в массив — Object.values

        

          Когда нужно отрендерить список, используется Object.values(entities)
          для преобразования объекта-словаря обратно в массив. Массив ids
          при этом гарантирует порядок элементов:
        


        

// selectAllTodos с сохранением порядка
const selectAllTodos = (state: RootState) =>
  state.ids.map(id => state.entities[id])
        


        
Производительность нормализации

        

          При наличии большого количества записей разница в производительности
          между find() и прямым доступом становится значительной.
          Для 10 000 записей find() может перебрать все 10 000 элементов,
          а обращение entities[id] — это одна операция.
        


        

          Паттерн selectAllTodos / selectTodoById / selectTodoIds
          — стандартный набор селекторов для каждой нормализованной сущности. В Redux Toolkit
          createEntityAdapter генерирует их автоматически.
        


        

          Object.values(entities) создаёт новый массив при каждом вызове.
          Для оптимизации используйте мемоизацию (createSelector из reselect),
          чтобы пересчитывать массив только при изменении entities.
