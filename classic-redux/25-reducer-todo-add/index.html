    Теория — Добавление элементов в массив (reducer)
    
      
    
        
    Полный цикл: Action → Reducer → Новый State
    
        
    
          В этом уроке мы проходим полный цикл Redux на примере добавления задачи в todo-список.
          Пользователь вводит текст, нажимает «Добавить», и происходит следующее:
        
    
        
      
          
    1. Создаётся action объект { type: 'todos/todoAdded', payload: text }
      2. 
          
    2. Action отправляется в store через store.dispatch(action)
      3. 
          
    3. Store вызывает reducer с текущим state и action
      4. 
          
    4. Reducer создаёт новый массив с добавленным элементом
      5. 
          
    5. Store сохраняет новый state и уведомляет подписчиков
      6. 
        
    

        
    
          «Reducers must always follow some specific rules: They should only calculate the new
          state value based on the state and action arguments. They must make immutable updates,
          by copying the existing state and making changes to the copied values.»
          
    — Redux Fundamentals, Part 3
        
    

        
    Структура состояния Todo
    
        
    
          Каждая задача — это объект с тремя полями: уникальный id,
          текст задачи text и статус выполнения completed.
          State — это массив таких объектов:
        
    

        
    
interface Todo {
  id: number
  text: string
  completed: boolean
}

// State — массив задач
type TodoState = Todo[]

// Начальное состояние — пустой массив
const initialState: TodoState = []
        
    

        
    Reducer: добавление через spread
    
        
    
          Для иммутабельного добавления элемента в массив мы используем spread-оператор.
          Вместо state.push(newTodo) (мутация!) мы создаём новый массив,
          содержащий все старые элементы плюс новый:
        
    

        
    
function todosReducer(state: TodoState = initialState, action: TodoAction): TodoState {
  switch (action.type) {
    case 'todos/todoAdded': {
      const newTodo: Todo = {
        id: nextId++,
        text: action.payload,
        completed: false
      }
      return [...state, newTodo]  // ← новый массив!
    }
    default:
      return state
  }
}
        
    

        
    Генерация уникальных ID
    
        
    
          В этом примере мы используем простой инкрементный счётчик nextId
          для генерации уникальных идентификаторов. В реальном приложении можно использовать
          uuid, nanoid или ID из базы данных.
        
    

        
    
let nextId = 1

// Каждый вызов создаёт todo с уникальным id
// id: 1, id: 2, id: 3, ...
        
    

        
    
          Никогда не используйте push() в reducer!
          Метод push() мутирует существующий массив. Redux требует иммутабельных
          обновлений — всегда возвращайте новый массив через [...state, newItem].
        
    

        
    Почему [...state, newTodo], а не state.push(newTodo)?
    
        
    
          Spread создаёт новую ссылку на массив. Redux сравнивает ссылки
          (reference equality), чтобы определить, изменился ли state. Если вернуть тот же
          массив (мутированный через push), Redux не заметит изменений и UI не обновится.
        
    

        
    
          Обратите внимание на DevTools-панель: каждый dispatch todos/todoAdded
          показывает, как массив растёт. Diff показывает только добавленный элемент —
          остальные элементы массива не изменились.
        
    

        
    Ключевые моменты
    
        
      
          
    • Action todos/todoAdded несёт текст задачи в payload
      • 
          
    • Reducer создаёт новый объект todo с id, text, completed: false
      • 
          
    • Для добавления используем spread: [...state, newTodo]
      • 
          
    • Никогда не мутируем state напрямую (никакого push!)
      • 
          
    • ID генерируются инкрементно через внешний счётчик
