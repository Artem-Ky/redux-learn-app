Теория — dispatch(thunk) возвращает значение thunk'а

      

        
Возвращаемое значение dispatch

        

          Когда thunk middleware перехватывает функцию, он вызывает её и возвращает результат.
          Если thunk — async-функция, то dispatch(thunk()) вернёт Promise.
        


        

          «Since dispatching a thunk returns the thunk's return value,
          if we make the thunk function async, we can await it to know
          when the async work is done.»
          
— Redux Essentials, Part 5 — Async Logic and Data Fetching
        


        
Паттерн: await dispatch

        

          Это позволяет выполнить код после завершения асинхронной операции —
          показать уведомление, перенаправить пользователя, закрыть модальное окно.
        


        

const saveItem = (data: string) => {
  return async (dispatch: any): Promise<void> => {
    dispatch({ type: 'save/pending' })
    await fakeApiSave(data)
    dispatch({ type: 'save/fulfilled', payload: data })
  }
}

// В обработчике клика:
await store.dispatch(saveItem('My item') as any)
showToast('Saved!') // выполнится после завершения
        


        
Как это работает внутри

        

          Thunk middleware возвращает результат вызова thunk-функции:
        


        

// Упрощённый thunk middleware
if (typeof action === 'function') {
  return action(dispatch, getState) // ← возвращает Promise
}
        


        

          dispatch(thunk()) возвращает то, что возвращает сама thunk-функция.
          Для async-функции это всегда Promise. Используйте
          await для последовательного UI-обновления.
        


        
Когда это полезно

        
    
          
  • Показать уведомление (toast) после сохранения
    • 
          
  • Закрыть модальное окно после успешного запроса
    • 
          
  • Перенаправить на другую страницу после создания ресурса
    • 
          
  • Обновить локальный state компонента на основе результата
    • 
        


        

          Не забывайте обрабатывать ошибки при await dispatch(...) —
          если thunk выбросит исключение, промис будет отклонён. Используйте try/catch
          вокруг await.
        


        
Ключевые моменты

        
    
          
  • dispatch(thunk()) возвращает возвращаемое значение thunk'а
    • 
          
  • Async thunk → dispatch возвращает Promise
    • 
          
  • Можно await store.dispatch(saveItem(data))
    • 
          
  • Полезно для UI-фидбека после асинхронных операций
