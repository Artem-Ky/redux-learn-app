Теория — Зачем выносить типы в константы

      

        
Проблема: строковые литералы

        

          Когда action type — это строковый литерал, который повторяется в нескольких местах
          (dispatch, reducer, middleware), легко допустить опечатку. JavaScript
          не выдаст ошибку — reducer просто не обработает экшен, и баг будет тихим.
        


        

// В reducer
case 'counter/incremented': ...

// В dispatch — опечатка!
store.dispatch({ type: 'counter/incremeted' })  // «e» пропущена
// Reducer не найдёт совпадения → вернёт state без изменений
// Ни ошибки, ни предупреждения — тихий баг!
        


        
Решение: выносим в константы

        

// actionTypes.ts
export const INCREMENTED = 'counter/incremented'
export const DECREMENTED = 'counter/decremented'
export const RESET       = 'counter/reset'

// reducer.ts
import { INCREMENTED, DECREMENTED, RESET } from './actionTypes'

case INCREMENTED: ...
case DECREMENTED: ...
case RESET: ...

// dispatch
store.dispatch({ type: INCREMENTED })
        


        
Преимущества констант

        
    
          
  • Защита от опечаток — если написать INCREMETED, IDE и компилятор выдадут ошибку
    • 
          
  • Автодополнение — IDE предлагает имена констант при вводе
    • 
          
  • Единый источник истины — строка определена в одном месте
    • 
          
  • Рефакторинг — переименование меняет значение везде
    • 
          
  • Поиск — легко найти все места использования конкретного типа
    • 
        


        
Типичная структура файлов

        

src/
├── store/
│   ├── actionTypes.ts    // константы типов
│   ├── actions.ts        // action creators (следующий урок)
│   ├── reducer.ts        // reducer
│   └── store.ts          // createStore
        


        

          В Redux Toolkit константы не нужны — createSlice автоматически
          генерирует типы и action creators. Но понимание констант важно для работы
          с legacy-кодом и для понимания, почему RTK упрощает жизнь.
        


        
Ключевые моменты

        
    
          
  • Строковые литералы подвержены тихим ошибкам из-за опечаток
    • 
          
  • Выносите action types в именованные константы
    • 
          
  • Используйте одну и ту же константу в reducer и dispatch
    • 
          
  • В Redux Toolkit этот паттерн встроен в createSlice
