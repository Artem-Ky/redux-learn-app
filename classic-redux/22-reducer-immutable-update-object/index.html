Теория — Spread-оператор для обновления объекта

      

        
Почему нельзя мутировать state?

        

          Redux определяет, изменился ли state, через сравнение ссылок (===).
          Если вы мутируете объект напрямую — ссылка остаётся прежней,
          и Redux не увидит изменение. UI не обновится.
        


        

// ❌ ПЛОХО — мутация
case 'profile/nameChanged':
  state.name = action.payload  // мутация!
  return state                    // та же ссылка — Redux не заметит
        


        
Правильный паттерн: spread + перезапись

        

// ✔ ПРАВИЛЬНО — новый объект
case 'profile/nameChanged':
  return {
    ...state,                           // копируем все поля
    name: action.payload              // перезаписываем нужное
  }
        


        

          Оператор ...state создаёт поверхностную копию (shallow copy) объекта.
          Затем мы перезаписываем только нужное поле. Результат — новый объект
          с новой ссылкой, и Redux обнаружит изменение.
        


        

          «Reducers are not allowed to modify the existing state. Instead, they must make
          immutable updates, by copying the existing state and making changes to the copied values.»
          
— Redux Fundamentals, Part 3
        


        
Как Redux обнаруживает изменения

        

const oldState = store.getState()
store.dispatch(action)
const newState = store.getState()

// Redux проверяет:
oldState === newState   // false → state изменился, обновить UI
                        // true  → state не изменился, ничего не делать
        


        

          Запомните: { ...state, field: newValue } — базовый паттерн
          иммутабельного обновления объекта в Redux. Он создаёт новый объект, копируя все поля
          и перезаписывая только те, что нужно изменить.
        


        
Ключевые моменты

        
    
          
  • Reducer должен возвращать новый объект, а не мутировать старый
    • 
          
  • Паттерн: { ...state, changedField: newValue }
    • 
          
  • Redux проверяет изменения через === (сравнение ссылок)
    • 
          
  • Мутация state → Redux не увидит изменение → UI не обновится
    • 
          
  • Spread создаёт поверхностную копию (вложенные объекты — те же ссылки)
