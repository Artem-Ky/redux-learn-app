Теория — Полный API объекта Store

      

        
Четыре метода Store

        

          Redux store предоставляет минимальный, но мощный API из четырёх методов.
          Каждый метод выполняет одну конкретную задачу:
        


        

          «The Redux store brings together the state, actions, and reducers that make up
          your app. The store has several responsibilities: holds the current application
          state, allows access via getState(), allows state to be updated
          via dispatch(action), registers listeners via
          subscribe(listener).»
          
— Redux API Reference: Store
        


        
1. getState()

        

          Возвращает текущее дерево состояния приложения. Не принимает аргументов.
          Это единственный способ «прочитать» state:
        


        

const state = store.getState()
console.log(state) // { value: 42 }
        


        
2. dispatch(action)

        

          Единственный способ изменить state. Принимает action-объект и передаёт
          его в reducer. Возвращает переданный action:
        


        

store.dispatch({ type: 'counter/incremented' })
// reducer обрабатывает action → state обновляется
// все listener'ы уведомляются
        


        
3. subscribe(listener)

        

          Регистрирует функцию-callback, которая вызывается после каждого dispatch.
          Возвращает функцию unsubscribe для отписки:
        


        

const unsubscribe = store.subscribe(() => {
  console.log('State обновился:', store.getState())
})

// Позже, когда listener больше не нужен:
unsubscribe()
        


        
4. replaceReducer(nextReducer)

        

          Заменяет текущий reducer. Используется для code splitting и hot reloading.
          После замены автоматически диспатчится @@INIT:
        


        

const newReducer = (state = { value: 0 }, action) => {
  // новая логика
}
store.replaceReducer(newReducer)
        


        
Сводная таблица


        

┌────────────────────┬──────────────────────────────────────┐
│ Метод              │ Назначение                           │
├────────────────────┼──────────────────────────────────────┤
│ getState()         │ Чтение текущего state                │
│ dispatch(action)   │ Отправка action → обновление state   │
│ subscribe(fn)      │ Подписка на изменения state          │
│ replaceReducer(fn) │ Замена reducer (code splitting / HMR)│
└────────────────────┴──────────────────────────────────────┘
        


        

          В повседневной работе вы используете в основном getState,
          dispatch и subscribe. Метод replaceReducer
          нужен только для продвинутых сценариев вроде code splitting или горячей
          перезагрузки модулей.
        


        

          Не вызывайте dispatch внутри reducer'а! Это приведёт к
          непредсказуемому поведению. Reducer должен быть чистой функцией —
          только вычислять новый state и возвращать его.
