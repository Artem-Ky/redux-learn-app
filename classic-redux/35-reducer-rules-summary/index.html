Теория — Все правила reducer'ов

      

        
Полный список правил reducer'а

        

          Reducer — это чистая функция, которая принимает текущий state
          и action, и возвращает новый state. Вот все правила, которые необходимо соблюдать:
        


        

          «Reducers must always follow some specific rules:
          They should only calculate the new state value based on the state and action arguments.
          They are not allowed to modify the existing state.
          They must not do any asynchronous logic, calculate random values, or cause other "side effects".»
          
— Redux Fundamentals, Part 3
        


        
1. Чистая функция (Pure Function)

        

          Reducer должен быть чистой функцией: результат зависит
          только от аргументов (state и action). Никаких внешних зависимостей.
        


        

// ✅ Чистая функция
function reducer(state: number, action: Action): number {
  return state + action.payload
}

// ❌ Зависимость от внешней переменной
let multiplier = 2
function reducer(state: number, action: Action): number {
  return state + action.payload * multiplier
}
        


        
2. Нет побочных эффектов (No Side Effects)

        

          Reducer не должен: логировать в консоль, делать HTTP-запросы,
          записывать в localStorage, изменять DOM, генерировать случайные числа.
        


        

// ❌ Побочные эффекты
function reducer(state, action) {
  console.log(action)              // side effect!
  localStorage.setItem('state', ...) // side effect!
  fetch('/api')                     // side effect!
  return { ...state, value: Math.random() } // непредсказуемо!
}
        


        
3. Иммутабельные обновления

        

          Reducer никогда не изменяет существующий state напрямую.
          Всегда создаётся новый объект/массив через spread или методы,
          возвращающие новый массив.
        


        

// ❌ Мутация!
state.count = state.count + 1
state.items.push(newItem)

// ✅ Иммутабельное обновление
return { ...state, count: state.count + 1 }
return { ...state, items: [...state.items, newItem] }
        


        
4. Возврат state для неизвестных actions

        

          Если reducer получает action, который он не обрабатывает, он
          должен вернуть текущий state без изменений.
          Это делается через default в switch.
        


        

switch (action.type) {
  case 'known/action':
    return { ...state, updated: true }
  default:
    return state  // ← обязательно!
}
        


        
5. Нет асинхронной логики

        

          Reducer — синхронная функция. Никаких async/await,
          setTimeout, Promise, fetch.
          Асинхронная логика выносится в middleware (thunk, saga и т.д.).
        


        
6. Нет случайных значений

        

          Math.random(), Date.now(), crypto.randomUUID()
          — всё это делает reducer непредсказуемым. Такие значения должны
          генерироваться до dispatch и передаваться в action.
        


        

// ❌ В reducer
return { ...state, id: Date.now(), random: Math.random() }

// ✅ В action creator
const action = {
  type: 'item/added',
  payload: { id: Date.now(), random: Math.random() }
}
store.dispatch(action)
        


        

          Пройдите квиз в демо выше — каждый вопрос показывает reducer и спрашивает,
          правильный ли он. Это поможет закрепить все правила на практике.
        


        

          Нарушение правил reducer'ов приводит к непредсказуемому поведению:
          DevTools перестают работать, time-travel debugging ломается,
          React-компоненты не перерисовываются. Правила — не рекомендации,
          а обязательные требования.
        


        
Ключевые моменты

        
    
          
  • Reducer — чистая функция: (state, action) → newState
    • 
          
  • Нет побочных эффектов: нет fetch, console.log, localStorage
    • 
          
  • Иммутабельные обновления: только spread, map, filter — никогда push, splice
    • 
          
  • default всегда возвращает state
    • 
          
  • Нет async: reducer — синхронный
    • 
          
  • Нет random/Date.now: непредсказуемые значения — в action creator
