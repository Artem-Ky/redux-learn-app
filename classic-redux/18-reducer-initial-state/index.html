Теория — Начальное состояние и default-параметр

ES6 default-параметр в Reducer

Reducer использует ES6 default parameter для определения начального состояния. Когда state равен undefined, вместо него подставляется значение по умолчанию:

const initialState = { value: 0 } // state = initialState — default-параметр ES6 function counterReducer(state = initialState, action) { switch (action.type) { case 'counter/incremented': return { ...state, value: state.value + 1 } default: return state } }

Экшен @@INIT

При вызове createStore(reducer) Redux автоматически отправляет специальный «инициализирующий» экшен с типом @@INIT (или @@redux/INIT...). Это заставляет reducer вызваться с state = undefined:

// Что происходит внутри createStore: const store = createStore(counterReducer) // 1. Redux вызывает: counterReducer(undefined, { type: '@@INIT' }) // 2. state === undefined → срабатывает default: state = { value: 0 } // 3. action.type === '@@INIT' → попадает в default case → return state // 4. Store сохраняет { value: 0 } как начальное состояние
«The first time that the reducer is called, state will be undefined. That's our chance to return an initial value for the state. Reducers normally use ES6 default argument syntax to provide initial state: (state = initialState, action).»
— Redux Fundamentals, Part 3

Что если забыть default-параметр?

Если reducer не имеет default-параметра, при вызове с undefined он вернёт undefined — и это ошибка! State не должен быть undefined:

// ❌ Без default-параметра function badReducer(state, action) { switch (action.type) { case 'increment': return { value: state.value + 1 } // TypeError: Cannot read 'value' of undefined default: return state // return undefined } }

preloadedState vs default-параметр

createStore принимает необязательный второй аргумент — preloadedState. Если он указан, Redux использует его вместо default-параметра reducer'а:

// С preloadedState — default-параметр НЕ используется const store = createStore( counterReducer, { value: 42 } // ← preloadedState ) // store.getState() → { value: 42 } // Без preloadedState — используется default из reducer const store = createStore(counterReducer) // store.getState() → { value: 0 } (из initialState)
preloadedState часто используется для гидрации — восстановления состояния из localStorage или серверного рендеринга (SSR). В обычных случаях достаточно default-параметра в reducer'е.

Ключевые моменты

  • Начальное состояние задаётся через ES6 default-параметр: state = initialState
  • createStore() отправляет @@INIT для инициализации
  • Reducer вызывается с state = undefined → срабатывает default
  • preloadedState (2-й аргумент createStore) переопределяет default
  • Без default-параметра state будет undefined — ошибка!
