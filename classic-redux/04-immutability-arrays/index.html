Теория — Иммутабельные операции с массивами

      

        
Мутирующие методы (нельзя в Redux)

        

// Все эти методы ИЗМЕНЯЮТ оригинальный массив:
arr.push(item)       // добавляет в конец
arr.pop()            // удаляет последний
arr.splice(i, 1)    // удаляет по индексу
arr.sort()           // сортирует на месте
arr.reverse()        // разворачивает на месте
arr[i] = newValue   // прямое присвоение
        


        
Иммутабельные альтернативы (правильно для Redux)

        

// Добавление
const added = [...arr, newItem]            // spread + новый элемент
const added2 = arr.concat(newItem)        // concat возвращает новый массив

// Удаление
const removed = arr.filter((_, i) => i !== index)

// Обновление элемента
const updated = arr.map((item, i) =>
  i === index ? newValue : item
)

// Сортировка (сначала копия!)
const sorted = [...arr].sort()

// Вставка в середину
const inserted = [
  ...arr.slice(0, index),
  newItem,
  ...arr.slice(index)
]
        


        

          «We can do this by hand using JavaScript's array / object spread operators,
          as well as array methods that return new copies of the array instead of
          mutating the original array.»
          
— Redux Fundamentals, Part 2
        


        

          Частая ошибка: array.sort() мутирует оригинал!
          Всегда делайте копию перед сортировкой: [...array].sort()
