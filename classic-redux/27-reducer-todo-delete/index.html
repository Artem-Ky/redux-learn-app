    Теория — Удаление элементов из массива (filter)
    
      
    
        
    filter() — иммутабельное удаление
    
        
    
          Для удаления элемента из массива в Redux мы используем метод filter().
          Он создаёт новый массив, содержащий только те элементы, которые
          прошли проверку (predicate). Элемент с нужным id просто не попадает
          в новый массив.
        
    

        
    
          «We can use array methods like filter() to create a new array without the item
          we want to remove, or use map() to create a copy of an array where we replace
          items that match a condition.»
          
    — Redux Fundamentals, Part 3
        
    

        
    Паттерн: filter по id
    
        
    
          Reducer получает id задачи для удаления в action.payload
          и возвращает новый массив без этого элемента:
        
    

        
    
case 'todos/todoDeleted': {
  return state.filter(todo => todo.id !== action.payload)
}
        
    

        
    
          Это читается как: «верни все задачи, чей id не равен переданному».
          Результат — новый массив, на один элемент короче.
        
    

        
    Почему не splice()?
    
        
    
          Метод splice() мутирует исходный массив. В Redux
          это запрещено — мы должны всегда возвращать новый массив:
        
    

        
    
// ✔ Правильно: filter создаёт новый массив
return state.filter(todo => todo.id !== action.payload)

// ✖ Неправильно: splice мутирует массив
const index = state.findIndex(t => t.id === action.payload)
state.splice(index, 1)  // ← мутация!
return state            // ← та же ссылка
        
    

        
    Что происходит внутри
    
        
    
          Допустим, у нас есть 3 задачи с id 1, 2, 3. Мы хотим удалить задачу с id 2:
        
    

        
    
// До: state = [{ id: 1 }, { id: 2 }, { id: 3 }]

store.dispatch({ type: 'todos/todoDeleted', payload: 2 })

// filter проверяет каждый элемент:
// id: 1 !== 2 → true  → включить
// id: 2 !== 2 → false → исключить
// id: 3 !== 2 → true  → включить

// После: state = [{ id: 1 }, { id: 3 }]
        
    

        
    
          Откройте DevTools и удалите задачу — вы увидите, как в diff элемент исчезает
          из массива. Длина массива (state.length) уменьшается на 1.
        
    

        
    
          Если в массиве несколько элементов с одинаковым id, filter()
          удалит все совпадения. Убедитесь, что id уникальны!
        
    

        
    Ключевые моменты
    
        
      
          
    • filter() создаёт новый массив — это иммутабельная операция
      • 
          
    • Паттерн: state.filter(item => item.id !== action.payload)
      • 
          
    • Никогда не используйте splice() в reducer
      • 
          
    • В payload передаём только id удаляемого элемента
      • 
          
    • filter не мутирует исходный массив — это ключевое требование Redux
