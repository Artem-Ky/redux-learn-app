Теория — Почему default case критически важен

      

        
Reducer ОБЯЗАН вернуть state для неизвестных экшенов

        

          Когда reducer получает экшен, type которого он не распознаёт,
          он должен вернуть текущий state без изменений.
          Это правило — не рекомендация, а обязательное требование.
        


        

          «Every reducer should handle the default case. The reducer must always return
          the existing state for any unknown actions. If the reducer doesn't recognize
          the action type, it should return the existing state unchanged.»
          
— Redux Fundamentals, Part 3
        


        
Что произойдёт БЕЗ default case?

        

// ❌ ПЛОХО — нет default
function badReducer(state = initialState, action) {
  switch (action.type) {
    case 'counter/incremented':
      return { ...state, value: state.value + 1 }
  }
  // Если type не совпал — функция возвращает undefined!
  // Redux заменит ваш state на undefined
}
        


        

          Без default: return state reducer вернёт undefined
          для любого неизвестного экшена. Это уничтожит ваш state!
          Redux даже выбросит ошибку при инициализации, если reducer вернёт undefined.
        


        
Правильная реализация

        

// ✔ ПРАВИЛЬНО — есть default
function goodReducer(state = initialState, action) {
  switch (action.type) {
    case 'counter/incremented':
      return { ...state, value: state.value + 1 }
    case 'counter/decremented':
      return { ...state, value: state.value - 1 }
    default:
      return state  // ← неизвестный экшен — возвращаем state
  }
}
        


        
Почему приходят «неизвестные» экшены?

        
    
          
  • Redux сам отправляет @@INIT при создании store
    • 
          
  • В реальном приложении бывает несколько reducer (combineReducers) — каждый получает все экшены
    • 
          
  • Middleware и сторонние библиотеки могут отправлять свои экшены
    • 
        


        

          Правило: default case — это «сеть безопасности» вашего reducer.
          Если тип экшена не распознан — просто верните state как есть.
          Это гарантирует, что state никогда не станет undefined.
        


        
Ключевые моменты

        
    
          
  • default: return state — обязательный паттерн
    • 
          
  • Без default reducer вернёт undefined для неизвестных экшенов
    • 
          
  • Все reducer получают все экшены в приложении
    • 
          
  • Redux отправляет @@INIT — reducer должен его корректно обработать
    • 
          
  • DevTools: diff для неизвестного экшена покажет «Без изменений»
