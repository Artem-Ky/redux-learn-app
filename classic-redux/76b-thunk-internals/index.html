Теория — Анатомия thunk middleware

      

        
Исходный код thunk middleware (5 строк)

        

          Вся магия thunk — это буквально 5 строк кода. Вот полная реализация:
        


        

const thunkMiddleware = storeAPI => next => action => {
  // Если action — функция, вызываем её с dispatch и getState
  if (typeof action === 'function') {
    return action(storeAPI.dispatch, storeAPI.getState)
  }
  // Иначе — обычный action, передаём следующему middleware
  return next(action)
}
        


        
Пошаговый разбор

        
Когда вы вызываете store.dispatch(something):


        
    
          
  1. 
            dispatch перехватывается middleware pipeline.
            Thunk middleware — первый (или один из первых) в цепочке.
          
    2. 
          
  2. 
            typeof проверка: middleware смотрит — something это функция или объект?
          
    3. 
          
  3. 
            Если функция (thunk):
            
      
              
    • Вызывает action(dispatch, getState)
      • 
              
    • dispatch здесь — это storeAPI.dispatch, то есть полный pipeline сначала
      • 
              
    • getState — для чтения текущего state
      • 
              
    • Возвращает результат вызова (может быть Promise!)
      • 
              
    • Action НЕ доходит до reducer — функцию reducer не поймёт
      • 
            
    
          
    4. 
          
  4. 
            Если объект:
            
      
              
    • Вызывает next(action) — передаёт дальше по pipeline
      • 
              
    • Доходит до reducer как обычно
      • 
            
    
          
    5. 
        


        
Ключевой момент: storeAPI.dispatch vs next

        

          Thunk middleware передаёт в функцию storeAPI.dispatch, а не next.
          Это значит, что когда thunk-функция вызывает dispatch(action),
          action проходит весь pipeline заново, включая сам thunk middleware.
          Это позволяет dispatch'ить другие thunks из thunk!
        


        

// Thunk может dispatch'ить другие thunks!
const outerThunk = () => async (dispatch) => {
  // dispatch проходит ВЕСЬ pipeline заново
  await dispatch(innerThunk())  // ← тоже thunk!
  dispatch({ type: 'done' })   // ← обычный action
}

const innerThunk = () => async (dispatch) => {
  const data = await fetch('/api')
  dispatch({ type: 'data/loaded', payload: data })
}
        


        
Синхронные thunks

        

          Thunk не обязан быть асинхронным! Можно использовать для доступа к getState:
        


        

// Синхронный thunk — для условной логики
const incrementIfOdd = () => (dispatch, getState) => {
  const { value } = getState()
  if (value % 2 !== 0) {
    dispatch({ type: 'counter/incremented' })
  }
}
        


        

          «Thunk functions can be used for both asynchronous and synchronous logic.
          Thunks provide a way to write any reusable logic that needs access to
          dispatch and getState.»
          
— Redux Fundamentals, Part 6
        


        
Что возвращает dispatch(thunk)?

        

          store.dispatch(thunk()) возвращает то, что возвращает thunk-функция.
          Если thunk — async функция, dispatch вернёт Promise. Это позволяет делать:
          
await store.dispatch(fetchData())
          
и выполнять код после завершения async операции.
