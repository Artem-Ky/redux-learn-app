    Теория — Проблема ссылочного равенства (Reference Identity)
    
      
    
        
    В чём проблема?
    
        
    
          Когда селектор использует .map(), .filter() или любой метод,
          создающий новый массив, результат всегда будет новой ссылкой —
          даже если исходные данные не изменились.
        
    

        
    
const selectTodoIds = (state: RootState) =>
  state.todos.map(t => t.id)

const result1 = selectTodoIds(store.getState())
const result2 = selectTodoIds(store.getState())

result1 === result2  // false! Всегда новый массив
        
    

        
    Почему это важно?
    
        
    
          В React-Redux хук useSelector по умолчанию сравнивает результат
          селектора через === (строгое равенство). Если ссылка новая —
          компонент перерендерится, даже если данные те же.
        
    

        
    
          Redux Fundamentals, Part 7: Standard Redux Patterns:
          «If we call useSelector(selectTodoIds) at the top of a component,
          and the selector returns a new array reference every time an action is dispatched,
          this component would re-render every time, even if the array of IDs didn't change!»
          
    
          redux.js.org — Memoized Selectors
        
    

        
    Методы, создающие новую ссылку
    
        
      
          
    • .map() — всегда новый массив
      • 
          
    • .filter() — всегда новый массив
      • 
          
    • .slice() — всегда новый массив
      • 
          
    • .concat() — всегда новый массив
      • 
          
    • Spread [...arr] — всегда новый массив
      • 
          
    • Object.assign() / {...obj} — всегда новый объект
      • 
        
    

        
    Что такое мемоизация?
    
        
    
          Мемоизация — это техника кэширования результата функции. Если аргументы
          не изменились, функция возвращает закэшированный результат
          (ту же ссылку), вместо пересчёта.
        
    

        
    
// Псевдокод мемоизации:
let lastInput, lastResult

function memoizedSelector(state) {
  const input = selectTodos(state)
  if (input === lastInput) return lastResult  // та же ссылка!
  lastInput = input
  lastResult = input.map(t => t.id)
  return lastResult
}
        
    

        
    
          Библиотека Reselect реализует именно этот паттерн: она кэширует
          последний результат и пересчитывает только когда входные данные действительно изменились.
          Мы рассмотрим createSelector в следующем уроке.
        
    

        
    
          Если селектор возвращает примитив (число, строку, boolean), проблемы нет —
          === сравнивает по значению. Проблема только с объектами и массивами,
          где === сравнивает по ссылке.
        
    

        
    Ключевые моменты
    
        
      
          
    • map/filter всегда создают новый массив (новую ссылку)
      • 
          
    • === для объектов/массивов сравнивает ссылки, не содержимое
      • 
          
    • useSelector вызывает ре-рендер при новой ссылке
      • 
          
    • Мемоизация кэширует последний результат и возвращает его при тех же входных данных
      • 
          
    • Reselect — стандартное решение для мемоизированных селекторов в Redux
