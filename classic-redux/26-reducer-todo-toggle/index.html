    Теория — Обновление одного элемента массива через map()
    
      
    
        
    Задача: изменить один элемент, не трогая остальные
    
        
    
          Переключение статуса задачи — классический пример обновления одного элемента
          в массиве. Нам нужно найти задачу по id и изменить только её поле
          completed, оставив все остальные элементы без изменений.
        
    

        
    
          «If we want to update one specific item in an array, we have to make copies
          of all the items that we're not changing, and only change the one item
          we need to update.»
          
    — Redux Fundamentals, Part 3
        
    

        
    Паттерн: map() с условием
    
        
    
          Метод map() создаёт новый массив, проходя по каждому
          элементу. Для элемента с нужным id мы возвращаем новый объект с
          изменённым полем. Для всех остальных — возвращаем элемент как есть:
        
    

        
    
case 'todos/todoToggled': {
  return state.map(todo =>
    todo.id === action.payload
      ? { ...todo, completed: !todo.completed }
      : todo  // ← остальные элементы без изменений
  )
}
        
    

        
    Почему именно map(), а не цикл с индексом?
    
        
    
          map() всегда возвращает новый массив — это гарантирует иммутабельность.
          Паттерн todo.id === action.payload ? новыйОбъект : todo чётко показывает,
          какой элемент обновляется. Это читаемо и безопасно.
        
    

        
    
// ✔ Правильно: map создаёт новый массив
return state.map(todo =>
  todo.id === id ? { ...todo, completed: !todo.completed } : todo
)

// ✖ Неправильно: мутация через индекс
state[index].completed = !state[index].completed  // ← мутация!
return state  // ← та же ссылка, Redux не заметит изменений
        
    

        
    Новый объект только для изменённого элемента
    
        
    
          Обратите внимание: мы создаём новый объект ({ ...todo, completed: !todo.completed })
          только для того элемента, который действительно изменился. Остальные элементы
          возвращаются по старой ссылке — это важно для оптимизации (React пропустит
          ре-рендер неизменённых элементов).
        
    

        
    
          В DevTools-панели посмотрите на diff после каждого toggle — изменяется только
          один элемент массива. Поле completed переключается между
          true и false.
        
    

        
    Action для toggle
    
        
    
          Для переключения нам нужен только id задачи — текущее значение
          completed мы берём из state внутри reducer:
        
    

        
    
// Action — в payload передаём только id
{ type: 'todos/todoToggled', payload: 2 }

// Reducer знает текущее значение completed
// и инвертирует его: true → false, false → true
        
    

        
    
          Не передавайте completed: true в payload! Reducer сам знает текущее значение
          и инвертирует его через !todo.completed. Это предотвращает рассинхронизацию
          между UI и state.
        
    

        
    Ключевые моменты
    
        
      
          
    • Используйте map() для обновления одного элемента массива
      • 
          
    • Паттерн: item.id === action.payload ? { ...item, field: newValue } : item
      • 
          
    • Новый объект создаётся только для изменённого элемента
      • 
          
    • В payload передаём только id, reducer берёт текущее значение из state
      • 
          
    • Никогда не обращайтесь к элементу по индексу для мутации
