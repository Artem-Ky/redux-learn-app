Теория — Timing middleware

      

        
Зачем замерять время dispatch?

        

          Middleware может замерять, сколько времени занимает обработка каждого action.
          Это помогает выявить медленные reducer'ы и оптимизировать
          производительность приложения.
        


        

          «Middleware provides a third-party extension point between dispatching an action,
          and the moment it reaches the reducer.»
          
— Redux Docs — Middleware
        


        
Паттерн: performance.now() до и после

        

const timingMiddleware =
  (storeAPI: any) => (next: any) => (action: any): any => {
    const start = performance.now()
    const result = next(action)
    const end = performance.now()
    const duration = (end - start).toFixed(2)

    console.log(`${action.type} took ${duration}ms`)
    return result
  }
        


        
Как это работает?

        
    
          
  • performance.now() — точный таймер (микросекунды, не миллисекунды как Date.now())
    • 
          
  • Замер до next(action) — фиксирует начало
    • 
          
  • Замер после — фиксирует конец
    • 
          
  • Разница = время выполнения всех middleware ниже + reducer
    • 
        


        
Профилирование dispatch'ей

        

          В демо выше «Fast +1» выполняется почти мгновенно (< 0.1ms), а «Slow +1»
          искусственно замедлен тяжёлым циклом. Timing middleware показывает разницу.
        


        

// Пример: предупреждение о медленных action'ах
const timingMiddleware =
  (storeAPI: any) => (next: any) => (action: any): any => {
    const start = performance.now()
    const result = next(action)
    const ms = performance.now() - start

    if (ms > 1) {
      console.warn(`⚠ Slow dispatch: ${action.type} (${ms.toFixed(2)}ms)`)
    }
    return result
  }
        


        
Мониторинг в продакшене

        

          В реальных приложениях timing данные отправляются в системы мониторинга
          (DataDog, New Relic) для отслеживания деградации производительности
          во времени.
        


        

          performance.now() возвращает время в миллисекундах с точностью до
          микросекунд. Это намного точнее, чем Date.now(), который считает
          только целые миллисекунды.
        


        

          Не оставляйте timing middleware в продакшен-сборке без необходимости — он добавляет
          небольшой overhead к каждому dispatch. Используйте условное подключение:
          process.env.NODE_ENV !== 'production'.
        


        
Ключевые моменты

        
    
          
  • Замер: performance.now() до и после next(action)
    • 
          
  • Показывает время выполнения reducer + нижестоящих middleware
    • 
          
  • Помогает найти медленные reducer'ы
    • 
          
  • Можно настроить порог для предупреждений
    • 
          
  • В продакшене — отправка данных в систему мониторинга
