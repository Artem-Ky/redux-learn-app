Теория — Trace в DevTools

Зачем нужен Trace?

В большом приложении один и тот же тип экшена может диспатчиться из десятков мест: из компонентов, middleware, thunks, обработчиков событий. Trace показывает точное место в коде, откуда был вызван dispatch.

// Включение trace в DevTools Extension: const store = createStore( reducer, window.__REDUX_DEVTOOLS_EXTENSION__?.({ trace: true, // включить трейсинг traceLimit: 25 // глубина стека (по умолчанию 10) }) )

Как это работает?

Когда trace: true включен, DevTools перехватывает каждый dispatch и вызывает new Error().stack для захвата текущего стека вызовов. При выборе экшена во вкладке «Trace» вы увидите полный путь от корня до вызова dispatch.

// Как мы эмулируем trace в нашем DevTools: const originalDispatch = store.dispatch store.dispatch = (action: AnyAction): AnyAction => { const trace = new Error().stack traceMap.set(action.type, trace) return originalDispatch(action) }
«The Trace tab in Redux DevTools shows the JavaScript stack trace that led to the action being dispatched. This is useful for tracking down exactly where a particular action is coming from.»
— Redux DevTools Extension — Trace

Отладка сложных цепочек dispatch

Trace особенно полезен для отладки сложных цепочек: когда один thunk диспатчит несколько экшенов, или middleware трансформирует один экшен в другой. Trace покажет, что todos/loaded был вызван из loadTodos thunk, а не из компонента напрямую.

Trace добавляет незначительные накладные расходы (захват стека вызовов). В большинстве случаев это не заметно, но для production-сборки рекомендуется отключить DevTools полностью.
Trace не работает с минифицированным кодом — имена функций будут нечитаемыми. Для отладки используйте development-сборку с source maps. В production DevTools всё равно следует отключать.

Ключевые моменты

  • trace: true включает захват стека вызовов при dispatch
  • traceLimit задаёт глубину стека (по умолчанию 10)
  • Trace показывает точное место в коде, откуда вызван dispatch
  • Незаменим для отладки цепочек dispatch в thunks и middleware
  • Работает через new Error().stack
  • Используйте source maps для читаемых имён функций
