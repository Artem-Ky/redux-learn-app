Теория — applyMiddleware

      

        
Что такое applyMiddleware?

        

          applyMiddleware — это store enhancer, встроенный в Redux.
          Он принимает список middleware и возвращает enhancer, который оборачивает
          dispatch store'а цепочкой middleware.
        


        

          «The applyMiddleware enhancer is itself an example of a store enhancer that is included
          with Redux. It lets us change how dispatch works, and enables us to write middleware
          that can contain whatever logic we want.»
          
— Redux Fundamentals, Part 4
        


        
Подключение к createStore

        

          applyMiddleware передаётся как второй или третий аргумент
          createStore. Если нет preloadedState — как второй:
        


        

import { legacy_createStore as createStore, applyMiddleware } from 'redux'

// Без preloadedState:
const store = createStore(
  rootReducer,
  applyMiddleware(middleware1, middleware2)
)

// С preloadedState:
const store = createStore(
  rootReducer,
  preloadedState,
  applyMiddleware(middleware1, middleware2)
)
        


        
Порядок middleware имеет значение

        

          Middleware выполняются в порядке передачи в applyMiddleware.
          Первый получает action первым, последний — последним перед reducer'ом.
          Это конвейер (pipeline):
        


        

applyMiddleware(print1, print2, print3)

// Порядок выполнения:
// dispatch(action)
//   → print1 получает action, логирует "1", вызывает next(action)
//     → print2 получает action, логирует "2", вызывает next(action)
//       → print3 получает action, логирует "3", вызывает next(action)
//         → reducer(state, action)
        


        
Простейший middleware

        

const print1 = storeAPI => next => action => {
  console.log('1')
  return next(action)
}
        


        

          applyMiddleware — это store enhancer, а не просто утилита. Он возвращает
          функцию, которую createStore использует для оборачивания store.
          Поэтому middleware получают доступ к dispatch и getState.
        


        
Pipeline — конвейерная обработка

        

          Middleware образуют pipeline — цепочку обработчиков. Каждый
          middleware решает: передать action дальше (next(action)),
          модифицировать его, или вовсе остановить. Это похоже на конвейер на заводе —
          каждая станция делает свою работу и передаёт деталь дальше.
        


        

          Если вы поменяете порядок middleware в applyMiddleware, изменится
          порядок обработки action. В демо выше попробуйте мысленно представить, что будет
          если порядок будет applyMiddleware(print3, print1, print2).
