Теория — switch/case vs if/else в reducer

      

        
Redux не диктует стиль ветвления

        

          Redux не требует использования switch/case в reducer.
          Это лишь соглашение (convention), принятое в сообществе. Вы можете
          использовать if/else, lookup-таблицы или любой другой паттерн.
        


        
Вариант 1: switch/case (классический)

        

function counterReducer(state = initialState, action) {
  switch (action.type) {
    case 'counter/incremented':
      return { ...state, value: state.value + 1 }
    case 'counter/decremented':
      return { ...state, value: state.value - 1 }
    default:
      return state
  }
}
        


        
Вариант 2: if/else

        

function counterReducer(state = initialState, action) {
  if (action.type === 'counter/incremented') {
    return { ...state, value: state.value + 1 }
  } else if (action.type === 'counter/decremented') {
    return { ...state, value: state.value - 1 }
  } else {
    return state
  }
}
        


        
Вариант 3: Lookup-таблица

        

const handlers = {
  'counter/incremented': (state) =>
    ({ ...state, value: state.value + 1 }),
  'counter/decremented': (state) =>
    ({ ...state, value: state.value - 1 }),
}

function counterReducer(state = initialState, action) {
  const handler = handlers[action.type]
  return handler ? handler(state, action) : state
}
        


        

          «We definitely recommend using switch statements for handling actions,
          but it is not a requirement. The docs use switch statements,
          but you can use if/else, lookup tables, or any other approach.»
          
— Redux FAQ: Reducers
        


        
Когда что использовать?

        

          switch/case — когда много типов экшенов. Чисто и структурировано.

          if/else — для 2-3 типов или сложных условий (например, range проверки).

          Lookup-таблица — для динамических / плагинных систем, когда обработчики добавляются извне.
        


        
Ключевые моменты

        
    
          
  • Redux не требует использования switch/case
    • 
          
  • switch — соглашение сообщества, не техническое требование
    • 
          
  • Главное: reducer — чистая функция (state, action) → newState
    • 
          
  • Все три подхода дают идентичный результат
    • 
          
  • В Redux Toolkit используется Immer + builder-паттерн, заменяющий switch
