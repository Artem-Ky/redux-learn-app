Теория — store.getState()

      

        
Что возвращает getState()?

        

          Метод store.getState() возвращает текущее дерево состояния (state tree)
          приложения. Это то же самое значение, которое вернул последний вызов reducer.
        


        

          «Returns the current state tree of your application. It is equal to the last value
          returned by the store's reducer.»
          
— Redux API Reference: Store — getState()
        


        
getState() возвращает ту же ссылку

        

          Важно: getState() не создаёт копию state.
          Он возвращает ту же самую ссылку, которую вернул reducer.
          Если reducer вернул тот же объект (например, в default case),
          повторные вызовы getState() вернут === true.
        


        

const stateBefore = store.getState()
store.dispatch({ type: 'counter/incremented' })
const stateAfter = store.getState()

console.log(stateBefore === stateAfter) // false — reducer вернул НОВЫЙ объект
        


        

const stateA = store.getState()
store.dispatch({ type: 'unknown/action' })
const stateB = store.getState()

console.log(stateA === stateB) // true — default case вернул ТОТ ЖЕ объект
        


        
Без защитного копирования

        

          Redux не делает защитное копирование (defensive copy) при вызове
          getState(). Это значит, что мутация возвращённого объекта
          изменит state store напрямую — именно поэтому reducer должен возвращать
          новый объект при каждом изменении.
        


        

          Никогда не мутируйте объект, полученный от getState()!
          Это нарушит принцип иммутабельности и сломает React-Redux, DevTools
          и любой код, зависящий от ссылочного сравнения.
        


        
Когда вызывать getState()

        
    
          
  • В subscribe callback — чтобы узнать новый state после dispatch
    • 
          
  • В обработчиках событий — чтобы прочитать текущий state перед dispatch
    • 
          
  • В middleware — для доступа к state при обработке action
    • 
        


        

          Нажмите кнопки выше и посмотрите, как getState() вызывается после
          каждого dispatch. Обратите внимание на проверку ссылочного равенства — при реальном
          изменении ссылка всегда новая (=== false).
        


        
Ключевые моменты

        
    
          
  • getState() возвращает текущее дерево state
    • 
          
  • Возвращается та же ссылка, что вернул reducer — не копия
    • 
          
  • После dispatch с реальным изменением → === будет false
    • 
          
  • После dispatch без изменения (default case) → === будет true
    • 
          
  • Никогда не мутируйте результат getState()
