Теория — One-Way Data Flow

Что такое однонаправленный поток данных?

Прежде чем углубляться в Redux, важно понять базовый паттерн управления состоянием. Любое приложение можно разбить на три части:

  • State — источник истины, описывает состояние приложения в конкретный момент
  • View — визуальное представление, основанное на текущем state
  • Actions — события, происходящие на основе действий пользователя, которые запускают обновления state
«State describes the condition of the app at a specific point in time. The UI is rendered based on that state. When something happens (such as a user clicking a button), the state is updated based on what occurred. The UI re-renders based on the new state.»
— Redux Fundamentals, Part 2

Проблема масштабирования

В примере выше всё просто: одно состояние, один view, прямые действия. Но когда несколько компонентов нуждаются в одних данных, особенно если они расположены в разных частях приложения — простота ломается.

Решение — вынести общее состояние из компонентов в централизованное место. Это и есть основная идея Redux: единое место для хранения глобального state, с определёнными правилами обновления.

Код примера

// State — простой объект let state = { count: 0 } // View — функция отрисовки function render() { document.getElementById('display').textContent = state.count } // Action — обновление state и перерисовка function increment() { state = { ...state, count: state.count + 1 } // иммутабельно! render() }
Обратите внимание: даже без Redux мы используем иммутабельное обновление ({ ...state, count: state.count + 1 } вместо state.count++). Это важнейший паттерн, который мы подробно разберём в следующих уроках.
