Теория — Кэширование в Reselect

Кэш размером 1

Reselect кэширует только последний результат (cache size = 1). Если входные данные не изменились — возвращается закэшированный результат без вызова output-функции.

const selectCompletedTodos = createSelector( state => state.todos, // input selector todos => todos.filter(t => t.completed) // output selector ) // Первый вызов: вычисляет output selectCompletedTodos(state) // recompute = 1 // Повторный вызов: state.todos === прежний → кэш! selectCompletedTodos(state) // recompute = 1 (не изменился)

Проверка равенства входов (===)

Input selectors проверяются через строгое равенство ===. Если ссылка на результат input selector'а не изменилась — кэш действителен. Если ссылка новая — output пересчитывается.

// Dispatch действия, не меняющего todos: store.dispatch({ type: 'ui/toggle' }) // state.todos === previousState.todos → кэш сработает! selectCompletedTodos(store.getState()) // без пересчёта

Когда кэш «ломается»

Кэш инвалидируется, когда input selector возвращает новую ссылку. Даже если данные структурно одинаковые — другая ссылка = пересчёт.

// ❌ Каждый вызов создаёт новый массив — кэш ломается! const badSelector = createSelector( state => state.todos.filter(t => t.active), // новый массив! active => active.length )
«Reselect provides a function called createSelector to generate memoized selectors. createSelector accepts one or more "input selector" functions, plus an "output selector", and returns a new selector function. Every time you call the selector, it checks the inputs — if any input changed, it re-runs the output selector.»
— Redux Fundamentals: Memoized Selectors

defaultMemoize

Внутри Reselect использует функцию defaultMemoize (или lruMemoize в новых версиях). Она хранит один последний результат и сравнивает аргументы по ссылке (===). Это оптимально для Redux, где getState() возвращает тот же объект до dispatch.

Влияние на производительность

  • createSelector предотвращает лишние пересчёты тяжёлых операций (filter, map, sort)
  • Cache size = 1 — подходит для Redux, где state меняется через dispatch
  • Не подходит для параметризованных селекторов с разными аргументами в одном рендере
  • Для мульти-кэша используйте createSelector с кастомным memoize
Правило: input selectors должны быть «чистыми извлекателями». Они просто достают кусочек state. Вся трансформация — в output selector.
Если input selector создаёт новый объект/массив при каждом вызове — кэш никогда не сработает. Это самая частая ошибка с Reselect.
