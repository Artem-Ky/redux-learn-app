    Теория — Обновление конкретных полей элемента массива
    
      
    
        
    Редактирование текста — обновление одного поля
    
        
    
          В предыдущих уроках мы переключали completed (boolean). Теперь мы
          обновляем text (string). Принцип тот же: map() с условием
          по id, но вместо инверсии boolean мы подставляем новую строку.
        
    

        
    
          «We need to make a copy of every level of nesting that needs to be updated.
          For updating a specific item in an array, we use array.map() to create
          a new version of the array.»
          
    — Redux Fundamentals, Part 3
        
    

        
    Action с составным payload
    
        
    
          Для редактирования нам нужны два данных: id задачи и новый text.
          Поэтому payload — это объект:
        
    

        
    
// Action для редактирования
{
  type: 'todos/todoEdited',
  payload: {
    id: 2,
    text: 'Новый текст задачи'
  }
}
        
    

        
    Reducer: деструктуризация payload
    
        
    
          В reducer мы извлекаем id и text из action.payload
          через деструктуризацию и используем map() для обновления:
        
    

        
    
case 'todos/todoEdited': {
  const { id, text } = action.payload
  return state.map(todo =>
    todo.id === id
      ? { ...todo, text }  // ← обновляем только text
      : todo
  )
}
        
    

        
    Локальное состояние vs Redux state
    
        
    
          Обратите внимание: состояние «сейчас редактируется» (editingId)
          хранится не в Redux, а в локальной переменной. Это правильный подход —
          временное UI-состояние (открыт ли модал, какой input активен) не нужно
          хранить в Redux store.
        
    

        
    
// Локальное UI-состояние — НЕ в Redux
let editingId: number | null = null

// Redux state — только данные задач
// [{ id, text, completed }, ...]
        
    

        
    
          Общее правило: если состояние нужно только одному компоненту и не влияет
          на другие части приложения — храните его локально. Данные, которые нужны
          нескольким компонентам или должны пережить навигацию — в Redux.
        
    

        
    Spread для обновления одного поля
    
        
    
          Запись { ...todo, text } создаёт новый объект со всеми полями
          из todo, но с обновлённым text. Остальные поля
          (id, completed) копируются из оригинала:
        
    

        
    
// Исходный объект
{ id: 2, text: 'Старый текст', completed: false }

// { ...todo, text: 'Новый текст' }
{ id: 2, text: 'Новый текст', completed: false }
// ↑ id и completed — из оригинала, text — новый
        
    

        
    
          Если пользователь вводит пустую строку, не сохраняйте изменения.
          В нашем примере мы проверяем text.trim() перед dispatch.
        
    

        
    Ключевые моменты
    
        
      
          
    • payload может быть объектом с несколькими полями: { id, text }
      • 
          
    • Деструктуризация const { id, text } = action.payload делает код читаемым
      • 
          
    • map() + spread обновляет только нужный элемент и нужное поле
      • 
          
    • Временное UI-состояние (editingId) хранится локально, не в Redux
      • 
          
    • Валидируйте данные перед dispatch (пустая строка, пробелы и т.д.)
