Теория — createStore: создание Redux Store

      

        
Что такое Store?

        

          Store — это объект, который объединяет всё в Redux: state, reducer
          и API для взаимодействия. Каждое Redux-приложение имеет ровно один store.
        


        

          «The Redux store brings together the state, actions, and reducers that make up your app.
          The store has several responsibilities: holds the current application state inside,
          allows access to the current state via store.getState(), allows state to be updated
          via store.dispatch(action), and registers listener callbacks via store.subscribe(listener).»
          
— Redux Fundamentals, Part 4
        


        
createStore(reducer)

        

          Функция createStore принимает reducer и возвращает store.
          При создании store автоматически диспатчит специальный action @@INIT,
          чтобы reducer вернул начальное состояние.
        


        

import { legacy_createStore as createStore } from 'redux'

function counterReducer(state: number = 0, action: Action): number {
  switch (action.type) {
    case 'counter/incremented':
      return state + 1
    default:
      return state
  }
}

const store = createStore(counterReducer)
// ↑ Redux автоматически диспатчит @@INIT
// reducer получает (undefined, { type: '@@redux/INIT...' })
// → срабатывает default параметр state = 0
// → начальный state = 0
        


        
@@INIT — скрытый первый action

        

          Когда createStore вызывается, Redux немедленно диспатчит action с type
          @@redux/INIT.... Этот action не совпадает ни с одним case
          вашего reducer, поэтому срабатывает default, и reducer возвращает
          начальное значение (default параметр функции).
        


        

          Откройте DevTools-панель выше — вы увидите @@INIT как самый первый
          action в истории. Это подтверждает, что store уже работает до любого ручного dispatch.
        


        
Ответственности Store

        
    
          
  • Хранит текущий state приложения
    • 
          
  • getState() — позволяет читать текущий state
    • 
          
  • dispatch(action) — позволяет обновлять state
    • 
          
  • subscribe(listener) — регистрирует callback на изменения
    • 
        


        

const store = createStore(counterReducer)

// 1. Прочитать state
console.log(store.getState()) // 0

// 2. Обновить state
store.dispatch({ type: 'counter/incremented' })

// 3. Прочитать обновлённый state
console.log(store.getState()) // 1
        


        

          createStore в новых версиях Redux помечен как deprecated в пользу
          Redux Toolkit configureStore. Мы используем
          legacy_createStore для изучения основ без лишних абстракций.
        


        
Ключевые моменты

        
    
          
  • createStore(reducer) создаёт store и автоматически вызывает @@INIT
    • 
          
  • Reducer при @@INIT получает undefined как state → возвращает default
    • 
          
  • Store — единственный источник правды (single source of truth)
    • 
          
  • Store предоставляет 3 метода: getState, dispatch, subscribe
