Интерактив — Что возвращает dispatch()?

Middleware может вернуть любое значение вместо результата next(action). Это значение станет результатом store.dispatch().

Теория — Возвращаемое значение middleware

      

        
Что возвращает dispatch() по умолчанию?

        

          Без middleware store.dispatch(action) возвращает сам action объект.
          Но middleware может изменить это поведение, вернув что угодно.
        


        

          «Middleware can also modify the return value of dispatch. The simplest example
          is a middleware that always returns a specific value... dispatch will return
          whatever the outermost middleware returns.»
          
— Redux Fundamentals, Part 4: Store — Middleware
        


        
Цепочка возвращаемых значений

        

          Возвращаемое значение dispatch() — это то, что вернёт
          первый (внешний) middleware в цепочке. Каждый middleware
          может передать значение от next(action) или заменить его.
        


        

// Middleware, который всегда возвращает строку
const alwaysReturnHello = (storeAPI: any) => (next: any) => (action: any) => {
  next(action) // action всё равно проходит
  return 'Hello!' // но возвращаем свою строку
}

const result = store.dispatch({ type: 'test' })
console.log(result) // 'Hello!'
        


        
Middleware, возвращающий Promise

        

          Особенно полезно для асинхронных middleware: они могут вернуть Promise,
          и вызывающий код сможет «дождаться» завершения асинхронной операции.
        


        

const asyncMiddleware = (storeAPI: any) => (next: any) => (action: any) => {
  if (action.type === 'async/run') {
    return new Promise(resolve => {
      setTimeout(() => {
        next(action)
        resolve('done!')
      }, 1000)
    })
  }
  return next(action)
}

// Можно использовать await:
const result = await store.dispatch({ type: 'async/run' })
console.log(result) // 'done!'
        


        
Thunks и возвращаемые значения

        

          Именно этот механизм позволяет thunk middleware возвращать Promise
          из dispatch(thunkFunction). Компоненты могут дождаться завершения
          асинхронной логики.
        


        

          Возвращаемое значение dispatch — мощный механизм. Thunk middleware использует
          его для возврата промисов, что позволяет компонентам ждать завершения
          асинхронных операций через await dispatch(thunk).
        


        

          Не полагайтесь на возвращаемое значение dispatch в обычном коде — это
          деталь реализации middleware. В стандартном Redux (без middleware) dispatch
          возвращает action, но это поведение может измениться при добавлении middleware.
