Теория — Action Creator с параметрами

      

        
Передача аргументов в Action Creator

        

          В предыдущем уроке мы создали простые action creator'ы без параметров.
          Но в реальных приложениях экшены почти всегда содержат payload —
          данные, описывающие, что именно произошло. Чтобы формировать payload,
          action creator принимает аргументы:
        


        

// Action creator с параметром
function todoAdded(text: string) {
  return {
    type: 'todos/todoAdded',
    payload: text
  }
}

// Использование:
store.dispatch(todoAdded('Купить молоко'))
// → { type: 'todos/todoAdded', payload: 'Купить молоко' }
        


        

          «An action creator is a function that creates and returns an action object.
          We typically use these so we don't have to write the action object by hand every time.»
          
— Redux Fundamentals, Part 2
        


        
Зачем передавать параметры?

        

          Без параметров action creator всегда возвращает один и тот же объект.
          С параметрами — он формирует payload на основе входных данных:
        

        
    
          
  • Текст новой задачи → payload: 'Купить молоко'
    • 
          
  • Сумма для перевода → payload: 500
    • 
          
  • ID для удаления → payload: 42
    • 
        


        
Инкапсуляция формата экшена

        

          Action creator инкапсулирует детали формирования экшена.
          Вызывающий код не знает и не должен знать, как именно устроен объект экшена.
          Если формат изменится (например, payload станет объектом), менять нужно
          только action creator:
        


        

// Раньше:
function todoAdded(text: string) {
  return { type: 'todos/todoAdded', payload: text }
}

// Позже — формат изменился, но вызывающий код НЕ меняется:
function todoAdded(text: string) {
  return { type: 'todos/todoAdded', payload: { text, createdAt: Date.now() } }
}

// В обоих случаях вызов одинаковый:
store.dispatch(todoAdded('Купить молоко'))
        


        

          Action creator — это единственное место, где формируется
          объект экшена. Это упрощает рефакторинг: при изменении структуры payload
          нужно обновить только функцию-creator, а не каждый dispatch().
        


        
Reducer для обработки

        

function todosReducer(state = [], action) {
  switch (action.type) {
    case 'todos/todoAdded':
      return [...state, {
        id: state.length + 1,
        text: action.payload,
        completed: false
      }]
    default:
      return state
  }
}
        


        
Ключевые моменты

        
    
          
  • Action creator принимает аргументы и формирует из них payload
    • 
          
  • Вызывающий код работает с простым интерфейсом: todoAdded('текст')
    • 
          
  • Формат экшена инкапсулирован внутри creator'а
    • 
          
  • Reducer получает готовый объект экшена и обрабатывает action.payload
