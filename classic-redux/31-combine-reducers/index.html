    Теория — combineReducers API
    
      
    
        
    Что такое combineReducers?
    
        
    
          combineReducers — утилита из Redux, которая принимает объект с
          слайс-reducer'ами и возвращает один корневой reducer (root reducer).
          Ключи этого объекта определяют форму state.
        
    

        
    
          «combineReducers takes an object full of slice reducer functions, and creates
          a function that outputs a corresponding state object with the same keys.»
          
    — Redux Fundamentals, Part 3
        
    

        
    API combineReducers
    
        
    
          Функция принимает единственный аргумент — объект, где ключ — имя слайса,
          а значение — reducer для этого слайса:
        
    

        
    
import { combineReducers } from 'redux'

const rootReducer = combineReducers({
  todos: todosReducer,
  filters: filtersReducer
})

// rootReducer — это обычная функция:
// (state, action) => newState
        
    

        
    Ключи формируют форму state
    
        
    
          Ключи объекта, переданного в combineReducers, становятся ключами
          корневого state. Это значит, что форма state определяется ключами,
          а не reducer'ами:
        
    

        
    
combineReducers({
  todos: todosReducer,
  filters: filtersReducer
})

// Результат: state = {
//   todos: [...],    ← результат todosReducer
//   filters: {...}   ← результат filtersReducer
// }
        
    

        
    Root Reducer — концепция
    
        
    
          Результат combineReducers — это root reducer
          (корневой reducer). Он передаётся в createStore. При каждом dispatch
          корневой reducer вызывает каждый слайс-reducer с его частью state и текущим action.
        
    

        
    
// Внутри combineReducers делает примерно это:
function rootReducer(state, action) {
  return {
    todos: todosReducer(state.todos, action),
    filters: filtersReducer(state.filters, action)
  }
}
        
    

        
    Каждый слайс получает только свою часть state
    
        
    
          Это ключевой момент: todosReducer получает только
          state.todos (массив), а не весь объект state.
          filtersReducer получает только state.filters.
          Слайс-reducer'ы изолированы друг от друга.
        
    

        
    
          Попробуйте добавить задачу и переключить фильтр в демо выше.
          Обратите внимание, как полный state содержит оба слайса, но
          каждый reducer управляет только своей частью.
        
    

        
    
          Если слайс-reducer не обрабатывает полученный action, он обязан
          вернуть текущий state без изменений (через default в switch).
          Иначе этот слайс станет undefined.
        
    

        
    Ключевые моменты
    
        
      
          
    • combineReducers создаёт root reducer из набора слайс-reducer'ов
      • 
          
    • Ключи объекта определяют форму итогового state
      • 
          
    • Каждый слайс-reducer получает только свою часть state
      • 
          
    • Root reducer вызывает все слайс-reducer'ы при каждом dispatch
      • 
          
    • Слайс-reducer должен возвращать state по умолчанию для неизвестных actions
