Теория — Logger Middleware

      

        
Зачем logger?

        

          Logger middleware — это самый популярный пример кастомного middleware. Он записывает
          в консоль какой action был отправлен и каким стал state
          после обработки. Это незаменимый инструмент для отладки.
        


        
Паттерн logger middleware

        

const loggerMiddleware = storeAPI => next => action => {
  console.log('dispatching', action)
  let result = next(action)
  console.log('next state', storeAPI.getState())
  return result
}
        


        

          «Here's a small example of a middleware that logs every action dispatched to the
          store, along with the state after the action was handled. This is a simplified
          version of the real redux-logger middleware.»
          
— Redux Fundamentals, Part 4
        


        
Как это работает

        
    
          
  1. console.log('dispatching', action) — логируем action ДО обработки
    2. 
          
  2. let result = next(action) — передаём action дальше (в reducer)
    3. 
          
  3. console.log('next state', storeAPI.getState()) — логируем state ПОСЛЕ
    4. 
          
  4. return result — возвращаем результат для следующего middleware в цепочке
    5. 
        


        
Расширенный logger

        

          Можно добавить больше информации: state до dispatch, время, группировку:
        


        

const detailedLogger = storeAPI => next => action => {
  console.group(action.type)
  console.log('prev state', storeAPI.getState())
  console.log('action', action)

  const start = Date.now()
  let result = next(action)
  const ms = Date.now() - start

  console.log('next state', storeAPI.getState())
  console.log('duration', ms + 'ms')
  console.groupEnd()

  return result
}
        


        
Почему return result?

        

          next(action) возвращает значение — обычно сам action. Middleware
          должен вернуть это значение, чтобы не сломать цепочку. Вызывающий код
          store.dispatch(action) может зависеть от возвращаемого значения.
        


        

// Полная цепочка return:
const logger = storeAPI => next => action => {
  console.log('dispatching', action)
  let result = next(action)   // result = возврат от reducer/middleware
  console.log('next state', storeAPI.getState())
  return result                // ← обязательно!
}
        


        

          Этот паттерн — «before / next / after» — основа для большинства middleware.
          Код до next(action) выполняется ДО reducer'а, код после —
          ПОСЛЕ. Это позволяет видеть и «до» и «после» в одном middleware.
        


        

          В продакшене обычно используют библиотеку redux-logger, которая
          форматирует вывод с цветами, diff'ами, и свёрнутыми группами. Но написать
          свой простой logger — отличный способ понять middleware.
        


        
Ключевые моменты

        
    
          
  • Logger — самый частый пример кастомного middleware
    • 
          
  • Логирует action и resulting state для каждого dispatch
    • 
          
  • Паттерн: log → next(action) → log → return result
    • 
          
  • Код до next — ДО reducer'а, после — ПОСЛЕ
    • 
          
  • Всегда возвращайте result из next(action)
