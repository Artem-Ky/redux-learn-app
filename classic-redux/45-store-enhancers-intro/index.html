Теория — Store Enhancers

      

        
Что такое Store Enhancer?

        

          Store Enhancer — это функция высшего порядка, которая оборачивает
          createStore и может модифицировать поведение store. Это третий
          (необязательный) аргумент createStore.
        


        

          «Store enhancers are a formal mechanism for adding capabilities to Redux itself.
          Most people will never need to write one. An enhancer wraps the store's dispatch,
          getState, and subscribe functions, and can override them.»
          
— Redux Fundamentals, Part 4
        


        
Сигнатура enhancer

        

// Enhancer — это функция, принимающая createStore
// и возвращающая улучшенный createStore
type StoreEnhancer = (
  createStore: CreateStore
) => (
  reducer: Reducer,
  preloadedState?: State
) => Store
        


        
Третий аргумент createStore

        

          Enhancer передаётся как третий аргумент. Redux определяет тип аргумента:
          если второй аргумент — функция, он считается enhancer'ом (без preloadedState).
        


        

// С preloadedState и enhancer:
createStore(reducer, preloadedState, enhancer)

// Только с enhancer (без preloadedState):
createStore(reducer, enhancer)
        


        
Пример: logEnhancer

        

function logEnhancer(createStore) {
  return (reducer, preloadedState) => {
    const store = createStore(reducer, preloadedState)

    const originalDispatch = store.dispatch
    store.dispatch = (action) => {
      console.log('[enhancer] action:', action)
      console.log('[enhancer] state до:', store.getState())
      const result = originalDispatch(action)
      console.log('[enhancer] state после:', store.getState())
      return result
    }

    return store
  }
}

const store = createStore(reducer, logEnhancer)
        


        
Что может enhancer

        
    
          
  • Обернуть dispatch — добавить логирование, аналитику, дебаунс
    • 
          
  • Обернуть getState — трансформировать возвращаемый state
    • 
          
  • Обернуть subscribe — фильтровать уведомления
    • 
          
  • Добавить методы — расширить API store
    • 
        


        

          Каждый dispatch в демо выше проходит через logEnhancer. Посмотрите в консоль —
          вы увидите action и state до/после каждого dispatch.
        


        
DevTools Extension — это enhancer!

        

          Самый известный enhancer — Redux DevTools Extension. Он оборачивает
          store, перехватывает все dispatch и записывает историю для time-travel.
        


        

const store = createStore(
  reducer,
  window.__REDUX_DEVTOOLS_EXTENSION__?.()
)
        


        

          Enhancers — мощный, но низкоуровневый инструмент. В большинстве случаев
          middleware (который сам является enhancer'ом через
          applyMiddleware) — более подходящий выбор для расширения dispatch.
        


        
Ключевые моменты

        
    
          
  • Enhancer — третий аргумент createStore
    • 
          
  • Оборачивает создание store, может переопределить dispatch/getState/subscribe
    • 
          
  • Сигнатура: (createStore) => (reducer, preloadedState) => store
    • 
          
  • Redux DevTools — самый известный enhancer
    • 
          
  • Для большинства задач лучше использовать middleware
    •
