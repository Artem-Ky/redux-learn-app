Интерактив — Один селектор на всех vs фабрика (свой на каждую строку)

5 задач. Каждая строка «подписана» на свой todo через селектор. Нажимайте toggle/edit и следите за счётчиком пересчётов у каждой строки:

Один общий селектор shared

selectTodoById(state, id) — один экземпляр createSelector на всех. Каждый вызов с другим id сбрасывает кэш (cache size = 1).

Всего пересчётов: 0
Фабрика — свой селектор на строку factory

makeSelectTodoById() — каждая строка создаёт свой экземпляр. Кэш не сбрасывается при вызове с другим id.

Всего пересчётов: 0
Теория — Фабрика селекторов (Selector Factory)

Проблема: один мемоизированный селектор на всех

createSelector кэширует только последний результат (cache size = 1). Если вы вызываете один и тот же селектор с разными аргументами — каждый вызов сбрасывает кэш предыдущего:

// Один экземпляр селектора на ВСЁ приложение const selectTodoById = createSelector( (state: RootState) => state.entities, (_: RootState, id: number) => id, (entities, id) => entities[id] // пересчитывается КАЖДЫЙ раз! ) // Строка 0: selectTodoById(state, 0) → cache MISS, вычисляем // Строка 1: selectTodoById(state, 1) → id изменился → cache MISS! // Строка 2: selectTodoById(state, 2) → id изменился → cache MISS! // ... мемоизация бесполезна — всегда пересчёт!
Если в списке 100 строк, каждая вызывает selectTodoById(state, id) с разным id — мемоизация не работает вообще. Каждый вызов вытесняет кэш предыдущего.

Решение: фабрика селекторов

Selector Factory — функция, которая создаёт новый экземпляр мемоизированного селектора при каждом вызове. Каждая строка списка получает свой собственный селектор со своим кэшем:

// ФАБРИКА: id запекается через замыкание // Каждый экземпляр зависит ТОЛЬКО от entities[todoId] const makeSelectTodoById = (todoId: number) => createSelector( (state: RootState) => state.entities[todoId], (entity) => entity ) // Каждая строка создаёт СВОЙ экземпляр: const selectTodo0 = makeSelectTodoById(0) // зависит от entities[0] const selectTodo1 = makeSelectTodoById(1) // зависит от entities[1] const selectTodo2 = makeSelectTodoById(2) // зависит от entities[2] // При Toggle #2: // selectTodo0(state) → entities[0] не изменился → cache HIT ✓ // selectTodo1(state) → entities[1] не изменился → cache HIT ✓ // selectTodo2(state) → entities[2] ИЗМЕНИЛСЯ → cache MISS, пересчёт

Как это работает в React-Redux

В React-Redux (который мы изучим позже) это выглядит так:

// В React компоненте строки списка: const TodoRow = ({ id }: { id: number }) => { // useMemo создаёт ОДИН раз при монтировании const selectTodo = useMemo(() => makeSelectTodoById(id), [id]) // Этот селектор зависит ТОЛЬКО от entities[id] const todo = useSelector(selectTodo) return <div>{todo.text}</div> }
Ключевой момент: фабрика работает вместе с нормализованным state. Когда state хранит entities: { [id]: entity }, каждый фабричный селектор зависит только от entities[todoId]. Изменение другого entity не инвалидирует кэш. С массивом todos[] это не работает — любой .map() создаёт новый массив для всех.
Паттерн именования: фабрику обычно называют с префиксом make: makeSelectTodoById, makeSelectUserPosts и т.д. Это сигнал другим разработчикам: «эта функция создаёт селектор, а не является селектором».

Когда нужна фабрика?

  • У вас список элементов, каждый элемент использует мемоизированный селектор с уникальным параметром (id)
  • Один и тот же селектор вызывается с разными аргументами в разных местах
  • Вы хотите, чтобы изменение элемента #5 не вызывало пересчёт селектора элемента #3

Когда фабрика НЕ нужна?

  • Селектор всегда вызывается с одними и теми же аргументами (или без аргументов)
  • Селектор используется только в одном месте
  • Вы не используете createSelector (обычные селекторы не кэшируют)
«Memoized selectors are only helpful when you actually derive additional values from the original data. If you are only looking up and returning an existing value, you can keep the selector as a plain function.»
— Redux Fundamentals, Part 7

Полный код фабрики из этого примера

import { createSelector } from 'reselect' interface Todo { id: number; text: string; completed: boolean } interface RootState { entities: Record<number, Todo> ids: number[] } // ❌ ОДИН на всех — кэш бьётся при смене id const selectTodoById = createSelector( (state: RootState) => state.entities, (_: RootState, id: number) => id, (entities, id) => entities[id] ) // ✅ ФАБРИКА — id через замыкание, зависимость только от entities[todoId] const makeSelectTodoById = (todoId: number) => createSelector( (state: RootState) => state.entities[todoId], (entity) => entity )
