Теория — Сложный payload в Action Creator

      

        
Несколько параметров → объект в payload

        

          Когда action creator принимает несколько аргументов,
          он комбинирует их в структурированный объект payload.
          Это позволяет передать всю необходимую информацию в одном экшене:
        


        

// Action creator с несколькими параметрами
function colorFilterChanged(color: string, changeType: string) {
  return {
    type: 'filters/colorFilterChanged',
    payload: {
      color,
      changeType
    }
  }
}

// Результат:
colorFilterChanged('red', 'added')
// → { type: 'filters/colorFilterChanged', payload: { color: 'red', changeType: 'added' } }
        


        

          «The type field should be a string that gives this action a descriptive name...
          An action object can have other fields with additional information about what happened.
          By convention, we put that information in a field called payload.»
          
— Redux Fundamentals, Part 2
        


        
Подготовка данных в Action Creator

        

          Action creator может не просто передавать аргументы «как есть», а
          подготавливать и трансформировать данные перед созданием экшена.
          Это централизует логику формирования payload:
        


        

// Action creator может трансформировать входные данные
function colorFilterChanged(color: string, changeType: string) {
  return {
    type: 'filters/colorFilterChanged',
    payload: {
      color: color.toLowerCase(),
      changeType
    }
  }
}
        


        
Инкапсуляция деталей формата

        

          Компонент, вызывающий colorFilterChanged('red', 'added'),
          не знает, как устроен объект экшена. Он знает только интерфейс
          (два аргумента). Если завтра payload изменится — менять нужно только creator.
        


        

          Каждый action creator — это контракт между UI и store.
          UI говорит «что хочет» через параметры, а creator формирует правильный экшен.
        


        
Reducer для обработки сложного payload

        

function filtersReducer(state, action) {
  switch (action.type) {
    case 'filters/colorFilterChanged': {
      const { color, changeType } = action.payload
      if (changeType === 'added') {
        return { ...state, selectedColors: [...state.selectedColors, color] }
      } else {
        return { ...state, selectedColors: state.selectedColors.filter(c => c !== color) }
      }
    }
    default:
      return state
  }
}
