Теория — Стандартный 3-action async паттерн

      

        
Три экшена для async-операции

        

          Это THE паттерн для асинхронных операций в классическом Redux.
          Каждая async-операция сопровождается тремя экшенами:
        

        
    
          
  • request (started) — запрос начат
    • 
          
  • success (succeeded) — данные получены
    • 
          
  • failure (failed) — произошла ошибка
    • 
        


        

// Три action types
const FETCH_STARTED   = 'users/fetchStarted'
const FETCH_SUCCEEDED = 'users/fetchSucceeded'
const FETCH_FAILED    = 'users/fetchFailed'
        


        
Reducer обрабатывает каждый

        

          Reducer — конечный автомат. Каждый case — переход между состояниями.
        


        

function usersReducer(state = initialState, action) {
  switch (action.type) {
    case 'users/fetchStarted':
      return { ...state, status: 'loading', error: null }
    case 'users/fetchSucceeded':
      return { ...state, status: 'succeeded', entities: action.payload }
    case 'users/fetchFailed':
      return { ...state, status: 'failed', error: action.payload }
    default:
      return state
  }
}
        


        
Thunk с try/catch

        

          Полный thunk action creator: dispatch started, try fetch, dispatch
          succeeded или catch → dispatch failed.
        


        

function fetchUsers() {
  return async (dispatch: any) => {
    dispatch({ type: 'users/fetchStarted' })
    try {
      const response = await fetch('/api/users')
      const users = await response.json()
      dispatch({ type: 'users/fetchSucceeded', payload: users })
    } catch (err) {
      dispatch({ type: 'users/fetchFailed', payload: err.message })
    }
  }
}
        


        
Именование: request / success / failure

        

          Redux не диктует конкретные имена, но паттерн устоялся:
          started/succeeded/failed или pending/fulfilled/rejected
          (как в Redux Toolkit createAsyncThunk).
        


        

          «For any async operation, you need to dispatch actions to track the
          loading state. A typical pattern is dispatching a 'pending' action
          before the request, then either a 'fulfilled' or 'rejected' action
          based on the request result.»
          
— Redux Essentials, Part 5: Async Logic and Data Fetching
        


        

          Этот 3-action паттерн настолько стандартный, что Redux Toolkit
          автоматизирует его через createAsyncThunk. Но важно
          понимать, как он работает изнутри.
        


        

          Всегда оборачивайте async-логику в try/catch. Без этого
          ошибка API не будет обработана, и приложение останется в состоянии
          loading навсегда.
        


        
Ключевые моменты

        
    
          
  • 3 экшена: started → succeeded | failed
    • 
          
  • Reducer обрабатывает каждый как переход state machine
    • 
          
  • Thunk: dispatch started → try/catch → dispatch succeeded/failed
    • 
          
  • Стандартное именование: started/succeeded/failed или pending/fulfilled/rejected
