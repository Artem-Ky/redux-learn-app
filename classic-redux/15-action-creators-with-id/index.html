Теория — Генерация ID в Action Creator

      

        
Почему ID генерируется в Action Creator, а не в Reducer?

        

          Reducer должен быть чистой функцией — он не может иметь
          побочных эффектов и должен возвращать предсказуемый результат при одинаковых входных данных.
          Генерация уникального ID — это побочный эффект, потому что каждый вызов
          даёт новое значение:
        


        

// ❌ НЕЛЬЗЯ — генерация ID в reducer нарушает чистоту
function notesReducer(state, action) {
  case 'notes/noteAdded':
    return [...state, {
      id: Math.random(),  // ← побочный эффект!
      text: action.payload
    }]
}

// ✅ ПРАВИЛЬНО — генерация ID в action creator
let nextId = 1
function noteAdded(text: string) {
  return {
    type: 'notes/noteAdded',
    payload: { id: nextId++, text }
  }
}
        


        

          «Reducers must always follow some specific rules: They should only calculate
          the new state value based on the state and action arguments.
          They must not do any asynchronous logic, calculate random values,
          or cause other "side effects".»
          
— Redux Fundamentals, Part 3
        


        
Паттерн nextId++

        

          Самый простой способ — использовать замыкание с инкрементируемым счётчиком.
          Каждый вызов action creator'а увеличивает счётчик:
        


        

let nextNoteId = 1

function noteAdded(text: string) {
  return {
    type: 'notes/noteAdded',
    payload: {
      id: nextNoteId++,  // 1, 2, 3, 4...
      text
    }
  }
}
        


        
Альтернативы: nanoid / uuid

        

          В реальных проектах часто используют библиотеки для генерации уникальных идентификаторов:
        


        

import { nanoid } from 'nanoid'

function noteAdded(text: string) {
  return {
    type: 'notes/noteAdded',
    payload: {
      id: nanoid(),  // 'V1StGXR8_Z5jdHi6B-myT'
      text
    }
  }
}
        


        

          Никогда не генерируйте случайные значения (ID, даты, Math.random()) внутри reducer'а.
          Это нарушает его чистоту и делает поведение непредсказуемым.
          Все «нечистые» операции выполняются до dispatch — в action creator'е.
        


        
Ключевые моменты

        
    
          
  • Reducer — чистая функция, никаких побочных эффектов
    • 
          
  • Генерация ID — побочный эффект → делаем в action creator
    • 
          
  • Простой подход: let nextId = 1; nextId++
    • 
          
  • Продвинутый подход: nanoid() или uuid()
    • 
          
  • ID приходит в reducer готовым через action.payload.id
