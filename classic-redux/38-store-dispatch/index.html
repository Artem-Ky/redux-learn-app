Теория — store.dispatch(action)

      

        
Единственный способ обновить state

        

          store.dispatch(action) — это единственный способ
          изменить state в Redux. Вызов dispatch передаёт action в reducer, который
          вычисляет новый state.
        


        

          «Dispatches an action. This is the only way to trigger a state change.
          The store's reducing function will be called with the current getState() result
          and the given action synchronously.»
          
— Redux API Reference: Store — dispatch(action)
        


        
dispatch вызывает reducer синхронно

        

          Когда вы вызываете dispatch(action), Redux синхронно
          вызывает reducer с текущим state и переданным action. Это значит, что после
          dispatch вызов getState() уже вернёт обновлённый state.
        


        

store.dispatch({ type: 'counter/incremented' })
// ↑ reducer уже отработал
console.log(store.getState()) // уже новый state
        


        
dispatch возвращает action

        

          Возвращаемое значение dispatch() — это тот же action объект,
          который вы передали. Это полезно для цепочки вызовов и middleware.
        


        

const returnedAction = store.dispatch({
  type: 'counter/addAmount',
  payload: 5
})

console.log(returnedAction)
// { type: 'counter/addAmount', payload: 5 }
        


        
Различные action types

        

          В реальном приложении store получает множество разных action types.
          Reducer обрабатывает каждый тип через switch/case:
        


        

store.dispatch({ type: 'counter/incremented' })
store.dispatch({ type: 'counter/decremented' })
store.dispatch({ type: 'counter/reset' })
store.dispatch({ type: 'counter/addAmount', payload: 10 })
        


        

          Нельзя dispatch из reducer! Reducer — чистая функция. Вызов
          store.dispatch() внутри reducer приведёт к ошибке или непредсказуемому
          поведению. Dispatch вызывается только из обработчиков событий, middleware или
          внешнего кода.
        


        

          Нажмите кнопки в демо и обратите внимание: после каждого dispatch в консоль
          выводится и action, и новый state. Также посмотрите на «Возвращаемое значение» —
          dispatch всегда возвращает тот же action, который получил.
