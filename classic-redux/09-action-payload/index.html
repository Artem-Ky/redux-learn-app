Теория — Поле payload в Action

      

        
Зачем нужен payload?

        

          Поле type говорит что произошло, а payload
          передаёт дополнительные данные, необходимые для обновления state. Без
          payload reducer не знал бы, какой именно текст добавить, какое число прибавить и т.д.
        


        

          «An action object can have other fields with additional information about what happened.
          By convention, we put that information in a field called payload.»
          
— Redux Fundamentals, Part 2
        


        
Примеры payload

        

// Payload — строка
{ type: 'todos/todoAdded', payload: 'Купить молоко' }

// Payload — число
{ type: 'counter/addAmount', payload: 5 }

// Payload — объект
{ type: 'users/userUpdated', payload: { id: 1, name: 'Анна' } }
        


        
Как payload попадает в reducer

        

          Reducer читает action.payload и использует эти данные для вычисления
          нового state:
        


        

function todosReducer(state = [], action) {
  switch (action.type) {
    case 'todos/todoAdded':
      return [...state, action.payload]
    default:
      return state
  }
}

store.dispatch({
  type: 'todos/todoAdded',
  payload: 'Купить молоко'
})
// state: ['Купить молоко']
        


        
Payload — это соглашение, а не требование

        

          Технически можно назвать поле как угодно (data, text,
          value), но payload — общепринятое соглашение,
          которое делает код предсказуемым. Это часть стандарта
          Flux Standard Action (FSA), который мы разберём в следующем уроке.
        


        

          Не все экшены нуждаются в payload. Например, counter/incremented достаточно
          самоописателен — reducer знает, что нужно прибавить 1. Payload нужен, когда данные
          динамические и приходят извне (ввод пользователя, ответ API и т.д.).
        


        
Ключевые моменты

        
    
          
  • type — что произошло, payload — данные события
    • 
          
  • payload — общепринятое имя для дополнительных данных
    • 
          
  • Payload может быть строкой, числом, объектом, массивом — чем угодно
    • 
          
  • Reducer использует action.payload для вычисления нового state
    • 
          
  • Не каждый экшен нуждается в payload
