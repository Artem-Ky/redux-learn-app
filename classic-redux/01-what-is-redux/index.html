Теория — Что такое Redux?

Определение

Redux — это библиотека для управления и обновления глобального состояния приложения. UI генерирует события, называемые «actions» (экшены), которые описывают «что произошло», а отдельная логика обновления — «reducers» (редьюсеры) — обновляет состояние в ответ.

«Redux is a pattern and library for managing and updating application state, using events called "actions". It serves as a centralized store for state that needs to be used across your entire application, with rules ensuring that the state can only be updated in a predictable fashion.»
— Redux Fundamentals, Part 1

Три принципа Redux

  1. Single Source of Truth (Единый источник истины) — глобальное состояние хранится в объекте внутри единственного store. Любая часть данных существует только в одном месте.
  2. State is Read-Only (Состояние только для чтения) — единственный способ изменить состояние — отправить action, объект, описывающий что произошло. UI не может случайно перезаписать данные.
  3. Changes via Pure Reducer Functions (Изменения через чистые функции) — чтобы указать, как дерево состояний обновляется в ответ на экшены, вы пишете чистые reducer-функции.

Когда использовать Redux?

Redux полезен, когда:
— У вас много данных, которые нужны в разных частях приложения
— Состояние часто обновляется
— Логика обновления может быть сложной
— Кодовая база средняя или большая, над ней работает много людей

Поток данных Redux

Диаграмма выше показывает, как данные проходят через Redux-приложение:

  1. Пользователь взаимодействует с UI (клик, ввод и т.д.)
  2. UI создаёт action — простой объект { type: 'событие' }
  3. dispatch — action отправляется в store
  4. Store вызывает reducer — чистую функцию (state, action) => newState
  5. Reducer возвращает новое состояние, store сохраняет его
  6. Store оповещает UI, UI обновляется

Код диаграммы (упрощённо)

// 1. Action — объект, описывающий событие const action = { type: 'counter/incremented' } // 2. Reducer — чистая функция function counterReducer(state = { value: 0 }, action) { switch (action.type) { case 'counter/incremented': return { ...state, value: state.value + 1 } default: return state } } // 3. Store — хранит состояние const store = Redux.createStore(counterReducer) // 4. Dispatch — отправка action store.dispatch(action) // 5. Новое состояние console.log(store.getState()) // { value: 1 }
