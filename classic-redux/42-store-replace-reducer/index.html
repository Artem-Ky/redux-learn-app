Теория — store.replaceReducer()

      

        
Что делает replaceReducer?

        

          Метод store.replaceReducer(nextReducer) заменяет текущий reducer store
          на новый. Это продвинутый API, который используется в специфических
          сценариях.
        


        

          «replaceReducer is an advanced API. You might need this if your app implements
          code splitting, and you want to load some of the reducers dynamically. You might
          also need this if you implement a hot reloading mechanism for Redux.»
          
— Redux API Reference: Store
        


        
Сигнатура

        

store.replaceReducer(nextReducer: Reducer): void
        


        
Основные use cases

        
    
          
  • Code splitting — подгрузка reducer'ов по мере необходимости
    • 
          
  • Hot Module Replacement — замена reducer без перезагрузки
    • 
          
  • Динамические фичи — добавление/удаление slice reducer'ов
    • 
        


        

// Code splitting: динамическая загрузка reducer
import('./features/newFeature/reducer').then(module => {
  const newRootReducer = combineReducers({
    ...existingReducers,
    newFeature: module.default
  })
  store.replaceReducer(newRootReducer)
})
        


        
replaceReducer диспатчит @@INIT

        

          После замены reducer Redux автоматически диспатчит внутренний action
          @@INIT. Это заставляет новый reducer инициализировать state —
          новые ключи получат default values из нового reducer.
        


        

store.replaceReducer(newReducer)
// Redux автоматически: store.dispatch({ type: '@@INIT' })
// Новый reducer обрабатывает существующий state
        


        

          Попробуйте переключить reducer в демо выше. Заметьте, что state (счётчик) сохраняется
          при замене — меняется только логика обработки действий. Кнопка Increment теперь
          увеличивает на 2 вместо 1.
        


        

          replaceReducer не сбрасывает state. Существующее состояние передаётся
          новому reducer как есть. Если новый reducer ожидает другую форму state, это может
          привести к ошибкам.
        


        
Ключевые моменты

        
    
          
  • replaceReducer — продвинутый API для замены reducer на лету
    • 
          
  • Используется для code splitting, HMR, динамических фич
    • 
          
  • После замены автоматически диспатчит @@INIT
    • 
          
  • State не сбрасывается при замене reducer
    • 
          
  • Новый reducer должен уметь работать с существующим state
