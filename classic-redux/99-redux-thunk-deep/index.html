Весь исходный код redux-thunk — это одна функция. Слева — настоящий код из npm-пакета. Справа — наша идентичная реализация.

Оригинал redux-thunk (npm)
function createThunkMiddleware(
  extraArgument
) {
  const middleware = ({ dispatch, getState }) =>
    (next) =>
      (action) => {
        if (typeof action === 'function') {
          return action(
            dispatch,
            getState,
            extraArgument
          )
        }
        return next(action)
      }

  middleware.withExtraArgument =
    createThunkMiddleware

  return middleware
}

export default createThunkMiddleware()

Теория — Разбор исходного кода redux-thunk

      

        
Весь redux-thunk — ~14 строк

        

          redux-thunk — один из самых маленьких и элегантных npm-пакетов.
          Вся логика — в одной функции createThunkMiddleware. Разберём
          его построчно.
        


        

function createThunkMiddleware(extraArgument) {
  // Возвращаем middleware с сигнатурой Redux:
  // ({ dispatch, getState }) => (next) => (action) => result

  const middleware = ({ dispatch, getState }) =>
    (next) =>
      (action) => {
        // Ключевая проверка: action — это функция?
        if (typeof action === 'function') {
          // Да → вызываем её, передавая dispatch, getState, extra
          return action(dispatch, getState, extraArgument)
        }
        // Нет → пропускаем дальше по цепочке middleware
        return next(action)
      }

  // Метод для создания middleware с «доп. аргументом»
  middleware.withExtraArgument = createThunkMiddleware

  return middleware
}

// Экспорт — middleware без extraArgument:
export default createThunkMiddleware()
        


        
Построчный разбор

        
    
          
  1. createThunkMiddleware(extraArgument) — фабрика, принимающая опциональный аргумент
    2. 
          
  2. ({ dispatch, getState }) — из store получаем dispatch и getState
    3. 
          
  3. (next) — следующий middleware в цепочке (или сам dispatch store)
    4. 
          
  4. (action) — action, который был dispatch-нут
    5. 
          
  5. typeof action === 'function' — если action — функция (thunk), вызываем её
    6. 
          
  6. action(dispatch, getState, extraArgument) — передаём в thunk всё необходимое
    7. 
          
  7. next(action) — обычный объект-action пропускаем дальше
    8. 
        


        
Паттерн замыкания (closure)

        

          Три стрелочные функции создают замыкание. Каждый уровень
          «запоминает» переменные верхнего уровня: dispatch, getState,
          next. Это стандартный паттерн Redux middleware.
        


        
withExtraArgument — Dependency Injection

        

          withExtraArgument позволяет передать дополнительный объект (API-клиент,
          конфиг, сервис) в каждый thunk. Thunk получит его третьим аргументом:
        


        

const api = { fetchUsers: () => fetch('/api/users') }

// Создаём thunk middleware с extra:
const thunkWithApi = thunk.withExtraArgument(api)

// Теперь thunk получает api третьим аргументом:
const loadUsers = () =>
  (dispatch, getState, api) => {
    api.fetchUsers()
      .then(users => dispatch({ type: 'users/loaded', payload: users }))
  }
        


        
Почему thunk такой маленький?

        

          Thunk делегирует всю логику пользовательским функциям. Middleware
          лишь проверяет typeof action === 'function' и вызывает её.
          Вся бизнес-логика, async/await, обработка ошибок — в ваших thunk-функциях.
        


        
RTK и thunk

        

          Redux Toolkit включает thunk по умолчанию в configureStore.
          Не нужно устанавливать отдельно — он уже встроен.
        


        

          «Redux Thunk middleware allows you to write action creators that return a function
          instead of an action. The thunk can be used to delay the dispatch of an action,
          or to dispatch only if a certain condition is met.»
          
— redux-thunk — GitHub README
        


        

          Понимание исходного кода redux-thunk — ключ к пониманию всей middleware-архитектуры
          Redux. Если вы разобрались в этих 14 строках — вы понимаете middleware.
        


        

          Для production-кода используйте npm-пакет redux-thunk, а не свою
          реализацию. Пакет поддерживается командой Redux и включает TypeScript-типы.
