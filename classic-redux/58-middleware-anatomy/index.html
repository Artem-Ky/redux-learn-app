Три уровня middleware (function-стиль)
function exampleMiddleware(storeAPI) {    // ← 1 раз при создании store
  return function wrapDispatch(next) {    // ← 1 раз при создании store
    return function handleAction(action) { // ← КАЖДЫЙ dispatch
      return next(action)
    }
  }
}
exampleMiddleware(storeAPI) — 1 раз
wrapDispatch(next) — 1 раз
handleAction(action) — каждый dispatch
Теория — Три вложенные функции middleware

Почему три уровня?

Каждый middleware в Redux — это три вложенные функции. Каждая принимает свой аргумент и имеет своё назначение. Это не случайность — каждый уровень вызывается в разное время.

«A Redux middleware is written as a series of three nested functions. Let's define those three functions along with the names and arguments they each receive. It helps to understand each layer and what it does.»
— Redux Fundamentals, Part 4

Уровень 1: Внешняя функция — storeAPI

Принимает объект с dispatch и getState. Вызывается один раз при создании store через applyMiddleware.

function exampleMiddleware(storeAPI) { // storeAPI = { dispatch, getState } // Вызывается ОДИН раз при настройке store // Здесь можно сохранить ссылку на storeAPI ... }

Уровень 2: Средняя функция — next

Принимает next — ссылку на dispatch следующего middleware в цепочке (или настоящий dispatch для последнего). Тоже вызывается один раз.

return function wrapDispatch(next) { // next = dispatch следующего middleware // Вызывается ОДИН раз при настройке цепочки ... }

Уровень 3: Внутренняя функция — action

Принимает action. Вызывается каждый раз при store.dispatch(action). Здесь живёт основная логика middleware.

return function handleAction(action) { // Вызывается КАЖДЫЙ раз при dispatch // Здесь логика: логирование, модификация, etc. return next(action) // передаём дальше }

Полная структура

function exampleMiddleware(storeAPI) { // ← 1 раз return function wrapDispatch(next) { // ← 1 раз return function handleAction(action) { // ← каждый dispatch // можно делать что угодно до next(action) const result = next(action) // можно делать что угодно после next(action) return result } } }
Внешние две функции — это «настройка». Они замыкают storeAPI и next в замыкании (closure). Внутренняя функция — это «работа», которая выполняется при каждом dispatch. Благодаря замыканию она имеет доступ к storeAPI и next без повторной передачи.
Не путайте storeAPI.dispatch и next! next(action) передаёт action следующему middleware. storeAPI.dispatch(action) отправляет action в самое начало цепочки — это может вызвать бесконечный цикл, если не осторожно!

Ключевые моменты

  • Middleware = три вложенные функции с разным временем вызова
  • Внешняя (storeAPI) — один раз, при создании store
  • Средняя (next) — один раз, при настройке цепочки
  • Внутренняя (action) — каждый dispatch
  • next ≠ storeAPI.dispatch
