Теория — Immer produce внутри Redux-редьюсеров

      

        
Зачем Immer в Redux?

        

          Redux требует иммутабельных обновлений. Классический подход — каскады
          spread-операторов. С Immer вы пишете «мутирующий» код, а библиотека
          создаёт иммутабельный результат автоматически.
        


        
Оборачиваем каждый case в produce

        

          Простейший способ — обернуть весь reducer в produce:
        


        

import { produce } from 'immer'

const reducer = produce((draft: TodoState, action: TodoAction) => {
  switch (action.type) {
    case 'todos/add':
      draft.todos.push({ id: draft.nextId, text: action.payload, completed: false })
      draft.nextId++
      break
  }
})
        


        
Сравнение объёма кода по операциям

        
    
          
  • Добавление (add): spread — 7 строк, Immer — 3 строки (push + break)
    • 
          
  • Переключение (toggle): spread — 5 строк (map + spread), Immer — 2 строки (find + присвоение)
    • 
          
  • Вложенное обновление: spread — 10+ строк каскада, Immer — 1 строка прямого присвоения
    • 
        


        

// spread: toggle — 5 строк
case 'todos/toggle':
  return {
    ...state,
    todos: state.todos.map(t =>
      t.id === action.payload ? { ...t, completed: !t.completed } : t

// Immer: toggle — 1 строка логики

  if (todo) todo.completed = !todo.completed

Это ИМЕННО то, что делает createSlice

        

          Redux Toolkit оборачивает каждый reducer в produce автоматически.
          Когда вы пишете «мутирующий» код внутри createSlice, это Immer под капотом.
          Понимание этого урока = понимание того, как работает RTK.
        


        

// Redux Toolkit — Immer скрыт внутри createSlice:
const todosSlice = createSlice({
  name: 'todos',
  initialState,
  reducers: {
    add(state, action) {
      state.todos.push(action.payload) // «мутация» = produce!
    },
    toggle(state, action) {
      const todo = state.todos.find(t => t.id === action.payload)
      if (todo) todo.completed = !todo.completed
    }
  }
})
        


        

          «Using Immer is like having a personal assistant. The assistant takes a letter
          (the current state) and gives you a copy (draft) to jot changes onto. Once you
          are done, the assistant will take your draft and produce the real immutable,
          final letter for you (the next state).»
          
— Immer Documentation
        


        

          Оба подхода дают идентичный результат — одинаковые state-объекты
          в DevTools. Разница только в количестве кода и читаемости. Имейте в виду: с Immer
          не забывайте break (или return) в каждом case!
        


        

          Не смешивайте мутации draft и return в одном case! Если вы используете Immer —
          только мутируйте draft. Если хотите вернуть новое значение — не мутируйте ничего.
