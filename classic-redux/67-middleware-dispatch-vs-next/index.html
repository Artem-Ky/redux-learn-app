Теория — next(action) vs storeAPI.dispatch(action)

      

        
Два способа передать action

        

          Внутри middleware доступны две функции для передачи action дальше:
          next(action) и storeAPI.dispatch(action).
          Они делают принципиально разные вещи.
        


        

          «If you call storeAPI.dispatch(action), it will be sent to the first middleware
          in the chain again, restarting the pipeline. If you call next(action), it goes
          to the next middleware in line.»
          
— Redux Fundamentals, Part 4: Store — Writing Custom Middleware
        


        
next(action) — следующий middleware

        

          next(action) передаёт action следующему middleware
          в цепочке (или reducer, если это последний middleware). Это стандартный путь —
          action проходит по pipeline один раз.
        


        

const myMiddleware = (storeAPI: any) => (next: any) => (action: any) => {
  console.log('passing to next middleware...')
  return next(action) // → следующий middleware
}
        


        
storeAPI.dispatch(action) — перезапуск pipeline

        

          storeAPI.dispatch(action) отправляет action с самого начала
          pipeline. Все middleware выполнятся заново. Это полезно, когда middleware хочет
          отправить новый action.
        


        

const myMiddleware = (storeAPI: any) => (next: any) => (action: any) => {
  if (action.type === 'special') {
    // Отправляем НОВЫЙ action с самого начала pipeline
    storeAPI.dispatch({ type: 'converted' })
    return
  }
  return next(action)
}
        


        
Когда использовать dispatch?

        
    
          
  • Отправка нового action — middleware решает, что нужно другое действие
    • 
          
  • Повторная обработка — action нужно пропустить через все middleware заново
    • 
          
  • Thunk-паттерн — thunk-функция вызывает dispatch для отправки action'ов
    • 
        


        
Когда использовать next?

        
    
          
  • Пропуск дальше — стандартный случай, action идёт дальше по цепочке
    • 
          
  • Логирование — middleware только наблюдает, не вмешиваясь
    • 
          
  • Дополнение action — middleware модифицирует и передаёт дальше
    • 
        


        

          Опасность бесконечного цикла! Если middleware вызывает
          storeAPI.dispatch(action) с тем же самым action без условия выхода,
          pipeline будет перезапускаться бесконечно. Всегда проверяйте условие перед dispatch.
        


        

          Простое правило: используйте next(action) для «пропускания» action
          дальше, а storeAPI.dispatch(action) — для «отправки нового» action
          с начала pipeline.
        


        
Ключевые моменты

        
    
          
  • next(action) — передаёт action следующему middleware в цепочке
    • 
          
  • storeAPI.dispatch(action) — перезапускает pipeline с начала
    • 
          
  • dispatch используется для отправки новых action'ов
    • 
          
  • next — стандартный путь для «пропускания» action дальше
    • 
          
  • dispatch того же action без условия = бесконечный цикл
