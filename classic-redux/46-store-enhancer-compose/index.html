Теория — compose() для объединения enhancer'ов

      

        
Зачем compose?

        

          Часто приложению нужно несколько enhancer'ов одновременно: один добавляет
          логирование, другой — метрики, третий — DevTools. Функция compose
          из Redux позволяет объединить их в один.
        


        

          «Since enhancers are the most powerful extension mechanism for a Redux store, you
          will almost certainly need to use at least one enhancer when setting up a store.
          Redux includes a compose function that can be used to merge multiple
          enhancers into a single enhancer.»
          
— Redux Fundamentals, Part 4
        


        
Что делает compose?

        

          compose(f, g, h) возвращает (...args) => f(g(h(...args))).
          То есть функции применяются справа налево. Это стандартная
          математическая композиция функций.
        


        

import { compose } from 'redux'

const composedEnhancer = compose(
  logEnhancer,        // применяется вторым (оборачивает результат)
  timestampEnhancer   // применяется первым (ближе к store)
)
        


        
Пример: logEnhancer

        

          Этот enhancer оборачивает dispatch, чтобы логировать каждое
          отправленное действие и новый state:
        


        

const logEnhancer = (createStore) => (reducer, preloadedState, enhancer) => {
  const store = createStore(reducer, preloadedState, enhancer)
  const originalDispatch = store.dispatch

  store.dispatch = (action) => {
    console.log('📋 dispatch:', action)
    const result = originalDispatch(action)
    console.log('📋 new state:', store.getState())
    return result
  }

  return store
}
        


        
Пример: timestampEnhancer

        

          Этот enhancer оборачивает getState, добавляя lastUpdated
          к результату:
        


        

const timestampEnhancer = (createStore) => (reducer, preloadedState, enhancer) => {
  const store = createStore(reducer, preloadedState, enhancer)
  const originalGetState = store.getState

  store.getState = () => ({
    ...originalGetState(),
    lastUpdated: new Date().toLocaleTimeString()
  })

  return store
}
        


        
Объединение через compose


        

const composedEnhancer = compose(logEnhancer, timestampEnhancer)

const store = createStore(reducer, undefined, composedEnhancer)
        


        

          Порядок в compose важен! Enhancer'ы применяются справа
          налево: сначала timestampEnhancer оборачивает store,
          затем logEnhancer оборачивает результат. Это значит, что
          logEnhancer «видит» store, уже модифицированный
          timestampEnhancer.
        


        
createStore с третьим аргументом

        

          Enhancer передаётся третьим аргументом в createStore.
          Второй аргумент — начальное состояние (preloadedState). Если начальное
          состояние не нужно, передайте undefined:
        


        

createStore(reducer, undefined, composedEnhancer)
// reducer — первый аргумент
// undefined — preloadedState (нет)
// composedEnhancer — enhancer (третий)
        


        

          Не путайте порядок аргументов! Если передать enhancer вторым аргументом,
          Redux попытается использовать его как начальное состояние, что приведёт
          к ошибке.
        


        
Ключевые моменты

        
    
          
  • compose объединяет несколько enhancer'ов в один
    • 
          
  • Порядок выполнения — справа налево
    • 
          
  • Каждый enhancer оборачивает store, добавляя своё поведение
    • 
          
  • Enhancer передаётся третьим аргументом в createStore
    • 
          
  • compose полезен и вне Redux — это общая утилита композиции функций
