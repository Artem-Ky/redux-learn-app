Теория — Как устроен Redux Store внутри

      

        
Store — это удивительно просто

        

          Redux Store можно реализовать примерно в 25 строках кода. Внутри
          — всего лишь замыкание с переменной state, массивом слушателей и тремя методами.
        


        

          «The Redux store itself is actually quite small — the store code is less than
          200 lines, and you could rewrite it yourself if you wanted to. It's really just
          a few functions in a closure.»
          
— Redux Fundamentals, Part 4: Store
        


        
Полная реализация miniCreateStore

        

function miniCreateStore(reducer, preloadedState?) {
  let state = preloadedState
  let listeners: Function[] = []

  // Инициализация: вызываем reducer с undefined
  // чтобы каждый reducer вернул свой default state
  if (state === undefined) {
    state = reducer(undefined, { type: '@@INIT' })
  }

  function getState() {
    return state
  }

  function dispatch(action) {
    state = reducer(state, action)
    listeners.forEach(listener => listener())
  }

  function subscribe(listener) {
    listeners.push(listener)
    return () => {
      listeners = listeners.filter(l => l !== listener)
    }
  }

  return { getState, dispatch, subscribe }
}
        


        
Разбор по частям


        
1. Замыкание (closure)

        

          state и listeners — приватные переменные внутри замыкания.
          Доступ к ним возможен только через возвращаемые методы.
        


        
2. getState()

        

          Просто возвращает текущее значение state. Никакой копии — та же ссылка.
        


        
3. dispatch(action)

        

          Вызывает reducer с текущим state и action, сохраняет результат, затем уведомляет
          всех подписчиков. Это единственный способ изменить state.
        


        
4. subscribe(listener)

        

          Добавляет слушателя в массив. Возвращает функцию для отписки (unsubscribe).
        


        
5. @@INIT

        

          При создании store вызывается reducer(undefined, { type: '@@INIT' }).
          Поскольку state === undefined, каждый reducer вернёт свой default state.
        


        

          Демо выше работает полностью на нашей реализации — без импорта Redux.
          Счётчик, dispatch, subscribe — всё работает на miniCreateStore.
