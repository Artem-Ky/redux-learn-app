Теория — Именование Action Types

      

        
Соглашение «domain/eventName»

        

          В Redux принято именовать action type в формате "domain/eventName",
          где domain — это функциональная область (feature, slice), а
          eventName — описание того, что произошло.
        


        

          «The type field should be a string that gives this action a descriptive name,
          like "todos/todoAdded". We usually write that type string like "domain/eventName",
          where the first part is the feature or category that this action belongs to,
          and the second part is the specific thing that happened.»
          
— Redux Fundamentals, Part 2
        


        
Почему именно слэш «/»?

        

          Слэш визуально разделяет домен и событие, делая action type читаемым. Он работает
          как namespace — при большом количестве экшенов легко понять, к какой части
          приложения относится каждый.
        


        

// Современный стиль — domain/eventName
'todos/todoAdded'
'filters/statusChanged'
'users/userLoggedIn'

// Старый стиль — SCREAMING_SNAKE_CASE
'ADD_TODO'       // ← непонятно к какому домену
'SET_FILTER'    // ← звучит как команда, а не событие
'LOGIN_USER'    // ← нет группировки
        


        
Старый стиль: SCREAMING_SNAKE_CASE

        

          В ранних версиях Redux использовался стиль SCREAMING_SNAKE_CASE:
          ADD_TODO, INCREMENT_COUNTER и т.д. Это работает, но имеет
          недостатки:
        

        
    
          
  • Не видна принадлежность к конкретной области приложения
    • 
          
  • Формулировка «команда» (ADD_TODO) вместо «событие» (todoAdded)
    • 
          
  • Легко получить конфликт имён в большом приложении
    • 
        


        
Правила хорошего именования

        
    
          
  • Описывайте событие, а не команду: todoAdded, не addTodo
    • 
          
  • Используйте camelCase для eventName: statusChanged
    • 
          
  • Домен = feature/slice: counter, todos, auth
    • 
          
  • Будьте конкретны: counter/incrementByAmount лучше, чем counter/update
    • 
        


        

          В Redux Toolkit action types генерируются автоматически из имени slice и имени reducer-функции.
          Например, slice counter с reducer incremented создаёт
          тип "counter/incremented".
