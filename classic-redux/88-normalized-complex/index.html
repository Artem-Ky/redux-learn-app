Теория — Комплексная нормализация

      

        
Полностью нормализованное приложение

        

          В реальных приложениях данные имеют сложные связи: пользователь создаёт посты,
          к постам оставляют комментарии. Каждый тип сущности хранится
          в своём слайсе с форматом { ids, entities }.
        


        

// Три сущности — три слайса
const rootReducer = combineReducers({
  users: usersReducer,     // { ids, entities, nextId }
  posts: postsReducer,     // { ids, entities, nextId }
  comments: commentsReducer // { ids, entities, nextId }
})
        


        
Реляционные селекторы

        

          Связи между сущностями строятся через селекторы. Каждый «уровень»
          фильтрует данные по ID родительской сущности:
        


        

// Посты пользователя
const selectPostsByUserId = (state, userId) =>
  selectAllPosts(state).filter(p => p.authorId === userId)

// Комментарии к посту
const selectCommentsByPostId = (state, postId) =>
  selectAllComments(state).filter(c => c.postId === postId)

// Автор комментария
const selectCommentAuthor = (state, comment) =>
  selectUserById(state, comment.authorId)
        


        

          «In a Redux store, we typically organize data into 'slices' based on
          what it's used for... Data that is used together should generally be
          stored close together.»
          
— Redux Essentials, Part 6: Performance & Normalizing Data
        


        
Когда нормализация оправдана

        

          Нормализация добавляет сложность. Она оправдывает себя когда:
        

        
    
          
  • Одна сущность отображается в нескольких местах UI
    • 
          
  • Между сущностями есть связи «один ко многим»
    • 
          
  • Данные часто обновляются — обновление одной записи O(1)
    • 
          
  • Большой объём данных — прямой доступ по ID вместо поиска
    • 
        


        
Подготовка к createEntityAdapter

        

          Паттерн { ids, entities } настолько распространён,
          что Redux Toolkit предоставляет createEntityAdapter —
          утилиту, которая автоматически генерирует reducers и selectors
          для нормализованных данных.
        


        

          Комплексная нормализация — фундамент для масштабируемых приложений.
          Redux Toolkit's createEntityAdapter автоматизирует всё, что мы
          делали вручную: создание { ids, entities }, CRUD-операции и селекторы.
        


        

          При полной нормализации важно следить за целостностью ссылок. При удалении
          пользователя нужно решить: удалить его посты или назначить «автор неизвестен».
          Каскадное удаление в Redux нужно реализовывать вручную.
        


        
Ключевые моменты

        
    
          
  • Каждая сущность — отдельный слайс через combineReducers
    • 
          
  • Связи выражаются через ID: post.authorId, comment.postId
    • 
          
  • Реляционные селекторы «join» данные из разных слайсов
    • 
          
  • createEntityAdapter из RTK автоматизирует этот паттерн
    • 
          
  • Нормализация оправдана при сложных связях и частых обновлениях
