    Теория — createSelector с несколькими input selectors

    Несколько input selectors

    createSelector принимает любое количество input selectors. Результат каждого передаётся как аргумент в output selector:

    const selectFilteredTodos = createSelector( selectTodos, // input 1 → todos selectFilters, // input 2 → filters (todos, filters) => { // output получает оба результата let result = todos if (filters.status !== 'all') { result = result.filter(...) } if (filters.color !== 'all') { result = result.filter(...) } return result } )
    Redux Fundamentals, Part 7: Standard Redux Patterns: «createSelector can accept multiple input selectors, which can be provided as separate arguments. The result function receives the extracted values as corresponding arguments.»
    redux.js.org — Memoizing Selectors with createSelector

    Когда пересчитывается output?

    Output selector вызывается только если хотя бы один input вернул новое значение (по ===). Если все inputs те же — возвращается кэш.

    // input 1: selectTodos → те же todos (ссылка не изменилась) // input 2: selectFilters → те же filters (ссылка не изменилась) // → output НЕ вызывается, возврат кэша // input 1: selectTodos → те же todos // input 2: selectFilters → НОВЫЙ объект filters // → output вызывается (input 2 изменился)

    Пример: фильтрация по статусу + цвету

    interface Filters { status: 'all' | 'active' | 'completed' color: 'all' | 'red' | 'blue' | 'green' } const selectTodos = (state: RootState) => state.todos const selectFilters = (state: RootState) => state.filters const selectFilteredTodos = createSelector( selectTodos, selectFilters, (todos, filters) => { let result = todos if (filters.status === 'active') result = result.filter(t => !t.completed) else if (filters.status === 'completed') result = result.filter(t => t.completed) if (filters.color !== 'all') result = result.filter(t => t.color === filters.color) return result } )
    Порядок аргументов в output selector соответствует порядку input selectors. Первый input → первый аргумент, второй → второй, и т.д.
    Если один из input selectors сам возвращает новый объект/массив при каждом вызове (например, через .filter()), мемоизация «сломается» — output будет пересчитываться каждый раз. Решение — мемоизировать и input selectors тоже.

    Ключевые моменты

    • createSelector(input1, input2, ..., outputFn) — любое количество inputs
    • Output получает результаты всех inputs как аргументы
    • Пересчёт только при изменении хотя бы одного input
    • Порядок аргументов output соответствует порядку inputs
    • Каждый input тоже можно мемоизировать для надёжности
