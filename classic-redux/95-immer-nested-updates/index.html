Интерактив — 4 уровня вложенности: spread vs Immer

Структура: company → departments[] → teams[] → members[]. Задача: обновить зарплату конкретного сотрудника. Сравните два подхода.

Без Immer (8+ строк spread)
const newState = {

          )
        }
      )
    }
  )
}

Теория — Глубоко вложенные обновления: главная боль Redux

      

        
Проблема глубокой вложенности

        

          Самая большая боль классического Redux — каскады spread-операторов
          при обновлении глубоко вложенных данных. Каждый уровень вложенности добавляет
          ещё один spread, делая код нечитаемым и подверженным ошибкам.
        


        

// 4 уровня вложенности: company.departments[].teams[].members[].salary
// Без Immer — каскад map + spread на КАЖДОМ уровне:

return {
  ...state,
  departments: state.departments.map((dep, di) =>
    di !== deptIdx ? dep : {
      ...dep,
      teams: dep.teams.map((team, ti) =>
        ti !== teamIdx ? team : {
          ...team,
          members: team.members.map((m, mi) =>
            mi !== memberIdx ? m : { ...m, salary: newSalary }
          )
        }
      )
    }
  )
}
        


        
Immer делает это тривиальным

        

          С Immer весь каскад выше заменяется одной строкой.
          Вы просто «мутируете» draft напрямую:
        


        

// С Immer — прямое присвоение через proxy:
const next = produce(state, draft => {
  draft.departments[0].teams[1].members[2].salary = 100000
})
        


        
Массивы — push, splice, sort

        

          На draft-массивах работают все мутирующие методы:
        


        

produce(state, draft => {
  // Добавить сотрудника в команду:
  draft.departments[0].teams[0].members.push(newMember)

  // Удалить второй отдел:
  draft.departments.splice(1, 1)

  // Отсортировать по зарплате:
  draft.departments[0].teams[0].members.sort(
    (a, b) => b.salary - a.salary
  )
})
        


        
Proxy-based copy-on-write

        

          Immer создаёт Proxy на каждом уровне вложенности. При первом изменении
          создаётся копия только того узла и всех его предков (copy-on-write).
          Неизменённые ветви переиспользуются (structural sharing).
        


        

const next = produce(state, draft => {
  draft.departments[0].teams[1].members[0].salary = 100000
})

// Что изменилось:
next !== state                                    // true (новый корень)
next.departments[0] !== state.departments[0]        // true (путь к изменению)
next.departments[0].teams[1] !== ...              // true

// Что НЕ изменилось (structural sharing):
next.departments[0].teams[0] === state.departments[0].teams[0]  // TRUE!
        


        

          «The beauty of Immer is that you can use all the natural JavaScript APIs
          and language features you are used to when working with plain objects, arrays
          and other data structures to define your state updates.»
          
— Immer Documentation
        


        

          Глубоко вложенные обновления — killer feature Immer.
          Это главная причина, почему Redux Toolkit использует Immer в каждом
          createSlice reducer. Больше никаких каскадов spread!
        


        

          Immer не делает глубокое клонирование! Он использует Proxy и copy-on-write.
          Это важно для производительности — клонируются только изменённые ветви.
