    Теория — Селекторы с параметрами
    
      
    
        
    Зачем параметры?
    
        
    
          Иногда нужно извлечь данные, зависящие от внешнего параметра:
          конкретный ID, статус фильтра, строка поиска. Для этого селектор принимает
          дополнительные аргументы помимо state.
        
    

        
    Паттерн: (state, param) => result
    

        
    
const selectTodoById = (state: AppState, id: number): Todo | undefined =>
  state.todos.find(todo => todo.id === id)

// Использование:
const todo = selectTodoById(store.getState(), 3)
        
    

        
    find для одного элемента
    
        
    
          Array.find() возвращает первый элемент, удовлетворяющий условию,
          или undefined если ничего не найдено:
        
    

        
    
const selectTodoById = (state: AppState, id: number): Todo | undefined =>
  state.todos.find(todo => todo.id === id)

// find вернёт undefined, если ID не найден
const result = selectTodoById(state, 999) // undefined
        
    

        
    filter для множества элементов
    
        
    
          Array.filter() возвращает новый массив со всеми элементами,
          прошедшими проверку:
        
    

        
    
const selectTodosByStatus = (state: AppState, completed: boolean): Todo[] =>
  state.todos.filter(todo => todo.completed === completed)

// Все выполненные:
selectTodosByStatus(state, true)

// Все активные:
selectTodosByStatus(state, false)
        
    

        
    Когда параметризировать селектор?
    
        
      
          
    • Поиск по ID — selectUserById(state, userId)
      • 
          
    • Фильтрация по значению — selectOrdersByStatus(state, 'pending')
      • 
          
    • Поиск по тексту — selectTodosBySearch(state, query)
      • 
          
    • Пагинация — selectPageItems(state, page, perPage)
      • 
        
    

        
    
// Ещё примеры параметризированных селекторов:

const selectUserById = (state: AppState, userId: number) =>
  state.users.find(u => u.id === userId)

const selectProductsByCategory = (state: AppState, category: string) =>
  state.products.filter(p => p.category === category)

const selectTodosBySearch = (state: AppState, query: string) =>
  state.todos.filter(t =>
    t.text.toLowerCase().includes(query.toLowerCase())
  )
        
    

        
    
          Параметризированный селектор — это по-прежнему обычная функция. Первый
          аргумент всегда state, дополнительные — параметры для фильтрации
          или поиска. Это соглашение Redux.
        
    

        
    
          Помните, что find может вернуть undefined. Всегда
          обрабатывайте случай «не найдено» в коде, который вызывает такой селектор.
          В TypeScript используйте тип Todo | undefined.
        
    

        
    Ключевые моменты
    
        
      
          
    • Параметризированный селектор: (state, param) => result
      • 
          
    • find — для поиска одного элемента (возвращает T | undefined)
      • 
          
    • filter — для выборки нескольких элементов (возвращает T[])
      • 
          
    • Первый аргумент — всегда state, остальные — параметры
      • 
          
    • Используйте для поиска по ID, фильтрации по статусу, поиска по тексту
