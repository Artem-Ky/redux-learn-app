Теория — Структура нормализованного state

      

        
entities — объект по ключу ID

        

          entities — это объект, где ключи — это ID сущностей, а значения —
          сами объекты. Это даёт O(1) доступ к любому элементу по ID,
          без необходимости перебирать массив.
        


        

const entities: Record<number, Item> = {
  1: { id: 1, name: 'Redux' },
  2: { id: 2, name: 'React' },
  3: { id: 3, name: 'TypeScript' }
}

// O(1) — мгновенный доступ
const item = entities[2]  // → { id: 2, name: 'React' }
        


        
ids — массив для порядка

        

          Объект не гарантирует порядок ключей. Массив ids хранит
          порядок элементов. Для итерации мы проходим по ids и достаём
          элементы из entities.
        


        

const ids: number[] = [1, 2, 3]

// O(n) итерация в правильном порядке
const orderedItems = ids.map(id => entities[id])
        


        
Паттерн { entities, ids }

        

          Вместе entities и ids формируют стандартный
          паттерн нормализованного состояния:
        


        

interface NormalizedState<T> {
  entities: Record<number, T>
  ids: number[]
}

// Пример
const state: NormalizedState<Item> = {
  entities: {
    1: { id: 1, name: 'Redux' },
    2: { id: 2, name: 'React' }
  },
  ids: [1, 2]
}
        


        
O(1) поиск vs O(n) перебор

        

          В массиве, чтобы найти элемент с id === 42, нужно перебрать
          все элементы — O(n). В нормализованном state —
          entities[42] — O(1), мгновенно.
        


        

          «By defining a normalized state structure, we can keep the same data in multiple places
          without having to duplicate it, and we can efficiently look up items by their IDs.»
          
— Redux Docs — Normalizing State Shape
        


        

          Паттерн { entities: Record<id, Entity>, ids: id[] } — это
          стандарт Redux для нормализованных данных. Redux Toolkit даже имеет встроенный
          createEntityAdapter, который генерирует этот формат автоматически.
        


        

          Не забывайте синхронизировать entities и ids!
          При добавлении элемента нужно записать его и в entities, и добавить ID в ids.
          При удалении — удалить из обоих.
