Теория — Иммутабельность

Что такое иммутабельность?

«Mutable» означает «изменяемый». Если что-то «immutable», оно не может быть изменено. В JavaScript объекты и массивы по умолчанию мутабельны — вы можете менять их содержимое:

const obj = { a: 1, b: 2 } obj.b = 3 // Мутация! Тот же объект в памяти, но содержимое изменилось const arr = ['a', 'b'] arr.push('c') // Мутация! Изменили оригинальный массив
«In order to update values immutably, your code must make copies of existing objects/arrays, and then modify the copies. We can do this by hand using JavaScript's array / object spread operators.»
— Redux Fundamentals, Part 2

Иммутабельное обновление

Чтобы обновить данные иммутабельно, нужно создать копию существующего объекта/массива и модифицировать копию:

const obj = { a: 1, b: 2 } // Spread-оператор создаёт НОВЫЙ объект const copy = { ...obj, b: 3 } obj === copy // false — это РАЗНЫЕ объекты в памяти obj.b // 2 — оригинал НЕ изменился! copy.b // 3 — только копия содержит новое значение

Почему Redux требует иммутабельности?

  • Предсказуемость — если state не мутируется, легко отследить каждое изменение
  • Time-travel debugging — DevTools может сохранять снимки state и «путешествовать во времени»
  • React-Redux оптимизации — React-Redux использует === для сравнения; если ссылка та же — компонент НЕ перерисовывается
  • Тестируемость — чистые функции без мутаций легко тестировать
