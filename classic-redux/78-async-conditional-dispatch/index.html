Теория — getState() для условного dispatch

      

        
Зачем getState() в thunk

        

          Thunk получает getState вторым аргументом. Это позволяет
          прочитать текущий state перед выполнением логики и принять решение —
          нужно ли делать запрос или нет.
        


        

          «The thunk can be used to delay the dispatch of an action, or to dispatch
          only if a certain condition is met. The inner function receives the store
          methods dispatch and getState as parameters.»
          
— redux-thunk — Motivation
        


        
Guard-паттерн: предотвращение дублирующих запросов


        

const fetchDataIfNeeded = () => {
  return async (dispatch: any, getState: any): Promise<void> => {
    const { status } = getState()

    if (status !== 'idle') {
      console.log('Data already loaded, skipping fetch')
      return // ← ранний выход
    }

    dispatch({ type: 'fetch/pending' })
    const data = await fetchFromApi()
    dispatch({ type: 'fetch/fulfilled', payload: data })
  }
}
        


        
Типичные проверки

        
    
          
  • status !== 'idle' — данные уже загружены или загружаются
    • 
          
  • status === 'loading' — запрос уже в процессе
    • 
          
  • data.length > 0 — данные уже есть в кэше
    • 
          
  • Date.now() - lastFetchTime < 60000 — кэш ещё не устарел
    • 
        


        

// Проверка нескольких условий
const fetchUsersIfNeeded = () => (dispatch: any, getState: any) => {
  const state = getState()

  if (state.status === 'loading') return // уже грузим
  if (state.data.length > 0) return  // данные есть

  dispatch(fetchUsers())
}
        


        

          Guard-паттерн — простой и эффективный способ предотвратить лишние API-запросы.
          Вместо отключения кнопки в UI, логика находится в thunk и работает
          независимо от того, откуда вызывается dispatch.
        


        

          getState() возвращает state на момент вызова. Если между вызовами
          getState() и dispatch() проходит время (например, await),
          state мог измениться. Проверяйте state непосредственно перед принятием решения.
        


        
Ключевые моменты

        
    
          
  • getState() — второй аргумент thunk-функции
    • 
          
  • Используйте для проверки текущего состояния перед dispatch
    • 
          
  • Guard-паттерн: ранний return если данные уже загружены
    • 
          
  • Предотвращает дублирующие запросы на уровне логики, а не UI
