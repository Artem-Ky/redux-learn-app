    Теория — Что такое Selector?
    
      
    
        
    Определение
    
        
    
          Selector — это функция, которая принимает state и возвращает
          нужную часть данных. Это простая чистая функция:
          state => state.someField.
        
    

        
    
          «A "selector function" is any function that accepts the Redux store state
          (or part of the state) as an argument, and returns data that is based on
          that state. Selectors don't have to be written using a special library,
          and it doesn't matter whether you write them as arrow functions or as
          the function keyword.»
          
    — Redux Fundamentals, Part 2
        
    

        
    Простейший селектор
    
        
    
          Селектор — это просто функция. Никакой магии:
        
    

        
    
const selectTodos = (state: AppState): Todo[] => state.todos

// Использование:
const todos = selectTodos(store.getState())
        
    

        
    Селектор с вычислением
    
        
    
          Селектор может не просто извлекать данные, но и вычислять производные:
        
    

        
    
const selectTodoCount = (state: AppState): number =>
  state.todos.length
        
    

        
    Зачем нужны селекторы?
    
        
      
          
    • 
            Инкапсуляция — компоненты не знают структуру state.
            Если структура изменится, нужно обновить только селектор.
          
      • 
          
    • 
            Переиспользование — один селектор можно использовать
            в разных местах приложения.
          
      • 
          
    • 
            Единый источник доступа — вся логика чтения данных
            в одном месте.
          
      • 
          
    • 
            Тестируемость — чистые функции легко тестировать.
          
      • 
        
    

        
    
// Без селектора — жёсткая привязка к структуре state:
const todos = store.getState().todos

// С селектором — абстракция:
const todos = selectTodos(store.getState())

// Если state.todos переименуется в state.items,
// меняем только selectTodos — остальной код не затронут
        
    

        
    
          Соглашение по именованию: селекторы начинаются с select,
          например selectTodos, selectTodoCount,
          selectUserName. Это помогает легко находить их в коде.
        
    

        
    Селекторы и combineReducers
    
        
    
          При использовании combineReducers state имеет вложенную структуру.
          Селекторы «знают» эту структуру и предоставляют удобный доступ:
        
    

        
    
// State после combineReducers:
// { todos: [...], filters: { ... } }

const selectTodos = (state: AppState) => state.todos
const selectFilters = (state: AppState) => state.filters
const selectStatusFilter = (state: AppState) => state.filters.status
        
    

        
    
          Селектор всегда получает весь state (корневой объект) как аргумент.
          Не передавайте в селектор уже извлечённую часть state — это нарушает паттерн
          и делает селектор бесполезным.
        
    

        
    Ключевые моменты
    
        
      
          
    • Selector — функция (state) => данные
      • 
          
    • Инкапсулирует знание о структуре state
      • 
          
    • Может извлекать данные или вычислять производные
      • 
          
    • Именование: selectSomething
      • 
          
    • Чистая функция — легко тестировать и переиспользовать
