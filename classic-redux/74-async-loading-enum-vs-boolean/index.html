Теория — Почему enum, а не boolean?

      

        
Boolean ограничивает до 2 состояний

        

          isLoading: boolean может быть только true или
          false. Но реальные приложения имеют минимум 4 состояния:
        

        
    
          
  • idle — данные ещё не запрашивались
    • 
          
  • loading — запрос в процессе
    • 
          
  • succeeded — данные получены
    • 
          
  • failed — произошла ошибка
    • 
        


        

// ❌ Boolean: невозможно отличить "ещё не загружали" от "загрузка завершена"
interface BooleanState {
  isLoading: boolean    // false... но загружено? или ещё нет?
  data: Item[]
  error: string | null  // нужен отдельный флаг
}

// ✔ Enum: каждое состояние чётко определено
interface EnumState {
  status: 'idle' | 'loading' | 'succeeded' | 'failed'
  data: Item[]
  error: string | null
}
        


        
Проблема «невозможных состояний»

        

          С boolean можно попасть в невозможное состояние:
          { isLoading: true, error: "Network error" } — загрузка
          и ошибка одновременно? Enum делает такое невозможным:
          status может быть только одним значением.
        


        
State Machine (конечный автомат)

        

          Enum-подход — это по сути конечный автомат.
          Каждое состояние имеет определённые допустимые переходы:
        


        

// Допустимые переходы:
// idle     → loading
// loading  → succeeded | failed
// succeeded → loading (перезагрузка)
// failed   → loading (повторная попытка)
        


        

          «Treat Reducers as State Machines. Many reducers don't model their
          logic as state machines. Treat the reducer as a state machine, where
          the combination of both the current state and the dispatched action
          determines whether a new state value is actually computed.»
          
— Redux Style Guide — Treat Reducers as State Machines
        


        

          Enum-подход масштабируется. Нужно состояние 'refreshing'
          для pull-to-refresh? Просто добавьте его к union type. С boolean
          пришлось бы добавлять ещё один флаг.
        


        

          Избегайте множественных boolean-флагов: isLoading,
          isError, isSuccess. Это создаёт 2³ = 8 возможных
          комбинаций, из которых только 4 валидны. Один enum решает проблему.
        


        
Ключевые моменты

        
    
          
  • Boolean = 2 состояния, реальные приложения требуют 4+
    • 
          
  • isLoading: false — это «ещё не загружали» или «загрузка завершена»?
    • 
          
  • Enum исключает невозможные комбинации состояний
    • 
          
  • Конечный автомат = чёткие переходы между состояниями
    • 
          
  • Redux Style Guide рекомендует подход state machine
