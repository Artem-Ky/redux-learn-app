Теория — DevTools + middleware вместе

      

        
Правильная композиция

        

          Когда в приложении нужны и middleware, и DevTools, их нужно объединить
          через compose. У Redux есть своя функция compose,
          которая комбинирует несколько enhancer-ов в один.
        


        

import { createStore, applyMiddleware, compose } from 'redux'

// DevTools enhancer (с фоллбэком для продакшена)
const devtoolsEnhancer =
  window.__REDUX_DEVTOOLS_EXTENSION__?.() || (f => f)

// compose объединяет enhancer-ы справа налево
const store = createStore(
  reducer,
  compose(
    applyMiddleware(thunk, loggerMiddleware),
    devtoolsEnhancer
  )
)
        


        
Порядок имеет значение

        

          applyMiddleware передаётся первым аргументом в
          compose, а DevTools enhancer — вторым. Это означает, что DevTools
          «оборачивает» store снаружи middleware, и может видеть все экшены,
          включая те, что прошли через middleware.
        


        

// compose(A, B) = A(B(createStore))
//
// Порядок выполнения:
// 1. devtoolsEnhancer оборачивает createStore
// 2. applyMiddleware оборачивает результат
//
// Цепочка dispatch:
// dispatch → middleware₁ → middleware₂ → DevTools → reducer
        


        
composeWithDevTools

        

          Пакет redux-devtools-extension предоставляет утилиту
          composeWithDevTools, которая упрощает этот паттерн:
        


        

import { composeWithDevTools } from 'redux-devtools-extension'

const store = createStore(
  reducer,
  composeWithDevTools(
    applyMiddleware(thunk, logger)
  )
)
// composeWithDevTools = compose + DevTools enhancer в одном
        


        

          «The Redux DevTools Extension can be composed with any other store enhancers.
          applyMiddleware goes inside compose, and the DevTools enhancer wraps everything.»
          
— Redux Fundamentals, Part 4: Store
        


        
Logger middleware

        

          Logger middleware перехватывает каждый dispatch и логирует: тип экшена, payload,
          предыдущее и следующее состояние. Это простой, но мощный инструмент для отладки —
          особенно когда DevTools расширение недоступно.
        


        

          Для продакшена используйте фоллбэк (f => f) для DevTools enhancer.
          Это «пустой» enhancer — функция-тождество, которая ничего не делает.
          Так приложение не упадёт, если расширение DevTools не установлено.
        


        

          Не путайте порядок аргументов в compose.
          applyMiddleware должен идти первым, DevTools —
          вторым. Если поменять местами, DevTools не увидит экшены,
          прошедшие через middleware (например, thunk-функции).
        


        
Ключевые моменты

        
    
          
  • compose объединяет middleware и DevTools enhancer
    • 
          
  • applyMiddleware идёт первым аргументом в compose
    • 
          
  • DevTools enhancer оборачивает всё снаружи
    • 
          
  • composeWithDevTools — удобная обёртка из отдельного пакета
    • 
          
  • Фоллбэк (f => f) защищает от отсутствия расширения
    • 
          
  • Logger middleware пишет в консоль тип, payload, prev/next state
