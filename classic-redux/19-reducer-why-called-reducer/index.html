Теория — Почему reducer называется «reducer»?

Array.reduce() — основа концепции

Метод Array.prototype.reduce() принимает массив значений и «сворачивает» (reduces) его в одно значение. Он делает это, вызывая callback-функцию для каждого элемента и передавая результат предыдущего вызова как аккумулятор:

const numbers = [1, 2, 3, 4, 5] const sum = numbers.reduce((accumulator, currentValue) => { return accumulator + currentValue }, 0) // 0 — начальное значение аккумулятора // Шаги: // (0, 1) → 1 // (1, 2) → 3 // (3, 3) → 6 // (6, 4) → 10 // (10, 5) → 15

Redux reducer — это та же идея!

Redux reducer — это функция с сигнатурой (state, action) => newState. Это точно такая же сигнатура, как у callback для Array.reduce()!

Если у вас есть массив экшенов, вы можете получить итоговый state, пропустив их через reduce():

const actions = [ { type: 'counter/incremented' }, { type: 'counter/incremented' }, { type: 'counter/addAmount', payload: 5 }, ] const initialState = { value: 0 } const finalState = actions.reduce(counterReducer, initialState) // { value: 7 }
«Redux reducers reduce a set of actions (over time) into a single state. The difference is that with Array.reduce() it happens all at once, and with Redux, it happens over the lifetime of your running app.»
— Redux Fundamentals, Part 3: Reducers

Сравнение: Array.reduce() vs Redux

// Array.reduce() // callback: (accumulator, item) → newAccumulator // initialValue: начальное значение аккумулятора // Redux reducer // reducer: (state, action) → newState // initialState: начальное состояние // Одна и та же сигнатура!
Ключевая идея: reducer «сводит» (reduces) поток экшенов во времени к единственному текущему состоянию. Каждый новый экшен — это как следующий элемент массива, а текущий state — аккумулятор.

Ключевые моменты

  • Reducer назван по аналогии с Array.prototype.reduce()
  • Сигнатура одинаковая: (accumulator, item) → newAccumulator
  • В Redux: (state, action) → newState
  • Разница: reduce() обрабатывает всё сразу, Redux — по мере поступления экшенов
  • Можно воспроизвести всю историю Redux, пропустив массив экшенов через reduce()
