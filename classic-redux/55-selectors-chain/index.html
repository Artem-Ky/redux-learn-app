Теория — Цепочки мемоизированных селекторов

      

        
Мемоизированный селектор как input

        

          Результат createSelector — это обычная функция-селектор. Её можно
          передать как input selector в другой createSelector,
          создавая цепочку мемоизации:
        


        

// Уровень 1: мемоизированная фильтрация
const selectFilteredTodos = createSelector(
  selectTodos,
  selectFilter,
  (todos, filter) => { /* фильтрация */ }
)

// Уровень 2: мемоизированное извлечение ID
const selectFilteredTodoIds = createSelector(
  selectFilteredTodos,           // input — мемоизированный!
  (todos) => todos.map(t => t.id)
)
        


        
Как работает цепочка

        

          При вызове selectFilteredTodoIds(state) происходит:
        


        

selectFilteredTodoIds(state)
  → selectFilteredTodos(state)  // input (мемоизированный)
    → selectTodos(state)         // input 1 мемоизированного
    → selectFilter(state)        // input 2 мемоизированного
    → если inputs те же → кэш    // уровень 1 не пересчитывается
  → если input тот же → кэш      // уровень 2 тоже не пересчитывается
        


        
Выгода двойной мемоизации

        
    
          
  • Dispatch без изменений — ни один output selector не вызывается
    • 
          
  • Изменился фильтр — пересчитывается уровень 1, затем уровень 2 (только если результат уровня 1 новый)
    • 
          
  • Изменился todo — оба пересчитываются (новый массив todos → новый отфильтрованный → новые ID)
    • 
        


        

          Redux Fundamentals, Part 7: Standard Redux Patterns:
          «Memoized selectors are a valuable tool for improving performance. Reselect's
          createSelector can be used to create memoized selectors that only
          recalculate results when their inputs change.»
          

          redux.js.org — Memoizing Selectors
        


        
Когда мемоизация «ломается»

        

          createSelector кэширует только последний результат.
          Если селектор вызывается с чередующимися inputs, кэш постоянно
          «промахивается»:
        


        

// Плохо: чередование → кэш бесполезен
selectFilteredTodos(stateA) // кэш: stateA
selectFilteredTodos(stateB) // промах, кэш: stateB
selectFilteredTodos(stateA) // промах, кэш: stateA

// Решение: отдельные экземпляры для разных компонентов
const makeSelectFiltered = () => createSelector(...)
        


        

          Цепочки мемоизированных селекторов — мощный инструмент. Каждый уровень
          кэширует свой результат, и пересчёт происходит только для тех уровней,
          где inputs действительно изменились.
        


        

          Не создавайте слишком длинные цепочки (3+ уровней) без необходимости.
          Каждый уровень добавляет сложность при дебаге. Оптимально: 1-2 уровня мемоизации.
        


        
Ключевые моменты

        
    
          
  • Мемоизированный селектор можно использовать как input для другого createSelector
    • 
          
  • Цепочка кэширует на каждом уровне — максимальная эффективность
    • 
          
  • При dispatch без изменений — ни один output selector не вызывается
    • 
          
  • Кэш размером 1: чередование разных inputs ломает мемоизацию
    • 
          
  • Фабрика () => createSelector(...) решает проблему нескольких подписчиков
