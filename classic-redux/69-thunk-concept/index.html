Теория — Что такое Thunk?

Thunk — «отложенная работа»

Thunk — это термин из программирования, означающий «кусок кода, который выполняет отложенную работу». В контексте Redux, thunk — это функция, которую можно отправить через dispatch() вместо обычного action-объекта.

«The word 'thunk' is a programming term that means 'a piece of code that does some delayed work'. Rather than execute some logic now, we can write a function body or code that can be used to perform the work later.»
— Redux Fundamentals, Part 6: Async Logic and Data Fetching

Наш thunk middleware — 5 строк

Thunk middleware невероятно прост. Он проверяет: если action — функция, вызвать её с (dispatch, getState). Если объект — передать дальше через next().

const thunkMiddleware = (storeAPI: any) => (next: any) => (action: any) => { if (typeof action === 'function') { return action(storeAPI.dispatch, storeAPI.getState) } return next(action) }

Dispatch функции вместо объекта

Обычно мы dispatch'им объекты: { type: 'counter/increment' }. С thunk middleware мы можем dispatch'ить функцию:

// Thunk action creator — возвращает функцию const incrementAsync = () => { return (dispatch: any, getState: any) => { setTimeout(() => { dispatch({ type: 'counter/incremented' }) }, 1000) } } // dispatch функции! store.dispatch(incrementAsync())

Зачем это нужно?

Redux reducers не могут содержать асинхронную логику — они чистые функции. Thunk middleware позволяет выносить асинхронную логику (API-вызовы, таймеры) в action creators, сохраняя reducers чистыми.

// Thunk имеет доступ к dispatch и getState const conditionalIncrement = () => (dispatch: any, getState: any) => { const state = getState() if (state.value < 10) { dispatch({ type: 'counter/incremented' }) } }
Обратите внимание: наш thunk middleware использует storeAPI.dispatch, а не next. Это значит, что action'ы, отправленные из thunk-функции, проходят через весь pipeline заново — включая сам thunk middleware.
Это учебная реализация! В реальных проектах используйте официальный redux-thunk — он протестирован и поддерживается командой Redux. Но наша версия показывает, что под капотом всё предельно просто.
