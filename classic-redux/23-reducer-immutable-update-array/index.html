Теория — Иммутабельные паттерны для массивов

      

        
Добавление элемента: spread

        

          Вместо push() (мутация) используем spread-оператор для создания нового массива:
        


        

// ❌ Мутация
state.push(newItem)
return state

// ✔ Иммутабельно
return [...state, newItem]           // добавить в конец
return [newItem, ...state]           // добавить в начало
        


        
Удаление элемента: filter

        

          Вместо splice() (мутация) используем filter() — он возвращает новый массив:
        


        

// ❌ Мутация
state.splice(index, 1)
return state

// ✔ Иммутабельно
return state.filter((_, i) => i !== indexToRemove)
return state.filter(item => item.id !== idToRemove)
        


        
Обновление элемента: map

        

          map() возвращает новый массив, где можно заменить нужный элемент:
        


        

// ❌ Мутация
state[index].text = newText
return state

// ✔ Иммутабельно
return state.map((item, i) =>
  i === index ? { ...item, text: newText } : item
)
        


        

          «You can only make copies of the original values, and then mutate the copies.
          We can use the JavaScript array / object spread operators, as well as array
          methods that return new copies of the array.»
          
— Redux Fundamentals, Part 3
        


        
Запрещённые методы (мутирующие)

        

          В reducer нельзя использовать мутирующие методы массива:
          push, pop, shift, unshift,
          splice, sort, reverse.
          Вместо них — concat/spread, filter,
          map, slice, [...arr].sort().
        


        
Шпаргалка

        
    
          
  • Добавить: [...state, newItem] или state.concat(newItem)
    • 
          
  • Удалить: state.filter(item => condition)
    • 
          
  • Обновить: state.map(item => condition ? updatedItem : item)
    • 
          
  • Вставить: [...state.slice(0, i), newItem, ...state.slice(i)]
    • 
          
  • Все эти методы возвращают новый массив
