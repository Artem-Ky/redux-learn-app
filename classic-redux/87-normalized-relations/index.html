Теория — Связи между сущностями

      

        
Ссылки по ID между сущностями

        

          В нормализованном состоянии сущности хранятся независимо друг от друга.
          Связи между ними выражаются через ссылки по ID — аналогично
          внешним ключам в реляционных базах данных.
        


        

// Комментарий ссылается на пост через postId
interface Comment {
  id: number
  postId: number  // ← ссылка на Post.id
  text: string
  author: string
}
        


        
«Join» данных в селекторах

        

          Чтобы получить все комментарии к посту, мы делаем «join» — фильтруем
          комментарии по postId:
        


        

const selectCommentsByPostId = (
  state: RootState,
  postId: number
) =>
  selectAllComments(state).filter(
    c => c.postId === postId
  )
        


        

          «Any references to other individual items should be done by storing the
          item's ID... This approach of storing items by ID in an object and using
          IDs as references is called 'normalization'.»
          
— Redux Docs: Normalizing State Shape
        


        
Независимость сущностей

        

          Посты и комментарии хранятся в отдельных слайсах состояния.
          Это даёт несколько преимуществ:
        

        
    
          
  • Обновление комментария не затрагивает объект поста
    • 
          
  • Один комментарий может быть связан с разными сущностями
    • 
          
  • Удаление поста не требует обхода вложенных массивов
    • 
          
  • Каждый reducer управляет только своей сущностью
    • 
        


        

// Состояние: два независимых слайса
{
  posts:    { ids: [...], entities: { ... } },
  comments: { ids: [...], entities: { ... } }
}
        


        
Загрузка связанных данных

        

          При загрузке данных с сервера ответ часто приходит вложенным.
          Перед сохранением в store его нужно нормализовать —
          разложить по отдельным таблицам и заменить вложенные объекты на ID.
        


        

          Паттерн comment.postId → post.id — это «внешний ключ»
          из мира SQL. В Redux мы используем тот же подход, но «join» выполняется
          в селекторах, а не в базе данных.
        


        

          Фильтрация selectAllComments(state).filter(c => c.postId === postId)
          выполняется за O(n). Для больших объёмов данных лучше хранить обратный индекс
          commentsByPostId: { [postId]: number[] } или использовать мемоизацию.
        


        
Ключевые моменты

        
    
          
  • Связи между сущностями — через ссылки по ID (comment.postId)
    • 
          
  • «Join» данных выполняется в селекторах, а не в reducer
    • 
          
  • Сущности хранятся независимо в отдельных слайсах
    • 
          
  • Подход аналогичен внешним ключам в реляционных базах данных
    • 
          
  • Нормализация вложенных ответов API — стандартная практика
