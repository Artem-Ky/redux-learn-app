Интерактив — Типичные ошибки при работе с Immer

Три интерактивных примера частых ошибок с produce. Нажимайте «Попробовать» и смотрите в консоль, что происходит.

Ошибка #1 Мутация draft + возврат нового значения

Нельзя одновременно мутировать draft и возвращать новое значение. Immer не знает, какой результат использовать.

Ошибка #2 Использование draft за пределами produce

Draft-объект действителен только внутри callback функции produce. За её пределами draft «протухает» и его нельзя использовать.

Ошибка #3 Забыли return при полной замене состояния

Если нужно заменить состояние целиком (не мутировать), нужно вернуть новое значение. Без return Immer вернёт исходный объект.

Теория — Правила работы с Immer

Правило 1: Мутируй draft ИЛИ возвращай новое значение

Внутри produce можно либо мутировать draft, либо вернуть новый объект. Нельзя делать и то, и другое одновременно — Immer выбросит ошибку.

// ✅ Правильно: мутируем draft const next1 = produce(state, draft => { draft.count++ }) // ✅ Правильно: возвращаем новый объект const next2 = produce(state, draft => { return { ...draft, count: draft.count + 1 } }) // ❌ Ошибка: и мутация, и return const bad = produce(state, draft => { draft.count++ return { ...draft, count: draft.count + 1 } })

Правило 2: Draft действителен только внутри produce

Draft-объект — это Proxy, который отслеживает мутации. После завершения produce этот Proxy «отзывается» (revoked). Попытка использовать draft снаружи приведёт к ошибке.

let savedDraft: any produce(state, draft => { savedDraft = draft // сохраняем ссылку draft.count++ }) // ❌ draft уже «протух» (revoked) savedDraft.count = 42 // TypeError!

Правило 3: Для полной замены состояния — return

Если вы хотите полностью заменить состояние новым объектом (а не мутировать draft), вы должны вернуть новое значение через return. Без return Immer вернёт первоначальный state.

// ❌ Неправильно: state не изменится const oops = produce(state, draft => { const newState = { count: 0, items: [] } // забыли return! }) // ✅ Правильно: return заменяет state целиком const fixed = produce(state, draft => { return { count: 0, items: [] } })

Правило 4: produce возвращает замороженный объект

Результат produce автоматически замораживается (Object.freeze). Это помогает обнаружить случайные мутации в другом коде.

«An Immer producer can either modify the draft or return a new value, but not both. If you both modify the draft and return a new value, Immer will throw an error.»
— Immer Docs — Pitfalls

Частые подводные камни

  • Async/await внутри produce — не используйте async produce без специального паттерна. Стандартный produce синхронный.
  • Деструктуризация draft — const { items } = draft даёт Proxy, но примитивы теряют связь: const { count } = draft — count не мутируемый.
  • this в produce — стрелочная функция не имеет своего this, что может сбить с толку в классах.
  • Map/Set — Immer поддерживает их, но нужно включить enableMapSet().
Если вы сомневаетесь — используйте только мутации draft, никогда не комбинируйте с return. Это самый безопасный подход.
Никогда не сохраняйте ссылку на draft вне produce! Proxy отзывается сразу после завершения callback. Любое обращение к нему вызовет TypeError.
