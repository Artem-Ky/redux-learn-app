Интерактив — shallow equality и ре-рендеры connect()

Компонент подключён через connect(). mapStateToProps возвращает только { name }. Нажмите кнопки и наблюдайте: изменение age не вызывает ре-рендер, потому что name не изменился.

Теория — Shallow Equality и ре-рендеры mapStateToProps

Как connect() решает, перерисовывать ли компонент?

Когда state в store изменяется, React Redux вызывает вашу функцию mapStateToProps и получает новый объект. Затем он сравнивает каждое поле нового объекта с соответствующим полем предыдущего результата через ===. Это называется shallow equality check (поверхностная проверка равенства).

"React Redux internally implements the shouldComponentUpdate method such that the wrapper component re-renders precisely when the data your component needs has changed. By default, React Redux decides whether the contents of the object returned from mapStateToProps are different using === comparison (a 'shallow equality' check) on each field of the returned object." — react-redux.js.org/using-react-redux/connect-mapstate
«React Redux внутренне реализует метод shouldComponentUpdate так, что компонент-обёртка перерисовывается именно тогда, когда данные, нужные вашему компоненту, изменились. По умолчанию React Redux определяет, изменилось ли содержимое объекта, возвращённого из mapStateToProps, используя сравнение === (поверхностную проверку равенства) для каждого поля возвращённого объекта.»

Визуально: как работает shallow equality

// Предположим mapStateToProps возвращает: const prev = { name: "Alice" } const next = { name: "Alice" } // Shallow equality check: prev.name === next.name // "Alice" === "Alice" → true // Все поля одинаковы → НЕ перерисовывать // Теперь если name изменилось: const prev2 = { name: "Alice" } const next2 = { name: "Bob" } prev2.name === next2.name // "Alice" === "Bob" → false // Есть отличие → ПЕРЕРИСОВАТЬ

Ловушка: мутация объекта

"Note that returning a mutated object of the same reference is a common mistake that can result in your component not re-rendering when expected." — react-redux.js.org/using-react-redux/connect-mapstate
«Обратите внимание: возврат мутированного объекта с той же ссылкой — распространённая ошибка, которая может привести к тому, что ваш компонент не перерисуется, когда вы этого ожидаете.»
// ПЛОХО: мутация объекта const mapStateToProps = (state) => { state.user.name = "changed" // мутация! return state.user // та же ссылка → shallow check видит === } // Результат: компонент НЕ перерисуется, хотя данные изменились // ХОРОШО: новый объект const mapStateToProps = (state) => ({ name: state.user.name })

Когда запускается mapStateToProps?

Сигнатура Когда вызывается
mapStateToProps(state) Когда state в store изменяется
mapStateToProps(state, ownProps) Когда state в store изменяется ИЛИ когда ownProps изменяются

Связь с системой Subscription

Когда store обновляется, React Redux использует систему Subscription (подробнее в уроке 03a): Subscription хранит linked list callback'ов. При каждом dispatch вызывается checkForUpdates, который запускает mapStateToProps с новым state, сравнивает каждое поле результата с предыдущим через ===, и если хотя бы одно поле отличается — вызывает forceRender для компонента.

Ключевой вывод: возвращайте из mapStateToProps только те поля, которые реально нужны компоненту. Чем меньше полей — тем реже компонент перерисовывается.
