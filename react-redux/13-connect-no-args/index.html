Интерактив — connect() без аргументов

connect()(MyComponent) — самый простой вызов. Компонент получает props.dispatch, но не подписывается на store (не ре-рендерится при изменении state).

Теория — Четыре комбинации вызова connect()

connect() без аргументов

Когда вы вызываете connect() без аргументов, результат — компонент, который получает только dispatch в props. Он не подписан на изменения store, поэтому изменения state не вызывают его ре-рендер.

// connect() без аргументов: const ConnectedComponent = connect()(MyComponent) // MyComponent получит: // props.dispatch — функция для dispatch'а actions // Больше ничего из store! // Компонент НЕ подписан на store — не перерисовывается при dispatch.

Четыре комбинации вызова connect()

В зависимости от переданных аргументов, connect() ведёт себя по-разному:

Вызов Подписка на store dispatch в props Что получает компонент
connect()(Comp) нет да только props.dispatch
connect(mapStateToProps)(Comp) да да данные из store + dispatch
connect(null, mapDispatchToProps)(Comp) нет нет* только action creators
connect(mapStateToProps, mapDispatchToProps)(Comp) да нет* данные из store + action creators

* Когда передан mapDispatchToProps, dispatch не добавляется в props автоматически (предполагается, что вы биндите action creators).

«If you don't supply your own mapDispatchToProps function or object full of action creators, the default mapDispatchToProps implementation just injects dispatch into your component's props.»
— react-redux.js.org — connect: Dispatching Actions
«Если вы не предоставите свою собственную функцию mapDispatchToProps или объект с action creators, реализация mapDispatchToProps по умолчанию просто инжектит dispatch в props вашего компонента.»
— перевод

Подписка на store и Subscription

Подписка на store контролируется первым аргументом (mapStateToProps):

  • Если mapStateToProps передан — connect создаёт Subscription (listener в linked list). При dispatch — selector вызывается с новым state, результат сравнивается с предыдущим (===). Если отличается — forceRender().
  • Если mapStateToProps не передан (null или undefined) — подписки нет. Компонент не знает об изменениях state.
Когда connect() без аргументов полезен?
Когда компоненту нужно только отправлять actions, но не читать данные из store. Например, кнопка «Выйти из аккаунта», которая dispatch'ит action, но не отображает данные пользователя.
Современная альтернатива: useDispatch() делает то же самое без обёртки. В новом коде предпочитайте хуки.
