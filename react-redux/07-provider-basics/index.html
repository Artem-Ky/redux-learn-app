Интерактив — Без <Provider> vs с <Provider>

Слева — компонент без <Provider> (ошибка перехватывается ErrorBoundary). Справа — с <Provider> (рабочий счётчик).

Теория — Компонент <Provider>

Зачем нужен <Provider>?

"The <Provider> component makes the Redux store available to any nested components that need to access the Redux store."

Компонент <Provider> делает Redux store доступным для любых вложенных компонентов, которым нужен доступ к Redux store.

— react-redux.js.org, Provider

<Provider> — это React-компонент, который принимает prop store и помещает его в React Context. После этого любой компонент внутри дерева может получить доступ к store через хуки (useSelector, useDispatch) или через connect().

Почему на верхнем уровне?

"Since any React component in a React Redux app can be connected to the store, most applications will render a <Provider> at the top level, with the entire app's component tree inside of it."

Поскольку любой React-компонент в React Redux приложении может быть подключён к store, большинство приложений рендерят <Provider> на верхнем уровне, с деревом компонентов всего приложения внутри.

— react-redux.js.org, Provider
// Типичная точка входа React-Redux приложения import { Provider } from 'react-redux' import { createRoot } from 'react-dom/client' import store from './store' import App from './App' createRoot(document.getElementById('root')).render( <Provider store={store}> <App /> </Provider> )

Как это связано с Context?

"The Hooks and connect APIs can then access the provided store instance via React's Context mechanism."

API хуков и connect затем получают доступ к предоставленному экземпляру store через механизм React Context.

— react-redux.js.org, Accessing the Store

Внутри <Provider> создаёт объект {'{store, subscription}'} и помещает его в ReactReduxContext. Когда вы вызываете useSelector(), хук читает store из этого контекста и подписывается на обновления через Subscription (linked list коллбэков, где каждый вызывает checkForUpdates(), сравнивает результат selector через === и при изменении вызывает forceRender() — подробнее в уроке 03a).

Что если забыть <Provider>?

Если вызвать useSelector или useDispatch без <Provider> в дереве, React-Redux выбросит ошибку:

Error: Could not find "store" in the context of "Connect(MyComponent)". Either wrap the root component in a <Provider>, or pass a custom React context provider to <Provider> and the corresponding React context consumer to Connect(MyComponent) in connect options.
Важно: <Provider> должен быть выше в дереве, чем любой компонент, использующий useSelector, useDispatch или connect. Обычно его ставят в самом корне — в файле index.tsx.
