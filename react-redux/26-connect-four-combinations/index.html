Интерактив — четыре варианта connect бок о бок

Ниже — один и тот же компонент-счётчик, подключённый через connect четырьмя разными способами. Все четыре варианта смотрят в один и тот же store. Кнопки сверху меняют state. Обратите внимание на счётчик рендеров в каждой карточке: варианты без mapStateToProps не подписаны на store и не ре-рендерятся при изменении state. Список props каждого варианта тоже отличается.

Теория — 4 комбинации connect

Итоговая таблица 2×2

Документация react-redux описывает четыре наиболее часто встречающихся способа вызова connect. Разница — в двух ортогональных осях: подписывается ли компонент на store (есть ли mapStateToProps) и инжектятся ли action creators (есть ли mapDispatchToProps).

Без инжекции
action creators		 connect()(Comp) connect(mapStateToProps)(Comp)
С инжекцией
action creators		 connect(null, mapDispatchToProps)(Comp) connect(mapStateToProps, mapDispatchToProps)(Comp)

— react-redux.js.org/tutorials/connect
«В зависимости от того, с какими компонентами вы работаете, существуют разные способы вызова connect — наиболее распространённые из них приведены ниже.»
— перевод

Вариант 1: connect()(Component) — без аргументов

«If you call connect without providing any arguments, your component will: not re-render when the store changes, receive props.dispatch that you may use to manually dispatch action»
— react-redux.js.org/tutorials/connect
«Если вы вызываете connect без аргументов, ваш компонент: не будет ре-рендериться при изменениях store, получит props.dispatch, который можно использовать для ручного диспатча action.»
— перевод
// Вариант 1: не подписан, props.dispatch export default connect()(Comp) // props = { dispatch } // store изменяется → Comp НЕ ре-рендерится

Вариант 2: connect(mapStateToProps)(Component)

Компонент подписывается на store (есть mapStateToProps). Без mapDispatchToProps он также получает props.dispatch по умолчанию (это мы подробно разбирали в уроке 20).

// Вариант 2: подписан, props.dispatch const mapStateToProps = (state) => ({ count: state.counter }) export default connect(mapStateToProps)(Comp) // props = { count, dispatch } // store изменяется → mapStateToProps запускается → если результат поменялся → ре-рендер

Вариант 3: connect(null, mapDispatchToProps)(Component)

Компонент не подписан на store (первый аргумент null), но получает action creators в виде готовых функций. Полезно для «dispatch-only» компонентов — форм, кнопок, dispatcher'ов, которым не нужно читать state.

// Вариант 3: не подписан, action creators в props const mapDispatchToProps = { increment, decrement } export default connect(null, mapDispatchToProps)(Comp) // props = { increment, decrement } — dispatch НЕ передаётся // store изменяется → Comp НЕ ре-рендерится

Вариант 4: connect(mapStateToProps, mapDispatchToProps)(Component)

Самая полная форма. Компонент подписан на store и получает action creators как props. Это типичная форма для «контейнерных» компонентов из классического Redux-подхода.

// Вариант 4: подписан, action creators в props export default connect( mapStateToProps, mapDispatchToProps )(Comp) // props = { count, increment, decrement } — dispatch НЕ передаётся // store изменяется → mapStateToProps → сравнение → ре-рендер при изменениях

Связь с системой Subscription

Подписка «включается» именно наличием mapStateToProps. Когда вы указываете mapStateToProps, connect при маунте вызывает store.subscribe(checkForUpdates) и добавляет callback в linked list Subscription (подробнее в уроке 03a). После dispatch Subscription проходит по linked list callback'ов → notify() → checkForUpdates → mapStateToProps с новым state → сравнение каждого поля результата через === → forceRender если хоть одно отличается. Варианты 1 и 3 не регистрируют callback в Subscription (нет mapStateToProps), поэтому они никогда не ре-рендерятся от изменений store — только от изменений собственных ownProps.

Как запомнить: первый аргумент connect отвечает за чтение (подписку на store), второй — за запись (dispatch). Они независимы: можно иметь одно без другого. Компонент-«читатель» — connect(mapStateToProps). Компонент-«писатель» — connect(null, mapDispatchToProps). Оба — connect(mapStateToProps, mapDispatchToProps). Ни того ни другого — connect().
Помните про dispatch: если mapDispatchToProps не указан, props.dispatch передаётся автоматически. Если указан — props.dispatch не передаётся, вместо него передаётся результат mapDispatchToProps. Подробнее — в уроке 25.
