Теория — Селекторы в mapStateToProps

Зачем нужны селекторы?

"We highly encourage the use of 'selector' functions to help encapsulate the process of extracting values from specific locations in the state tree. Memoized selector functions also play a key role in improving application performance." — react-redux.js.org/using-react-redux/connect-mapstate
«Мы настоятельно рекомендуем использовать функции-"селекторы", чтобы инкапсулировать процесс извлечения значений из конкретных мест дерева state. Мемоизированные функции-селекторы также играют ключевую роль в улучшении производительности приложения.»

Простые селекторы

Простой селектор — обычная функция, которая принимает state и возвращает нужный фрагмент. Они живут рядом с reducer'ами:

// selectors.ts (рядом с todosReducer) export const selectAllTodos = (state: RootState) => state.todos.items export const selectFilter = (state: RootState) => state.todos.filter export const selectTodoCount = (state: RootState) => state.todos.items.length

Мемоизированные селекторы (createSelector)

Когда нужно вычислять производное значение (фильтрация, сортировка, агрегация), используйте createSelector из библиотеки reselect:

import { createSelector } from 'reselect' export const selectFilteredTodos = createSelector( [selectAllTodos, selectFilter], (todos, filter) => { if (filter === 'done') return todos.filter(t => t.done) if (filter === 'active') return todos.filter(t => !t.done) return todos } )

createSelector запоминает результат. Функция-вычислитель вызывается только если хотя бы один входной селектор вернул новое значение. Если входы те же — возвращается кэш.

Использование в mapStateToProps

const mapStateToProps = (state: RootState) => ({ todos: selectFilteredTodos(state), count: selectTodoCount(state), }) export default connect(mapStateToProps)(TodoList)

Паттерн: селекторы живут рядом с reducer'ами

// Структура файлов: // // features/ // todos/ // todosReducer.ts ← reducer // todosSelectors.ts ← селекторы // TodoList.tsx ← компонент // // Селекторы знают структуру state → живут рядом с reducer'ом, // который эту структуру определяет.
Ключевое преимущество: если структура state изменится, вы правите только селекторы — компоненты не затрагиваются. Это инкапсуляция.

Связь с системой Subscription

При каждом dispatch система Subscription (подробнее в уроке 03a) проходит по linked list callback'ов и вызывает checkForUpdates. Для компонентов, подключённых через connect(), checkForUpdates запускает mapStateToProps, а внутри неё — ваши селекторы. Мемоизированный селектор возвращает тот же объект если входы не изменились → shallow equality check проходит через === → компонент не перерисовывается.

Без мемоизации: если селектор возвращает новый массив/объект каждый раз (через .filter(), .map(), spread), shallow check видит новую ссылку и компонент перерисовывается при каждом dispatch.
