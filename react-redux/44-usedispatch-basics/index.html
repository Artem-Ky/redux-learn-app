Интерактив — счётчик и проверка стабильности dispatch

Слева — типовой счётчик через useDispatch. Справа — инструмент, который при каждом рендере сравнивает текущий dispatch с предыдущим. В одной и той же сессии (пока не меняется store) ссылка должна оставаться одной и той же — это важно для корректной работы useEffect, useCallback и прочих хуков с deps.

Теория — useDispatch

Что это

"This hook returns a reference to the dispatch function from the Redux store. You may use it to dispatch actions as needed."
— react-redux.js.org/api/hooks
«Этот хук возвращает ссылку на функцию dispatch из Redux store. Её можно использовать для отправки actions по мере необходимости.»
— перевод

Сигнатура

const dispatch = useDispatch() // dispatch: Dispatch<AnyAction> // Вызов такой же, как у store.dispatch: dispatch({ type: 'counter/increment' }) dispatch({ type: 'todos/addTodo', payload: 'buy milk' })

Стабильность ссылки

"The dispatch function reference will be stable as long as the same store instance is being passed to the <Provider>. Normally, that store instance never changes in an application."
— react-redux.js.org/api/hooks
«Ссылка на функцию dispatch стабильна до тех пор, пока в <Provider> передаётся один и тот же экземпляр store. В обычном приложении store никогда не меняется.»
— перевод

dispatch — это метод конкретного store, и если у вас один store на всё приложение (обычный случай), то useDispatch будет возвращать одну и ту же функцию всю жизнь приложения. Смена происходит только при смене самого store — например, при замене store props у <Provider>.

Можно и нужно добавлять в dependency arrays

"However, the React hooks lint rules do not know that dispatch should be stable, and will warn that the dispatch variable should be added to dependency arrays for useEffect and useCallback. The simplest solution is to do just that"
— react-redux.js.org/api/hooks
«Однако правила React hooks lint не знают, что dispatch стабилен, и будут предупреждать, что переменную dispatch нужно добавить в dependency array для useEffect и useCallback. Самое простое решение — так и сделать.»
— перевод
// ✔ Идиоматичный async-load-on-mount function TodosList() { const dispatch = useDispatch() useEffect(() => { dispatch(fetchTodos()) }, [dispatch]) // dispatch стабилен, эффект запустится один раз // ... }

Включение dispatch в deps — это compromise между формальной корректностью (линт) и гарантиями рантайма (react-redux): dep-array честно включает используемое значение, а поскольку оно не меняется, лишних запусков эффекта не будет.

Работа в паре с useSelector

function Counter() { const count = useSelector((state: RootState) => state.counter.value) const dispatch = useDispatch() return ( <div> <button onClick={() => dispatch({ type: 'counter/decrement' })}>−</button> <span>{count}</span> <button onClick={() => dispatch({ type: 'counter/increment' })}>+</button> </div> ) }

useSelector читает state, useDispatch отправляет actions — это симметричная пара. Вместе они заменяют связку mapStateToProps + mapDispatchToProps в connect.

Типизация

// Если вы хотите свой типизированный dispatch с поддержкой thunks: import type { ThunkDispatch } from 'redux-thunk' import { useDispatch } from 'react-redux' type AppDispatch = ThunkDispatch<RootState, unknown, AnyAction> export const useAppDispatch: () => AppDispatch = useDispatch

Связь с системой Subscription

useDispatch не трогает подписку вообще — он не подписывается на store, не регистрирует callback'ов. Он просто возвращает метод store.dispatch из контекста <Provider>. Полный путь после вызова dispatch: reducer → новый state → linked list callback'ов → Subscription.notify() → checkForUpdates → selector → === → forceRender. useDispatch — только «входная дверь» в эту цепочку. Подробнее про Subscription — урок 03a.

Идиомы:
  • const dispatch = useDispatch() — где-то в начале компонента.
  • В useEffect / useCallback добавляйте dispatch в deps — линт успокоится, эффект не перезапустится лишний раз.
  • Для thunks / async — типизируйте свой useAppDispatch поверх useDispatch.
Антипаттерн: создавать dispatch-callback прямо в рендере (() => dispatch(...)) и передавать его в React.memo-ребёнка. Каждый рендер — новая ссылка функции, ребёнок ре-рендерится. Лечится useCallback — об этом следующий урок (45).
