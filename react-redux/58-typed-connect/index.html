Интерактив — те же props, две разные типизации

Оба переключателя читают state.isOn и диспатчат TOGGLE_IS_ON. Слева — ручная типизация через StateProps, DispatchProps, OwnProps, плюс три дженерика в connect. Справа — ConnectedProps<typeof connector>, который автоматически вычисляет типы из mapStateToProps и mapDispatchToProps. Внутри — одинаковый JavaScript.

Теория — типизация connect

Три источника props у connected-компонента

connect склеивает вместе три разных объекта: mapStateToProps (данные из Redux), mapDispatchToProps (callback'и), и props от родителя (OwnProps). Типизировать это можно двумя принципиально разными путями: «вручную описать всё» или «попросить TypeScript вывести всё из первого вызова connect».

«The package includes a helper type, ConnectedProps, that can extract the return types of mapStateToProps and mapDispatchToProps from the first function. This means that if you split the connect call into two steps, all of the "props from Redux" can be inferred automatically without having to write them by hand.»
— react-redux.js.org/using-react-redux/usage-with-typescript
«Пакет включает вспомогательный тип ConnectedProps, который способен извлечь типы, возвращаемые mapStateToProps и mapDispatchToProps, из первой функции. Это значит, что если разбить вызов connect на два шага, все "props из Redux" могут быть выведены автоматически, без необходимости писать их вручную.»
— перевод

Вариант 1. Ручная типизация (полный boilerplate)

interface StateProps { isOn: boolean } interface DispatchProps { toggleOn: () => void } interface OwnProps { backgroundColor: string } type Props = StateProps & DispatchProps & OwnProps const mapStateToProps = (state: RootState): StateProps => ({ isOn: state.isOn, }) const mapDispatchToProps: DispatchProps = { toggleOn: () => ({ type: 'TOGGLE_IS_ON' }), } const ToggleButton = (props: Props) => ( <button style={{ backgroundColor: props.backgroundColor }} onClick={props.toggleOn}> Toggle is {props.isOn ? 'ON' : 'OFF'} </button> ) export default connect<StateProps, DispatchProps, OwnProps>( mapStateToProps, mapDispatchToProps, )(ToggleButton)

Вариант 2. ConnectedProps (автовывод из connector)

const mapStateToProps = (state: RootState) => ({ isOn: state.isOn }) const mapDispatchToProps = { toggleOn: () => ({ type: 'TOGGLE_IS_ON' as const }), } const connector = connect(mapStateToProps, mapDispatchToProps) type PropsFromRedux = ConnectedProps<typeof connector> // выведется: { isOn: boolean; toggleOn: () => void } interface OwnProps { backgroundColor: string } type Props = PropsFromRedux & OwnProps const ToggleButton = (props: Props) => /* ... */ export default connector(ToggleButton)

Вариант 3. Короткий инференс через ReturnType / typeof

Иногда можно ужать код без ConnectedProps:

type StateProps = ReturnType<typeof mapStateToProps> type DispatchProps = typeof mapDispatchToProps type Props = StateProps & DispatchProps & OwnProps
Предупреждение: этот вариант ломается, если mapDispatchToProps содержит thunks. typeof считает тип значений как есть — и в props попадут функции-экшен-криейторы вместо биндед-коллбеков. ConnectedProps эту задачу решает корректно, потому что вытаскивает типы именно после биндинга. Если думаете между вариантами 2 и 3 — берите вариант 2.

Связь с системой Subscription

Типизация connect — чисто TypeScript-вещь, в рантайме ничего не меняется. Всё так же: dispatch → reducer → Subscription проходит по linked list callback'ов → Subscription.notify() → для каждого connected-компонента checkForUpdates → запустить mapStateToProps → сравнить результат с prev через === (или areStatesEqual из options) → forceRender если значения разные. Подробнее о Subscription — урок 03a.

Рекомендация: в новых проектах берите hooks API — там типизация ещё прямее (урок 55–57). Но если у вас уже legacy-кодовая база на connect — конвертируйте постепенно на ConnectedProps, это даст TypeScript-покрытие без переписывания логики.
