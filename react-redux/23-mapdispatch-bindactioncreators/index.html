Интерактив — три способа подключить action creators

Три счётчика, подключённые к одному store тремя разными способами: ручной обёрткой, через bindActionCreators и объектной короткой формой (превью урока 24). Внешне и по props они идентичны. Нажимайте кнопки и сравнивайте вызовы в консоли.

Теория — bindActionCreators

Что такое bindActionCreators?

bindActionCreators — это утилита из пакета redux (не из react-redux!). Импортируется так:

import { bindActionCreators } from 'redux'
«bindActionCreators turns an object whose values are action creators, into an object with the same keys, but with every action creator wrapped into a dispatch call so they may be invoked directly.»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«bindActionCreators превращает объект, значениями которого являются action creators, в объект с теми же ключами, но с каждым action creator, обёрнутым в вызов dispatch, так что их можно вызывать напрямую.»
— перевод

Параметры

  1. action creator (функция) или объект, у которого значения — action creators;
  2. dispatch — функция из store.

Ручная обёртка vs bindActionCreators

// Action creators const increment = () => ({ type: 'counter/increment' }) const decrement = () => ({ type: 'counter/decrement' }) const reset = () => ({ type: 'counter/reset' }) // --- (A) Руками: много boilerplate --- const mapDispatchToProps = (dispatch) => ({ increment: (...args) => dispatch(increment(...args)), decrement: (...args) => dispatch(decrement(...args)), reset: (...args) => dispatch(reset(...args)), }) // --- (B) Через bindActionCreators: одна строка --- const mapDispatchToProps = (dispatch) => bindActionCreators({ increment, decrement, reset }, dispatch)

Автоматический форвардинг аргументов

«The wrapper functions generated by bindActionCreators will automatically forward all of their arguments, so you don't need to do that by hand.»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«Функции-обёртки, сгенерированные bindActionCreators, автоматически пробрасывают все свои аргументы, так что вам не нужно делать это вручную.»
— перевод

Это важно, когда action creator принимает аргументы:

const addTodo = (text) => ({ type: 'todos/add', payload: text }) const bound = bindActionCreators({ addTodo }, dispatch) // bound.addTodo("Купить хлеб") // ≡ dispatch(addTodo("Купить хлеб")) // ≡ dispatch({ type: 'todos/add', payload: 'Купить хлеб' })

Что возвращает bindActionCreators?

Объект с тем же набором ключей, но значения — функции-обёртки:

bindActionCreators({ increment, decrement, reset }, dispatch) // → { increment: (...args) => dispatch(increment(...args)), decrement: (...args) => dispatch(decrement(...args)), reset: (...args) => dispatch(reset(...args)), }

Можно передать и одну функцию

Если передать не объект, а один action creator — вернётся одна обёрнутая функция:

const boundIncrement = bindActionCreators(increment, dispatch) boundIncrement() // ≡ dispatch(increment())

Превью следующего урока

Если в connect передать объект вместо функции, react-redux автоматически вызовет bindActionCreators за вас. Это — объектная короткая форма, рекомендуемая документацией. Полный разбор в уроке 24.

Связь с Subscription-системой

Обёрнутые через bindActionCreators функции делают то же самое, что и ручные — вызывают dispatch(actionCreator(...args)). Далее обычный путь: store обновляет state, дёргает Subscription.notify(), тот пробегает по linked list подписчиков, для каждого checkForUpdates запускает mapStateToProps, сравнивает поля через === с предыдущими значениями и при изменении вызывает forceRender() (урок 03a). Обёртки — чисто синтаксический сахар, на цикл обновления они не влияют.

Итог: bindActionCreators спасает от boilerplate при ручной обёртке action creators. Но в повседневной работе с connect вы будете чаще использовать ещё более короткую объектную форму (следующий урок).
