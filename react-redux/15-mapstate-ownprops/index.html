Интерактив — mapStateToProps с ownProps (Todo-список)

Каждый <TodoItem> получает id от родителя (ownProps) и читает свой todo из store через mapStateToProps(state, ownProps). Изменение одного todo ре-рендерит только его.

id={0}

Теория — Второй аргумент mapStateToProps: ownProps

Что такое ownProps?

ownProps — это props, которые передаёт родительский компонент (а не Redux). Если ваша mapStateToProps объявлена с двумя параметрами, она получит ownProps как второй аргумент.

// mapStateToProps с ownProps: const mapStateToProps = (state, ownProps) => ({ todo: state.todos[ownProps.id] }) // Родитель передаёт id: <ConnectedTodoItem id={0} /> <ConnectedTodoItem id={1} /> <ConnectedTodoItem id={2} />

Когда mapStateToProps вызывается с ownProps

«If your mapStateToProps function is declared as taking two parameters, it will be called whenever the store state changes or when the wrapper component receives new props (based on shallow equality comparisons).»
— react-redux.js.org — connect: Extracting Data with mapStateToProps
«Если ваша функция mapStateToProps объявлена с двумя параметрами, она будет вызываться при каждом изменении состояния store ИЛИ когда компонент-обёртка получает новые props (на основе shallow equality сравнения).»
— перевод

Это ключевое отличие от mapStateToProps с одним параметром:

  • Один параметр (state) — вызывается только при изменении state
  • Два параметра (state, ownProps) — вызывается при изменении state И при получении новых props от родителя
Производительность: с ownProps, mapStateToProps запускается чаще — при каждом изменении props родителя. Это может стать проблемой, если родитель часто перерисовывается. Используйте ownProps только когда действительно нужно сочетать данные из store с props от родителя.

Автоматическое слияние props

«You do not need to include values from ownProps in the object returned from mapStateToProps. connect will automatically merge those different prop sources into a final set of props.»
— react-redux.js.org — connect: Extracting Data with mapStateToProps
«Вам не нужно включать значения из ownProps в объект, возвращаемый mapStateToProps. connect автоматически объединит эти различные источники props в итоговый набор props.»
— перевод
// Не нужно так: const mapStateToProps = (state, ownProps) => ({ id: ownProps.id, // ❌ не нужно — connect и так передаст id todo: state.todos[ownProps.id] }) // Правильно — только данные из store: const mapStateToProps = (state, ownProps) => ({ todo: state.todos[ownProps.id] // ✔ только то, что нужно из store }) // Компонент получит: { id, todo, dispatch } // ↑ ownProps ↑ из store ↑ автоматически

Как connect определяет количество аргументов

React-Redux проверяет mapStateToProps.length (количество объявленных параметров). Если length === 1 — ownProps не передаётся, функция вызывается только при изменении state. Если length >= 2 — ownProps передаётся, и функция вызывается и при изменении props.

// length === 1 → ownProps НЕ передаётся: const mapStateToProps = (state) => ({ count: state.count }) mapStateToProps.length // 1 // length === 2 → ownProps передаётся: const mapStateToProps = (state, ownProps) => ({ todo: state.todos[ownProps.id] }) mapStateToProps.length // 2
Паттерн: список с индивидуальным подключением.
Список рендерит массив id, каждый элемент получает свой id через ownProps и читает данные из store через mapStateToProps(state, ownProps). При изменении одного элемента ре-рендерится только он — остальные не затрагиваются. Это стандартный подход для оптимизации списков в Redux.
