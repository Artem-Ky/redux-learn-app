Интерактив — три dispatch, два исхода

Два изолированных store. Одна и та же операция «загрузить пользователя» — три dispatch: SET_LOADING(true) → SET_USER → SET_LOADING(false). Нажми кнопку в каждой колонке и сравни сколько раз перерисовался компонент и были ли промежуточные («глитчевые») состояния.

Почему flushSync слева? React 18 automatic batching по умолчанию объединяет все dispatch в один рендер даже без batch(). Чтобы честно показать проблему, левая колонка оборачивает каждый dispatch в flushSync() — это принудительно сбрасывает очередь рендеров после каждого вызова, как это происходило в React 17 с setTimeout / Promise.then.
Теория — что такое batch() и зачем он нужен

Проблема трёх dispatch подряд

Классическая ситуация: thunk получил данные с сервера и хочет уведомить store тремя действиями. Каждый dispatch вызывает store.subscribe-листенеры — то есть обходит Subscription, вызывает checkForUpdates → selector → compare → forceRender. Три dispatch подряд = три отдельных рендер-прохода.

// ❌ три рендера в React 17 (setTimeout, Promise.then, нативные события) dispatch({ type: 'SET_LOADING', value: true }) // рендер #1 — spinner без данных dispatch({ type: 'SET_USER', ...user }) // рендер #2 — spinner + данные (нелогично!) dispatch({ type: 'SET_LOADING', value: false }) // рендер #3 — итоговое состояние

batch() — оборачивает в unstable_batchedUpdates

batch() из react-redux — реэкспорт unstable_batchedUpdates() из react-dom. Внутри обёртки React складывает все setState/forceRender в очередь и делает один коммит в конце.

import { batch } from 'react-redux' // ✅ всегда один рендер — независимо от версии React batch(() => { dispatch({ type: 'SET_LOADING', value: true }) dispatch({ type: 'SET_USER', ...user }) dispatch({ type: 'SET_LOADING', value: false }) })

React 18 — автоматический батчинг

С React 18 (createRoot) automatic batching работает везде — в обработчиках событий, Promise.then, setTimeout и нативных callback'ах. Три dispatch подряд дают один рендер без всякого batch().

«If you're using React 18, you do not need to use the batch API. React 18 automatically batches all state updates, no matter where they're queued.»
— react-redux.js.org/api/batch

Когда batch() всё ещё нужен

  • Проект на React < 18.
  • Legacy-точка входа ReactDOM.render — даже в React 18 она не включает automatic batching.
  • Явное документирование намерения: «эти dispatch — одна логическая операция».
  • React Native со старыми версиями.
Важно: batch() объединяет только синхронные dispatch внутри callback'а. Если внутри стоит await или setTimeout, dispatch после паузы уже в другом тике и не батчатся вместе с первыми.
