Теория — Полный обзор API React-Redux

Три группы API

Весь публичный API React-Redux можно разделить на три группы: Provider (компонент для подключения store), Hooks API (современный подход) и Connect API (классический HOC-подход).

"useSelector reads a value from the store state and subscribes to updates, while useDispatch returns the store's dispatch method to let you dispatch actions."

useSelector читает значение из state store и подписывается на обновления, а useDispatch возвращает метод dispatch store для отправки actions.

— react-redux.js.org, Getting Started

1. <Provider>

Компонент <Provider store={store}> оборачивает всё приложение и делает Redux store доступным для всех вложенных компонентов через React Context. Без него ни один хук или connect не сможет получить доступ к store.

import { Provider } from 'react-redux' import store from './store' createRoot(document.getElementById('root')).render( <Provider store={store}> <App /> </Provider> )

2. Hooks API (рекомендуется)

"We recommend using the React-Redux hooks API as the default approach in your React components."

Мы рекомендуем использовать React-Redux hooks API как подход по умолчанию в ваших React-компонентах.

— react-redux.js.org, Hooks
  • useSelector(selector, equalityFn?) — читает данные из store state, подписывается на обновления. При каждом dispatch вызывает selector, сравнивает результат с предыдущим через === и при изменении вызывает forceRender() (подробнее в 03a).
  • useDispatch() — возвращает функцию store.dispatch для отправки actions.
  • useStore() — возвращает ссылку на сам store-объект. Используется редко — только когда нужен прямой доступ к store.getState() или store.replaceReducer().

3. Connect API (классический)

"The existing connect API still works and will continue to be supported, but the hooks API is simpler and works better with TypeScript."

Существующий connect API по-прежнему работает и будет поддерживаться, но hooks API проще и лучше работает с TypeScript.

— react-redux.js.org, Hooks
connect(mapStateToProps?, mapDispatchToProps?, mergeProps?, options?)

connect — это Higher-Order Component (HOC), который оборачивает ваш компонент и передаёт ему props из Redux store. Принимает до четырёх аргументов:

  • mapStateToProps — функция, которая извлекает данные из state и передаёт их как props
  • mapDispatchToProps — функция или объект, определяющий dispatch-функции как props
  • mergeProps — функция для объединения stateProps, dispatchProps и ownProps
  • options — объект с дополнительными настройками (context, areStatesEqual и др.)
Рекомендация: Для нового кода используйте Hooks API (useSelector + useDispatch). Connect API изучайте для поддержки legacy-кода — его вы встретите в старых проектах повсюду.
