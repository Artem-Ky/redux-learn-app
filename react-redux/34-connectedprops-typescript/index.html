Интерактив — две эквивалентные типизации одного компонента

Оба переключателя делают одно и то же: читают state.isOn и диспатчат toggleOn. Слева — «ручная» типизация через StateProps, DispatchProps, OwnProps. Справа — ConnectedProps автоматически выводит типы из mapStateToProps и mapDispatchToProps. Код работает одинаково, но автовариант требует в разы меньше объявлений типов.

Теория — ConnectedProps

Что это такое

ConnectedProps — это утилитарный тип из react-redux, который умеет вытаскивать типы «props из Redux» (всё, что приходит от mapStateToProps и mapDispatchToProps) из сохранённого в переменную connector'а. Работает это только в «двухэтапной» форме вызова connect из урока 33 — именно поэтому типизация и разбиение на два шага идут рука об руку.

«The package includes a helper type, ConnectedProps, that can extract the return types of mapStateToProps and mapDispatchToProps from the first function. This means that if you split the connect call into two steps, all of the "props from Redux" can be inferred automatically without having to write them by hand.»
— react-redux.js.org/using-react-redux/usage-with-typescript
«Пакет включает вспомогательный тип ConnectedProps, который способен извлечь типы, возвращаемые mapStateToProps и mapDispatchToProps, из первой функции. Это означает, что если разбить вызов connect на два шага, все "props из Redux" могут быть выведены автоматически, без необходимости писать их вручную.»
— перевод
«If you're using connect, we recommend using the ConnectedProps approach for inferring the props from Redux, as that requires the fewest explicit type declarations.»
— react-redux.js.org/using-react-redux/usage-with-typescript
«Если вы используете connect, мы рекомендуем использовать подход с ConnectedProps для вывода типов props из Redux, так как это требует минимального количества явных объявлений типов.»
— перевод

Как это работает по шагам

  1. Импортировать: import { connect, ConnectedProps } from 'react-redux'.
  2. Написать mapStateToProps и mapDispatchToProps обычным способом. Тип state в mapStateToProps должен быть явно указан — от него пойдёт весь инференс.
  3. Разбить вызов на два этапа: сохранить connector в переменную const connector = connect(mapStateToProps, mapDispatchToProps).
  4. Извлечь «props из Redux»: type PropsFromRedux = ConnectedProps<typeof connector>.
  5. Объединить с OwnProps: type Props = PropsFromRedux & OwnProps.
  6. Экспортировать: export default connector(MyComponent).

Сравнение boilerplate

Вариант Объявлений типов Дженерики в connect
Ручная типизация StateProps, DispatchProps, OwnProps, Props connect<StateProps, DispatchProps, OwnProps>(...)
ConnectedProps PropsFromRedux, OwnProps, Props connect(mapState, mapDispatch) — без дженериков

Ручная типизация (много boilerplate)

import { connect } from 'react-redux' interface StateProps { isOn: boolean } interface DispatchProps { toggleOn: () => void } interface OwnProps { backgroundColor: string } type Props = StateProps & DispatchProps & OwnProps const mapStateToProps = (state: RootState): StateProps => ({ isOn: state.isOn }) const mapDispatchToProps: DispatchProps = { toggleOn: () => ({ type: 'TOGGLE' }) } const ManualToggle = (props: Props) => /* ... */ export default connect<StateProps, DispatchProps, OwnProps>( mapStateToProps, mapDispatchToProps )(ManualToggle)

ConnectedProps (минимум boilerplate)

import { connect, ConnectedProps } from 'react-redux' const mapStateToProps = (state: RootState) => ({ isOn: state.isOn }) const mapDispatchToProps = { toggleOn: () => ({ type: 'TOGGLE' }) } const connector = connect(mapStateToProps, mapDispatchToProps) type PropsFromRedux = ConnectedProps<typeof connector> // hover → { isOn: boolean; toggleOn: () => { type: 'TOGGLE' } } interface OwnProps { backgroundColor: string } type Props = PropsFromRedux & OwnProps const AutoToggle = (props: Props) => /* ... */ export default connector(AutoToggle)

Что именно инференс делает

  • ReturnType<typeof mapStateToProps> — даёт объект state-props.
  • Если mapDispatchToProps — объект action creators, библиотека вычисляет, какие функции будут в props после bindActionCreators (функции, возвращающие void, а не экшены).
  • Пересечение этих двух объектов ConnectedProps и возвращает.

Связь с системой Subscription

ConnectedProps — это чисто типовой уровень, TypeScript. В рантайме он удаляется компиляцией и не добавляет ни одной строки. Механика подписки остаётся прежней: при dispatch Subscription проходит по linked list callback'ов → Subscription.notify() → для каждого слушателя checkForUpdates → запустить selector (mapStateToProps) → сравнить с prevValue через === → forceRender если отличается. Типы только помогают вам не забыть, какие поля реально приходят в props. Подробнее о Subscription — урок 03a.

Рекомендация команды React Redux: если вы пишете новый код на TypeScript и используете connect — используйте ConnectedProps. Если же вы начинаете новый проект с нуля — команда рекомендует вообще переходить на hooks API (useSelector / useDispatch), там типизация проще и ещё прямее. Об этом — урок 35.
Edge case: если mapDispatchToProps задан как объект и внутри встречаются thunk'и (redux-thunk), ConnectedProps автоматически не вычислит корректный тип возврата thunk'а. Для таких случаев либо типизируйте thunk вручную через ThunkAction, либо используйте hooks-подход с типизированным useAppDispatch.
