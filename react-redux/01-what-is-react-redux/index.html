Три проблемы, которые решает React-Redux

Нажмите «Показать решения», чтобы увидеть как React-Redux справляется с каждой проблемой:

Проблема #1
Props Drilling
Без React-Redux store нужно передавать через props на каждом уровне вложенности. Если store нужен компоненту на глубине 5 уровней — придётся «просверлить» props через 4 промежуточных компонента.
✔ Provider оборачивает всё приложение и делает store доступным через React Context для любого вложенного компонента без props drilling.
Проблема #2
Лишние ре-рендеры
Нативное использование Context перерисовывает ВСЕ компоненты-потребители при любом изменении state, даже если конкретному компоненту нужна только часть данных, которая не изменилась.
✔ React-Redux использует собственную Subscription систему: каждый компонент подписывается на store и ре-рендерится только если его конкретные данные изменились.
Проблема #3
Ручная подписка и утечки памяти
При ручном подключении нужно вызывать store.subscribe() в useEffect, не забыть вернуть unsubscribe при размонтировании, обработать edge cases.
✔ React-Redux автоматически управляет подпиской и отпиской. useSelector/connect сами подписываются и очищаются при размонтировании компонента.

Теория — Что такое React-Redux?

Определение

React-Redux — это официальная библиотека UI-привязок (UI bindings) для Redux, созданная специально для React. Она предоставляет API, позволяющий React-компонентам читать данные из Redux store и отправлять actions для обновления состояния.

«React Redux is the official React UI bindings layer for Redux. It lets your React components read data from a Redux store, and dispatch actions to the store to update state.»
— react-redux.js.org — Getting Started
«React Redux — это официальный слой UI-привязок React для Redux. Он позволяет вашим React-компонентам читать данные из Redux store и отправлять actions в store для обновления состояния.»
— перевод

Зачем нужна отдельная библиотека?

Redux сам по себе — это библиотека управления состоянием, не привязанная к конкретному UI-фреймворку. Он работает с Vanilla JS, Angular, Vue, React и чем угодно. Но чтобы эффективно интегрировать Redux с React, нужен «мост» между ними.

Можно ли подключить Redux к React вручную? Технически — да (мы покажем это в уроке 02). Но у такого подхода много проблем: props drilling, лишние ре-рендеры, утечки памяти, отсутствие батчинга обновлений. React-Redux решает все эти проблемы.

«Designed to work with React's component model. You define how to extract the values your component needs from Redux, and your component updates automatically as needed.»
— react-redux.js.org
«Спроектирован для работы с компонентной моделью React. Вы определяете, как извлечь значения, нужные вашему компоненту, из Redux, и компонент обновляется автоматически при необходимости.»
— перевод

Два API: Connect и Hooks

React-Redux предоставляет два способа подключения компонентов к Redux store:

  • Hooks API (рекомендуемый, с v7.1.0) — хуки useSelector, useDispatch, useStore. Проще, лучше работает с TypeScript.
  • Connect API (HOC-подход) — функция connect(mapStateToProps?, mapDispatchToProps?, mergeProps?, options?), оборачивающая компонент. Принимает до 4 аргументов (все опциональные). Работает с классовыми и функциональными компонентами. По-прежнему поддерживается, но не рекомендуется для нового кода.
Почему мы изучаем ОБА подхода?
В реальных проектах вы встретите и connect, и hooks. Легаси-код часто использует connect. Понимание обоих подходов — ключ к эффективной работе с React-Redux. В этом курсе мы подробно разберём каждый API и научимся мигрировать между ними.

Предпосылки

Этот курс предполагает, что вы прошли главу Classic Redux и понимаете: actions, reducers, store (createStore, getState, dispatch, subscribe), middleware, thunks, selectors, combineReducers. Если нет — вернитесь к Classic Redux.

Код примера: минимальное React-Redux приложение

import { createStore } from 'redux' import { Provider, useSelector, useDispatch } from 'react-redux' // 1. Reducer (уже знакомый из Classic Redux) function counterReducer(state = { value: 0 }, action) { switch (action.type) { case 'increment': return { value: state.value + 1 } case 'decrement': return { value: state.value - 1 } default: return state } } // 2. Store (уже знакомый) const store = createStore(counterReducer) // 3. React-компонент с хуками React-Redux function Counter() { const count = useSelector(state => state.value) // читаем из store const dispatch = useDispatch() // получаем dispatch return ( <div> <button onClick={() => dispatch({ type: 'decrement' })}>−</button> <span>{count}</span> <button onClick={() => dispatch({ type: 'increment' })}>+</button> </div> ) } // 4. Provider оборачивает приложение <Provider store={store}> <Counter /> </Provider>
Важно: В курсе Classic Redux мы работали без React — чистый JS + HTML + Redux. Теперь мы добавляем React в стек. Каждый урок будет содержать React-компоненты, которые рендерятся в <div id="root">.
