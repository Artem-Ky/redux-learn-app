Интерактив — два способа напечатать useSelector

Оба компонента читают одни и те же поля — counter.value и user.name. Слева тип RootState указывается в каждом вызове useSelector: получается шумно и легко забыть. Справа используется pre-typed хук useAppSelector, созданный через useSelector.withTypes<RootState>(): тип state выводится автоматически в каждом селекторе.

Теория — типизация useSelector

Две стратегии печати

Хук useSelector сам по себе дженерик: он принимает функцию-селектор и возвращает её результат. Всё, что нужно TypeScript, — чтобы параметр state селектора имел конкретный тип. Как только TS увидит (state: RootState) => ..., он автоматически выведет тип возврата, и именно этот тип станет типом useSelector(...).

Отсюда ровно два рабочих подхода:

  1. Ручная типизация — в каждом вызове пишем (state: RootState) => ....
  2. Pre-typed hook — один раз создаём useAppSelector = useSelector.withTypes<RootState>() и дальше используем его без аннотаций.
«When writing selector functions for use with useSelector, you should explicitly define the type of the state parameter. TS should be able to then infer the return type of the selector, which will be reused as the return type of the useSelector hook.»
— react-redux.js.org/using-react-redux/usage-with-typescript
«Когда вы пишете функции-селекторы для использования с useSelector, вам следует явно определять тип параметра state. Тогда TS сможет вывести тип возврата селектора, который и будет использоваться как тип возврата хука useSelector.»
— перевод

Вариант 1. Ручная типизация state в каждом вызове

interface RootState { counter: { value: number } user: { name: string; email: string } } function CounterManual() { const count = useSelector((state: RootState) => state.counter.value) const userName = useSelector((state: RootState) => state.user.name) return <div>{userName}: {count}</div> }

Плюсы: работает из коробки, никаких дополнительных файлов. Минусы: каждый вызов — это лишние 18 символов шума, легко забыть указать тип и получить скрытый any в state.

Вариант 2. Отдельные selector-функции с типом

Типизация state в отдельной функции ещё выгоднее — вывод типа распространяется и на возврат, и на все места вызова.

const selectIsOn = (state: RootState) => state.isOn // TS infers: (state: RootState) => boolean const isOn = useSelector(selectIsOn) // TS infers: isOn — boolean

Вариант 3. Pre-typed hook — useAppSelector

Самый компактный способ. Один файл app/hooks.ts — и дальше никто не пишет : RootState вручную.

// app/hooks.ts import { useSelector } from 'react-redux' import type { RootState } from './store' export const useAppSelector = useSelector.withTypes<RootState>() // Counter.tsx function CounterTyped() { const count = useAppSelector(state => state.counter.value) const userName = useAppSelector(state => state.user.name) return <div>{userName}: {count}</div> }

Что выводится автоматически

Выражение Тип, который выведет TS
useAppSelector(state => state.counter.value) number
useAppSelector(state => state.user) { name: string; email: string }
useAppSelector(state => state.user.email) string
useAppSelector(state => state.missing) ошибка компиляции — поля нет

Рантайм один и тот же

Типизация — это только TypeScript. В рантайме оба подхода компилируются в одинаковый JavaScript, и внутри работает одна и та же механика подписок: при dispatch Subscription проходит по linked list callback'ов → Subscription.notify() → для каждого слушателя checkForUpdates → запускается selector → сравнение с prevValue через === → forceRender если значения разные. Подробнее о Subscription — урок 03a.

Рекомендация: создайте файл app/hooks.ts один раз в начале проекта. Дальше все компоненты импортируют useAppSelector вместо useSelector, и вы никогда больше не напишете : RootState внутри селектора. Это буквально та же пара строк, о которой речь шла в уроке 55.
Опасность: если вы забыли указать тип state в ручном варианте — useSelector(state => state.counter.value) — TS не даст ошибку (state станет any), и любая опечатка (state.contre.value) пройдёт молча. Pre-typed хук useAppSelector эту дырку закрывает: забыть тип уже невозможно.
