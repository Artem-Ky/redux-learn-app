Теория — Todo через Hooks API

Цель

Мы уже видели это приложение в уроке 27. Теперь переписываем его на hooks. Структура state, action creator'ы, селекторы — всё то же самое. Отличается только точка подключения компонента к store: вместо HOC connect каждый компонент сам вызывает useSelector / useDispatch.

"The existing connect API still works and will continue to be supported, but the hooks API is simpler and works better with TypeScript."
— react-redux.js.org/api/hooks

Структура state (без изменений)

const initialState = { todos: { byIds: {}, // { [id]: { id, content, completed } } allIds: [] // [id, id, ...] — порядок }, visibilityFilter: 'all' // 'all' | 'completed' | 'incomplete' }

Action creators (без изменений)

const addTodo = (content) => ({ type: 'ADD_TODO', payload: { id: nextId(), content } }) const toggleTodo = (id) => ({ type: 'TOGGLE_TODO', payload: { id } }) const setFilter = (filter) => ({ type: 'SET_FILTER', payload: { filter } })

Селекторы (без изменений)

const getTodos = (state) => state.todos.allIds.map(id => state.todos.byIds[id]) const getTodosByVisibilityFilter = (state, filter) => { const all = getTodos(state) switch (filter) { case 'completed': return all.filter(t => t.completed) case 'incomplete': return all.filter(t => !t.completed) default: return all } }

AddTodo — useDispatch + useState

function AddTodo() { const dispatch = useDispatch() const [value, setValue] = useState('') const submit = (e) => { e.preventDefault() const trimmed = value.trim() if (!trimmed) return dispatch(addTodo(trimmed)) setValue('') } return (...) }

В версии на connect был HOC connect(null, { addTodo }) и внутри props.addTodo(trimmed). Здесь — явный useDispatch и dispatch(addTodo(trimmed)). Типизация проще: компоненту не нужно объявлять AddTodoProps { addTodo: (...) => Action }.

TodoList — один useSelector для отфильтрованного списка

function TodoList() { const filter = useSelector((state: RootState) => state.visibilityFilter) const todos = useSelector((state: RootState) => getTodosByVisibilityFilter(state, filter) ) return ( <div> {todos.map(t => <Todo key={t.id} todo={t} />)} </div> ) }

Todo — получает todo через props, dispatch через useDispatch

function Todo({ todo }: { todo: TodoItem }) { const dispatch = useDispatch() return ( <li onClick={() => dispatch(toggleTodo(todo.id))}> {todo.content} </li> ) }

VisibilityFilters — useSelector + useDispatch

function VisibilityFilters() { const active = useSelector((state: RootState) => state.visibilityFilter) const dispatch = useDispatch() return ( <div> {['all', 'completed', 'incomplete'].map(f => ( <button key={f} onClick={() => dispatch(setFilter(f))} style={{ fontWeight: f === active ? 'bold' : 'normal' }} >{f}</button> ))} </div> ) }

Что изменилось по сравнению с уроком 27

  • Нет HOC-обёрток. В React DevTools дерево плоское: <TodoList> вместо <Connect(TodoList)> → <TodoListRaw>.
  • Меньше типов. Вместо разделения StateProps / DispatchProps / OwnProps и объединения type Props = StateProps & DispatchProps & OwnProps — один RootState и обычные props компонента.
  • Логика dispatch явная. Читая код, сразу видите dispatch(addTodo(...)), а не «props.addTodo» — не надо догадываться, что это забинженный action creator.
  • Каждый компонент сам себе connect. Нет единого места с mapStateToProps. Иногда это плюс (локально), иногда минус (сложнее тестить презентационный компонент изолированно).

Связь с системой Subscription

В hooks-версии подписки создаются для каждого useSelector отдельно. TodoList имеет два useSelector (filter и todos) — это две подписки. Todo не имеет useSelector — вообще не подписан. При dispatch TOGGLE_TODO цепочка linked list callback'ов → Subscription.notify() → checkForUpdates → selector → === → forceRender пробегает по всем подписанным компонентам. useSelector в TodoList, читающий visibilityFilter, вернёт ту же строку → === → не ре-рендерится от этой подписки. Selector для todos вернёт новый массив (после .map/.filter) → === не равны → forceRender. Подробнее — урок 03a.

Один и тот же функционал — выбирайте подход

Официальная документация рекомендует hooks для новых проектов. Но connect продолжает работать — если ваш проект на нём, нет никакой необходимости переписывать. Дерево уроков: 27 (connect) → 50 (side-by-side чтение) → 51 (side-by-side dispatch) разбирают отличия точно.

Практический совет: селекторы (getTodos, getTodosByVisibilityFilter) — общее ядро. Они работают одинаково и с mapStateToProps, и с useSelector. Вкладывайте логику именно в них — тогда миграция между connect и hooks, а также переиспользование между компонентами, становится тривиальной.
