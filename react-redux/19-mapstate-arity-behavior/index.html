Интерактив — Три варианта mapStateToProps: arity и ownProps

Три компонента с разными сигнатурами mapStateToProps. Обратите внимание: вариант 3 использует значение по умолчанию (state, ownProps = {}), но function.length === 1, поэтому ownProps не передаётся.

Теория — Arity (количество аргументов) mapStateToProps

Что такое arity?

Arity — количество аргументов функции. В JavaScript это свойство function.length, которое считает количество обязательных параметров (до первого с значением по умолчанию или rest-параметра).

Как React Redux использует arity

"With just (state), the function runs whenever the root store state object is different. With (state, ownProps), it runs any time the store state is different and ALSO whenever the wrapper props have changed." — react-redux.js.org/using-react-redux/connect-mapstate
«С параметром только (state) функция запускается каждый раз, когда корневой объект state store отличается. С (state, ownProps) она запускается каждый раз, когда state store отличается, И ТАКЖЕ когда props обёртки изменились.»

Не добавляйте ownProps без необходимости

"You should not add the ownProps argument unless you actually need to use it, or your mapStateToProps function will run more often than it needs to." — react-redux.js.org/using-react-redux/connect-mapstate
«Вы не должны добавлять аргумент ownProps, если вы реально его не используете, иначе ваша функция mapStateToProps будет запускаться чаще, чем нужно.»

Ловушка: значение по умолчанию

"Note: ownProps is not passed to mapStateToProps and mapDispatchToProps if the formal definition of the function contains one mandatory parameter (function has length 1)." — react-redux.js.org/api/connect
«Примечание: ownProps не передаётся в mapStateToProps и mapDispatchToProps, если формальное определение функции содержит один обязательный параметр (function.length === 1).»
// Вариант 1: length === 1 → ownProps НЕ передаётся const mapStateToProps = (state) => ({...}) // Вызывается ТОЛЬКО при изменении store state // Вариант 2: length === 2 → ownProps ПЕРЕДАЁТСЯ const mapStateToProps = (state, ownProps) => ({...}) // Вызывается при изменении store state ИЛИ ownProps // Вариант 3: length === 1 (!) → ownProps НЕ передаётся const mapStateToProps = (state, ownProps = {}) => ({...}) // Значение по умолчанию → length === 1 → ownProps не будет передан!

Таблица: arity и поведение

Сигнатура fn.length ownProps передаётся? Когда вызывается
(state) => ... 1 Нет При изменении store state
(state, ownProps) => ... 2 Да При изменении state ИЛИ ownProps
(state, ownProps = {}) => ... 1 Нет При изменении store state (ownProps игнорируется!)
Ловушка: (state, ownProps = {}) выглядит так, будто принимает ownProps, но function.length === 1, поэтому React Redux не передаёт ownProps. Второй аргумент всегда будет undefined (и сработает значение по умолчанию {'{}'}).
Правило: если вам нужен ownProps — явно объявите два параметра без значений по умолчанию: (state, ownProps).
