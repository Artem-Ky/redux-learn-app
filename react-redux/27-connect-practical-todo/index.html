Интерактив — полноценный Todo-лист

Полноценное Todo-приложение из официального туториала react-redux. Четыре компонента — AddTodo, TodoList, Todo, VisibilityFilters — каждый подключён через connect с разным набором аргументов. Добавляйте задачи, отмечайте выполненные, переключайте фильтр.

Теория — Todo-приложение через connect

Структура state

Мы используем нормализованную форму: todos хранятся в объекте byIds и массиве allIds. Это стандартный паттерн Redux для коллекций — он даёт O(1) доступ по id и позволяет менять отдельный todo без копирования всего массива.

const initialState = { todos: { byIds: {}, allIds: [] }, visibilityFilter: 'all' // 'all' | 'completed' | 'incomplete' }

Action creators

const addTodo = (content) => ({ type: 'ADD_TODO', payload: { id: nextId(), content } }) const toggleTodo = (id) => ({ type: 'TOGGLE_TODO', payload: { id } }) const setFilter = (filter) => ({ type: 'SET_FILTER', payload: { filter } })

Селекторы

const getTodos = (state) => state.todos.allIds.map(id => state.todos.byIds[id]) const getTodosByVisibilityFilter = (state, filter) => { const all = getTodos(state) switch (filter) { case 'completed': return all.filter(t => t.completed) case 'incomplete': return all.filter(t => !t.completed) default: return all } }

AddTodo — connect(null, { addTodo })

«By passing it to connect, our component receives it as a prop, and it will automatically dispatch the action when it's called.»
— react-redux.js.org/tutorials/connect
«Передавая action creator в connect, наш компонент получает его в props, и при вызове он будет автоматически диспатчить action.»
— перевод
const mapDispatchToProps = { addTodo } export default connect(null, mapDispatchToProps)(AddTodo) // props.addTodo('Купить хлеб') → dispatch({ type: 'ADD_TODO', payload: ... })

Форме AddTodo не нужно читать state — только диспатчить. Поэтому первым аргументом идёт null. Вариант 3 из предыдущего урока.

TodoList — connect(mapStateToProps)

«The TodoList component is responsible for rendering the list of todos. Therefore, it needs to read data from the store. We enable it by calling connect with the mapStateToProps parameter»
— react-redux.js.org/tutorials/connect
«Компонент TodoList отвечает за отрисовку списка задач. Поэтому ему нужно читать данные из store. Мы подключаем его, вызывая connect с параметром mapStateToProps.»
— перевод
const mapStateToProps = (state) => ({ todos: getTodosByVisibilityFilter(state, state.visibilityFilter) }) export default connect(mapStateToProps)(TodoList)

Todo — connect(null, { toggleTodo })

Каждый отдельный Todo получает свои данные через ownProps от родителя (TodoList передаёт todo как prop). Из store ему нужен только toggleTodo.

const mapDispatchToProps = { toggleTodo } export default connect(null, mapDispatchToProps)(Todo) // <Todo todo={...} /> — todo приходит через ownProps, toggleTodo через connect

VisibilityFilters — connect(mapStateToProps, mapDispatchToProps)

Компоненту фильтра нужно и читать текущее значение (чтобы подсветить активную кнопку), и диспатчить изменение — классический вариант 4.

const mapStateToProps = (state) => ({ activeFilter: state.visibilityFilter }) const mapDispatchToProps = { setFilter } export default connect(mapStateToProps, mapDispatchToProps)(VisibilityFilters)

Связь с системой Subscription

Подписаны только те компоненты, у которых указан mapStateToProps: TodoList и VisibilityFilters. AddTodo и Todo не подписаны на store — они только диспатчат. Когда пользователь нажимает «Добавить», dispatch проходит через reducer, Subscription проходит по linked list callback'ов → notify() → checkForUpdates → селектор getTodosByVisibilityFilter для TodoList → === сравнение массива todos → массив новый → forceRender для TodoList. VisibilityFilters тоже проверяется: его mapStateToProps возвращает { activeFilter: state.visibilityFilter } — filter не менялся, activeFilter остался тот же по === → forceRender не вызывается. Подробнее о Subscription — урок 03a.

Главное наблюдение: каждый компонент использует ровно ту комбинацию аргументов connect, которая ему нужна. Не стоит везде писать connect(mapStateToProps, mapDispatchToProps) «на всякий случай» — подписка стоит ресурсов и вызывает лишние ре-рендеры. Форма ввода, кнопка, одиночный item — часто не нуждаются в подписке на store.
