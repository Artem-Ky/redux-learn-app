Интерактив — TodoItem с ownProps.todoId

Каждый TodoItem получает todoId как проп от родителя (ownProps). Внутри mapDispatchToProps мы замыкаемся на ownProps.todoId и создаём персональные toggle() и remove(). Компонент вызывает их без аргументов — id уже «вшит». В консоли видно, как для каждого элемента mapDispatchToProps вызывается отдельно.

Теория — ownProps и стратегии биндинга

Второй аргумент функции — ownProps

Если функция mapDispatchToProps объявлена с двумя параметрами, react-redux передаёт ей dispatch первым аргументом, а пропсы родителя вторым. И, что важно, пересобирает её результат при смене этих пропсов.

«If your mapDispatchToProps function is declared as taking two parameters, it will be called with dispatch as the first parameter and the props passed to the connected component as the second parameter, and will be re-invoked whenever the connected component receives new props.»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«Если ваша функция mapDispatchToProps объявлена с двумя параметрами, она будет вызвана с dispatch в качестве первого аргумента и пропсами, переданными в connected-компонент, в качестве второго, и будет вызвана повторно всякий раз, когда connected-компонент получает новые пропсы.»
— перевод

Синтаксис

const mapDispatchToProps = ( dispatch, ownProps: { todoId: number } ) => ({ toggle: () => dispatch({ type: 'todos/toggle', payload: ownProps.todoId // замыкание на id }), remove: () => dispatch({ type: 'todos/remove', payload: ownProps.todoId }), }) export default connect(mapStateToProps, mapDispatchToProps)(TodoItem)

Две стратегии биндинга аргументов

Если todoId приходит как ownProps, то есть два места, где можно «привязать» его к экшену: в render'е компонента (при каждом клике) или в mapDispatchToProps (один раз, пока props не изменятся).

«This means, instead of re-binding new props to action dispatchers upon component re-rendering, you may do so when your component's props change.»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«Это значит, что вместо повторного связывания новых пропсов с диспатчерами экшенов при каждом ре-рендере, вы можете делать это при смене пропсов компонента.»
— перевод
Bind on component re-rendering
// mapDispatchToProps без ownProps: const mapDispatchToProps = dispatch => ({ toggleTodo: todoId => dispatch(toggleTodo(todoId)) }) // В render: <button onClick={() => this.props.toggleTodo(this.props.todoId) }/>
Bind on props change
// mapDispatchToProps с ownProps: const mapDispatchToProps = (dispatch, ownProps) => ({ toggleTodo: () => dispatch(toggleTodo(ownProps.todoId)) }) // В render: <button onClick={() => this.props.toggleTodo() }/>

Полный пример — Bind on props change

Ниже — полный рабочий код: родитель TodoList передаёт todoId как проп → connect перехватывает его как ownProps → mapStateToProps и mapDispatchToProps получают его вторым аргументом → toggle() вызывается в компоненте без аргументов, id уже в замыкании.

Bind on props change — полный код
// ─── 1. Типы ───────────────────────────────────────────── interface Todo { id: number; text: string; done: boolean } interface RootState { todos: { items: Todo[] } } type TodosAction = { type: 'todos/toggle'; payload: number } // ─── 2. Reducer ────────────────────────────────────────── function todosReducer(state = { items: [] }, action: TodosAction) { if (action.type === 'todos/toggle') { return { items: state.items.map(t => t.id === action.payload ? { ...t, done: !t.done } : t ) } } return state } const store = createStore(combineReducers({ todos: todosReducer })) // ─── 3. Типы пропсов компонента ───────────────────────── interface OwnProps { todoId: number } // от родителя interface StateProps { todo: Todo | undefined } // из mapState interface DispatchProps { toggle: () => void } // из mapDispatch type Props = OwnProps & StateProps & DispatchProps // ─── 4. Компонент — НЕ знает про dispatch и id ────────── function TodoItemRaw({ todo, toggle }: Props) { if (!todo) return null return ( <div onClick={toggle}> // БЕЗ аргументов! {todo.done ? '✓ ' : '○ '}{todo.text} </div> ) } // ─── 5. mapStateToProps — 2-й аргумент: ownProps ──────── const mapStateToProps = ( state: RootState, ownProps: OwnProps // ← приходят от родителя ): StateProps => ({ todo: state.todos.items.find(t => t.id === ownProps.todoId) }) // ─── 6. mapDispatchToProps — 2-й аргумент: ownProps ───── const mapDispatchToProps = ( dispatch: Dispatch<TodosAction>, ownProps: OwnProps // ← те же ownProps ): DispatchProps => ({ toggle: () => dispatch({ type: 'todos/toggle', payload: ownProps.todoId // ← id запечён в замыкание }) }) // ─── 7. connect склеивает: OwnProps + StateProps + DispatchProps ─ const TodoItem = connect< StateProps, DispatchProps, OwnProps, RootState >(mapStateToProps, mapDispatchToProps)(TodoItemRaw) // ─── 8. Родитель — передаёт только todoId ─────────────── function TodoList() { const items = useSelector((s: RootState) => s.todos.items) return ( <div> {items.map(t => ( // ↓ todoId становится ownProps внутри connect <TodoItem key={t.id} todoId={t.id} /> ))} </div> ) } // ─── 9. Render ────────────────────────────────────────── createRoot(document.getElementById('root')!).render( <Provider store={store}> <TodoList /> </Provider> ) // ─── Поток ownProps ───────────────────────────────────── // <TodoList> рендерит <TodoItem todoId={t.id} /> // │ // ▼ ownProps = { todoId: t.id } // [connect] ──► mapStateToProps(state, ownProps) → { todo } // └─► mapDispatchToProps(dispatch, ownProps) → { toggle } // │ // ▼ props = { todoId, todo, toggle } // [TodoItemRaw] onClick={toggle} → dispatch({ payload: ownProps.todoId })

Про арность — как это определяется?

react-redux смотрит на mapDispatchToProps.length. Ровно тот же механизм, что и для mapStateToProps (урок 19):

  • length === 1 → вызывается как mapDispatchToProps(dispatch) один раз при монтировании;
  • length === 2 → вызывается как mapDispatchToProps(dispatch, ownProps) при каждом обновлении пропсов.
Грабли: если вы записали (dispatch, ownProps = {'{}'}) — length считается как 1, и ownProps вы не получите. Используйте форму без значения по умолчанию.

Производительность

С ownProps-формой react-redux пересоздаёт возвращаемый объект при каждом изменении ownProps. Это значит, что props.toggle меняет свою ссылку, и React.memo/shouldComponentUpdate детей увидит «новую функцию». Для большинства случаев это не проблема, но стоит помнить.

Связь с Subscription-системой

Цикл обновления не меняется: при каждом dispatch store пересчитывает state и зовёт Subscription.notify(). Subscription — это linked list callback'ов. Для каждого подключённого компонента checkForUpdates запускает mapStateToProps, сравнивает каждое поле с предыдущим значением через ===, и если отличается — вызывает forceRender() (разбор в уроке 03a). mapDispatchToProps дополнительно пересчитывается до рендера, если функция с двумя параметрами и пропсы родителя изменились.

Когда стоит использовать: когда action creator требует аргумент, который уже есть в ownProps (id todo, параметр в URL, айди пользователя). Это делает onClick без аргументов — чище в JSX, меньше закрытий в хендлерах.
