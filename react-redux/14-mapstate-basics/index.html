Интерактив — mapStateToProps извлекает данные из store

mapStateToProps — первый аргумент connect(). Функция принимает весь state и возвращает объект с нужными данными. Этот объект передаётся как props в компонент.

Redux state
{ counter: { value: N } }
→
mapStateToProps
(state) => ({ count: state.counter.value })
→
Component props
{ count: N }
Теория — mapStateToProps: первый аргумент connect()

Что такое mapStateToProps?

mapStateToProps — это функция, которая определяет, какие данные из Redux store нужны вашему компоненту. Она «маппит» (отображает) state store в props компонента.

«As the first argument passed in to connect, mapStateToProps is used for selecting the part of the data from the store that the connected component needs.»
— react-redux.js.org — connect: Extracting Data with mapStateToProps
«Как первый аргумент, переданный в connect, mapStateToProps используется для выбора той части данных из store, которая нужна подключённому компоненту.»
— перевод

Когда вызывается mapStateToProps?

«It is called every time the store state changes. It receives the entire store state, and should return an object of data this component needs.»
— react-redux.js.org — connect: Extracting Data with mapStateToProps
«Она вызывается каждый раз при изменении состояния store. Она получает весь state store и должна возвращать объект с данными, которые нужны компоненту.»
— перевод

Точнее, механизм работает так: при dispatch Subscription (linked list слушателей) уведомляет connect-обёртку. Обёртка вызывает mapStateToProps(newState), сравнивает результат с предыдущим (shallow equality), и если хотя бы одно поле отличается — вызывает ре-рендер оригинального компонента.

Сигнатура функции

«mapStateToProps should be defined as a function: function mapStateToProps(state, ownProps?)»
— react-redux.js.org — connect: Extracting Data with mapStateToProps
«mapStateToProps должна быть определена как функция: function mapStateToProps(state, ownProps?)»
— перевод
// Базовая форма — один аргумент (state): const mapStateToProps = (state) => ({ count: state.counter.value, userName: state.user.name, }) // connect() передаст { count, userName } как props в компонент: const ConnectedCounter = connect(mapStateToProps)(Counter) // Counter получит: // props.count — из state.counter.value // props.userName — из state.user.name // props.dispatch — добавлен автоматически

Правила mapStateToProps

  • Должна быть чистой функцией — никаких side effects
  • Возвращает простой объект, каждое поле которого станет prop'ом
  • Должна быть быстрой — вызывается при каждом dispatch
  • Не вызывайте dispatch внутри mapStateToProps
  • Результат сравнивается через shallow equality (не === как в useSelector)
Не создавайте новые объекты внутри mapStateToProps!
return { items: state.items.filter(...) } — создаёт новый массив при каждом вызове. Shallow equality увидит «новый» массив → ненужный ре-рендер. Используйте reselect для мемоизации таких вычислений.
mapStateToProps vs useSelector:
mapStateToProps возвращает объект с несколькими полями (shallow equal сравнение). useSelector возвращает одно значение (=== сравнение). Для нескольких полей с useSelector нужно вызывать его несколько раз или использовать createSelector.
