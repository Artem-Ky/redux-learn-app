Интерактив — Все props компонента <Provider>

Кликайте на каждый prop чтобы увидеть его тип, описание и пример. Внизу — рабочий счётчик с применённым Provider.

Теория — Полный API props компонента <Provider>

TypeScript интерфейс

interface ProviderProps<A extends Action = AnyAction, S = any> { store: Store<S, A> // обязательный context?: React.Context<ReactReduxContextValue> // опциональный serverState?: S // опциональный (SSR) stabilityCheck?: 'never' | 'once' | 'always' // опциональный children: ReactNode // обязательный }

store (обязательный)

Единственный обязательный prop. Принимает Redux store, созданный через createStore() или configureStore(). Provider помещает этот store в React Context и создаёт корневую Subscription (linked list коллбэков — checkForUpdates, сравнение через ===, forceRender() при изменении; подробнее в уроке 03a).

const store = createStore(rootReducer) <Provider store={store}> <App /> </Provider>

context (опциональный)

Позволяет передать собственный React Context вместо встроенного ReactReduxContext. Полезно, когда нужно подключить несколько Redux store к одному приложению — каждый Provider с собственным context изолирует свой store.

import { createContext } from 'react' const MyContext = createContext(null) <Provider store={storeA} context={MyContext}> <App /> </Provider> // Компонент внутри: const value = useSelector(selector, { context: MyContext })

serverState (опциональный)

Используется при SSR-гидрации с hydrateRoot. Передаёт серверное состояние, чтобы избежать hydration mismatch — ситуации, когда серверный HTML не совпадает с клиентским рендером. Подробнее — в следующем уроке.

hydrateRoot( document.getElementById('root'), <Provider store={store} serverState={window.__PRELOADED_STATE__}> <App /> </Provider> )

stabilityCheck (опциональный)

Контролирует проверку стабильности selector-функций в development-режиме. React-Redux может дважды вызвать selector с одним и тем же state и предупредить, если результаты отличаются (нестабильный selector).

  • 'once' — проверка при первом вызове (по умолчанию в dev)
  • 'always' — проверка при каждом вызове
  • 'never' — отключить проверку
// Отключить проверку стабильности (не рекомендуется) <Provider store={store} stabilityCheck="never"> <App /> </Provider>
Предупреждение: Не отключайте stabilityCheck без веской причины. Нестабильные селекторы — частая причина лишних ре-рендеров и багов.
Совет: В большинстве приложений вам потребуется только prop store. Остальные props используются в специфических сценариях: множественные store, SSR-гидрация или отладка производительности.
