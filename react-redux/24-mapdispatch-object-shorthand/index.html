Интерактив — эволюция mapDispatchToProps
★ Рекомендованная документацией форма — mapDispatchToProps как объект action creators.

Те же три счётчика, но теперь расположены как «эволюция»: функция → bindActionCreators → объектная короткая форма. Под капотом все три одинаковы — это демонстрирует таблица с их props ниже.

Теория — объектная короткая форма

Почему появилась эта форма?

«You've seen that the setup for dispatching Redux actions in a React component follows a very similar process: define an action creator, wrap it in another function that looks like (…args) => dispatch(actionCreator(…args)), and pass that wrapper function as a prop to your component.»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«Вы видели, что настройка диспатча экшенов в React-компоненте идёт по очень похожему сценарию: определить action creator, обернуть его в функцию вида (…args) => dispatch(actionCreator(…args)) и передать эту обёртку как проп в компонент.»
— перевод

Как это работает

«Because this is so common, connect supports an "object shorthand" form for the mapDispatchToProps argument: if you pass an object full of action creators instead of a function, connect will automatically call bindActionCreators for you internally.»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«Поскольку этот сценарий очень распространён, connect поддерживает «объектную короткую форму» для аргумента mapDispatchToProps: если передать объект action creators вместо функции, connect автоматически вызовет для вас bindActionCreators внутри.»
— перевод

По сути, react-redux делает ровно это:

// То, что react-redux делает за вас автоматически: (dispatch) => bindActionCreators(mapDispatchToProps, dispatch)

Синтаксис

import { connect } from 'react-redux' import { increment, decrement, reset } from './counterActions' // Вариант 1: отдельная переменная const mapDispatchToProps = { increment, decrement, reset } export default connect(mapStateToProps, mapDispatchToProps)(Counter) // Вариант 2: инлайн прямо в connect export default connect( mapStateToProps, { increment, decrement, reset } )(Counter) // Вариант 3: import * as — особенно удобно для большого списка import * as actions from './counterActions' export default connect(mapStateToProps, actions)(Counter)

Правила

«Each field of the mapDispatchToProps object is assumed to be an action creator»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«Каждое поле объекта mapDispatchToProps считается action creator'ом.»
— перевод

Поэтому в объекте должны лежать именно функции-action creators (возвращающие объект экшена), а не сами объекты экшенов.

Важно: как и при функциональной форме, props.dispatch не передаётся компоненту, когда используется объектная короткая форма. Об этом отдельно поговорим в уроке 25.

Почему это рекомендуемая форма

«We recommend always using the "object shorthand" form of mapDispatchToProps, unless you have a specific reason to customize the dispatching behavior.»
— react-redux.js.org/using-react-redux/connect-mapdispatch
«Мы рекомендуем всегда использовать «объектную короткую форму» mapDispatchToProps, если у вас нет конкретной причины кастомизировать поведение диспатча.»
— перевод
  • Минимум boilerplate — один объект, без функции-обёртки;
  • Аргументы автоматически форвардятся (благодаря внутреннему bindActionCreators);
  • Легко импортировать все action creators через import * as actions;
  • Рекомендуется официальной документацией.

Когда всё же нужна функциональная форма

  • когда нужно использовать ownProps в обёртке;
  • когда нужно кастомизировать или логировать dispatch;
  • когда надо вручную вернуть dispatch в props (урок 25);
  • когда нужно комбинировать разные источники экшенов.

Связь с Subscription-системой

Короткая форма — это синтаксический сахар над bindActionCreators, а тот — сахар над (...args) => dispatch(actionCreator(...args)). То есть поведение во время выполнения одинаковое: при клике вызывается dispatch, store прогоняет reducer, зовёт Subscription.notify(). Subscription — linked list callback'ов; для каждого подключённого компонента checkForUpdates запускает mapStateToProps, сравнивает каждое поле через === и при необходимости вызывает forceRender() (подробнее — урок 03a).

Итог урока: если не нужны ownProps и специфические кастомизации — пишите connect(mapStateToProps, { actionA, actionB }). Это идиоматичный и самый лаконичный способ.
