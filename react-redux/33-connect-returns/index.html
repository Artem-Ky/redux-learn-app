Интерактив — один HOC для трёх компонентов

Один и тот же connectUser = connect(mapStateToProps, mapDispatchToProps) применён к трём разным компонентам: Login, Profile, Settings. Все три видят один и тот же срез store и получают одинаковые action creators. Нажмите «Login» / «Logout» — все три карточки обновляются одновременно, потому что все три подписаны на одни и те же поля state.

Теория — что возвращает connect()

Два этапа вызова

Когда вы пишете connect(mapStateToProps, mapDispatchToProps)(Comp) — это два вызова подряд. Первый — connect(...) — возвращает не компонент, а функцию, которую в документации называют «wrapper function». Второй — применение этой функции к вашему Comp — возвращает уже подключённый компонент.

«The return of connect() is a wrapper function that takes your component and returns a wrapper component with the additional props it injects.»
— react-redux.js.org/api/connect
«Возвращаемое значение connect() — это функция-обёртка, которая принимает ваш компонент и возвращает компонент-обёртку с дополнительными props, которые она инжектит.»
— перевод
«In most cases, the wrapper function will be called right away, without being saved in a temporary variable»
— react-redux.js.org/api/connect
«В большинстве случаев функцию-обёртку вызывают сразу же, не сохраняя её в промежуточной переменной.»
— перевод

Разворачиваем однострочник

Этап 1 — настройка HOC
connect(...) запоминает mapStateToProps / mapDispatchToProps и возвращает wrapper-функцию.
const connectUser = connect( mapStateToProps, mapDispatchToProps ) // typeof connectUser === 'function'
Этап 2 — применение HOC
Wrapper-функция получает ваш компонент и возвращает новый, подключённый к store.
const ConnectedLogin = connectUser(Login) const ConnectedProfile = connectUser(Profile) const ConnectedSettings = connectUser(Settings)
connect(mapState, mapDispatch)
→
wrapper: (Comp) => Connected
→
wrapper(Comp)
→
Connected<Comp>

Обычная запись — в одну строку

В 99% случаев вы сразу применяете wrapper к компоненту в той же строке — именно так выглядит классический export default в контейнерах.

// Две формы эквивалентны: const connectUser = connect(mapStateToProps, mapDispatchToProps) export default connectUser(Login) // ≡ export default connect(mapStateToProps, mapDispatchToProps)(Login)

Зачем сохранять wrapper в переменную

  • Переиспользование: один HOC с одинаковыми mapStateToProps / mapDispatchToProps можно применить к нескольким компонентам (этот урок — живой пример).
  • TypeScript / ConnectedProps: если wrapper в переменной, можно извлечь типы его props через ConnectedProps<typeof connectUser> (урок 34).
  • Композиция HOC'ов: удобно передавать wrapper в цепочки compose(withRouter, connectUser, withTheme).

Связь с системой Subscription

Wrapper-функция — это фабрика компонентов. Когда вы применяете её к разным компонентам, вы получаете разные подключённые компоненты, но у них общая «форма» подписки: один и тот же mapStateToProps. В рантайме каждый инстанс всё равно подписывается отдельно — при mount вызывается store.subscribe(checkForUpdates) и callback добавляется в linked list Subscription. При dispatch Subscription проходит по linked list callback'ов → Subscription.notify() → для каждого слушателя checkForUpdates → запустить selector (mapStateToProps) → сравнить с prevValue через === → forceRender если отличается. Три инстанса ConnectedLogin / ConnectedProfile / ConnectedSettings — это три независимых записи в linked list, у каждой свои prevValue. Подробнее о Subscription — урок 03a.

Мысленная модель: connect(mapState, mapDispatch) — это «шаблон контейнера», connectUser(Login) — «инстанциация шаблона» на конкретном презентационном компоненте. Пока шаблоны одинаковые — можно держать один wrapper в переменной и применять его несколько раз.
Важно: две разные connect(...)-вызова вернут две разные wrapper-функции, даже если передали им одинаковые mapStateToProps. Никакой «кэш по ссылке на mapStateToProps» внутри библиотеки нет. Сохранение wrapper в переменную — это ваше решение, а не автоматическая оптимизация.
