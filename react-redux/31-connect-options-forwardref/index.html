Интерактив — ref на connect-HOC vs forwardRef

Оба варианта — один и тот же класс-компонент InputComponent, обёрнутый connect. Слева — без опции forwardRef: ref из родителя попадает на обёрточный компонент, а не на класс, ref.current.focus() падает. Справа — с { forwardRef: true }: ref пробрасывается к реальному инстансу класса, и focus() работает. Нажмите «Focus input» в каждой карточке.

Теория — options.forwardRef

Зачем нужен forwardRef

connect — это высокоуровневый компонент (HOC). Он оборачивает ваш компонент в собственный обёрточный: читает store из <Provider>, запускает mapStateToProps, mapDispatchToProps, сравнивает результаты и, если что-то поменялось, форсирует рендер. Когда родитель ставит ref на connect-HOC, по умолчанию ref попадает именно на обёртку, а не на ваш класс-компонент внутри. У обёртки нет .focus(), нет .scrollIntoView(), нет никаких instance-методов — ref в такой форме бесполезен.

«If {forwardRef : true} has been passed to connect, adding a ref to the connected wrapper component will actually return the instance of the wrapped component.»
— react-redux.js.org/api/connect
«Если в connect передано {forwardRef: true}, то добавление ref на обёрнутый компонент-обёртку вернёт инстанс внутреннего (wrapped) компонента.»
— перевод
«Note: This parameter is supported in >= v6.0 only»
— react-redux.js.org/api/connect
«Замечание: эта опция поддерживается только начиная с v6.0.»
— перевод

Как включить

forwardRef — это поле в четвёртом аргументе connect, объекте options (там же, где areStatesEqual, context и т. д.). Тип — boolean, по умолчанию false.

// Без forwardRef — ref попадает на обёртку, а не на InputComponent const Connected = connect(mapStateToProps)(InputComponent) const inputRef = useRef<InputComponent>(null) <Connected ref={inputRef} /> inputRef.current.focus() // ✖ TypeError: inputRef.current.focus is not a function // С forwardRef — ref пробрасывается к реальному InputComponent const ConnectedWithRef = connect( mapStateToProps, null, null, { forwardRef: true } )(InputComponent) <ConnectedWithRef ref={inputRef} /> inputRef.current.focus() // ✔ работает

Что происходит внутри

Когда forwardRef: true, библиотека внутри оборачивает обёрточный компонент в React.forwardRef. Входящий ref не потребляется обёрткой, а прокидывается в «настоящий» компонент. Для класс-компонента ref.current будет инстансом класса (и на нём есть все методы и ссылки). Для функционального — нужен React.forwardRef / useImperativeHandle внутри самого компонента, потому что у функции нет «инстанса».

Что куда попадает: таблица

Конфигурация Куда попадает ref родителя Что даёт ref.current
connect(mapStateToProps)(Comp) На обёрточный компонент-connect Обёртка; у неё нет ваших методов
connect(mapStateToProps, null, null, { forwardRef: true })(Comp) На ваш Comp (wrapped) Инстанс класса — доступны .focus(), поля и т. д.

Связь с системой Subscription

На подписку опция forwardRef не влияет. При dispatch Subscription проходит по linked list callback'ов → Subscription.notify() → для каждого слушателя checkForUpdates → запустить selector (то есть mapStateToProps) → сравнить с prevValue через === (точнее, shallowEqual над результатом) → forceRender если отличается. Весь этот конвейер живёт в обёрточном компоненте и одинаково работает с forwardRef и без. Опция меняет только то, куда указывает внешний ref. Подробнее о Subscription — урок 03a.

Когда использовать

  • Родителю нужно вызвать instance-метод класса: .focus(), .scrollIntoView(), свой .reset(), .validate().
  • Библиотечный компонент должен прокинуть ref «наружу» для интеграции с DOM API или другими библиотеками (drag'n'drop, tooltip-позиционирование и т. п.).
  • Функциональный компонент использует React.forwardRef + useImperativeHandle — и вы хотите сохранить эту возможность после connect.
Альтернатива для функциональных компонентов: часто вместо forwardRef: true проще отказаться от connect и перейти на хуки (useSelector / useDispatch, уроки 35–49). В хук-версии вашего компонента не появляется HOC-обёртка, и ref работает без дополнительных опций — это ещё одна причина, почему команда React Redux рекомендует хуки по умолчанию.
Раньше было по-другому: в очень старых версиях React Redux (v5 и ниже) вместо forwardRef использовалась опция withRef: true и метод getWrappedInstance(). С v6 они удалены — forwardRef: true вместе с передачей ref напрямую закрывает все эти сценарии.
