Интерактив — обычный useDispatch vs useAppDispatch

В стор подключён redux-thunk. Слева — попытка вызвать thunk через обычный useDispatch(): на уровне TypeScript это компилятор ошибочно отклоняет, потому что стандартный тип Dispatch знает только про plain-экшены. Справа — тот же thunk, но через useAppDispatch, сконфигурированный под тип стор-а: компилятор счастлив, запрос работает, в уведомлениях виден переход loading → loaded.

Теория — AppDispatch и thunks

Почему стандартный useDispatch не знает про thunks

У ядра Redux тип Dispatch максимально простой: (action: A) => A, где A extends Action. То есть он принимает объект-экшен и возвращает его же. А thunk — это функция (dispatch, getState) => any. По стандартному типу такая функция не проходит как Action, поэтому TypeScript отказывается её принять.

Middleware redux-thunk в рантайме расширяет dispatch: «если пришла функция — вызвать её, иначе передать дальше». Типов это само по себе не меняет: чтобы компилятор узнал про новую возможность, тип dispatch тоже должен быть расширен. Именно это делает store.dispatch после applyMiddleware(thunk).

«For useDispatch, the default Dispatch type does not know about thunks or other middleware. In order to correctly dispatch thunks, you need to use the specific customized AppDispatch type from the store that includes the thunk middleware types, and use that with useDispatch.»
— react-redux.js.org/using-react-redux/usage-with-typescript
«Для useDispatch стандартный тип Dispatch не знает о thunks или других middleware. Чтобы корректно диспатчить thunks, нужно взять кастомный тип AppDispatch из стор-а, который уже включает типы middleware thunk, и использовать его с useDispatch.»
— перевод
«By default, the return value of useDispatch is the standard Dispatch type defined by the Redux core types, so no declarations are needed.»
— react-redux.js.org/using-react-redux/usage-with-typescript
«По умолчанию возвращаемое значение useDispatch имеет стандартный тип Dispatch, определённый в ядре Redux, поэтому никаких дополнительных деклараций не нужно.»
— перевод
«If you have a customized version of the Dispatch type, you may use that type explicitly: const dispatch: AppDispatch = useDispatch().»
— react-redux.js.org/using-react-redux/usage-with-typescript
«Если у вас есть кастомизированная версия типа Dispatch, вы можете использовать её явно: const dispatch: AppDispatch = useDispatch().»
— перевод

Шаги настройки

  1. Подключить thunk в стор: applyMiddleware(thunk).
  2. Извлечь точный тип dispatch: export type AppDispatch = typeof store.dispatch. Этот тип уже знает про thunks — потому что typeof считывает результат с применённым middleware.
  3. Создать pre-typed хук: export const useAppDispatch = useDispatch.withTypes<AppDispatch>().
  4. В компонентах писать const dispatch = useAppDispatch() и диспатчить как объекты-экшены, так и thunks — всё типизируется корректно.

Пример thunk action creator

// store.ts import { thunk } from 'redux-thunk' const store = createStore(rootReducer, applyMiddleware(thunk)) export type AppDispatch = typeof store.dispatch // знает про thunk! // fetchUser.ts const fetchUser = (id: number) => async (dispatch: AppDispatch) => { dispatch({ type: 'user/loading' }) await new Promise(r => setTimeout(r, 500)) dispatch({ type: 'user/loaded', payload: { id, name: 'Alice' }, }) }

Стандартный useDispatch — не компилируется

function Component() { const dispatch = useDispatch() // Dispatch<AnyAction> — не знает о thunks dispatch(fetchUser(1)) // TS2345: Argument of type // '(dispatch: AppDispatch) => Promise<void>' // is not assignable to parameter of type 'AnyAction'. }

Типизированный useAppDispatch — компилируется

const useAppDispatch = useDispatch.withTypes<AppDispatch>() function Component() { const dispatch = useAppDispatch() // тип: AppDispatch (с middleware) dispatch(fetchUser(1)) // OK — thunk dispatch({ type: 'ping' }) // OK — plain action }

Что именно делает .withTypes и что делает typeof store.dispatch

Конструкция Что возвращает
typeof store.dispatch точный тип dispatch с учётом всех middleware (thunks, RTK Query, и т. д.)
useDispatch.withTypes<AppDispatch>() фиксирует возврат useDispatch() как AppDispatch
useDispatch() (без типизации) Dispatch<UnknownAction> — минимальный набор

Связь с системой Subscription

Типы не меняют то, как работает подписка. Thunk вызывает dispatch несколько раз: user/loading → user/loaded. Каждый такой dispatch запускает reducer, обновляет state и запускает подписку — Subscription проходит по linked list callback'ов → Subscription.notify() → для каждого слушателя checkForUpdates → запустить selector → сравнить через === → forceRender если результат изменился. Подробнее о Subscription — урок 03a.

Паттерн для проектов: держите AppDispatch и useAppDispatch в файлах app/store.ts и app/hooks.ts. В самих компонентах никогда не импортируйте голый useDispatch — только useAppDispatch. Это делает весь код совместимым с thunks сразу, без экспериментов с as-кастами.
Не пытайтесь обойти типы через as any. Кастинг скроет проблему, но при этом потеряется и полезная информация: возврат thunk-а (Promise<T>), который часто нужен вызывающей стороне. С правильным AppDispatch вы получите const user = await dispatch(fetchUser(1)) с корректным типом.
