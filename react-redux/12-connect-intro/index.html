Теория — connect() и HOC-паттерн

Что такое connect()?

connect() — это Higher-Order Component (HOC), который подключает React-компонент к Redux store. Это исторически первый и основной API React-Redux (до появления хуков в v7.1).

«The connect() function connects a React component to a Redux store. It provides its connected component with the pieces of the data it needs from the store, and the functions it can use to dispatch actions to the store.»
— react-redux.js.org — Connect
«Функция connect() подключает React-компонент к Redux store. Она предоставляет подключённому компоненту те данные из store, которые ему нужны, и функции, которые он может использовать для dispatch'а actions в store.»
— перевод
«It does not modify the component class passed to it; instead, it returns a new, connected component class that wraps the component you passed in.»
— react-redux.js.org — Connect
«Она не модифицирует переданный ей класс компонента; вместо этого она возвращает новый, подключённый класс компонента, который оборачивает переданный вами компонент.»
— перевод

Сигнатура connect()

function connect( mapStateToProps?, // (state, ownProps?) => stateProps mapDispatchToProps?, // object | (dispatch, ownProps?) => dispatchProps mergeProps?, // (stateProps, dispatchProps, ownProps) => mergedProps options? // { context?, areStatesEqual?, ... } )

connect() возвращает функцию-обёртку (wrapper function). Эту функцию вы вызываете с вашим компонентом:

// Двойной вызов — это currying: const enhance = connect(mapStateToProps, mapDispatchToProps) // → wrapper function const ConnectedCounter = enhance(Counter) // → connected component // Или в одну строку: const ConnectedCounter = connect(mapStateToProps, mapDispatchToProps)(Counter)

HOC-паттерн

Higher-Order Component — это функция, которая принимает компонент и возвращает новый улучшенный компонент:

// HOC-паттерн в общем виде: const withSomething = (WrappedComponent) => { return function EnhancedComponent(props) { const extraData = useExtraData() return <WrappedComponent {...props} {...extraData} /> } } // connect() — частный случай HOC: // он добавляет данные из Redux store в props компонента

Обёртка (ConnectedCounter) делает следующее:

  1. Читает store из React Context (через <Provider>)
  2. Подписывается на изменения store через Subscription (linked list слушателей; при dispatch — selector проверяет изменения через ===, и если значение отличается — вызывает forceRender())
  3. Вызывает mapStateToProps(state) при каждом изменении store
  4. Передаёт результат как props в оригинальный компонент
connect() vs хуки: Сегодня для новых проектов рекомендуются хуки (useSelector + useDispatch). Но connect() всё ещё полностью поддерживается, широко используется в legacy-коде, и имеет некоторые преимущества (nested Subscription для защиты от zombie children, автоматический memoization через areStatesEqual).
