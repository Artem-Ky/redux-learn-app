Интерактив — три компонента, три среза state

Каждый компонент ниже использует useSelector для чтения только своего среза state. Нажмите кнопку — изменится только один срез; ре-рендерится только тот компонент, чей useSelector вернул новое значение. Карточка подсветится.

Теория — useSelector основы

Что делает useSelector

"Allows you to extract data from the Redux store state for use in this component, using a selector function."
— react-redux.js.org/api/hooks
«Позволяет извлечь данные из state Redux-хранилища для использования в этом компоненте с помощью selector-функции.»
— перевод

Сигнатура

useSelector<Selected>(selector, equalityFn?, options?) => Selected

Первый аргумент — selector-функция. Второй (необязательный) — функция сравнения equalityFn. Третий (необязательный) — объект options (включает equalityFn и настройки dev-проверок). Возвращается выбранное значение типа Selected.

Правила selector-функции

"The selector function should be pure since it is potentially executed multiple times and at arbitrary points in time."
— react-redux.js.org/api/hooks
«Selector-функция должна быть чистой, поскольку потенциально выполняется много раз и в произвольные моменты времени.»
— перевод
"The selector will be called with the entire Redux store state as its only argument."
— react-redux.js.org/api/hooks
«Selector вызывается с полным state Redux-хранилища в качестве единственного аргумента.»
— перевод
"The selector may return any value as a result, including directly returning a value that was nested inside state, or deriving new values."
— react-redux.js.org/api/hooks
«Selector может возвращать любое значение — как непосредственно значение, лежащее внутри state, так и производное (derived) значение.»
— перевод

Пример: чтение вложенного значения

function CounterDisplay() { const count = useSelector((state: RootState) => state.counter.value) return <div>Count: {count}</div> }

Когда selector запускается

"The selector will be run whenever the function component renders... useSelector() will also subscribe to the Redux store, and run your selector whenever an action is dispatched."
— react-redux.js.org/api/hooks
«Selector будет запущен при каждом рендере функционального компонента... useSelector() также подписывает компонент на Redux-хранилище и запускает selector при каждом dispatch action.»
— перевод
  • 1. При каждом рендере компонента — чтобы вернуть актуальное значение в текущем рендере.
  • 2. При каждом dispatch — чтобы проверить, изменился ли результат, и решить, нужен ли ре-рендер.

Концептуальное соответствие mapStateToProps

"The selector is approximately equivalent to the mapStateToProps argument to connect conceptually."
— react-redux.js.org/api/hooks
«Selector концептуально приблизительно эквивалентен аргументу mapStateToProps функции connect.»
— перевод

Но есть важное отличие: mapStateToProps обычно возвращает объект, а selector чаще возвращает одно значение (примитив или уже мемоизированную ссылку). Это связано с тем, что useSelector по умолчанию сравнивает результат через === (strict reference equality), а connect использует shallow-сравнение полей — подробнее в уроке 37.

Связь с системой Subscription

Когда компонент с useSelector маунтится, хук регистрирует свой checkForUpdates в nearest Subscription через store.subscribe (подробнее в уроке 03a). После каждого dispatch: Subscription проходит по linked list callback'ов → Subscription.notify() → для каждого слушателя checkForUpdates → запустить selector → сравнить через === → forceRender если отличается. Так каждый из трёх компонентов в демо выше слушает один и тот же store, но ре-рендерится только если его selector вернул другое значение.

Итог: useSelector — минимальный API для чтения state из хука. Селектор вызывается при рендере и после каждого dispatch. Компонент ре-рендерится, только если предыдущее и новое значения !==. В следующих уроках мы разберём, как это сравнение влияет на производительность и на выбор формы selector'а.
