Интерактив — dispatch в props без mapDispatchToProps

Компонент подключён через connect(mapStateToProps)(Counter) — без второго аргумента. В этом случае dispatch автоматически передаётся в props. Кнопки вызывают props.dispatch() напрямую.

Теория — dispatch по умолчанию

Когда dispatch передаётся в props?

"If you don't specify the second argument to connect(), your component will receive dispatch by default." — react-redux.js.org/using-react-redux/connect-mapdispatch
«Если вы не указываете второй аргумент connect(), ваш компонент получит dispatch по умолчанию.»

Три эквивалентные формы

Все три варианта ниже дают одинаковый результат — компонент получает props.dispatch:

// Форма 1: без аргументов connect()(MyComponent) // Форма 2: null, null connect(null, null)(MyComponent) // Форма 3: только mapStateToProps (без mapDispatchToProps) connect(mapStateToProps)(MyComponent) // Все три → props.dispatch доступен внутри компонента

Использование props.dispatch

function Counter({ count, dispatch }) { return ( <div> <span>{count}</span> <button onClick={() => dispatch({ type: 'INCREMENT' }) }>+</button> </div> ) } const mapStateToProps = state => ({ count: state.counter }) export default connect(mapStateToProps)(Counter) // mapDispatchToProps не указан → dispatch в props

Когда dispatch НЕ передаётся?

Если вы указали mapDispatchToProps (функцию или объект), dispatch не передаётся в props автоматически. Вместо этого передаются результаты mapDispatchToProps:

// mapDispatchToProps указан → dispatch НЕ в props const mapDispatchToProps = { increment: () => ({ type: 'INCREMENT' }) } connect(mapStateToProps, mapDispatchToProps)(Counter) // props = { count, increment } — dispatch отсутствует!
Важно: использование props.dispatch напрямую — самый простой, но наименее «чистый» подход. Компонент знает о Redux напрямую. В следующих уроках (21+) мы научимся использовать mapDispatchToProps для абстрагирования dispatch.

Связь с системой Subscription

Когда вы вызываете props.dispatch(action), action попадает в store. Store вызывает reducer, сохраняет новый state и уведомляет Subscription (подробнее в уроке 03a). Subscription проходит по linked list callback'ов и вызывает checkForUpdates для каждого подключённого компонента. checkForUpdates запускает mapStateToProps с новым state, сравнивает каждое поле через ===, и если есть изменения — вызывает forceRender.

Итог: без mapDispatchToProps вы получаете dispatch в props «бесплатно». Это удобно для быстрого прототипирования, но для продакшн-кода лучше использовать mapDispatchToProps (следующий урок).
